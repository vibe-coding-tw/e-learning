簡介：The Agentic Era

在過去，AI 只能在你打字的游標旁悄悄補全幾行程式碼。但現在，AI Agent 的時代已經來臨。 它不只能寫程式碼，還能自主規劃方案、跨多個檔案修改、執行終端機命令、 讀取編譯錯誤並自動修復——直到任務完成。

情境
小明接到一個緊急任務：「把整個專案的 REST API 改成 GraphQL」。 以前他得逐個檔案手動修改，花上一整天。但現在他只需在 Agent Mode 中輸入這句話， AI Agent 就會列出計畫、逐步修改每個檔案、執行測試、修正錯誤，他只需要審核最終結果。

使用 Copilot Agent Mode

這是最直接的 AI Agent 結合方式。在 VS Code 中，你可以將 Copilot 切換到 Agent Mode。 它是一個「自主程式設計師」，可以跨多個檔案進行修改、執行終端機命令、讀取編譯錯誤並自動修復。

操作流程

1

進入 Chat 視窗

打開 Copilot Chat，快捷鍵為 Ctrl+Alt+I（Mac 為 Cmd+Alt+I）。

2

切換模式

在 Chat 頂部的下拉選單中選擇 "Agent" 模式。

3

下達任務

輸入高階指令。例如：「幫我把這個專案的 API 從 REST 改成 GraphQL」。

自主迴圈 (Agentic Loop) Agent 會先列出計畫 (Plan)，獲得你同意後開始修改檔案、跑測試、看到錯誤後自動修正，直到任務完成。 這就是所謂的「自主回饋機制」。
提示 你可以隨時在 Agent 的執行過程中按下「停止」按鈕，審查目前的修改狀態，再決定是否讓 Agent 繼續。 養成審核的習慣，是 Vibe Coding 的核心能力之一。

MCP 協議與自訂 Agent 指令

MCP (Model Context Protocol)

你可以透過 MCP 協議，讓 AI Agent 擁有「手和腳」，去存取外部工具或私有數據。 這讓 Agent 不再受限於你本地的程式碼，還能查詢 Jira、搜尋 Google、讀取資料庫等等。

應用場景

  • 自動查詢 Jira / GitHub Issues
  • 搜尋 Google 即時資訊
  • 讀取電腦本地的特定 資料庫

設定方式

// 在 VS Code 設定中開啟 MCP
"chat.mcp.enabled": true

// 連接 MCP 伺服器，賦予 API 呼叫能力

自訂 Agent 指令 (AGENTS.md)

如果你希望 Agent 在你的 Workspace 中遵循特定的架構或風格，你可以「調教」它。 在專案根目錄建立一個 AGENTS.md 檔案，寫入你的團隊規範。

範例指示
  • 所有 API 必須使用 TypeScript 定義介面
  • 所有組件必須使用 Tailwind CSS 進行樣式設定
  • 測試必須使用 Jest 且覆蓋率需大於 80%
注意 AGENTS.md 的規則越明確、越具體，Agent 的遵循效果越好。 避免模糊的描述，例如「寫好的程式碼」，應改為具體的標準，如「所有導出的函數必須包含 JSDoc 註解」。

02-unit-agent-mode (AI 代理人與 Vibe Coding 實戰)

本程式庫是專為「AI 代理人與 Vibe Coding」單元設計的實戰作業環境。在 2026 年，開發的核心能力已從單純的編碼轉向「引導自主代理 (Agent Guidance)」。你將學習如何指揮 AI Agent 完成複雜的重構、設定團隊規範，並整合外部工具。

🎯 學習目標

  1. 掌握 AI Agent 的 "Plan-Execute" 工作流程。
  2. 學習透過 AGENTS.md 定義專案級別的 AI 工作規範。
  3. 實踐 MCP (Model Context Protocol) 整合，擴展 AI 的外部感知能力。

🛠️ 環境準備

  • GitHub Copilot 訂閱：請確保你的 VS Code 已更新至支援 Agent Mode 的版本。
  • VS Code 擴充功能：安裝最新版的 GitHub Copilot 與 GitHub Copilot Chat。

📝 作業任務

任務 1：全自動代碼重構 (Agent Refactoring)

  • 背景：src/legacy-service.js 是一段結構混亂的舊代碼。
  • 任務：開啟 Copilot Agent Mode，指令 AI 進行重構。要求 Agent 遵循「職責分離」原則，將 UI 邏輯與業務邏輯拆分。
  • 要求：在重構前，讓 Agent 產出 "Plan" (計畫清單)，並將其複製到 PLAN_LOG.md。

任務 2：定義團隊規範 (Custom Instructions)

  • 背景：為了確保團隊協作中 AI 的行為一致，我們需要建立 AGENTS.md。
  • 任務：編輯本程式庫根目錄下的 AGENTS.md，至少加入一條具體的開發規則。
  • 驗證：完成後，嘗試讓 Agent 修改 src/legacy-service.js，觀察它是否會自動遵循你的新規則。

任務 3：MCP 外部工具整合

  • 背景：AI 需要更多外部資訊來協助開發。
  • 任務：參考 MCP_GUIDE.md 串接一個 MCP Server。
  • 回傳：提交一張截圖 mcp-proof.png，展示 Copilot Chat 視窗中出現了 「Tool Call」 的圖示。

針對 02-unit-agent-mode (AI 代理人與 Vibe Coding 實戰) 單元，這是一個代表 2026 年最新開發維度的單元，重點不在於「撰寫單行代碼」，而在於「引導自主代理 (Agent Guidance)」。

1. 核心設置與工具要求

  • 軟體要求：學員必須擁有 GitHub Copilot 訂閱。
  • 範本程式庫配置：
    • 📂 src/legacy-service.js：故意放入結構混亂的舊代碼，供學員重構。
    • 📂 AGENTS.md：預置 TODO 的 Markdown 檔案，引導學員定義規範。
    • 📂 MCP_GUIDE.md：提供常用 MCP Server 連接清單。
