簡介

攝影機回傳的彩色畫面包含數十萬個像素，但機器需要的只是「哪裡有目標、哪裡沒有」。 本模組將教你掌握灰階轉換、二值化門檻判定與卷積濾波等影像處理基礎， 學會將複雜的現實世界「降維」成機器易於理解的純粹特徵矩陣。

情境

影像二值化 (Thresholding) 的原理

影像二值化（Thresholding）是數位影像處理中最基礎也最關鍵的運算之一。它的目的是將一張包含 256 階灰度值的影像，轉化為只有兩種數值（0 與 255，即黑與白）的二元矩陣。這個過程讓機器能夠快速判斷「哪裡有目標、哪裡是背景」，是所有後續特徵提取與控制邏輯的起點。

灰階轉換：從彩色到單通道

在進行二值化之前，必須先將三通道的 RGB 彩色影像轉換為單通道的灰階影像。人類視覺對綠色最敏感、對藍色最不敏感，因此業界標準採用加權亮度公式（ITU-R BT.601 標準）：

$$Gray = 0.299 \times R + 0.587 \times G + 0.114 \times B$$

這個公式確保轉換後的灰階值能真實反映人眼對亮度的感知。例如，純綠色 (0, 255, 0) 的灰階值約為 150，而純藍色 (0, 0, 255) 僅約 29——這與我們的直覺一致：綠色看起來比藍色「亮」得多。

// 灰階轉換公式實作
function toGrayscale(r, g, b) {
  return Math.round(r * 0.299 + g * 0.587 + b * 0.114);
}

門檻判定：非黑即白的數學規則

取得灰階值後，二值化的核心邏輯極其簡單——將灰階值與一個預設的門檻值（Threshold）進行比較。若灰階值大於門檻值，則設為白色（255），代表背景；若灰階值小於或等於門檻值，則設為黑色（0），代表目標特徵。

// 二值化核心邏輯
for (let i = 0; i < pixels.length; i += 4) {
  const gray = toGrayscale(pixels[i], pixels[i+1], pixels[i+2]);
  const binary = (gray > threshold) ? 255 : 0;
  pixels[i] = pixels[i+1] = pixels[i+2] = binary;
}
注意 在循跡應用中，黑色賽道線在白色背景上的灰階值較低。因此「目標像素」通常是灰階值低於門檻的黑色像素。門檻值的選擇直接決定了特徵提取的品質。

門檻值的選擇策略

門檻值沒有「絕對正確」的數字，必須根據實際環境條件進行調校：

門檻值效果描述適用情境
50（低）只有極暗的區域被標為目標，特徵少但純淨強光環境、高對比賽道
128（中）標準的明暗分界，適合一般室內環境均勻光照的室內賽道
200（高）大量區域被標為目標，雜訊干擾可能增多昏暗環境、低對比材料
警告 環境光源的變化是二值化最大的敵人。同一條賽道在早上與下午的最佳門檻值可能相差 50 以上。在正式比賽前，務必在實際場地光照條件下重新校準門檻值。
提示 在開發階段，建議使用 HTML <input type="range"> 滑桿來即時調整門檻值，並在 Canvas 上同步顯示二值化結果。這能讓你直觀地找到最佳門檻區間。

影像濾波 (Filtering)：消除噪點技術

無論攝影機感測器的品質多高，擷取到的影像中總是存在不可避免的噪點（Noise）。這些噪點可能來自電路的熱噪聲、光源的閃爍、或是感測器本身的量化誤差。如果直接對含噪影像進行二值化，噪點會被放大成大量的「假目標」像素，嚴重干擾後續的質心計算。影像濾波的目的，就是在保留重要特徵的前提下，盡可能消除這些隨機干擾。

卷積運算的核心概念

影像濾波的數學本質是卷積運算（Convolution）。卷積的過程是：取一個小型的矩陣（稱為卷積核 / Kernel），在影像上逐像素滑動。每到一個位置，就將 Kernel 覆蓋的區域內所有像素值進行加權計算，得出中心像素的新值。以 $3 \times 3$ 的 Kernel 為例，每次計算會參考中心像素周圍的 9 個鄰居。

注意 卷積核的大小決定了濾波的「力道」。$3 \times 3$ 適合輕度降噪；$5 \times 5$ 降噪更強但會模糊邊緣。在循跡應用中，通常使用 $3 \times 3$ 以兼顧速度與品質。

均值濾波 vs. 中值濾波

兩種最常見的濾波方法各有其適用場景：

特性均值濾波（Mean Filter）中值濾波（Median Filter）
計算方式取 $3 \times 3$ 區域所有像素的算術平均取 $3 \times 3$ 區域所有像素的排序中位數
效果整體平滑，類似「磨皮」移除極端值，保留邊緣
對椒鹽雜訊效果有限，噪點會被「平均進去」效果極佳，噪點被直接排除
邊緣保留較差，邊緣會被模糊較好，邊緣輪廓清晰
計算成本低（純加法與除法）較高（需要排序操作）

均值濾波實作

// 3x3 均值濾波：取周圍 9 個像素的平均值
function meanFilter(imageData, x, y, width) {
  let sum = 0, count = 0;
  for (let dy = -1; dy <= 1; dy++) {
    for (let dx = -1; dx <= 1; dx++) {
      sum += getPixel(imageData, x + dx, y + dy, width);
      count++;
    }
  }
  return Math.round(sum / count);
}

中值濾波實作

// 3x3 中值濾波：取周圍 9 個像素的中位數
function medianFilter(imageData, x, y, width) {
  const neighbors = [];
  for (let dy = -1; dy <= 1; dy++)
    for (let dx = -1; dx <= 1; dx++)
      neighbors.push(getPixel(imageData, x+dx, y+dy, width));
  neighbors.sort((a, b) => a - b);
  return neighbors[4]; // 9 個值排序後取第 5 個（中位數）
}
提示 在遍歷邊緣像素時，$3 \times 3$ Kernel 會「超出」影像邊界。常見的處理方式是跳過邊界像素（從 x=1 開始到 x=width-2），或將超出邊界的值視為 0。
警告 濾波本質上是一種「資訊損失」操作。過度濾波會讓賽道線的邊緣變得模糊，導致質心座標產生系統性偏移。建議先濾波、再二值化，並透過視覺化對比確認濾波強度的合理性。

侵蝕與擴張：形態學優化

即使經過濾波與二值化處理，影像中仍然可能存在兩類缺陷：殘餘噪點（孤立的白色或黑色小斑點）與特徵斷裂（原本連續的線條出現間斷）。形態學運算（Morphological Operations）是一組基於集合論的影像處理技術，專門用來解決這兩個問題。在循跡應用中，它是「原圖 → 濾波 → 二值化 → 形態學」標準流程的最後一環。

侵蝕（Erosion）：去除噪點

侵蝕的原理是：使用一個結構元素（通常是 $3 \times 3$ 的全白方塊）在二值化影像上滑動。只有當結構元素覆蓋區域內的所有像素都是目標（黑色）時，中心像素才保持為目標；否則，中心像素被設為背景（白色）。這個操作會讓所有目標區域的邊緣向內「收縮」，而孤立的小噪點因為面積不足以支撐結構元素，會被完全消除。

// 侵蝕操作：只保留完全被目標包圍的像素
function erode(binaryImage, width, height) {
  const result = new Uint8Array(binaryImage.length);
  for (let y = 1; y < height - 1; y++)
    for (let x = 1; x < width - 1; x++) {
      let allBlack = true;
      for (let dy = -1; dy <= 1; dy++)
        for (let dx = -1; dx <= 1; dx++)
          if (binaryImage[(y+dy)*width+(x+dx)] !== 0) allBlack = false;
      result[y * width + x] = allBlack ? 0 : 255;
    }
  return result;
}

擴張（Dilation）：填補斷裂

擴張是侵蝕的逆操作。只要結構元素覆蓋區域內有至少一個目標像素，中心像素就會被設為目標。這會讓目標區域的邊緣向外「膨脹」，將斷開的線段重新連接起來。

開運算與閉運算：組合技

實務中，侵蝕與擴張經常組合使用，形成兩種標準操作：

操作定義效果適用場景
開運算（Opening）先侵蝕，再擴張去除小噪點，同時恢復主體大小清除地面反光造成的散碎雜點
閉運算（Closing）先擴張，再侵蝕填補小孔洞與裂縫，同時不過度膨脹修復賽道線的微小斷裂
提示 在循跡場景中，推薦的標準處理流程為：原圖 → 中值濾波 → 二值化 → 開運算（去噪點）→ 閉運算（補裂縫）。這套流程能產出乾淨且連續的特徵遮罩，為質心計算提供可靠的輸入數據。
警告 過度使用形態學運算會改變目標特徵的幾何形狀。多次侵蝕會讓細線完全消失；多次擴張則會讓兩條相鄰的線條合併成一塊。每一步操作都應透過 Canvas 視覺化驗證效果後再決定參數。

3. DIP 工程導師點評標準 (DIP Engineering Benchmarks)

此單元的價值在於 「數據降維的準確性與程式效能」：

  • [ ] 特徵提取純淨度：二值化邊緣是否清晰？有沒有大量的散碎雜點？
  • [ ] 去噪與銳利的權衡：濾波器是否在去除雜訊的同時，過度模糊了賽道線的邊緣導致中心點偏移？
  • [ ] 運算效率意識：在 $3 \times 3$ 像素遍歷中，是否有加入邊界檢查防止 Array Index Out of Bounds 的崩潰？
