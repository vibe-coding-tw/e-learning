掌控事件主導權

event.preventDefault()

這是告訴瀏覽器：「這個動作由我的程式處理，你不需要執行預設反應」。在觸控操作中，這至關重要。

Passive Event Listeners (被動監聽器)

現代瀏覽器預設將 window, document, body 的 touch 事件視為 passive: true (被動)，這意味著你無法在其中呼叫 preventDefault() 來阻止捲動，除非你明確設定 { passive: false }。

JavaScript 語法 
// 阻止 Context Menu (右鍵選單)
element.addEventListener('contextmenu', (e) => {
    e.preventDefault();
    console.log('Right click blocked!');
});

// 阻止全頁面捲動 (需設定 passive: false)
document.addEventListener('touchmove', (e) => {
    e.preventDefault();
}, { passive: false });

CSS 靜態鎖定技術

使用 CSS 屬性是最直接、效能最好的方式來告訴瀏覽器某些元素不應該被選取或觸發手勢。

user-select: none;

防止使用者長按按鈕時，不小心選取到了按鈕上的文字或圖示。

touch-action: none;

最強力的屬性。告訴瀏覽器此元素不處理任何預設手勢（如平移、縮放）。瀏覽器會直接將所有 touch 事件交給 JS 處理。

-webkit-touch-callout: none;

針對 iOS Safari，防止長按圖片或連結時跳出預覽或選單。

🚀 任務指南：防止瀏覽器預設行為 (Prevent Default)

本單元的目標是學習如何將網頁轉化為具有「原生 App」觸感的專業介面。透過處理瀏覽器的預設行為，你將能打造出穩定、流暢且不被打斷的使用者體驗。

🎯 任務目標

  1. 鎖住全螢幕捲動：確保頁面在操作時不會意外滑動或產生橡皮筋回彈。
  2. 封印系統選單與文字選取：防止長按彈出選單，並鎖定 UI 文字不被反白。
  3. 消除點擊延遲：提升按鈕的反應速度，達到即時的操作感。

2. 作業任務部署細節

任務 1：鎖住全螢幕捲動與橡皮筋回彈 (Viewport Lockdown Lab)

  • 目標：打造「絕對靜止」的操作平面。掌握 preventDefault() 與 { passive: false } 的底層配置邏輯。
  • Classroom 部署建議：
    • 滾動攔截實作：在 document 層級監聽 touchmove，並有條件地執行 preventDefault()。
    • CSS 配合挑戰：對 body 或控制區加入 overscroll-behavior: none 以封印 iOS 常見的「底部回彈」效果。
    • 驗證要點：操作穩定性審核。學員是否理解了被動監聽器 (Passive Listener) 的限制？在操作區猛點、猛滑時，頁面是否依然穩如泰山？

任務 2：打造像原生 App 的手勢鎖定區 (Native-Like Surface Lab)

  • 目標：封印所有系統干擾。學會處理 iOS 與 Android 在長按、選取與選單觸發上的差異。
  • Classroom 部署建議：
    • 系統選單攔截實作：加入 contextmenu 攔截與 -webkit-touch-callout: none，防止遙控到一半跳出「儲存圖片」或「選取文字」的對話框。
    • 點擊延遲優化：應用 touch-action: manipulation 或 none 來消除瀏覽器預設的 300ms 雙擊等待時間，提升操控即時感。
    • 驗證要點：跨平台相容性審核。學員是否考慮了 iOS 專有的 CSS 屬性？按鈕上的文字是否已設定為無法選取？

任務 3：緊急操作壓力測試 (Operational Stress Test Lab)

  • 目標：驗證極端環境下的可靠性。確保在亂序、快速的多指操作下（模擬機器人驚險競速），介面不會位移或崩潰。
  • Classroom 部署建議：
    • 壓力操作模擬：要求學員在手機上同時使用三根手指進行暴力滑動，觀測是否有任何「露底」或頁面位移的情況。
    • 介面完整性評估：確認頁面佈局是否因為鎖定而犧牲了可讀性，並在 README 中寫下對「攔截」與「讓渡」權限的平衡心得。
    • 驗證要點：是否具備了處理邊界狀況的能力？提交的報告中是否有明確的對比數據或截圖？
