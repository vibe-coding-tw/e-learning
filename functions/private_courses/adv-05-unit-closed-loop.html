視覺閉迴路控制理論

🔗 1. 閉迴路控制的核心架構 (25 min)

開迴路 (Open-Loop)

只給指令（如：前進 5 秒），不看結果。容易受地面摩擦力、電池電量影響而偏離。

閉迴路 (Closed-Loop)

不斷觀察結果（影像位置），並根據結果調整指令，具備自動補償能力。

🏗️ 控制鏈元件 (Control Chain)

  • 控制器 (Controller)：網頁中的控制邏輯 (P-Control)。
  • 執行器 (Actuator)：ESP32 與馬達。
  • 感測器 (Sensor)：攝影機 (Camera)。
  • 回饋 (Feedback)：即時計算出的質心誤差。

⏱️ 2. 系統延遲 (Latency) 的影響 (20 min)

視覺延遲：影像傳輸、解析 JSON、計算質心都需要時間。

連鎖後果

如果延遲太高，車子修正的是「過去的誤差」，這會導致嚴重的震盪甚至失控（Over-correction）。就像在延遲嚴重的視訊通話中，兩個人總是同時說話又同時停下。

  • ✓ 提升藍牙傳輸 MTU (增加吞吐量)
  • ✓ 使用跳行掃描演算法 (減少運算量)

🎯 3. 設定點 (Set-point) 與偏差 (Deviation) (15 min)

  • ● 設定點 (Set-point)：我們希望物體出現在影像中的目標位置（通常是中心 $x = 160$）。
  • ● 偏差 (Deviation)：當前偵測位置與設定點之間的動態距離。
  • 💡 預習任務：為什麼閉迴路系統可以「自動容忍」馬達左右出力不均的問題？

PD 控制的物理意義

Kp (比例) 決定目前的轉向強度。Kp 越大，修正越果斷。
Kd (微分) 預測誤差變化趨勢，抑制過度震盪（就像是阻尼）。
線上任務：觀察「追蹤震盪」案例影片，分析為何 $K_p$ 過大會導致車輛在線上蛇行。

視覺閉迴路實驗室：自動化跟隨與穩定性調校

任務 1：視覺追蹤閉迴路整合實作 (Vision Feedback Integration)

角色扮演：控制系統工程師 | 難度：⭐️⭐️⭐️ | 估計時長：75 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 閉環邏輯部署：將影像質心提取出的 $Error$ 為輸入，實作 P 控制器輸出轉向脈衝。
  3. 連線同步：確保 Web Bluetooth 的發送頻率與影像處理幀率 (FPS) 達成同步匹配，避免指令堆疊延遲。
  4. Push Final：提交具備「自主追隨動態目標」能力的閉迴路控制系統。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 追蹤收斂度：物體緩慢移動時，系統是否能穩定將其維持在畫面中央（誤差趨於零）？
  • [ ] 指令原子性：發送的控制 Payload 是否簡潔且具備防止重複發送的防抖處理？

任務 2：階躍響應 (Step Response) 紀錄實驗

角色扮演：控制性能分析師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 注入擾動：在穩定狀態下，手動撥動目標物產生一個瞬間的「位置階躍（Step Check）」。
  2. 響應紀錄：觀察並口頭/書面描述系統找回設定點所需的「上升時間 (Rising Time)」與是否存在「過衝 (Overshoot)」。
  3. 參數對照：嘗試增加 $K_p$ 觀察震盪是否加劇，並將實驗觀察記錄於 README 的性能日誌中。
  4. Push Final：提交包含「性能調校路徑」的實驗報告與代碼。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 震盪識別能力：學員是否能正確分辨「過阻尼」與「欠阻尼」在車輛追隨過程中的視覺特徵？
  • [ ] 調校邏輯性：記錄中的參數改動是否遵循「先靈敏後平穩」的歸類演繹邏輯？

任務 3：斷點與恢復：Failsafe 機制測試

角色扮演：系統可靠性架構師 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 遮擋模擬：手動遮擋攝影機，觀察閉迴路在失去 Feedback 的瞬間，「執行器」的預設反應。
  2. Failsafe 實作：開發「超時鎖定」邏輯，若連續 $N$ 幀未偵測到質心，自動發送 {stop: true}。
  3. 恢復測試：當目標重新出現時，系統應能立即在無需刷新的情況下重新啟動閉環控制。
  4. Push Final：提交具備「環境異常容錯」能力的高安全性控制模組。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 失效模式處理：失去目標時車輛是否能立即停止，而非維持最後一個指令盲衝？
  • [ ] 重連無感度：目標恢復後的追蹤邏輯是否流暢，有無資料緩衝區殘留導致的「瞬移」錯覺？

GitHub Classroom 詳細操作指南

本單元教學重點：掌握「回饋」的力量。理解視覺誤差如何轉化為物理動作，並初步探索穩定性問題。

  1. 重點 1：控制鏈的完整性 - 讓學生體會「數據不斷循環」的概念，只要其中一節（如藍牙斷線）斷掉，閉迴路就會失效。
  2. 重點 2：延遲是閉迴路的天敵 - 現場降低瀏覽器 FPS 或提高影像解析度，讓學生親眼看見「反應過慢導致的劇烈蛇行」。
  3. 重點 3：Failsafe 的哲學 - 強調「失去回饋時必須停止」是所有自動駕駛與機器人系統最基本的道德義務。
