😈 1. 為什麼 delay() 是邪惡的？

執行 delay(1000) 時，處理器全停。像等紅燈閉眼，無視緊急狀況。

The Impact

  • 指令無視：CPU 忙著 delay，你的按鈕指令被丟棄。
  • 連線中斷：超時導致反應遲鈍甚至斷線。

Single Thread Problem

"Doing Absolutely Nothing but Waiting."

⏱️ 2. millis() 碼錶邏輯

不要叫電腦「等待」。要它「檢查時間差是否足夠」。

Non-blocking Formula

\[now - prev \ge Interval\]

Variable: unsigned long

MILLIS PATTERN (C++) 
unsigned long last = 0;
void loop() {
  unsigned long n = millis();
  if (n - last >= 1000) {
    last = n; // Task!
  }
  // Free to handle other tasks...
}
