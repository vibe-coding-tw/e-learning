Module 2: Core Principles

無人車控制面板的構造 (HTML Structure)

HTML5 就像房子的鋼骨結構。為了讓機器人（無人車）能被理解，我們使用具有意義的標籤。

<> <header> 標題區

放置 Logo、連線狀態指示燈。告訴使用者「我在控制哪一台車」。

<> <main> 核心操作區

<video> ESP32-CAM 即時串流影像
<button> 前進、後退、轉彎控制
<input type="range"> 速度控制拉桿

<> <footer> 狀態資訊

顯示 IP 位址、版權宣告、除錯訊息。

行動優先 (Mobile First) 的關鍵

為什麼手機網頁字這麼小？

預設情況下，手機會假裝自己是 980px 寬的電腦螢幕，然後縮小顯示。這對操控遙控車是致命的。

<head>
<meta name="viewport"
content="width=device-width, initial-scale=1.0">
</head>
  • ✔ width=device-width: 強制網頁寬度等於手機螢幕寬度 (1:1)。
  • ✔ initial-scale=1.0: 初始縮放比例為 100%。

Web App 開發環境與 HTML5 基礎 (Web App Dev & HTML5 Basics)

歡迎來到 Start 01 Unit: HTML5 Basics！ 在這個單元中，我們將跨出全棧開發的第一步。你將學習現代 Web 工程化的思維，架設即時預覽開發環境，並掌握 HTML5 的語義化結構與「行動優先 (Mobile First)」的開發流程。這將為後續實作遙控介面打下堅實的基礎。

🎯 作業目標 (Assignment Objectives)

本作業分為三個主要任務，幫助你熟悉 Web 工具鏈並掌控行動端細節：

任務 1：實機部署與跨裝置連線

  • 環境架設：使用 VS Code Live Server 啟動區域網路伺服器。
  • 跨裝置預覽：找出電腦的內網 IP，讓手機透過瀏覽器成功造訪你正在開發的網頁。

任務 2：觸控友善性優化

  • 限制不必要的互動：在 CSS 中加入相關屬性，防止使用者在手機上因為連續點擊而觸發雙擊縮放，或是誤選文字（例如：user-select: none 與 touch-action: manipulation）。
  • 優化按鈕熱區：確保按鈕的「點擊邊界」夠大，適合大拇指操作。

任務 3：FPV 影像容器佈局

  • 實作滿版容器：利用 100vw 與 100vh 建立一個自適應背景容器。
  • 影像縮放與裁切：使用 object-fit: cover 確保影像充滿螢幕且不變形。
  • 響應式驗證：在手機上旋轉螢幕，確認影像在直向與橫向皆能正確顯示。

以下是在 GitHub Classroom 部署其作業 (Assignments) 的具體建議：

1. 範本程式庫 配置建議

HTML5 基礎作業的核心在於「開發環境與觸控優化」，範本應包含基礎結構與測試資源，建議包含：

  • 📂 index.html：標竿 HTML5 模板。預設包含 viewport meta 標籤，讓學員在「行動優先」的預設環境下開始開發。
  • 📂 style.css：樣式基礎區。學員在此練習觸控優化 CSS（如 touch-action），並實作 FPV 影像容器的 object-fit 屬性。
  • 📂 assets/：測試資產。提供一張預置的 esp32-stream.jpg 測試圖片，模擬無人車的回傳影像，讓學員在沒有硬體的情況下也能練習佈局。
  • 📂 README.md：實機測試日誌。要求學員貼上手機成功連入 Live Server 的截圖，並簡述他們在測試「點擊延遲」時的觀察。

2. 作業任務部署細節

任務 1：實機部署與跨裝置連線 (Real-device Deployment Lab)

  • 目標：建立「電腦開發、手機同步」的高效工作流，理解區域網路連線與防火牆的基本排錯。
  • Classroom 部署建議：
    • 環境架設實作：使用 VS Code Live Server 啟動服務。
    • 跨裝置預覽：找出電腦內網 IP，讓手機透過瀏覽器成功造訪。
    • 驗證要點：關鍵部署審核。學員必須在 README.md 中提交手機預覽畫面截圖。確認學員能區分 localhost 與 內網 IP 的差異。

任務 2：觸控友善性優化 (Touch Friendliness Lab)

  • 目標：解決移動端網頁常見的「雙擊縮放」與「文字誤選」問題，提升遙控按鈕的交互品質。
  • Classroom 部署建議：
    • 交互限制實作：在 CSS 中補上 user-select: none（防止拖曳按鈕變選取文字）與 touch-action: manipulation（消除點擊延遲）。
    • 熱區擴大練習：確保按鈕的「點擊邊界」適合大拇指操作（不小於 48x48 像素）。
    • 驗證要點：在手機上連續點擊按鈕，是否會產生不必要的縮放或藍色選取區域？

任務 3：FPV 影像容器佈局 (FPV Container Layout Lab)

  • 目標：掌握數位影像的比例縮放邏輯，確保遙控車傳回的影像在任何螢幕上都不會變形。
  • Classroom 部署建議：
    • 滿版容器實作：建立一個 width: 100vw; height: 100vh; 的背景容器。
    • 比例裁切練習：運用 object-fit: cover 或 contain 處理影像。
    • 響應式驗證：要求學員切換手機「橫向」與「直向」，觀察影像如何自適應。
    • 驗證要點：圖像是否有被錯誤拉伸？容器在載入圖片前是否維持穩定高度（防止跳版）？

3. 工程素養與行動開發點評標準 (Mobile Development Standards)

此單元的價值在於 「對 Web 工具鏈的初步掌控與對移動端細節的感知力」：

  • [ ] 部署穩定性：是否能自主解決手機與電腦間的連線問題？
  • [ ] CSS 精細度：是否使用了正確的 CSS 屬性解決了觸控體驗中的痛點？
  • [ ] 響應式思維：是否理解了 viewport 在這類遙控應用中「 1:1 縮放」的重要性？
