簡介

在現代 Vibe Coding 開發模式中，AI 能幫你生成大量的 HTML、CSS 與 JavaScript 代碼， 但要讓這些代碼真正在「手機瀏覽器」上流暢地控制無人車或物聯網設備， 你需要首先建立一套高效的「本地開發工作流」。

情境
小明寫好了一個遙控介面，但每次改了一行 CSS，都要手動按 F5 重新整理瀏覽器， 而且在手機上測試時，按鈕因為「雙擊縮放」而無法快速連續點擊。 老師說：「你需要的是 Live Server 的熱重載，以及 touch-action: manipulation。」

本模組的核心目標就是解決這兩個問題。你將學習如何讓電腦成為一台區域網路伺服器， 讓手機直接訪問正在開發中的網頁，並且在存檔的瞬間就能看到即時更新。

直觀實驗室：所見即所得的開發體驗

傳統的網頁開發需要手動重新整理瀏覽器，而現代工作流使用 Live Server 建立即時同步通道。 更重要的是，你可以讓手機直接存取電腦上正在運行的開發伺服器，實現真正的跨裝置同步預覽。

架設步驟

1

安裝 Live Server 擴充套件

在 VS Code 中按 Ctrl+Shift+X，搜尋 Live Server（由 Ritwick Dey 發行），點擊安裝。

2

啟動本地伺服器

在 index.html 上按右鍵，選擇 「Open with Live Server」。瀏覽器會自動在 http://localhost:5500 開啟你的網頁。

3

找出電腦的內網 IP

Windows：在命令提示字元輸入 ipconfig；macOS：輸入 ifconfig | grep inet，找到類似 192.168.x.x 的位址。

4

手機連入開發伺服器

確保手機和電腦在同一個 WiFi 網路。在手機瀏覽器輸入 http://[電腦IP]:5500，即可看到你的網頁並即時同步更新。

防火牆注意事項 如果手機無法連入，很可能是電腦的防火牆阻擋了 Port 5500 的連線。請前往系統設定，暫時允許 VS Code / Node.js 通過防火牆，或改用手機熱點讓電腦連線。

Live Server 的兩大核心功能

本地伺服器
模擬真實網站環境

讓你的電腦變成小型伺服器，HTML 以正確的 HTTP 協議運行，避免直接開啟檔案時的各種限制（如 CORS 問題）。

熱重載
存檔即更新

每次按下 Ctrl+S 存檔，所有連線中的瀏覽器（包含手機）立即自動重新整理，無需手動按 F5。

HTML5 骨架與行動優先設計

HTML5 就像無人車控制面板的鋼骨結構。為了讓程式碼可讀、可維護， 我們使用具有語義意義的標籤，而不是一堆無意義的 <div>。

語義化骨架（控制面板比喻）

為什麼要語義化？ 搜尋引擎、螢幕閱讀器、以及你未來的隊友都會感謝你使用正確的標籤。 <header> 比 <div class="header"> 更清楚地傳達意圖。
HTML5 標籤 無人車控制面板對應功能 常見內容
<header> 頂部導航與狀態列 Logo、連線狀態燈、車輛 ID
<main> 核心操控區域 FPV 影像、方向鈕、速度拉桿
<section> 功能分區 感測器讀值區、相機控制區
<footer> 狀態資訊列 IP 位址、電池電量、版本號

行動優先（Mobile First）：最關鍵的一行 HTML

預設情況下，手機會假裝自己是 980px 寬的桌面螢幕，然後整頁縮小顯示。 對於遙控介面來說，這是災難性的 — 按鈕變得極小無法點擊，FPV 影像也會扭曲。

<head>
  <meta charset="UTF-8">
  <meta name="viewport"
       content="width=device-width, initial-scale=1.0">   ← 這行很關鍵！
</head>
  • width=device-width：強制網頁寬度等於手機螢幕實際寬度，1:1 完整呈現。
  • initial-scale=1.0：初始縮放比例 100%，禁止瀏覽器自動縮小。

觸控友善性優化

遙控介面的按鈕需要能快速、連續地被觸控，避免以下常見問題：

問題：雙擊縮放
iOS 和 Android 的預設行為：快速雙擊會放大頁面，導致方向鈕點擊後介面突然縮放失控。
解法：touch-action
touch-action: manipulation — 允許基本觸控滾動，但禁止雙擊縮放。是遙控介面的標準配備。
問題：長按選取文字
長按按鈕時，瀏覽器會出現文字選取的藍色框，干擾控制操作，讓使用者無法專注於駕駛。
解法：user-select
user-select: none — 禁止所有文字選取行為，讓控制介面不受干擾，操作更流暢直接。

觸控友善性優化

目標：解決遙控介面上的手機觸控痛點，包含雙擊縮放、文字誤選與點擊熱區不足。

  • 在 CSS 中加入 touch-action: manipulation 消除點擊延遲

  • 確認所有按鈕的點擊熱區不小於 48×48 像素
  • 在手機上連續點擊，確認不再出現雙擊縮放或文字選取
測試技巧 用兩根手指快速交替點擊同一個按鈕，若頁面不縮放，代表 touch-action 已正確生效！

🎯 作業目標 (Assignment Objectives)

本作業分為三個主要任務，幫助你熟悉 Web 工具鏈並掌控行動端細節：

任務 2：觸控友善性優化

  • 限制不必要的互動：在 CSS 中加入相關屬性，防止使用者在手機上因為連續點擊而觸發雙擊縮放，或是誤選文字（例如：user-select: none 與 touch-action: manipulation）。

任務 3：FPV 影像容器佈局

  • 實作滿版容器：利用 100vw 與 100vh 建立一個自適應背景容器。
  • 影像縮放與裁切：使用 object-fit: cover 確保影像充滿螢幕且不變形。
  • 響應式驗證：在手機上旋轉螢幕，確認影像在直向與橫向皆能正確顯示。
