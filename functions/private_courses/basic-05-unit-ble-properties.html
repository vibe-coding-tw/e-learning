基礎 5 - 藍牙通訊進階

BLE 核心機制：
讀取、寫入與通知

掌握 GATT 特徵值屬性，設計高效、低延遲且節能的工業級通訊地圖。

📖 1. 讀取 (Read)：靜態資源查詢

定義：客戶端（手機）主動發起要求，索取伺服器（ESP32）中的數據。就像是在查閱一本靜態字典。

Pull Mechanism

只有在手機感興趣「詢問」時，ESP32 才會回傳數據。這是最節能但也最不具實時性的通訊方式。

適合內容

韌體版本、設備 ID、校準參數

頻率建議

按需發起 (On Demand)

⚙️ Firmware Setup 

pChar->setValue("1.0.0");
// Device waits for
// Remote client to read.

✍️ 2. 寫入 (Write)：主動下令控制

手機主動推送資料給 ESP32。這是設計無人車遙控指令與參數調節的核心。

🛡️ With Response (Safety)

安全握手模式。ESP32 收到後會回傳 ACK。保證指令 100% 傳達，適合作為關鍵參數修改。

⚡ No Response (Command)

戰鬥高速模式。手機只管送，不等待確認。適合作為高頻率搖桿座標傳輸。

ON_WRITE CALLBACK 
void onWrite(BLECharacteristic *pC) {
  std::string val = pC->getValue();
  if (val == "GO") {
    // Trigger Motor logic immediately
    drive(255);
  }
}

🔔 3. 通知 (Notify)：即時數據推播

訂閱機制：手機先「訂閱」特定特徵值。之後只要 ESP32 數據一有變動，車子就會立刻主動推送到手機螢幕上。

即時監控場景

  • 超音波距離實時跳幅
  • 陀螺儀角度動態偏轉
  • 低電量主動警報
  • 循跡誤差值校正回傳

Efficiency King

"Notify 正確設計後，手機就不再需要輪詢 (Polling)，電力消耗將大幅下降。"

pCharacteristic->notify();

核心任務：通訊協議大師

建立高效能的資通訊架構，解鎖感測器數據流與精確的邊緣控制。

📊 45 min

【作業 5-1】

「多重權限特徵值佈署」

在同個 Service 下建立學號 (Read)、LED (Write) 與秒計數 (Notify)。實證手機端能在不主動請求下獲得秒值更新。

🏎️ 60 min

【作業 5-2】

「高頻指令響應測試」

每秒發送 50 次指令，實測不同寫入 Property 下的數據堆疊。分析搖桿控制與參數設定的最佳通訊選配策略。

💡 60 min

【作業 5-3】

「變動閾值過濾演算法」

實作「變動超過 10%」才推播感測值的智慧邏輯。量化數據平靜期與劇烈變動期的封包量差，達成省電通訊。

  1. 壓力環境建立：編寫測試，使手機每秒送出 50 筆馬達動力座標。

  3. 撰寫心得報告：在 README 中彙整實驗數據，說明「回應 (ACK)」在低延遲需求場景中的影響。

任務 3：「智慧推播」：條件式 Notify 優化 (Smart Notify)

角色扮演：低功耗系統設計師 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 閾值過濾實作：封裝一個 Notify 觸發器。只有當 ADC 感測值變動超過 10% 時才調用 pCharacter->notify()。
  2. 封包監測：觀察在「傳感器靜置」與「劇烈搖動」兩種狀態下，通訊指示燈與序列埠的跳動頻率差異。
  3. 優化效益分析：在 README 中推導此機制對於延長電池壽命、減少頻寬占用的預期貢獻。

GitHub Classroom 詳細操作指南

本單元教學重點：從單純的「可通訊」晉升為「高效通訊」的 GATT 屬性配置。

  1. 重點 1：適才適所的 Property - 引導學員根據數據類型 (靜態 vs 動態) 選擇 Read 或 Notify。
  2. 重點 2：延遲與可靠性的權衡 - 在任務 2 中，務必讓學員親手感受到 ACK 握手在高頻控制下的效能瓶頸。
  3. 重點 3：變動觸發的智慧 - 這是物聯網 (IoT) 低功耗設計的核心：只傳遞「有意義」的數據。
