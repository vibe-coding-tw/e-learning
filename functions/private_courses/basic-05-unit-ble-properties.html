基礎 5 - 藍牙通訊進階

BLE 核心機制：
讀取、寫入與通知

掌握 GATT 特徵值屬性。設計高效、低延遲且節能的工業級通訊。

📖 1. 讀取 (Read)：查詢

定義：客戶端主動索取數據。

Pull Mechanism

手機詢問時才回傳。節能但實時性低。

適合內容

韌體版本、設備 ID

頻率建議

按需發起

⚙️ Firmware Setup 

pChar->setValue("1.0.0");
// Device waits for
// Remote client to read.

✍️ 2. 寫入 (Write)：下令

手機推送資料。設計遙控指令的核心。

🛡️ With Response (Safety)

安全握手。收到後回傳 ACK。保證指令傳達。

⚡ No Response (Command)

戰鬥高速。只管送，不等待。適合搖桿座標。

ON_WRITE CALLBACK 
void onWrite(BLECharacteristic *pC) {
  std::string val = pC->getValue();
  if (val == "GO") {
    // Trigger Motor logic immediately
    drive(255);
  }
}

🔔 3. 通知 (Notify)：推播

訂閱機制：手機訂閱。數據變動即主動推播。

即時監控

  • 超音波距離
  • 陀螺儀角度
  • 主動警報
  • 循跡誤差值

Efficiency King

"手機不再需要輪詢，電力消耗將大幅下降。"

pCharacteristic->notify();
