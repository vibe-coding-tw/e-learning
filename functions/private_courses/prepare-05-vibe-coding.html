Antigravity：指揮多位 Agent 協作

在 Antigravity 中，你可以同時指揮多個 Agent 分工合作。Agent A 負責重構後端，Agent B 負責設計前端，你在統一的任務指揮中心進行批次核准。

核心概念

  • 任務導向架構 (Task Boundary)：每個 Agent 有明確的職責範圍，避免檔案衝突。
  • rules.md — 協作 DNA：定義團隊規範，所有 Agent 在工作時必須遵循。
  • 批次核准 (Batch Approve)：在統一視窗中一次審核所有 Agent 的修改。

rules.md 範例

# .antigravity/rules.md @Architect { role: "Planning" } @Coder { role: "Implementation" } # 所有 API 必須定義 TS Interface # 修改前必須讀取當前 Git 狀態
關鍵技巧 明確的 rules.md 越具體，多 Agent 協作的效果越好。模糊的規則會讓 Agent 產生衝突。

部署策略與 CI/CD 最佳實踐

為什麼 Agent 無法直接部署？

部署涉及基礎設施權限與敏感密鑰 (API Keys)。為確保安全，Agent 運行在隔離環境，無權直接存取你的正式伺服器。這是一個刻意的安全設計。

安全原則 永遠不要把生產環境的 API Key 或資料庫密碼直接貼給 Agent。所有部署應透過 CI/CD 流水線自動完成。

最佳實踐：CI/CD 部署

讓 Agent 撰寫 GitHub Actions YAML 腳本。你審核並合併 PR 後，由自動化流水線完成部署。這才是最專業的流程。
