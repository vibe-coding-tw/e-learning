像機器一樣看見：
特徵提取的藝術

影像不只是像素。學習如何從雜亂的畫面中提取關鍵資訊，讓無人車理解世界。

🔍 1. 什麼是特徵？ (20 min)

特徵是影像中具有「代表性」且「不隨旋轉、縮放、光照輕易改變」的資訊。

🟢 低階特徵

顏色、紋理、邊緣（Edges）。這些是構成影像的最基本元素。

🟡 中階特徵

形狀（如圓形、方形）、孔洞。開始具備物件的雛形。

🔴 高階特徵

臉部器官、車牌、交通標誌。這與人類的認知概念直接掛鉤。

✨ 2. 邊緣偵測的魔術 (25 min)

數學原理

邊緣發生在像素值「劇烈變化」的地方。透過計算梯度 (Gradient)，我們可以勾勒出物體的輪廓。

  • ● Sobel 算子：理解水平與垂直方向的差分。
  • ● Canny 算子：目前最流行的邊緣偵測演算法，包含降噪、尋找最強梯度、排除雜訊。

🛠️ 學習任務

從邊緣到輪廓：理解如何將相連的邊緣點串聯成一個封閉的「輪廓 (Contour)」。

矩 (Moments)

🎯 預習任務：幾何判別

視覺特徵實驗室：從像素輪廓到幾何辨識

任務 1：「邊緣獵人」：Canny 參數動態調校 (Edge Hunter Lab)

角色扮演：視覺演算法工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。

  4. Push Final：提交具備「高度抗噪能力」的可調式邊緣提取模組。

  • [ ] 參數理解度：README 中是否記錄了兩段門檻值 (Hysteresis Thresholding) 對邊緣連續性的具體影響？

任務 2：「幾何篩選器」：形狀識別實戰 (Geometry Filter Lab)

角色扮演：自動化檢測工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 輪廓層級檢索：實作 findContours 邏輯，並對所有檢索到的輪廓進行循環遍歷。
  2. 幾何特徵運算：實作圓度公式 Roundness = (4π * Area) / (Perimeter^2)，並過濾出目標形狀。
  3. 視覺化標註：在畫面上為識別出的「圓形標誌」繪製專屬的綠色外框，實現對特定形狀的鎖定。
  4. Push Final：提交具備「精確幾何篩選」能力的視覺感知模組。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 異常處理：對於極小面積（雜訊）的輪廓，是否有設置適當的 Area 門檻進行初步剔除？

任務 3：特徵提取與決策連動 (Feature-Based Decision)

角色扮演：機器人系統架構師 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 特徵邏輯組合：結合面積比與長寬比特徵，定義一個「橫向長方形（停車線）」的特徵指紋。
  2. 行為觸發實作：當畫面中出現符合特徵的物件時，自動調用 BLE API 發送 {command: 'STOP'} 指令。
  3. 現場聯調：驗證無人車在行駛途中看見模擬停車線時，是否能產生即時且正確的制動反應。
  4. Push Final：提交具備「語義辨識與決策連動」能力的感知-動作系統。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 端到端整合度：是否成功打通了從「封裝像素輪廓」到「發送物理行為指令」的全鏈路開發？

GitHub Classroom 詳細操作指南

本單元教學重點：從像素到「語義」。掌握特徵提取的核心演算法，並學會將數學結果轉化為機器指令。

  1. 重點 1：邊緣的物理意義 - 引導學生理解邊緣不只是線，它是影像中「資訊密度最高」的地方，能大幅削減無用數據。
  2. 重點 2：特徵的不變性 - 強調為什麼要計算圓度而非單純面積，因為圓度在不同距離下依然穩定，這就是魯棒性 (Robustness)。
  3. 重點 3：感知與行動的耦合 - 透過 1-3 任務，讓學生看見演算法如何真實地干預物理世界的運作（停車）。
