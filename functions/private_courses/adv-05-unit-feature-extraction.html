什麼是特徵？

在電腦視覺中，特徵 (Feature) 是影像中具有「代表性」且「不隨旋轉、縮放、光照輕易改變」的資訊。 人類可以輕鬆地從一張照片中辨識出「那是一輛車」或「那是一個停車標誌」， 但對電腦而言，影像只是一個龐大的數值矩陣。特徵提取的目的，就是從這片數字海洋中，篩選出具有物理意義的關鍵資訊。

好的特徵必須具備不變性 (Invariance)——即便物體旋轉了角度、距離改變了遠近、光線明暗有所不同， 提取出的特徵仍然能夠穩定地描述同一個物體。這種穩定性，在控制理論中稱為魯棒性 (Robustness)， 是自動駕駛系統能否在真實世界中可靠運作的關鍵。

特徵的層級架構

電腦視覺中的特徵可依據其抽象程度，分為三個層級。隨著層級上升，特徵從純粹的數學描述逐步過渡到人類可理解的語義概念：

層級 名稱 範例 說明
低階 Low-level Features 顏色、紋理、邊緣 (Edges) 構成影像的最基本元素，可透過數學濾波器直接計算。
中階 Mid-level Features 形狀（圓形、矩形）、孔洞 由低階特徵組合而成，開始具備物件的雛形。
高階 High-level Features 臉部器官、車牌、交通標誌 與人類的認知概念直接掛鉤，需要多種中階特徵的邏輯組合。
備註 在本課程中，我們聚焦於低階與中階特徵的提取——特別是邊緣偵測 (Edge Detection) 與輪廓分析 (Contour Analysis)。 這些技術不需要 GPU 或深度學習模型，適合在 ESP32 等嵌入式裝置的有限資源下即時執行。

從像素到決策的處理流水線

一張 320×240 的灰階影像包含 76,800 個像素值。如果直接將這些原始數據餵給控制系統， 不僅運算量巨大，而且大多數像素是「背景雜訊」——它們不包含任何有用的導航資訊。 特徵提取的本質就是資訊壓縮：將數萬個像素縮減為少數幾個有物理意義的數值（如圓度、面積、質心座標）， 讓控制邏輯能夠高效且準確地做出決策。

1

擷取影像

攝影機捕獲一幀原始像素矩陣 (Raw Frame)，通常為 RGB 或 YUV 格式。

2

預處理 (Preprocessing)

灰階化、高斯模糊降噪、二值化 (Thresholding)，去除無關的顏色與高頻雜訊資訊。

3

特徵提取 (Feature Extraction)

透過邊緣偵測與輪廓分析，取得物體的形狀、位置、大小等幾何描述符 (Descriptor)。

4

決策與控制

根據特徵值（例如：圓度 > 0.85 判定為圓形標誌），觸發對應的車輛行為（停車、轉彎等）。

思考練習 如果要讓無人車辨識一張「信用卡」，你會提取哪些特徵？例如：固定的長寬比（約 1.586:1）、晶片的矩形輪廓、卡號區域的紋理特徵。 試著列出至少三種不同層級的特徵，並思考哪些特徵在不同光線條件下仍然穩定。

邊緣偵測的魔術

邊緣是影像中像素值發生劇烈變化的地方。從數學的角度來看，邊緣就是影像亮度函數的「梯度極值」所在。 人類視覺系統天生擅長識別邊緣——我們只需看到一條簡筆畫的輪廓線，就能立刻辨認出「那是一隻貓」。 邊緣偵測讓電腦也能做到這一點：從雜亂的像素背景中，提取出物體的「骨架」結構。

邊緣之所以重要，是因為它包含了影像中資訊密度最高的區域。 一張影像的大部分像素屬於均勻的背景（低梯度），真正有意義的結構資訊集中在物體的邊界上。 透過邊緣偵測，我們可以將原始影像的數據量壓縮 90% 以上，同時保留關鍵的形狀資訊。

梯度 (Gradient) 的數學基礎

影像梯度描述的是「像素亮度在空間上的變化速率」。對於灰階影像，梯度可以分別在水平 (X) 與垂直 (Y) 方向上計算：

  • 水平梯度 Gx：偵測垂直方向的邊緣（左右亮度差異）
  • 垂直梯度 Gy：偵測水平方向的邊緣（上下亮度差異）
  • 梯度幅值：G = √(Gx² + Gy²)，值越大代表邊緣越明顯

常見邊緣偵測演算法比較

演算法 原理 優點 缺點
Sobel 算子 3×3 卷積核計算 X、Y 方向的一階差分 計算快速，容易理解與實作 對雜訊敏感，邊緣較粗
Laplacian 算子 計算二階導數，偵測亮度突變 方向無關性，同時偵測所有方向邊緣 極度敏感於雜訊，常產生雙重邊緣
Canny 算子 多階段：高斯降噪 → 梯度計算 → 非極大值抑制 → 雙門檻值 邊緣細薄、連續性好、抗雜訊能力強 計算成本較高，需調校兩個門檻值

Canny 邊緣偵測的四個階段

Canny 演算法是目前最廣泛使用的邊緣偵測方法，其核心分為四個處理階段：

1

高斯模糊 (Gaussian Blur)

使用高斯濾波器對影像進行平滑處理，消除高頻雜訊。這一步對最終結果影響極大——若省略，邊緣偵測會將雜訊誤判為邊緣。

2

梯度計算 (Gradient Calculation)

透過 Sobel 算子計算每個像素的梯度幅值與方向，找出潛在的邊緣位置。

3

非極大值抑制 (Non-Maximum Suppression)

沿梯度方向檢查每個像素，只保留局部最大值，將粗邊緣細化為單像素寬度的精確邊緣線。

4

雙門檻值 (Double Threshold)

設定高門檻值與低門檻值：梯度高於高門檻的直接標記為邊緣；低於低門檻的直接丟棄；介於兩者之間的，只有當它與強邊緣相連時才保留。

OpenCV.js 實作範例

// 1. 轉為灰階
cv.cvtColor(src, gray, cv.COLOR_RGBA2GRAY);

// 2. 高斯模糊降噪（核心步驟！）
cv.GaussianBlur(gray, blurred,
    new cv.Size(5, 5), 1.4);

// 3. Canny 邊緣偵測
cv.Canny(blurred, edges, 50, 150);
//                    ↑低門檻 ↑高門檻
關鍵 永遠在 Canny 之前進行高斯模糊。邊緣偵測對高頻雜訊極度敏感。 如果省略模糊步驟，即使是攝影機的感光元件雜訊（Sensor Noise）都會被誤判為邊緣， 導致輪廓檢索產生數百個無意義的碎片輪廓，嚴重拖慢後續運算。
調校技巧 Canny 的雙門檻值建議比例為 1:2 或 1:3（例如 50:150）。 光線不足時需降低門檻值以捕捉較弱的邊緣；背景雜亂時則需提高門檻值以過濾無用邊緣。 在作業中，你需要針對不同環境條件找到最佳的參數組合。

輪廓檢索與幾何特徵

邊緣偵測告訴我們「哪裡有變化」，但它輸出的是一堆離散的邊緣像素點。 輪廓檢索 (Contour Detection) 的任務是將這些相鄰的邊緣點串聯成有序的封閉曲線——也就是物體的「輪廓」。 有了輪廓，我們就能計算物體的幾何特徵（面積、周長、圓度）， 將抽象的像素集合轉化為具有物理意義的幾何描述符 (Geometric Descriptor)。

從邊緣到輪廓

OpenCV 的 findContours() 函式會遍歷二值化影像中的所有白色像素， 將相鄰的像素串聯成一系列有序的座標點列表。每一個列表就代表一個獨立的輪廓。 一張影像中可能同時存在數十個甚至上百個輪廓——包括目標物體和各種背景雜訊。

// 使用 OpenCV.js 檢索輪廓
let contours = new cv.MatVector();
let hierarchy = new cv.Mat();

cv.findContours(
  edges, // Canny 輸出的邊緣圖
  contours, // 輸出：輪廓點集合
  hierarchy, // 輸出：輪廓層級關係
  cv.RETR_EXTERNAL, // 只檢索最外層輪廓
  cv.CHAIN_APPROX_SIMPLE // 壓縮直線段上的冗餘點
);
效能提示 使用 RETR_EXTERNAL 模式只檢索最外層輪廓，可以大幅減少運算量。 使用 CHAIN_APPROX_SIMPLE 則會壓縮直線段上的冗餘座標點——例如一條水平線只需儲存起點和終點，而非所有中間像素。

幾何描述符 (Geometric Descriptors)

取得輪廓後，可以透過影像矩 (Image Moments) 計算一系列幾何特徵值。 這些數值能夠精確描述物體的形狀屬性，是後續分類決策的基礎：

描述符 公式 / API 物理意義 應用範例
面積 (Area) cv.contourArea(c) 輪廓所包圍的像素數量 過濾雜訊小輪廓（面積 < 100 即丟棄）
周長 (Perimeter) cv.arcLength(c, true) 輪廓邊界的總長度 配合面積計算圓度
圓度 (Roundness) 4π × Area / Perimeter² 形狀接近圓形的程度（完美圓 = 1.0） 區分圓形標誌 vs 矩形路標
長寬比 (Aspect Ratio) width / height of bounding rect 外接矩形的寬高比 區分橫向停車線 vs 直立標誌

圓度公式與實作

圓度 (Roundness) 是本單元最核心的幾何描述符。其數學定義利用了一個重要性質： 在所有面積相等的封閉形狀中，圓形的周長最短。因此，圓度值越接近 1.0，形狀越接近完美的圓。

function calculateRoundness(contour) {
  let area = cv.contourArea(contour);
  let perimeter = cv.arcLength(contour, true);

  if (perimeter === 0) return 0;

  let roundness =
    (4 * Math.PI * area) / (perimeter * perimeter);

  return roundness; // 完美圓 ≈ 1.0
}
備註 圓度之所以適合用於自動駕駛場景，是因為它具備尺度不變性 (Scale Invariance)—— 無論圓形標誌距離攝影機是 50 公分還是 2 公尺，其圓度值始終接近 1.0。 這種穩定性讓我們無需知道物體的實際距離，就能正確分類其形狀。

輪廓篩選策略

實務中，findContours() 通常會回傳大量輪廓，其中大部分是背景雜訊。 有效的篩選策略是先用面積過濾掉小輪廓，再用幾何描述符進行分類：

  1. 面積門檻過濾：丟棄面積小於設定閾值（如 100 像素）的輪廓，排除雜訊碎片。
  2. 圓度分類：圓度 > 0.85 → 判定為圓形物體；圓度 < 0.5 → 判定為長條形物體。
  3. 長寬比輔助：搭配 bounding rect 的長寬比，進一步區分橫向線條與直立標誌。
重要 在嵌入式裝置上，對所有輪廓計算幾何特徵可能會消耗大量 CPU 時間。 務必先用 cv.contourArea() 進行初步篩選，只對「有意義的」大輪廓執行後續的周長、圓度等計算， 以確保控制迴路的即時性不受影響。

    • 驗證方式：在網頁畫面上即時標註（Bounding Box）出所有圓度大於 0.85 的物體。
    • Autograding：餵入一個圓形與一個五角星形。學員的 isCircle() 函式必須能正確區分兩者。這能檢驗學員對「特徵不變性（Invariant Feature）」的理解。

任務 3：特徵提取與決策連動 (Feature-Based Decision)

  • 目標：實現從「感知」到「控制」的端到端連動。
  • Classroom 部署建議：
    • 行為邏輯檢核： 
      // 學生需實作：視覺觸發行為
if (detectParkingLine(features)) {
    ble.send({ command: 'STOP', duration: 3000 });
}
    • 穩定性測試：要求學員實作「防抖邏輯」（例如：連續 3 幀偵測到停車線才觸發）。
    • Tutor View (PR 審核)：導師点評其對「虛警 (False Alarm)」的處理。當背景出現類似長方形的雜訊時，學員是否有加入面積門檻 (Area Threshold) 來進行二次過濾？

3. 感知邏輯導師點評標準 (Vision Logic Benchmarks)

此單元的價值在於 「從混亂中提取穩定資訊的能力」：

  • [ ] 特徵魯棒性 (Robustness)：設定的圓度或長寬比門檻值是否具備容錯空間？（即：物體歪一點、遠一點是否還能認出來？）。
  • [ ] 運算效率意識：是否對大量的小輪廓進行了初步過濾（cv.contourArea）以節省後續複雜計算的 CPU 資源？
  • [ ] 語義化整合：是否成功將「數學特徵（$Roundness > 0.8$）」與「物理語義（它是個球）」進行了邏輯掛鉤？
