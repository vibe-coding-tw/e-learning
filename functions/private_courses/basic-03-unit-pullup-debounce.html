基礎 3 - 訊號穩定性

數位控制：
上拉與按鈕抖動

馴服不穩定的物理訊號。從浮空狀態到毫秒級彈跳，讓程式穩如泰山。

📡 1. 消失的邏輯：為什麼需要上拉電阻？

在電子世界中，「沒有接線」不代表讀值是「零」。

浮空狀態 (Floating)

未與確定的電壓相連時，像天線一樣吸收雜訊。讀值在 0 與 1 間瘋狂跳動。

解決方案：INPUT_PULLUP

透過內部上拉電阻穩定拉至 3.3V (HIGH)。按下按鈕才拉至接地 (LOW)。

pinMode(PIN, INPUT_PULLUP);

🧠 Logic Snapshot

  • ✓ 未按下：讀取值為 HIGH (1)
  • ✓ 按下：讀取值為 LOW (0)
  • 💡 這就是「負邏輯」配置。

3

Non-Blocking Only

為什麼不建議用 delay()？

系統必須偵測指令且控制馬達。任何阻塞 (blocking) 都會造成車輛反應遲鈍。

最佳實踐

使用 millis() 建立非阻塞(Non-blocking)計時狀態機是所有韌體的必修課。

實作任務：精準訊號處理
