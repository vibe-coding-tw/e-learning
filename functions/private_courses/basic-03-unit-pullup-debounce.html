數位控制：
上拉與按鈕抖動

馴服不穩定的物理訊號。從浮空狀態到毫秒級彈跳，讓程式穩如泰山。

📡 1. 消失的邏輯：為什麼需要上拉電阻？

在電子世界中，「沒有接線」不代表讀值是「零」。

浮空狀態 (Floating)

未與確定的電壓相連時，像天線一樣吸收雜訊。讀值在 0 與 1 間瘋狂跳動。

解決方案：INPUT_PULLUP

透過內部上拉電阻穩定拉至 3.3V (HIGH)。按下按鈕才拉至接地 (LOW)。

pinMode(PIN, INPUT_PULLUP);

🧠 Logic Snapshot

  • ✓ 未按下：讀取值為 HIGH (1)
  • ✓ 按下：讀取值為 LOW (0)
  • 💡 這就是「負邏輯」配置。

課程單元 基礎 3 - 訊號穩定性

數位控制：上拉電阻與按鈕去彈跳 (Digital Control: Pull-up Resistors & Button Debouncing)

課程目標

在電子電路中，機械開關（如按鈕）並非完美的。當你按下或放開按鈕時，接觸點會在毫秒等級的時間內發生多次微小的彈跳（Bouncing），這會導致微控制器誤以為你快速按下了多次按鈕。本單元將帶領你透過軟體演算法「馴服」這些彈跳，並學習專業韌體開發必備的 非阻塞（Non-blocking） 程式設計技巧。

核心技能

  • 理解浮空狀態 (Floating State)：學習為何數位輸入需要上拉或下拉電阻來確保邏輯確定性。
  • 物理特性量化：觀察並記錄按鈕按下時產生的機械彈跳現象。
  • 非阻塞計時 (Non-blocking Timing)：掌握 millis() 取代 delay()，讓程式在等待的同時仍能執行其他任務。

任務說明

任務 1：天線實驗 - 體會浮空狀態 (Floating State Lab)

  1. 將 Arduino 的一個數位引腳（預設為 BUTTON_PIN）設定為 INPUT 模式，且不連接任何電子零件（懸空）。
  2. 開啟 Serial Plotter，觀察當你的手靠近或觸摸引腳時，波形如何像天線一樣吸收雜訊並產生亂跳。
  3. 驗證穩定性：將模式改為 INPUT_PULLUP，再次觀察波形是否回歸穩定的 HIGH（高電位）。
  4. 紀錄：在 docs/Oscilloscope.md 中描述「浮空引腳」的行為及其對數位系統的危害。

任務 2：計數器挑戰 - 量化機械彈跳 (Bouncing Capture Lab)

  1. 實作一個基礎計數器：每偵測到按鈕按下（LOW），pressCount 加 1。
  2. 觀察連發：嘗試輕按一次按鈕，記錄 Serial Monitor 顯示的計數值增加了多少（通常會超過 1）。
  3. 數據蒐集：分別用「輕按」、「正常按」、「用力重按」三種方式進行測試，觀察彈跳現象的差異。
  4. 報告：將實驗數據填入 docs/Oscilloscope.md 的表格中。

任務 3：專業級去彈跳演算法 (Pro Debounce & Millis Lab)

  1. 實作目標：使用 millis() 實作狀態機去彈跳邏輯。
  2. 嚴格要求：全程嚴禁使用 delay()。
  3. 功能實作：
    • 精準計數：確保不管按鈕如何彈跳，每次完整的按壓動作只觸發一次 pressCount++。
    • 狀態切換 (Toggle)：實作按一下亮燈、再按一下熄燈的功能。
    • 並行任務：在 loop() 中加入一個每 2 秒印出一次 "System Heartbeat" 的功能，驗證按鈕偵測不會阻塞背景任務。

檔案結構

  • src/main.cpp：你的程式實作核心。
  • include/debouncer.h：定義去彈跳的時間閾值（例如 50ms）與引腳編號。
  • docs/Oscilloscope.md：物理實驗報告，需填寫你的觀察結果。

提交規範

  1. 確保程式碼通過編譯，且邏輯符合「非阻塞」原則。
  2. 完成 docs/Oscilloscope.md 中的所有實驗紀錄。
  3. 將所有變更 push 回 GitHub 程式庫。
