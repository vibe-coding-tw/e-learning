精通 硬體定時器
& ISR 中斷服務

超越 millis() 的極致精確度，讓處理器為你「搶佔」資源執行關鍵防線。

⏰ 1. 什麼是硬體定時器？

處理器內部有一個獨立的「小鬧鐘」。當時間一到，處理器會「強行暫停」主程式 (Loop)，去執行被稱為 ISR 的特定緊急任務。

Active vs Passive

millis() - 被動查詢

要在 loop 中手動檢查。容易受其他耗時代碼阻塞而失效。

Timer - 主動插隊

鬧鐘強迫中斷主程式。精度極高，不受 loop 阻塞影響。

ESP32 Hardware Resource

\[4 \times Hardware Timers\]

Resolution: Microseconds (\(\mu s\))

🚫 2. 中斷服務常式 (ISR) 的三大禁忌

中斷代碼必須遵循「快進快出」原則。如果 ISR 執行太久，會導致主系統 Watchdog 重啟甚至崩潰。

NO delay() & Serial.print()

NO Complex Calculation

IRAM ATTRIBUTE 
// 確保 ISR 存在 RAM 中以獲取極速響應
void IRAM_ATTR onTimer() {
  digitalWrite(LED_PIN, !digitalRead(LED_PIN));
}

// Fast in, Fast out.

🛡️ 3. 變數共享：volatile 關鍵字

當主程式與中斷共用同一個變數（如計數器）時，必須告訴編譯器：這東西「隨時會變」，不准擅自優化暫存。

Memory Visibility

  • volatile 修飾字：指示 CPU 每次都必須從真實內存中讀取數值，不可讀取暫存器副本。
  • 數據完整性：防止主程式讀到「舊時空」的過期數據。
  • 臨界區保護：針對多位元數值（如 long），需配合原子操作指令或臨界鎖。

Data Integrity

"硬體計數的核心在於確保主程式與中斷程式『看見』的是同一個真相。"

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 定時器硬體初始化：使用 timerBegin 分配硬體定時器 0，設定 80 分頻與計數值，實現精確的 1s 週期。
  3. ISR 中斷服務：實作 IRAM_ATTR 修飾的回呼函式，僅執行最精簡的接腳電位翻轉。
  4. 搶佔性壓力測試：在 loop() 執行模擬耗時 5s 的無意義死迴圈，觀察 LED 是否依然在背景維持 1s 閃爍，不受主程式影響。

任務 2：轉速計實作：馬達 Encoder 採樣 (Tachometer & RPM)

角色扮演：自動駕駛演算法工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 窗口採樣策略：配置定時器每 100ms 觸發中斷，作為實時速換算的物理窗口期。
  2. 跨域變數宣告：正確使用 volatile 關鍵字維護脈衝計數變數，確保主程式讀取的是真實內存快照。
  3. RPM 量化演算：根據 100ms 內的增量數據反推馬達每分鐘轉速 (RPM)。
  4. 原子操作保護：在讀取計數器時使用 portENTER_CRITICAL，防止讀取過程中數據被中斷改寫導致出錯。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 數據映射準確度：輸出的 RPM 數據是否與物理端的轉速變動保持高度線性對應？
  • [ ] 變數定義嚴謹性：所有參與中斷修改的全域共享變數是否皆已正確添加 volatile 修飾字？

任務 3：安全保護：硬體級 Watchdog (Watchdog Guard)

角色扮演：工業安全工程師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 監控邏輯編寫：配置定時器中斷作為「監督者」。實作遞減計數器遞減機制。
  2. 心跳餵狗程序：在控制指令成功解析的瞬間重置（Feeding）該計數器。
  3. 緊急制動接管：若計數歸零（即 loop 掛起超過 2s），定時器需主動接管並強制將馬達暫停，防止設備暴走。
  4. 成果交付：將具備自動超時限停保護的 main.cpp 推送至作業倉庫，並附帶觸發日誌截圖。
