預防「磚頭化」 (Bricking)

ESP32 將 Flash 分為 app0 與 app1。當 app0 忙碌時，寫入 app1。

xhr.upload.addEventListener('progress', function(e) {
  if (e.lengthComputable) {
    const percent = Math.round((e.loaded / e.total) * 100);
    progressBar.style.width = percent + '%';
    statusText.innerText = `已完成: ${percent}%`;
  }
});

🔐 安全與驗證的護城河

開放 OTA 就像把鑰匙留在無線網路上。安全機制是生產環境的必需品。

mDNS 本地發現

使用 `esp32.local` 代替難記 IP。

HTTP Basic Auth

進入更新頁面前，必須通過密碼驗證。

Security Warning

"If you don't encrypt your transmission, anyone on the LAN can flash a malicious firmware."

  • ● 務必在 `onStart()` 實行 UID 比對。
  • ● 生產環境建議強制使用 TLS/HTTPS。

空中韌體分發實驗室：路由實作、進度視覺化與 A/B 分區回滾實戰

任務 1：「無線進化」基礎 Web OTA (Foundation of Web OTA)

角色扮演：IoT 部署工程師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫教 Clone 到本地。
  2. 路由實作：在 ESP32 上建立 /update 路由處理 HTTP POST 請求。
  3. 前端部署：建立 HTML 頁面，包含檔案選擇器並透過 AJAX 發送二進位檔。
  4. 無線驗證：斷開 USB 線，僅透過 Wi-Fi 成功將修改後韧體上傳。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 服務通聯：能否在瀏覽器輸入 esp32.local/update 並加載上傳介面？
  • [ ] 功能驗證：燒錄後，設備能否自動重啟並執行新版本邏輯？

任務 2：「更新進度條」視覺化體驗 (OTA Progress Visualization)

角色扮演：前端體驗設計師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 事件監聽：使用 XMLHttpRequest 監聽 progress 事件獲取進度。
  2. UI 同步：實作動態 CSS 進度條反映 (e.loaded / e.total)。
  3. 狀態引導：上傳完畢顯示「進入燒錄模式」並在 3 秒後導向新首頁。
  4. 效能記錄：記錄不同環境下上傳 1MB 韌體的平均延遲。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 數據同步性：進度條的視覺位移是否與真實數據發送匹配？
  • [ ] 用戶告知：上傳後是否具備「更新中，請勿關機」提示？

任務 3：「安全防線」故障與回滾測試 (Security & Rollback Challenge)

角色扮演：安全測試專家 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 分區表透視：配置 ota_0 與 ota_1 邊界。
  2. 故障模擬演練：在進度達到一半時故意中斷 Wi-Fi 或拔除電源。
  3. 魯棒性驗證：確認 ESP32 能否偵測分區損毀並安全回滾 (Rollback) 至原版本。
  4. 深度報告：撰寫 A/B 分區如何實現「零故障更新」技術原理。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 容錯機制認知：是否理解 Update.abort() 與數據校驗的重要性？
  • [ ] 系統恢復力：發生異常中斷後，裝置是否能在 10 秒內恢復功能？

GitHub Classroom 詳細操作指南

本單元教學重點：掌握「韌體生命週期管理」，理解硬體如何透過軟體分區實現長治久安。

  1. 重點 1：打破燒錄恐懼 - 讓學生體會部署硬體可以像 Push 網頁一樣輕鬆。
  2. 重點 2：強化回饋感 - 經由 JavaScript 進度條，實踐全棧開發中的「狀態同步」。
  3. 重點 3：穩定性第一 - 實作 A/B 分區回滾實驗，建立「Failure is expected, recovery is mandatory」意識。
