🔄 OTA 的核心：雙分區機制

如何確保更新過程中斷電，機器也不會變成「磚頭」？

1

預防「磚頭化」 (Bricking)

絕不覆蓋運行中程式！OTA 採用「備用空間下載，驗證後切換」策略。

2

A/B 分區切換 (Partition Switching)

ESP32 將 Flash 分為 app0 與 app1。當 app0 忙碌時，寫入 app1。

xhr.upload.addEventListener('progress', function(e) {
  if (e.lengthComputable) {
    const percent = Math.round((e.loaded / e.total) * 100);
    progressBar.style.width = percent + '%';
    statusText.innerText = `已完成: ${percent}%`;
  }
});

🔐 安全與驗證的護城河

開放 OTA 就像把鑰匙留在無線網路上。安全機制是生產環境的必需品。

mDNS 本地發現

使用 `esp32.local` 代替難記 IP。

HTTP Basic Auth

進入更新頁面前，必須通過密碼驗證。

Security Warning

"If you don't encrypt your transmission, anyone on the LAN can flash a malicious firmware."

  • ● 務必在 `onStart()` 實行 UID 比對。
  • ● 生產環境建議強制使用 TLS/HTTPS。
