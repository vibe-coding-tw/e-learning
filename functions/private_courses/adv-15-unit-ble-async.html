🤝 1. 非同步握手：從 Promise 到 Async/Await

藍牙連線是一個緩慢且充滿不確定性的過程。使用線性化語法 async/await 代替繁瑣的點 .then()，讓硬體連線邏輯像讀散文一樣清晰。

Connection Pipeline

  • 選取裝置 (Request)：過濾特定的 UUID 與名稱，彈出原生 UI。
  • 服務掛載 (GATT)：建立連線橋樑並取得主要服務 (Primary Service)。
  • 特徵通知 (Notify)：開啟自動推送，實現 0 延遲數據回傳。

⚡ 3. 非同步並行：雙向流控處理

解決「指令與數據打架」的問題。實現高頻指令寫入 (Write) 與實時遙測接收 (Notify) 並行處理，互不阻塞，確保系統響應毫秒級不滑坡。

Async Optimization Rules

  • 限流機制 (Throttling)：限制寫入頻率，防止藍牙緩衝區溢位。
  • 異步隔離：將遙測解析與 UI 渲染分離，維持流暢 FPS。
  • 平行序列：對不同的特徵 (Characteristics) 執行併行操作。

非同步劇場：健壯的 Web Bluetooth 通訊鏈

任務 1：線性化連線序列 (Linearizing Connection Sequence)

角色扮演：非同步程式設計師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 線性語法重構：將原有的 .then() 地獄重構為 async/await。定義一個主函數 initConnection()。
  3. 異常捕獲守衛：利用 try...catch 正確捕獲「User Cancelled」（使用者手動關閉選取視窗）的錯誤，不要讓它在 Console 報紅。
  4. Loading UI 實作：連線期間在 UI 上顯示進度提示，並在失敗時跳出具體的 Toast 警告（如：找到裝置但連不上 GATT）。

  • [ ] 容錯處理：當拔掉電腦藍牙適配器或手動取消時，程式是否能冷靜退出且保持 UI 可重新點擊？

任務 2：斷線自動重連機制 (Auto-Reconnect Mechanism)

角色扮演：系統穩定性專家 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 斷線事件監聽：為藍牙裝置掛載 gattserverdisconnected 事件。
  2. 重連策略開發：實作「指數退避」演算法，分別在 1s, 2s, 4s 後嘗試自動執行連線，最多嘗試三次。
  3. 權限維持：確保重連後無需使用者再次手動選取（利用緩存的 device 物件重新 connect）。
  4. 機制驗證：手動關閉 ESP32 電源，數秒後重開。檢查網頁 Dashboard 是否能在 5 秒內找回數據流。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 自癒能力檢核：在意外斷連後，是否能在不刷新網頁的情況下完全恢復 Notify 功能？
  • [ ] 策略優雅度：重連間隔是否能平衡「嘗試頻率」與「系統負擔」，不讓 CPU 在等待時產生阻塞。

任務 3：非同步並行處理 (Concurrent Bi-directional Stream)

角色扮演：通訊優化師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 競態條件檢測：模擬在高頻下達「前進」指令時，監測電量與陀螺儀回傳是否發生卡頓。
  2. 異步隔離重構：確保指令發送 (Write) 與數據解析 (HandleValueChange) 使用獨立的 Promise 常務處理，互相不依賴對方的結果。
  3. 流量管理 (Throttling)：實作一個簡易的限流器。當滑鼠瘋狂移動時，每 50ms 最多只發送一個 BLE 請求，保護通訊帶寬。
  4. 提交 PR：展示優化後的通訊穩定度評測，完成本單元任務。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 流控機制有效性：在瘋狂操作下，瀏覽器後台是否依然保持低延遲？遙測數據有無出現斷崖式延遲？
  • [ ] 代碼架構力：並行處理是否使用了正確的 Promise 模式，有無潛在的 Memory Leak 風險？

GitHub Classroom 詳細操作指南

本單元教學重點：掌握 JS 異步程式設計在硬體控制中的「深度適配」。如何讓前端 UI 面對硬體層的各種不穩定性時，依然保持堅挺。

  1. 重點 1：Async 不是魔法 - 引導學員理解 await 會暫停當前函數執行，但不會凍結瀏覽器。這在硬體連線這種耗時任務中至關重要。
  2. 重點 2：自癒系統思維 - 灌輸學員「斷線是常態」的意識。優秀的系統必須在事件層面做好重連預備。
  3. 重點 3：流量管理 - 藍牙的 MTU 與帶寬是有限的。教導學員如何有節制品的使用 writeValue，防止指令擁塞導致的嚴重控制延遲。
