GitHub Classroom 詳細操作指南

本單元教學重點：掌握 JavaScript 異步模式在「實體硬體控制」場景中的深度適配與診斷。

  1. 重點 1：Async 暫停不等於阻塞 - 引導學員理解 await 會暫停函數內部的執行，但主線程依然能流暢處理 UI 動畫與遙測接收。
  2. 重點 2：斷線是常態設計 - 強調穩定系統必須假設通訊隨時中斷，並在 Event 層面做好重連預備，而非依賴使用者刷新。
  3. 重點 3：流量與頻寬節制 - 讓學員理解 BLE MTU 的物理限制，學習使用「節流 (Throttle)」而非「瘋狂寫入」，確保指令的即時性。
