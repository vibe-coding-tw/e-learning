📏 1. 影像座標系：從 (0,0) 開始 (20 min)

影像的原點 $(0,0)$ 位於左上角，$X$ 軸向右，$Y$ 軸向下。這與傳統數學座標系不同。理解此映射是進行空間誤差量化的第一步。

System Constraints

  • 解析度邊界：如 $320 \times 240$。超出此範圍的像素數據將被截斷。
  • 幾何中心：$W/2$ 與 $H/2$ 是我們對齊「車體中軸線」的零誤差基準。

🧮 2. 誤差 (Error) 定義與正規化 (30 min)

正規化讓算法具備通用性。我們將原始像素座標映射至 $[-1.0, 1.0]$。$0$ 代表目標在正中央，$-1.0$ 代表極左，$1.0$ 代表極右。

Mathematical Formula

Norm_Error = (Target_X - Center_X) / Center_X;

Cross-Resolution Logic

// 160x120 camera
Norm_Error = (40 - 80) / 80 = -0.5
// 640x480 camera
Norm_Error = (160 - 320) / 320 = -0.5

🚗 3. 誤差與控制的聯覺 (Mapping)

誤差的正負號決定了馬達轉向的需求。正值 ($+$) 代表目標在右，需向右修正；負值 ($-$) 則代表需向左修正。這就是比例控制 (P-Control) 的物理基礎。

Visual Servo Feedback

  • 誤差強度：$|Error|$ 越大，代表車體偏離越遠，修正力道應越強。
  • 增益 (Gain)：透過滑桿動態調整 $K_p$，讓修正力道在「過慢」與「劇烈抖動」之間取得平衡。
  • 死區 (Dead-zone)：當誤差極小時忽略不計，防止馬達在高頻雜訊下頻繁跳變。

adv-08-unit-error-calculation (誤差計算)

這是一個將「視覺幾何」轉化為「控制學」的核心橋樑單元。你將學習如何量化偏差，並引進控制增益與濾波的概念，這是邁向 PID 調控的關鍵基礎。

學習目標

  • 掌握座標空間映射，解決「硬體解析度依賴」問題。
  • 理解「比例控制 (Proportional Control)」的物理意義。
  • 透過數據隊列處理傳感器雜訊，建立穩定的控制輸出。

任務清單

任務 1：座標提取與正規化

  • 實作跨平台誤差映射公式，將座標映射至 $[-1, 1]$ 區間。
  • 預期結果：系統能夠正確將畫面中心點誤差標示為 0，最左側為 -1，最右側為 1。

任務 2：動態補償映射與增益調校

  • 實作控制映射邏輯：steering = normError * gain;
  • 處理輸出上限（飽和處理，確保轉向力道不超過 100%）。
  • 實驗紀錄：觀察並記錄當 $K_p$ 增益過大時（如 >500），系統產生的震盪現象。

任務 3：移動平均濾波

  • 實作 5 幀的滑動窗口平均，透過數據隊列處理傳感器雜訊。
  • 預期結果：訊號抖動程度顯著降低，控制曲線變得更加平滑。

檔案結構說明

  • js/error_app.js：核心邏輯，請在此實作正規化與平均濾波。
  • js/gain_control.js：實作 $K_p$ 滑桿與輸出映射邏輯。
  • data/mock_noise.json：測試用的模擬座標雜訊流。
  • tests/math.test.js：自動化測試腳本。

2. 作業任務部署細節

任務 1：座標提取與正規化 (Coordinate Extraction Lab)

  • 目標：掌握座標空間映射，解決「硬體解析度依賴」問題。
  • Classroom 部署建議：
    • 正規化公式檢核： 
      // 學生需實作：跨平台誤差映射
const normError = (targetX - canvasWidth/2) / (canvasWidth/2);
    • Autograding：餵入邊界像素（例如 X=0 或 X=Width），檢核輸出的 normError 是否精確對齊 -1.0 與 1.0。
    • 視覺驗證：學員需提交桌面上帶有「十字準星」標註目標物，且 UI 顯示 Normalized Error 的截圖。

任務 2：動態補償映射與增益調校 (Control Mapping Challenge)

  • 目標：理解「比例控制 (Proportional Control)」的物理意義。
  • Classroom 部署建議：
    • 映射邏輯檢核：學員需實作 steering = normError * gain; 邏輯。
    • 飽和處理 (Saturation)：檢核代碼中是否處理了輸出上限（例如：轉向力道不得超過 100%）。
    • 實驗紀錄：學員需在 README 中記錄：當 $K_p$ 過大（例如 >500）時，系統對微小位移產生的激烈反應現象（這就是 PID 調機中常見的震盪現象預演）。

任務 3：移動平均濾波 (Signal Smoothing Filter)

  • 目標：透過數據隊列處理傳感器雜訊，建立穩定的控制輸出。
  • Classroom 部署建議：
    • 隊列處理檢核： 
      // 學生需實作：5 幀滑動窗口平均
errorHistory.push(newError);
if (errorHistory.length > 5) errorHistory.shift();
const smoothedError = sum(errorHistory) / errorHistory.length;
    • Autograding：餵入 noisy_coordinates.json。透過計算學員輸出數值的「標準差」，檢核其抖動程度是否成功降低 50% 以上。
    • 延遲討論：引導學員在 PR 中討論：為什麼移動平均視窗不能開得太大（例如 50 幀）？（答案：會產生不可接受的反應遲滯）。

3. 控制工程導師點評標準 (Control Ops Benchmarks)

此單元的價值在於 「數據的標準化與系統的即時性」：

  • [ ] 誤差量化正確性：正規化後的數值是否具備清晰的物理意義？（0 是否真的代表正中央？負號是否代表左側？）。
  • [ ] 抗噪優化度：開啟濾波後，控制曲線是否明顯變「絲滑」？
  • [ ] 參數敏感度認識：是否理解 $K_p$ 增益對「反應速度 vs 穩定性」的權衡關係。
