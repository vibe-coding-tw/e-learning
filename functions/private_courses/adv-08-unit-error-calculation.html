誤差計算 (Error Calculation)

從像素到控制指令：學習如何量化「視覺目標」與「車體中心」的距離，並將其轉化為精確的轉向補償。

📏 1. 影像座標系：從 (0,0) 開始 (20 min)

影像的原點 $(0,0)$ 位於左上角，$X$ 軸向右，$Y$ 軸向下。這與傳統數學座標系不同。理解此映射是進行空間誤差量化的第一步。

System Constraints

  • 解析度邊界：如 $320 \times 240$。超出此範圍的像素數據將被截斷。
  • 幾何中心：$W/2$ 與 $H/2$ 是我們對齊「車體中軸線」的零誤差基準。

🧮 2. 誤差 (Error) 定義與正規化 (30 min)

正規化讓算法具備通用性。我們將原始像素座標映射至 $[-1.0, 1.0]$。$0$ 代表目標在正中央，$-1.0$ 代表極左，$1.0$ 代表極右。

Mathematical Formula

Norm_Error = (Target_X - Center_X) / Center_X;

Cross-Resolution Logic

// 160x120 camera
Norm_Error = (40 - 80) / 80 = -0.5
// 640x480 camera
Norm_Error = (160 - 320) / 320 = -0.5

Invariant Control Output O(1)

🚗 3. 誤差與控制的聯覺 (Mapping)

誤差的正負號決定了馬達轉向的需求。正值 ($+$) 代表目標在右，需向右修正；負值 ($-$) 則代表需向左修正。這就是比例控制 (P-Control) 的物理基礎。

Visual Servo Feedback

  • 誤差強度：$|Error|$ 越大，代表車體偏離越遠，修正力道應越強。
  • 增益 (Gain)：透過滑桿動態調整 $K_p$，讓修正力道在「過慢」與「劇烈抖動」之間取得平衡。
  • 死區 (Dead-zone)：當誤差極小時忽略不計，防止馬達在高頻雜訊下頻繁跳變。

視覺實驗室：座標正規化與誤差量化

任務 1：座標提取與正規化 (Coordinate Extraction Lab)

角色扮演：視覺演算法開發者 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 視覺對齊實作：在 Canvas 繪製十字中心線，並實作 findCenter() 提取目標物幾何邊界的質心。
  3. 誤差正規化：實作公式 $E_n = (X_{obj} - W/2) / (W/2)$，將座標轉換為 $[-1, 1]$ 純數字。
  4. 看板更新：在 UI 下方即時印出解析度、原始像素、與正規化誤差，並在 README 貼上測試截圖。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 映射正確性：當目標移至最左緣時，正規化值是否精確顯示為 -1.0？
  • [ ] 視算穩定度：當畫面中出現複數物體時，座標提取是否能穩定鎖定「最大面積」的目標？

任務 2：動態補償映射 (Control Mapping Challenge)

角色扮演：自動駕駛教練 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 線性映射設計：撰寫 errorToSteer(en) 函式。將誤差線性轉換為馬達轉速差。
  2. 增益調校介面：建立一個 $K_p$ 滑桿，範圍 0-500。用於控制「看到誤差後的反應力道」。
  3. 物理實證：錄製一段「手持目標物移動」的錄影，展示虛擬車頭（或數據流）如何跟隨目標物偏轉。
  4. Push Final：提交具備即時調校能力的控制邏輯。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 邏輯一致性：當誤差為正時，输出的轉向指令是否正確導向右側（或左側，視馬達極性而定）？
  • [ ] 動態範圍：當誤差達到最大值時，馬達輸出是否能正確觸發「全速補償」而不發生數據溢位？

任務 3：移動平均濾波 (Signal Smoothing Filter)

角色扮演：濾波架構師 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 隊列緩衝實作：建立一個長度為 5 的 Array 儲存最近 5 次的誤差值。
  2. 滑動窗口運算：實作 smoothError() 函式，每次計算後的輸出為該 Array 的平均值。
  3. 對比數據分析：在 README 中說明濾波器帶來的「反應延遲」與「畫面平滑度」之間的權衡細節。
  4. Push Final：提交具備抗噪能力的穩健視覺核心。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 雜訊抑制效果：開啟濾波後，控制指令的跳動頻率是否顯著降低？
  • [ ] 延遲控制：5 幀的平均窗口是否產生了肉眼可見的「追蹤滯後」？如何優化此參數？

GitHub Classroom 詳細操作指南

本單元教學重點：讓學生掌握「從感知到行動」的量化橋樑。這不只是數學，更是控制理論在視覺上的落實。

  1. 重點 1：座標系的衝突 - 學生常死記硬背 $(0,0)$ 在左上。必須讓他們畫圖，自行導出「中心點為 0」的變換公式。
  2. 重點 2：正規化的威力 - 強調正規化後，代碼就不再受「相機換成高清」或「螢幕變大」的硬體綁架。
  3. 重點 3：沒濾波就沒平滑 - 展示真實世界感測器有多「髒」。教導移動平均法作為通向 Kalman Filter 的基礎。
