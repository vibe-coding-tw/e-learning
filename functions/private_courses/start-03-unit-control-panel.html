感知實驗室：按鈕的物理回饋

1 狀態設計 (State Design)

一個好的按鈕不只是好看，還必須「好按」。透過 CSS 的 :active 偽類，我們可以模擬真實物理按鈕被按下去的感覺。

🖱️

:hover (懸停)

暗示它是可點擊的 (游標變化、微亮)。

👇

:active (按下)

關鍵時刻！使用 transform: scale(0.95) 創造下沉感。

🚫

:disabled (禁用)

斷連時防止誤觸，搭配 pointer-events: none。

Module 2: Core Principles

三大構建基石

🏗️

1. 語義化佈局

HTML5 不只是為了顯示，更是為了「結構」。使用 section 區分功能區塊，用 ID (如 #btn-up) 給每個按鈕唯一的身份證。

<div id="d-pad">
  <button id="btn-up">...</button>
</div>
🎨

2. 偽類魔法

透過 CSS 偽類 (:pseudo-classes) 來處理互動狀態，無需寫 JavaScript 就能實現基礎的按壓反饋和視覺提示。

  • ✔ :hover (PC端提示)
  • ✔ :active (按壓特效)
  • ✔ :disabled (狀態鎖定)
🖼️

3. 向量圖示 (SVG)

為什麼不用圖片？SVG 是數學描述的向量圖，放大不失真，且可以透過 CSS (fill/stroke) 隨意改色，非常適合 RWD 介面。

優勢：
檔案極小、載入快、樣式可控。

UI 佈局 — HTML 按鈕與 CSS 控制面板

歡迎來到本單元的作業！在這個練習中，你將挑戰將前端知識轉化為一個具備沉浸感的 FPV (First Person View) 無人機控制面板。

這不僅僅是關於排版，更是關於 介面架構 (UI Architecture) 與 人因設計 (Ergonomic Design)。你將學習如何利用 CSS Grid 構建穩定的控制網格，透過偽類實作物理級的按壓回饋，並優化觸控操作體驗。

🎯 學習目標

  1. CSS Grid 實戰：打造一個「旋轉不散架」的堅韌佈局，掌握語義化區塊命名。
  2. 觸控優化：應用 Fitts's Law 確保按鈕熱區足夠大，並實作 :active 狀態的物理回饋。
  3. 圖層管理：利用 z-index 與 absolute 定位，將 HUD 控制層完美疊加在影像背景上。

🚀 任務說明

任務 1：Grid 佈局挑戰 (D-Pad Layout Lab)

  • 目標：在 style.css 中實作一個 3x3 的控制網格。
  • 要求：
    • 使用 grid-template-areas 為按鈕命名（例如：up, left, stop, right, down）。
    • 確保在手機橫向與直向模式下，十字鍵的相對位置保持一致。
    • 避免使用大量的 margin 來微調位置，應依賴 Grid 系統。

任務 2：觸控區域與物理回饋優化 (Touch Feedback Lab)

  • 目標：提升操作的「手感」。
  • 要求：
    • 熱區擴大：確保每個按鈕的可點擊面積不小於 44x44px。
    • 互動動畫：利用 transform: scale() 與陰影變化實作 :active 狀態，模擬按鈕被按下去的物理下沉感。
    • 視覺回饋：設計一個明顯的 :disabled 樣式。

任務 3：沉浸式 FPV 疊加面板 (FPV HUD Overlay Lab)

  • 目標：打造專業級的抬頭顯示器 (HUD)。
  • 要求：
    • 圖層疊加：確保控制層浮於影像背景之上。
    • 視覺風格：實作半透明背景 (Glassmorphism)，確保介面清晰且不遮擋核心視野。
    • SVG 應用：使用 assets/ 中的圖示，並確保其顏色能隨狀態變化。

2. 作業任務部署細節

任務 1：Grid 佈局挑戰 (D-Pad Layout Lab)

  • 目標：打造「旋轉不散架」的堅韌佈局。掌握 CSS Grid 的語義化區塊命名與非對稱網格。
  • Classroom 部署建議：
    • 3x3 網格實作：使用 grid-template-areas 賦予按鈕特定的網格座標（如：. up ., left stop right）。
    • 響應式驗證：要求學員確保在手機橫向與直向模式下，十字鍵的相對位置與比例保持一致。
    • 驗證要點：佈局健壯性審核。學員是否使用了語義化的網格命名？按鈕在極端螢幕比例下是否會產生重疊或過度拉伸？

任務 2：觸控區域與物理回饋優化 (Touch Feedback Lab)

  • 目標：提升「操作手感」。應用 Fitts's Law 確保熱區足夠大，並透過動畫模擬按鈕被按下去的物理下沉感。
  • Classroom 部署建議：
    • 熱區擴大實作：確保每個按鈕的可點擊面積不小於 44x44px（即便視覺上看起來小一點）。
    • 互動動畫實作：利用 transform: scale(0.95) 與陰影變化實作 :active 狀態。設計一個明顯的 :disabled 樣式用於模擬斷線。
    • 驗證要點：在手機上操作時，是否有「我有按到」的明確視覺回饋？按鈕間距是否足夠防止誤觸？

任務 3：沉浸式 FPV 疊加面板支援 (FPV HUD Overlay Lab)

  • 目標：掌握圖層管理與視覺整合。打造如同專業無人機遙控器的抬頭顯示器 (HUD) 介面。
  • Classroom 部署建議：
    • 圖層疊加實作：利用 position: absolute 與 z-index 將控制層浮於影像層之上。
    • 視覺風格整合：實作半透明背景 (glassmorphism) 與統一的顏色編碼（如：停止鍵固定為紅色脈衝）。
    • 驗證要點：介面是否能清晰地疊加在背景之上且不遮擋核心視野？SVG 圖示的填充色是否會隨按壓狀態產生變化？

3. 排版美學與交互精準點評標準 (Aesthetics & Interaction Standards)

此單元的價值在於 「對介面幾何結構的掌控力與對使用者肌肉記憶的尊重力」：

  • [ ] 佈局邏輯純度：CSS 程式碼是否乾淨地使用了 Grid，而非依賴大量的 Margin 補針？
  • [ ] 觸控人因合規性：按鈕是否存在於拇指自然的掃描範圍內？點擊熱區是否符合移動端建議最小標準？
  • [ ] 視覺完整度：當按鈕被連續快速點擊時，動畫與狀態切換是否流暢、不產生閃爍或跳動？
