簡介

歡迎來到本模組！本單元聚焦於 PID 演算法視覺化模擬。 PID 控制是讓機器人從「大致能動」進化到「精確如手術刀」的核心演算法，但它的三個參數 Kp、Ki、Kd 的交互作用常被視為工程界的「玄學」。 本模組將透過即時波形圖把抽象的數學公式轉化為看得見的動態曲線，讓你直觀理解每個參數如何影響過衝、震盪與收斂速度，並掌握科學調校方法與工業級安全防護機制。

情境設定
你的循跡機器人在直線上表現完美，但一進入彎道就開始劇烈左右擺動，像喝醉了一樣蛇行前進。你把 Kp 調大，反應變快了但震盪更嚴重；調小後雖然穩定卻慢到追不上彎道。 真正的解方不是盲目試參數，而是用視覺化波形圖觀察過衝與穩態特徵，再結合齊格勒-尼科爾斯法科學定位最佳參數組合。 請先觀看教學影片，開啟你的控制工程之旅。

🧠 1. 拆解 PID：三大控制靈魂

PID (Proportional-Integral-Derivative) 控制器是自動化工程中最廣泛使用的回饋控制演算法。從工廠的溫度調節、無人機的姿態穩定，到你正在開發的循跡機器人，PID 的核心思想始終如一：持續測量目標值與實際值的偏差，並即時計算修正量。它的數學表達式看似簡單，但三個參數 Kp、Ki、Kd 的交互作用卻蘊含深刻的控制理論。

PID 控制公式

完整的 PID 控制輸出由三個分量的加總組成：

\( u(t) = K_p \cdot e(t) + K_i \int_0^t e(\tau)\,d\tau + K_d \frac{de(t)}{dt} \)

其中 \( e(t) = \text{Setpoint} - \text{Measured} \) 是當前時刻的誤差值。在離散化的嵌入式實作中，積分項以累加取代、微分項以差分取代：

// 離散化 PID 核心運算（每個控制週期執行一次）
error = setpoint - measured;
integral += error * dt;
derivative = (error - prevError) / dt;
output = Kp * error + Ki * integral + Kd * derivative;
prevError = error;

三大控制分量詳解

分量時間觀點物理作用過大的副作用
P (比例) 看「現在」 誤差越大修正越強，提供快速反應力。像彈簧一樣將系統「拉回」目標值。 高頻震盪、無法完全消除穩態誤差
I (積分) 看「過去」 累積歷史誤差，專門消滅持續性偏差。對抗摩擦力、重力等恆定擾動。 積分飽和 (Windup)、響應遲緩、過衝加劇
D (微分) 看「未來」 預測誤差變化趨勢，在過衝發生前提前施加「煞車力」，提升阻尼。 放大高頻雜訊、系統抖動
備註 在實際循跡機器人中，感測器讀值通常含有高頻雜訊。如果直接對原始誤差做微分，D 分量會劇烈抖動。常見解法是對微分項加入低通濾波器：filteredD = alpha * derivative + (1 - alpha) * prevFilteredD，其中 alpha 通常設為 0.1-0.3。

閉迴路控制的運作流程

PID 控制器運作在一個持續循環的閉迴路 (Closed-loop) 中。每個控制週期（通常 10-50ms）依序執行以下步驟：

1

量測 (Sense)

讀取感測器數值（如紅外線循跡陣列的偏移量）作為 Measured Value。

2

計算 (Compute)

將 Setpoint（目標值，通常為 0 即「在線上」）與 Measured 的差值帶入 PID 公式，得到控制輸出 u(t)。

3

執行 (Actuate)

將控制輸出轉換為左右馬達的 PWM 差值，驅動機器人修正方向。

4

回饋 (Feedback)

機器人移動後感測器數值改變，新的 Measured Value 進入下一輪迴圈。

提示 控制週期的穩定性至關重要。使用 millis() 計時並確保每次迴圈間隔恆定（例如固定 20ms）。不穩定的 dt 會導致積分項和微分項的計算結果偏離預期，使調參變得毫無規律。

📈 2. 數據視覺化：望塵莫及的北極星

PID 調參如果只靠反覆試錯，效率極低且無法複現。透過即時波形圖將控制過程「視覺化」，是從「憑感覺」進化到「憑數據」的關鍵一步。當你能在螢幕上看到 Setpoint（紅色目標線）與 Measured（藍色回授線）的動態追隨關係，PID 三個參數的效果立刻從抽象數學變成直覺可感的物理現象。

階躍響應的四大關鍵指標

當目標值瞬間從 0 跳變至某一固定值時（稱為階躍輸入, Step Input），系統的響應曲線揭示了控制品質的一切資訊。以下四個指標是評估 PID 調參成果的「北極星」：

指標定義理想值影響因素
上升時間 (Rise Time) 從 10% 到 90% 目標值的時間 越短越好 Kp 越大，上升越快
過衝量 (Overshoot) 超過目標值的最大百分比 < 5% Kd 可有效抑制
穩態時間 (Settling Time) 進入 ±2% 誤差帶並不再離開的時間 < 2 秒 三者共同影響
穩態誤差 (Steady-State Error) 穩定後與目標值的持續偏差 ≈ 0 Ki 專門消除

建立即時波形監測儀表板

使用 Chart.js 或類似的前端圖表庫，可以建立高頻率（15-60 FPS）的即時波形監測介面。核心架構是：BLE Notify 持續回傳 Setpoint 與 Measured 數據，前端將其即時繪製到動態折線圖上。加上 Kp/Ki/Kd 的滑動條控制，就能實現「所調即所見 (WYSIWYG)」的即時回饋體驗。

// 即時波形繪製核心邏輯
function onTelemetry(event) {
  const dv = new DataView(event.target.value.buffer);
  const setpoint = dv.getInt16(0, true);
  const measured = dv.getInt16(2, true);
  const output = dv.getInt16(4, true);

  // 推入時序數據並限制窗口大小
  chart.data.labels.push(Date.now());
  chart.data.datasets[0].data.push(setpoint);
  chart.data.datasets[1].data.push(measured);
  if (chart.data.labels.length > 200) {
    chart.data.labels.shift();
    chart.data.datasets.forEach(ds => ds.data.shift());
  }
  chart.update('none'); // 無動畫更新，保持高幀率
}
提示 Chart.js 的 update('none') 跳過動畫過渡，是高頻即時繪圖的關鍵。預設的動畫模式在 60Hz 數據流下會造成嚴重的渲染延遲。另外，將數據窗口限制在 200 個點以內可防止記憶體持續增長。

從波形特徵診斷調參問題

學會「閱讀」波形是控制工程師的核心技能。以下是常見波形特徵與對應的調參建議：

  • 持續高頻震盪，無法穩定：Kp 過大。先降低 Kp 至震盪消失，再逐步回調。
  • 穩定但有持續偏差：缺少 I 分量。適度增加 Ki，讓積分項消除穩態誤差。
  • 大幅過衝後緩慢收斂：缺少 D 分量。增加 Kd 提供「煞車力」以抑制過衝。
  • 反應遲鈍，追不上目標變化：Kp 太小。適度增大 Kp 以提升響應速度。
警告 BLE 通訊延遲（典型 15-40ms）會在波形上引入額外的「相位延遲」。在分析波形時必須將通訊延遲納入考量——它可能讓 D 分量的效果看起來比實際更弱。建議在 ESP32 端同步傳回時間戳，以精確對齊 Setpoint 與 Measured 的時間軸。

🛡️ 3. 調校策略與硬體安全邊界

盲目地反覆調整 Kp/Ki/Kd 是初學者最常見的錯誤——每次只改一個參數，重新跑一輪測試，效率極低且容易陷入「越調越差」的迴圈。經典的 Ziegler-Nichols (Z-N) 方法提供了一套系統化的起點估算流程。而在實際部署中，積分飽和抑制 (Anti-Windup) 和輸出限幅 (Output Clamping) 是保護硬體免於損壞的安全底線。

Ziegler-Nichols 臨界振盪法

Z-N 法是控制工程界使用超過 80 年的經典調參起點。其步驟如下：先將 Ki 和 Kd 設為 0，逐步增大 Kp 直到系統出現等幅持續振盪（既不發散也不收斂）。此時的 Kp 值稱為臨界增益 Ku，振盪週期稱為臨界週期 Tu。

控制器類型KpKiKd
純 P 0.5 × Ku — —
PI 0.45 × Ku 1.2 × Kp / Tu —
PID 0.6 × Ku 2 × Kp / Tu Kp × Tu / 8
備註 Z-N 法提供的是一個「合理的起點」，而非最終的最佳參數。它產生的響應通常有 25% 左右的過衝量。在對過衝敏感的場景中（如避障系統），需要在此基礎上進一步手動微調——通常是降低 Kp 並增加 Kd。

積分飽和抑制 (Anti-Windup)

當馬達被卡住或達到物理極限時，PID 的輸出被限幅鎖定，但積分項仍在持續累積。當障礙移除後，巨大的積分累積值會導致系統產生劇烈的過衝——馬達會以最大功率爆衝直到積分值慢慢減小。Anti-Windup 的解法是：當輸出達到限幅邊界時，凍結積分項的累積。

// 帶 Anti-Windup 的完整 PID 控制器
error = setpoint - measured;
pTerm = Kp * error;
dTerm = Kd * (error - prevError) / dt;

// Anti-Windup：僅在輸出未飽和時才累積積分
if (output > -MAX_OUTPUT && output < MAX_OUTPUT) {
  integral += error * dt;
}
iTerm = Ki * integral;

// 輸出限幅 (Clamping)
output = constrain(pTerm + iTerm + dTerm, -255, 255);
prevError = error;

輸出限幅的必要性

PWM 的有效範圍是 0-255（8-bit 解析度）。如果 PID 運算的原始輸出超出此範圍（例如 Kp 很大時，error × Kp 可能達到數千），不加限幅就直接寫入 PWM 暫存器會導致整數溢位，可能讓馬達收到完全錯誤的驅動訊號。constrain() 函式是最後一道防線。

1

設定 Ki=0, Kd=0

先用純 P 控制找到 Ku 和 Tu。

2

套用 Z-N 公式

計算 PID 三參數的起始值。

3

加入 Anti-Windup

實作積分凍結邏輯，防止飽和。

4

微調至目標指標

參照波形圖，調整至過衝 <5%、穩態時間 <2s。

警告 在實體機器人上進行 Z-N 調參的「尋找 Ku」階段時，系統會處於持續振盪的臨界狀態。務必確保機器人在安全的測試環境中運行（例如架高底盤、遠離障礙物），並隨時準備手動斷電。臨界振盪意味著機械結構承受最大的反覆應力，長時間運行可能損壞齒輪或軸承。

PID 模擬與調校 (PID Simulation & Tuning) - 作業指引 (Assignment Guide)

單元簡介

本單元旨在將抽象的 PID 控制理論轉化為實用的工業級控制技術。你將扮演「控制工程師」，透過即時監測、數據分析與演算法加固，開發出具備高穩定度且安全的自動化控制邏輯。

學習目標

  1. 即時遙測 (Telemetry)：建立視覺化儀表板，即時觀察控制量與受控量的關係。
  2. 科學調參 (Tuning)：運用齊格勒-尼科爾斯法 (Ziegler-Nichols Method) 系統性地找出最佳 PID 參數。
  3. 系統加固 (Hardening)：實作抗積分飽和 (Anti-windup) 機制，確保系統在極限狀態下的安全性與魯棒性。

任務說明

任務 1：高效 PID 動態遙測儀表板 (High-Frequency PID Telemetry Lab)

  • 目標：實作即時波形繪製，視覺化呈現 Kp, Ki, Kd 對系統的影響。
  • 要求：確保儀表板能以高頻率 (60fps) 更新，並能精確捕捉到參數變動引發的震盪或收斂。
  • 提交：上傳一張呈現系統震盪或收斂過程的 UI 截圖至 screenshots/ 程式庫目錄中。

任務 2：「收斂至極」Z-N 調參挑戰 (Optimal Tuning via Ziegler-Nichols Lab)

  • 目標：使用 Z-N 法找出讓系統達成快速收斂（過衝 < 5%, 1.2 秒內穩定）的參數。
  • 要求：在 docs/Stability_Lab_Report.md 中記錄你的 Ku (臨界增益) 與 Tu (臨界週期)，並比較優化前後的響應差異。
  • 提交：更新後的穩定性報告。

  • 目標：建立系統「透視能力」，將看不見的控制邏輯轉化為直覺的波形曲線。

任務 2：「收斂至極」Z-N 調參挑戰 (Optimal Tuning via Ziegler-Nichols Lab)

  • 目標：捨棄「憑感覺」，使用工業標準的齊格勒-尼科爾斯法找到最佳參數比例。
  • Classroom 部署建議：
    • 臨界增益審計：學員需在數據日誌中紀錄 Ku 與 Tu 的實驗數值。
    • 性能對比：在 PR 描述中展示「優化前後」的階躍響應截圖。目標是實現 1.2 秒內的快速收斂且過衝小於 5%。

  • 目標：解決「積分飽和」導致的爆衝問題，建立系統自癒與安全防線。
  • Classroom 部署建議：
    • 加固實作：重構演算函式。當輸出達上限（如 255）且誤差持續時，暫停積分累加（抗飽和）。
    • 暴力測試驗證：要求學員模擬「馬達遇阻 3 秒後釋放」的極端情境。具備加固邏輯的系統應能立即回歸準線，而非產生毀滅性爆衝。
    • Final Push：提交高品質、模組化且具備高度魯棒性 (Robustness) 的控制器組件。
