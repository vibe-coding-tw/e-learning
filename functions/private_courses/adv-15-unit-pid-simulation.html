🧠 1. 拆解 PID：三大控制靈魂

PID 參數調校常被視為工程界的「玄學」，但透過即時模擬分析，我們能清晰透視 P 的反應、I 的補償與 D 的預測效果。將冷冰冰的數學公式轉化為動態的物理直覺。

Controller Functional Components

  • P (比例 - Present)：當前的修正力量。比例越大反應越快，但也最容易引發劇烈震盪。
  • I (積分 - Integral)：過去的累積。專門消除靜態殘差，負責對抗持續性的外部阻力。
  • D (微分 - Derivative)：未來的預測。抑制震盪的智慧型煞車，負責提升系統的動態阻尼與穩定度。

Closed-Loop Control Logic

Error(t) = Setpoint - MeasuredValue
ControlValue = \( K_p \cdot e(t) + K_i \int e(t)dt + K_d \frac{de}{dt} \)

🛡️ 3. 調校策略與硬體安全邊界

頂尖工程師絕不盲目微調。學習經典的齊格勒-尼科爾斯 (Ziegler-Nichols) 啟發式綜合法。同時實作「積分飽和抑制 (Anti-windup)」與限幅邏輯，確保系統在任何物理負載下絕不產生計算溢位或爆衝。

Safety Implementation Rules

  • 積芬抑制 (Anti-windup)：馬達遇阻時自動凍結積分累計，防止恢復後發生毀滅性超調。
  • 軟體限幅 (Clamping)：強制鎖定控制量輸出範圍至 [0, 255]，保護底層 ESC 驅動管。
  • 靜態補償 (Static Bias)：補償機械摩擦阻力，提升起始點的靈敏度與穩態準確性。

單元課程 進階 15 - PID 模擬

PID 模擬與調校 (PID Simulation & Tuning) - 作業指引 (Assignment Guide)

單元簡介

本單元旨在將抽象的 PID 控制理論轉化為實用的工業級控制技術。你將扮演「控制工程師」，透過即時監測、數據分析與演算法加固，開發出具備高穩定度且安全的自動化控制邏輯。

學習目標

  1. 即時遙測 (Telemetry)：建立視覺化儀表板，即時觀察控制量與受控量的關係。
  2. 科學調參 (Tuning)：運用齊格勒-尼科爾斯法 (Ziegler-Nichols Method) 系統性地找出最佳 PID 參數。
  3. 系統加固 (Hardening)：實作抗積分飽和 (Anti-windup) 機制，確保系統在極限狀態下的安全性與魯棒性。

任務說明

任務 1：高效 PID 動態遙測儀表板 (High-Frequency PID Telemetry Lab)

  • 目標：實作即時波形繪製，視覺化呈現 Kp, Ki, Kd 對系統的影響。
  • 要求：確保儀表板能以高頻率 (60fps) 更新，並能精確捕捉到參數變動引發的震盪或收斂。
  • 提交：上傳一張呈現系統震盪或收斂過程的 UI 截圖至 screenshots/ 程式庫目錄中。

任務 2：「收斂至極」Z-N 調參挑戰 (Optimal Tuning via Ziegler-Nichols Lab)

  • 目標：使用 Z-N 法找出讓系統達成快速收斂（過衝 < 5%, 1.2 秒內穩定）的參數。
  • 要求：在 docs/Stability_Lab_Report.md 中記錄你的 Ku (臨界增益) 與 Tu (臨界週期)，並比較優化前後的響應差異。
  • 提交：更新後的穩定性報告。

任務 3：具備 Anti-windup 能力的加固型控制器 (Hardened Controller Lab)

  • 目標：重構 src/ControlSystem_Hardened.cpp，加入 Anti-windup 邏輯。
  • 要求：模擬馬達飽和情境，驗證系統是否能在解除飽和後立即恢復穩定，而不會產生嚴重的「爆衝」。
  • 提交：具備加固邏輯的控制器原始碼至本程式庫。

1. 範本程式庫 (Template 程式庫) 配置建議

PID 調校需要高度視覺化的反饋，建議範本包含：

  • 📂 src/pid_core.cpp：重構手術對象。一個「天真」且缺乏安全防護（無 Anti-windup）的基礎 PID 類別，學員需將其升級為「加固型控制器」。
  • 📂 public/tuning_lab/：調參實驗室介面。內置 Chart.js 框架，預留 $Setpoint$ 與 $Measured$ 數據繪製區，並提供 Kp/Ki/Kd 即時連動滑軌。
  • 📂 docs/STABILITY_LOG.md：穩定性分析報告。要求學員記錄：Z-N 法測量值 (Ku/Tu) | 調校前過衝 % | 調校後收斂時間 (ms) | 穩態誤差結論。
  • 📂 tests/ctrl_sanity.test.js：控制健全性測試。檢核核心演算法在「極限輸入」下是否能維持在 [0, 255] 的安全輸出範圍內，並不發生數值溢位。

2. 作業任務部署細節

任務 1：高效 PID 動態遙測儀表板 (High-Frequency PID Telemetry Lab)

  • 目標：建立系統「透視能力」，將看不見的控制邏輯轉化為直覺的波形曲線。
  • Classroom 部署建議：
    • 視覺化整合：學員需在網頁端實作 60fps 的即時繪圖矩陣。
    • 參數閉迴路：建立「網頁端微調 ➡ 裝置端生效」的即時同步頻道，讓調校過程實現「所調即所得 (WYSIWYG)」。
    • 驗證要點：提交一份 UI 動態截圖。證明波形能真實反映出 Kp 加大時引發的震盪現象。

任務 2：「收斂至極」Z-N 調參挑戰 (Optimal Tuning via Ziegler-Nichols Lab)

  • 目標：捨棄「憑感覺」，使用工業標準的齊格勒-尼科爾斯法找到最佳參數比例。
  • Classroom 部署建議：
    • 臨界增益審計：學員需在數據日誌中紀錄 Ku 與 Tu 的實驗數值。
    • 性能對比：在 PR 描述中展示「優化前後」的階躍響應截圖。目標是實現 1.2 秒內的快速收斂且過衝小於 5%。
    • 提交結果：推送具備科學數據佐證的最終 PID 權重配置檔。

任務 3：具備 Anti-windup 能力的加固型控制器 (Hardened Controller Lab)

  • 目標：解決「積分飽和」導致的爆衝問題，建立系統自癒與安全防線。
  • Classroom 部署建議：
    • 加固實作：重構演算函式。當輸出達上限（如 255）且誤差持續時，暫停積分累加（抗飽和）。
    • 暴力測試驗證：要求學員模擬「馬達遇阻 3 秒後釋放」的極端情境。具備加固邏輯的系統應能立即回歸準線，而非產生毀滅性爆衝。
    • Final Push：提交高品質、模組化且具備高度魯棒性 (Robustness) 的控制器組件。

3. 控制工程師導師點評標準 (Control Theory & Safety Standards)

此單元的價值在於 「對物理行為的數字掌控力與極限安全性要求」：

  • [ ] 測量嚴謹度：報告是否展現了對穩定性指標（Settling Time, Overshoot）的精確量化而非籠統描述？
  • [ ] 防禦深厚度：代碼中是否顯現了對溢位與積分飽和的明確防禦邏輯？（這是區分業餘與專業的界線）。
  • [ ] 系統回應性：最終參數是否確實在實測中達成了超速收斂且無可見殘差 (Steady-state error)？
