🧠 1. 拆解 PID：三個控制靈魂

PID 參數調整常被視為「玄學」，但透過即時分析，我們能清晰看到 P 的反應、I 的補償與 D 的煞車效果。將數學公式轉化為物理直覺。

Controller Components

  • P (比例)：當前的力量。反應偏離當下的修正力，但也最容易引起震盪。
  • I (積分)：過去的累積。消除靜態誤差，負責對抗持續性的阻力（如重力）。
  • D (微分)：未來的預測。抑制震盪的煞車，負責提升系統的動態穩定度。

Control Loop Logic

Error = Target - Measured
Output = (Kp*E) + (Ki*∫E) + (Kd*dE/dt)

🛠️ 3. 調校策略與安全極限

除錯不靠盲猜。學習經典的齊格勒-尼科爾斯法則。同時實作「積分飽和抑制 (Anti-windup)」與限幅邏輯，確保系統在任何極端情境下都不會由於計算溢位而爆衝。

Safety Implementation

  • 積分抑制 (Anti-windup)：防止在馬達卡住時，控制量無限堆積導致恢復後失控。
  • 輸出限幅 (Clamping)：確保最終 PWM 值嚴格鎖定在 0-255，保護電子零件。
  • 死區補償 (Deadband)：忽略微小的無意義震盪，延長機械壽命。

任務 3：PID 安全限制器 (Safe PID Implementation)

角色扮演：系統防禦工程師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 飽和抑制重構：修改 PID 演算邏輯。當輸出值達到飽和點 (255) 時，自動暫停 I 項（積分）的累加，防止 Windup 效應。
  2. Clamping 邏輯實作：確保最終輸出透過 constrain() 函式鎖定在安全範圍內，防止運算溢位導致的控制量反轉。
  3. 魯棒性驗證：手動模擬「目標值巨變」場景。檢查輸出量是否平滑飽和而非產生隨機亂跳。
  4. Push Final：提交具備防禦性邏輯的最終版 PID 代碼。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 防護有效性檢核：在積分飽和情境下，控制器是否能比「無抑制版本」更快找回控制平準？
  • [ ] 代碼穩健度：Clamping 邏輯是否涵蓋了負向輸出值的處理（如有倒車需求）？
