🧠 1. 拆解 PID：三大控制靈魂

PID 參數調校常被視為工程界的「玄學」，但透過即時模擬分析，我們能清晰透視 P 的反應、I 的補償與 D 的預測效果。將冷冰冰的數學公式轉化為動態的物理直覺。

Controller Functional Components

  • P (比例 - Present)：當前的修正力量。比例越大反應越快，但也最容易引發劇烈震盪。
  • I (積分 - Integral)：過去的累積。專門消除靜態殘差，負責對抗持續性的外部阻力。
  • D (微分 - Derivative)：未來的預測。抑制震盪的智慧型煞車，負責提升系統的動態阻尼與穩定度。

Closed-Loop Control Logic

Error(t) = Setpoint - MeasuredValue
ControlValue = \( K_p \cdot e(t) + K_i \int e(t)dt + K_d \frac{de}{dt} \)

🛡️ 3. 調校策略與硬體安全邊界

頂尖工程師絕不盲目微調。學習經典的齊格勒-尼科爾斯 (Ziegler-Nichols) 啟發式綜合法。同時實作「積分飽和抑制 (Anti-windup)」與限幅邏輯，確保系統在任何物理負載下絕不產生計算溢位或爆衝。

Safety Implementation Rules

  • 積芬抑制 (Anti-windup)：馬達遇阻時自動凍結積分累計，防止恢復後發生毀滅性超調。
  • 軟體限幅 (Clamping)：強制鎖定控制量輸出範圍至 [0, 255]，保護底層 ESC 驅動管。
  • 靜態補償 (Static Bias)：補償機械摩擦阻力，提升起始點的靈敏度與穩態準確性。

Safety Constraint Alert

GitHub Classroom 詳細操作指南

本單元教學重點：將抽象的「控制理論」轉化為學生可感知的動態回饋。引導學員建立工業級的安全性保護意識。

  1. 重點 1：誤差是控制的靈魂 - 強化學員對「誤差 (Error)」數值的敏感度。透過看見波形，讓他們理解 P, I, D 各項是如何對誤差進行分割處理的。
