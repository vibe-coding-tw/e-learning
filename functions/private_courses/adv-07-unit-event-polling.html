進階 7 - 單元 4

事件驅動 vs 輪詢
Event-Driven Architecture

優化數據存取效能：理解中斷 (Interrupt) 與輪詢 (Polling) 的差異，打造高效的即時監控系統。

🍕 1. 觀念拆解：外送員與餐廳的溝通 (25 min)

輪詢像是不停打電話問「好了嗎？」，而事件驅動則是等對方主動傳簡訊通知。這在嵌入式系統中直接影響到 CPU 的佔用率。

Polling

主動詢問、邏輯簡單，但極度浪費資源且具備固定延遲。

Event-Driven

被動中斷、零閒置浪費，反應即時但程式流程較跳躍。

🚨 3. 硬體端的中斷 (Hardware Interrupts)

當網頁傳送「緊急停止」或車子撞到東西時，ESP32 如何利用 attachInterrupt 立即跳出主程式進行處理？這是安全性通訊的最終防線。

Fail-safe Logic

  • 致命延遲：輪詢 100ms 代表你可能有 100ms 的空白期無法煞車。
  • 優先級調色：中斷處理器 (ISR) 會強制暫停 CPU 目前的工作。
  • 連線監聽：瀏覽器 gattserverdisconnected 的零秒感應能力。

架構實驗室：效率通訊與非阻塞 I/O

任務 1：從輪詢到監聽 (From Polling to Event)

角色扮演：介面效能優化師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 對比實驗 A (Polling)：使用 setInterval 每 100ms 檢查輸入框 value.length > 8，開啟按鈕。
  3. 對比實驗 B (Event)：移除 Timer，改用 input 事件回調實現相同邏輯。
  4. 效能分析：錄製 Chrome DevTools Performance 面板的截圖，對比兩者在靜止與輸入時的 Scripting Scripting 負載，並寫入 README。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 資源消耗觀察：是否能清楚識別出 Timer 在背後不斷「空轉」造成的能耗浪費？
  • [ ] 代碼可維護性：事件驅動後的代碼是否更具備處理特定動作的語意化特徵？

任務 2：搖桿的自適應輪詢 (Hybrid Joystick Logic)

角色扮演：遙控通訊協議開發者 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 數據緩衝區實作：在 global 變數中保存最新的 X, Y 座標。
  2. 事件與輪詢解耦：mousemove 事件只負責更新變數（極快）；另起一個 20Hz 的 setInterval 負責讀取變數並發送 BLE 指令。
  3. 穩定性測試：快速晃動搖桿 30 秒，觀察是否有發生 BLE 底層的「Write Busy」錯誤。
  4. Push Final：提交具備「流量節流 (Throttling)」能力的通訊引擎。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 流量控制效果：是否有效防止了高頻事件導致的藍牙協議棧堵塞？
  • [ ] 操作平滑度：20Hz 的發送速率是否依然能提供足夠流暢的車體反應？

任務 3：安全緊急停止系統 (Emergency Fail-safe)

角色扮演：安全架構工程師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 中斷事件監聽：建立 gattserverdisconnected 回調。
  2. 安全性重置：一旦進入回調，強制將馬達變數歸零，並在畫面上顯示紅色的「CONNECTION LOST - SAFETY STOP」全屏警告。
  3. Push Final：提交具備「系統自我意識」的安全應對方案。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 事故處理及時性：當藍牙斷開時，UI 與系統狀態是否能「瞬間」轉變為安全模式？
  • [ ] 狀態清理徹度：是否正確清理了所有待處理的 Timer 或未發送的數據包？

GitHub Classroom 詳細操作指南

本單元教學重點：破解「程式就是一條路走到底」的線性思考。學習非阻塞 I/O 與異步思維。

  1. 重點 1：事件雨 (Event Flood) - 教導學員面對高頻率事件（如滑鼠移動、高速感測器）時，必須學會「節流 (Throttling)」而非盲目響應。
  2. 重點 2：資源佔用可視化 - 透過 Performance Tab 讓學員第一次看到「程式沒在動、CPU 卻在燒」的真相 (Polling 缺點)。
  3. 重點 3：安全防範意識 - 強調「中斷」在生命安全系統（如自動駕駛煞車）中的不可取代性。
