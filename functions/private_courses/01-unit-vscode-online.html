雲端開發環境：VS Code Online 與 Codespaces 實戰

任務 1：Web Serial 硬體讀取測試 (Web Serial Connectivity Lab)

角色扮演：遠端除錯員 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 開啟環境：在瀏覽器開啟任一 GitHub 倉庫。
  2. 核心指令：按下鍵盤的 . 進入 vscode.dev 網頁環境。
  3. 連結硬體：打開序列埠監控視窗 (Serial Monitor) 並連結 ESP32。
  4. 權限授權：觀察並同意瀏覽器跳出的權限請求視窗，成功讀取到 "Hello" 訊號。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 硬體連線：是否成功透過瀏覽器 Web Serial 讀取到實體硬體訊號？
  • [ ] 權限管理：是否正確理解瀏覽器對硬體存取的安全性存取機制？

任務 2：Codespaces 雲端編譯實戰 (Cloud Container Build Lab)

角色扮演：雲端架構師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 啟動容器：在 GitHub 專案頁面點選 "Code" -> "Codespaces" 建立雲端虛擬機。
  2. 指令操作：開啟雲端終端機 (Terminal) 並執行 npm install 或 npm --version。
  3. 雲端體驗：嘗試在雲端環境中運行一個簡單的腳本（如 node cloud-test.js）。
  4. 環境獨立性：觀察並理解雲端環境與個人電腦環境的獨立性。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 容器啟動：是否能正確啟動 Codespaces 並進入 Linux 終端介面？
  • [ ] 環境理解：是否能清晰區分 Web 版 VS Code (純網頁) 與 Codespaces (具備運算力容器) 的差異？

任務 3：跨裝置同步除錯挑戰 (Cross-Device Sync Challenge)

角色扮演：敏捷開發者 | 難度：⭐️ | 估計時長：30 min

🛠️ 執行技術流程 (Git Workflow)

  1. 電腦端修改：在主要裝置（電腦）修改程式碼並更新文件。
  2. 行動端接力：使用手機或平板登入 GitHub 並進入編輯模式（按下 .）。
  3. 即時提交：在行動端修改 README 並執行 commit 提交。
  4. 全域同步：回電腦端確認更新是否即時反應，完成跨裝置開發閉環。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 同步品質：Commit Message 是否清楚描述跨裝置修改的內容？
  • [ ] 跨端操作：是否掌握在行動裝置進行緊急修復與即時提交的流程？

🎭 翻轉教室的實施策略

階段 學生行動 (Git Workflow) 教師/平台角色
課前 (預習) 完成互動模擬器的「變身」體驗，理解按下 . 的底層邏輯。 Dashboard: 抽查學生的參與度，確認基礎概念認知。
課中 (實作) 實際啟動 Codespace，執行 npm 指令並在主機端與雲端間同步代碼。 助教角色: 引導學生排除環境啟動延遲，解釋開發容器 (Container) 的隔離性。
課後 (精進) 挑戰跨端接力 Commit，在電腦與移動裝置間完成一次連貫的代碼維修閉環。 Feedback PR: 點評同步紀錄與對雲端 IDE 限制的具體理解。
