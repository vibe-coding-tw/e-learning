系統架構全覽
與數據流分析

從觸控到驅動：解構數據如何跨越空間賦予機械生命。掌握端到端 (End-to-End) 的全局工程視角。

🏗️ 1. 三層神經架構：從螢幕到馬達

理解系統全貌是架構師的核心競爭力。將自動駕駛系統拆解為感知控制、無線傳輸、硬體執行三大神經層級，定義清晰的層級協議邊界。

Data Pathway Tiering

  • 感知控制層 (Dashboard)：負責 UI 事件捕捉、遙測視覺化與控制指令序列化。
  • 通訊傳輸層 (Web Bluetooth)：跨平台 GATT 協議實施，負責封包的高效低延時傳輸。
  • 硬體執行層 (Firmware)：負責解析協議、狀態機管理與底層 PWM 物理訊號產生。

🔄 2. 數據變身術：序列化與動態映射

同一個控制概念在不同層級有完全不同的數位形態。學習定義具備容錯性的「通訊協議」，確保網頁端發出的浮點數能被 ESP32 精確映射為 8-bit 的物理功率。

Type Conversion Pipeline

Normalized (0~1.0) -> JSON Packet -> ESP32 UInt8 (0~255)

解決 Byte Order 不一致與精確度損失的實戰關鍵

📈 3. 性能指標：端到端延遲測量

工程師從不靠直覺除錯。學會使用時間戳記 (Timestamp) 埋點測量「全鏈路延遲」，精確定位瓶頸：是瀏覽器渲染太慢，還是固件端的序列化解析太過沉重？

End-to-End Latency Tracing

  • 雙向時間埋點：在 Web 發送與 ESP 接收處同步紀錄 millis()，量化 RTT 值。
  • 壓力限制測試：持續提升發送頻率直到觸發序列緩衝區 (Buffer) 溢位邊界。
  • 解析韌性 (Resilience)：故意注入損毀封包，測試系統解析器是否具備自動復位能力。

核心任務：全鏈路數據實證

利用測量工具驗證數據完整性，並在極限環境下實施性能壓力測試。透過 Git Workflow 建立具備工業強度的系統診斷報告。

🔍 60 min

【作業 15-1】

「數據鏈路追蹤」

在網頁發送端與固件接收端分別埋入時間戳記，量測一組 (x, y) 座標跨平台移動的確切耗時。產出具備精確數據支持的「系統全鏈路延時報告」。

📉 60 min

【作業 15-2】

「通訊瓶頸診斷」

執行高頻發送壓力測試，找出封包長度與控制即時性之間的最優平衡點。繪製「頻率 vs 丟包率」曲線，針對你的系統環境給出官方控制配置建議。

🛡️ 45 min

【作業 15-3】

「韌性通訊實作」

手動模擬損毀封包含亂碼注入。重構並實體化具備「錯誤隔離」能力的固件解析器，確保系統遇到非法通訊時能秒級自動復位而不當機，實現工業級韌性。
