M 1. 頻率法 (M-Method)：高速穩定王

固定取樣時間窗格（如 100ms），計算這段時間內「收到了幾個脈衝」。這是高速環境下最穩定的運算法。

Execution Logic

  • 原理：固定時間 \(\Delta T\)，數脈衝數 \(N\)。
  • 優點：自帶低通濾波效果，抗干擾強，高轉速極準。
  • 缺點：低速時解像力差，數值會階梯狀跳變。

T 2. 週期法 (T-Method)：低速精準儀

測量相鄰脈衝間隔的「微秒時間」。這是賦予機器人細膩極低速行走能力的關鍵技術。

⚡ 3. 混合測速 (M/T Method) 策略

工業級控制器會根據「轉速門檻」動態切換演算法。這能結合 M 法的高速強度與 T 法的低速靈敏。

M/T Crossover Strategy

  • 切換閾值 (Threshold)：尋找誤差曲線的交點（通常為 50ms 左右）作為換手點。
  • 平滑過渡 (Smoothing)：利用加權平均防止切換瞬間的速度值跳變。
  • 零速判定：實作超時機制，防止週期法在馬達靜止時「卡死」在舊數據。

任務 2：「低速精準儀」：週期法測速 (T-Method Implementation)

角色扮演：精準測量專家 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 微秒計時捕捉：在中斷 ISR 中使用 micros() 捕捉兩次相鄰上升沿的精確時間戳。
  2. 倒數運算實作：計算 1/dt 得到頻率。注意處理 0 除或溢位等異常邊界條件。
  3. 停機邏輯開發：實作超時偵測：若超過 500ms 沒收到脈衝，強制將轉速值校準回 0。
  4. 性能分析提交：對比高速下 T 法數據的跳動幅度，分析計時誤差的影響程度。

任務 3：全速域智能切換混合測速 (M/T Method Integration)

角色扮演：演算法架構師 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 雙規並行架構：在代碼中同時計算 M 法（100ms 窗格）與 T 法（瞬時脈衝間隔）。
  2. 智能分流判定：實作切換邏輯：if(dt > 50ms) use_T_method() else use_M_method()。
  3. 數據平滑優化：為切換過程加入移動平均濾波 (Moving Average)，消除不同演算法間的定標誤差。
  4. 全域測試報告：將混合測速框架 Push 至 GitHub，並在 README 附上平滑的速度曲線圖。
