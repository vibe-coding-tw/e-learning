頻率法 (M-Method)：高速穩定王

頻率法（M-Method，也稱為脈衝計數法）的原理極其直觀：設定一個固定的取樣時間窗格 ΔT（例如 100ms）， 計算在這段時間內收到了多少個脈衝 N。脈衝數越多，代表轉速越快。 這是工業控制與機器人領域中最廣泛使用的測速方法。

M-Method 公式： \[Speed = \frac{N}{PPR \times \Delta T} \quad \text{(圈/秒)}\] \[RPM = \frac{N \times 60}{PPR \times \Delta T}\] 其中 N = 窗格內脈衝數，PPR = 每圈脈衝數，ΔT = 取樣時間（秒）。
M-Method 優缺點分析
面向 說明
優點：高速穩定 高速時每個窗格內脈衝數量多，統計平均效果好，抗雜訊能力強
優點：實作簡單 只需一個計數器和定時器，程式邏輯清晰
缺點：量化誤差 速度解析度 = 1 tick / (PPR × ΔT)，低速時解析度不足
缺點：階梯跳變 低速時 N 值只有個位數，速度呈現離散階梯式變化
C++ (Arduino) — M-Method 實作
const int   PPR = 20;
const float DT  = 0.1;           // 100ms 取樣窗格
const float WHEEL_D = 6.5;       // 輪徑 cm
const float CM_TICK = PI * WHEEL_D / PPR;

volatile int mCount = 0;

void IRAM_ATTR pulseISR() { mCount++; }

unsigned long lastSample = 0;

void loop() {
    if (millis() - lastSample >= 100) {   // 每 100ms
        lastSample = millis();
        noInterrupts();
        int n = mCount;
        mCount = 0;
        interrupts();

        float rpm     = (float)n * 60.0 / (PPR * DT);
        float speed   = n * CM_TICK / DT;  // cm/s

        Serial.printf("N=%d  RPM=%.1f  Speed=%.1f cm/s\n",
                      n, rpm, speed);
    }
}
警告：低速盲區！假設 PPR=20、ΔT=100ms，最小可偵測速度為 1 tick / 100ms = 10 ticks/s， 對應 RPM = 10×60/20 = 30 RPM。低於此轉速時，窗格內可能出現 0 或 1 個脈衝，速度讀數會在 0 和最小值之間劇烈跳動。 這就是 M-Method 在低速場景的根本限制。

週期法 (T-Method)：低速精準儀

週期法（T-Method）反轉了 M-Method 的邏輯：不再計算「固定時間內有多少脈衝」， 而是測量相鄰兩個脈衝之間的時間間隔 Δt。間隔越短代表轉速越快，間隔越長代表轉速越慢。 這種方法在低速時能提供極高的解析度。

T-Method 公式： \[Speed = \frac{1}{PPR \times \Delta t} \quad \text{(圈/秒)}\] \[RPM = \frac{60}{PPR \times \Delta t}\] 其中 Δt = 相鄰脈衝間隔（秒），使用 micros() 測量以獲得微秒級精度。
T-Method 優缺點分析
面向 說明
優點：低速精準 低速時 Δt 很大，micros() 的解析度相對極高，計算結果平滑
優點：即時響應 每個脈衝都更新一次速度，不需等待整個取樣窗格
缺點：高速雜訊 高速時 Δt 極小，micros() 的有限解析度會導致計算結果抖動
缺點：停轉問題 馬達停止後不再產生脈衝，速度值會「凍結」在最後一次讀數，無法歸零
C++ (Arduino) — T-Method 實作（含超時保護）
const int   PPR = 20;
const float WHEEL_D = 6.5;
const float CM_TICK = PI * WHEEL_D / PPR;
const unsigned long TIMEOUT_US = 500000;  // 0.5 秒無脈衝視為停止

volatile unsigned long lastPulseTime = 0;
volatile unsigned long pulsePeriod   = 0;

void IRAM_ATTR periodISR() {
    unsigned long now = micros();
    pulsePeriod   = now - lastPulseTime;
    lastPulseTime = now;
}

void loop() {
    noInterrupts();
    unsigned long dt = pulsePeriod;
    unsigned long lp = lastPulseTime;
    interrupts();

    float speed = 0;
    // 超時保護：若距離上次脈衝已超過閾值，判定為停止
    if (micros() - lp < TIMEOUT_US && dt > 0) {
        float rps = 1000000.0 / (PPR * dt);   // 圈/秒
        speed = rps * PI * WHEEL_D;            // cm/s
    }

    Serial.printf("Period=%lu us  Speed=%.1f cm/s\n", dt, speed);
    delay(50);
}
重要：超時歸零機制。T-Method 最常見的 Bug 是「馬達停了，速度讀數卻不歸零」。 因為停轉後沒有新脈衝觸發 ISR，pulsePeriod 會永遠停留在最後一個值。 解法是在 loop() 中檢查「距離上次脈衝的時間」，超過閾值就強制將速度設為 0。 閾值建議設為「最低可接受速度下的理論脈衝間隔」的 2–3 倍。

混合測速 (M/T Method) 策略

工業級馬達控制器從不只依賴單一演算法。混合 M/T 測速法根據當前轉速動態切換演算法： 高速時使用 M-Method（脈衝計數穩定），低速時切換至 T-Method（週期測量精準）。 這樣就能在全速域範圍內獲得最佳的測量精度。

M/T 混合策略：速域分工
速域 使用方法 原因
高速 (> 閾值) M-Method（頻率法） 脈衝數量充足，統計平均效果好
低速 (< 閾值) T-Method（週期法） 脈衝間隔大，micros() 解析度足以精確計算
停止 (超時) 強制歸零 T-Method 無法自行偵測停轉
切換閾值的選擇：理論上，M-Method 與 T-Method 的誤差曲線會在某個轉速交叉。 該交叉點就是最佳切換閾值。實務上，可以設定 THRESHOLD_HIGH 和 THRESHOLD_LOW 兩個值（遲滯帶 / Hysteresis），避免在臨界速度時反覆切換造成速度值跳動。
C++ (Arduino) — 混合 M/T 測速引擎
const int   PPR = 20;
const float DT  = 0.1;
const float WHEEL_D = 6.5;
const float CM_TICK = PI * WHEEL_D / PPR;
const int   THRESHOLD_HIGH = 5;   // ≥5 ticks/窗格 → 用 M-Method
const int   THRESHOLD_LOW  = 3;   // ≤3 ticks/窗格 → 用 T-Method
const unsigned long TIMEOUT_US = 500000;

volatile int mCount = 0;
volatile unsigned long lastPulseTime = 0;
volatile unsigned long pulsePeriod   = 0;
bool useM = true;

void IRAM_ATTR hybridISR() {
    unsigned long now = micros();
    pulsePeriod   = now - lastPulseTime;
    lastPulseTime = now;
    mCount++;
}

unsigned long lastSample = 0;

void loop() {
    if (millis() - lastSample >= 100) {
        lastSample = millis();
        noInterrupts();
        int n = mCount;
        mCount = 0;
        unsigned long dt = pulsePeriod;
        unsigned long lp = lastPulseTime;
        interrupts();

        float speed = 0;

        // 遲滯切換邏輯
        if (n >= THRESHOLD_HIGH) useM = true;
        if (n <= THRESHOLD_LOW)  useM = false;

        if (micros() - lp > TIMEOUT_US) {
            speed = 0;                        // 超時歸零
        } else if (useM) {
            speed = n * CM_TICK / DT;         // M-Method
        } else if (dt > 0) {
            float rps = 1000000.0 / (PPR * dt);
            speed = rps * PI * WHEEL_D;       // T-Method
        }

        Serial.printf("[%s] N=%d dt=%lu Speed=%.1f cm/s\n",
                      useM ? "M" : "T", n, dt, speed);
    }
}
提示：遲滯帶 (Hysteresis) 是防止「閃爍切換」的關鍵設計。 就像恆溫器不會在 25.0°C 時不斷開關冷氣一樣，我們設定「升到 5 才切 M，降到 3 才切 T」， 中間的 4 保持現有模式，確保速度讀數在切換點附近平滑過渡。
警告：混合測速的切換閾值必須根據實際硬體校準。 不同的 PPR、取樣時間、輪徑都會改變 M/T 誤差的交叉點。 建議的校準流程：以固定 PWM 從低速逐步加速，同時記錄 M 法與 T 法的讀數，找到兩者偏差最小的交叉轉速。

速度演算法 (Speed Algorithms)

課程目標

本單元旨在學習如何在機器人控制中，透過數位訊號處理技術精確測量速度。你將實作工業界常用的兩種核心演算法：頻率法 (M-Method) 與 週期法 (T-Method)，並設計一個混合切換邏輯 (Hybrid Crossover)，以實現在全速域下的平滑速度回饋。

任務說明

任務 1：頻率法 (M-Method) 實作

  • 目標：理解在固定時間視窗內計算脈衝數的方法。
  • 要求：
    • 在指定的取樣視窗（如 100ms）內結算編碼器 Ticks。
    • 觀察在低速狀態下的「量化誤差」（數據呈階梯狀跳變）。
    • 在專案報告中說明為何低速時頻率法表現不佳。

任務 2：週期法 (T-Method) 實作與逾時保護

  • 目標：透過測量脈衝間隔時間來獲取高解析度的低速數據。
  • 要求：
    • 使用 micros() 記錄相鄰脈衝的時間差 $\Delta t$。
    • 失效保護 (Watchdog)：實作超時邏輯。若超過 500ms 未收到脈衝，速度值必須強行歸零。
    • 驗證在馬達停止時，速度讀數是否能正確歸零，而非停留在最後一個計算值。

任務 3：混合測速演算法 (Hybrid M/T Crossover)

  • 目標：結合 M 法與 T 法的優點，打造全速域平滑的測速引擎。
  • 要求：
    • 設計一個切換門檻（如 200 RPM）。
    • 低於門檻時使用 T-Method，高於門檻時使用 M-Method。
    • 實作「遲滯區塊 (Hysteresis)」或濾波邏輯，防止在切換點附近發生數據頻繁跳變。

專案結構

  • src/SpeedEngine.cpp: 測速核心邏輯實作。
  • include/speed_math.h: 參數定義與類別宣告。
  • src/main.ino: 主程式，包含中斷處理與數據輸出。
  • tests/: 自動化測試腳本。

如何開始

  1. 複製此程式庫到你的本地環境。
  2. 使用 Arduino IDE 或 PlatformIO 開啟專案。
  3. 依序完成 src/SpeedEngine.cpp 中的 TODO 項目。
  4. 開啟 Serial Plotter 觀察三種速度曲線（M法、T法、混合法）的表現。

繳交要求

  1. 完成程式碼實作並通過所有本地測試 (npm test)。
  2. 在 reports/ 資料夾下存放你的實驗截圖與數據分析。
  3. 將所有更改 push 回你的 GitHub 專屬程式庫。

  • Classroom 部署建議：
    • 捕捉邏輯實作：學員需使用 micros() 紀錄相鄰脈衝的間距 $\Delta t$。
    • 失效保護 (Watchdog)：學員必須實作一個邏輯：若超過 500ms 沒有收到脈衝，速度值必須強行歸零，防止「車子停了但速度讀數還停留在轉動那一刻」的 Bug。
    • Push Final：提交具備「平滑低速數據」的週期法單行引擎。

任務 3：混合測速演算法 (Hybrid M/T Crossover)

  • 目標：設計「算法換手」邏輯，打造全速域平滑的速度回饋中心。
  • Classroom 部署建議：
    • 動態切換閾值：學員需找出一個轉速閾值（例如 200 RPM），低於此值用 T 法，高於此值用 M 法。
    • 切換平滑化 (Hysteresis)：為防止在門檻點兩側發生數據頻繁跳變，學員需嘗試實作「遲滯區塊」或「移動平均濾波」。
    • PR 深度核核：點評其「換手邏輯」的設計。切換瞬間的速度曲線是否連續？是否有處理切換時導致的數據跳變？
    • 即時儀表板驗證：在網頁儀表板上展示從 0 到 100% 功率的完美速度爬升曲線。

3. 控制工程導師點評標準 (Control Engineering Benchmarks)

此單元的價值在於 「對數位離散現象與物理連續性的橋接能力」：

  • [ ] 時間精度感知：學員是否理解了 millis() 與 micros() 在不同算法中的適用性？
  • [ ] 例外條件防護：馬達剛起步與剛停止時，代碼是否會產生無意義的極大值或過期舊值？
  • [ ] 系統統合感：兩種算法整合後，介面調用是否統一？是否能讓上層 PID 演算法無感地獲取最優速度？
