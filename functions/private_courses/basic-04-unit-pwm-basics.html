數位轉類比：什麼是 PWM？

微控制器的 GPIO 腳位本質上是數位的——它只能輸出邏輯高電位（3.3V）或邏輯低電位（0V），不存在「中間值」。然而在實際應用中，我們經常需要控制 LED 的亮度、馬達的轉速、蜂鳴器的音量等「類比」特性。PWM（Pulse Width Modulation，脈衝寬度調變）正是解決這個矛盾的核心技術。

PWM 的原理其實很直觀：透過極高頻率的開關切換（每秒數千甚至數萬次），讓輸出在 HIGH 與 LOW 之間快速交替。由於切換速度遠超過負載（LED、馬達）的物理響應時間，負載實際感受到的是一段時間內的平均功率，而非離散的脈衝。這就像電影膠卷——每秒 24 幀靜態畫面，在人眼中卻呈現為連續動態。

占空比（Duty Cycle）：寬度即能量

在一個完整的 PWM 週期中，高電位所佔的時間比例稱為占空比（Duty Cycle）。占空比以百分比表示，直接決定了負載接收到的平均能量：

  • 0% 占空比：腳位持續輸出 LOW，等效電壓為 0V，LED 熄滅、馬達停止。
  • 25% 占空比：每個週期中有 1/4 時間為 HIGH，等效電壓約為 0.825V。
  • 50% 占空比：HIGH 與 LOW 各佔一半，等效電壓為 1.65V。
  • 75% 占空比：大部分時間維持 HIGH，等效電壓約為 2.475V。
  • 100% 占空比：持續輸出 HIGH，等效電壓為滿幅 3.3V。
注意 PWM 並非真正的類比輸出——腳位電壓在任何瞬間仍然只有 0V 或 3.3V。它是透過「時間平均」的方式模擬中間值。如果你需要真正平滑的類比電壓輸出，需要外加 RC 低通濾波器或使用 DAC（數位類比轉換器）。

頻率的角色

PWM 頻率決定了每秒鐘產生多少個完整的脈衝週期。頻率的選擇直接影響控制品質：

頻率範圍 適用場景 現象描述
1–10 Hz 教學觀察用 肉眼可見 LED 閃爍，馬達明顯抽搐
100–500 Hz 低階伺服控制 LED 有微閃感，馬達有輕微嗡鳴
1–5 kHz LED 調光 人眼無法察覺閃爍，適合照明應用
15–25 kHz 馬達驅動 超越人耳聽覺範圍（20kHz），靜音運轉
提示 選擇 PWM 頻率時的經驗法則：LED 使用 5kHz 以上避免視覺閃爍；馬達使用 20kHz 以上避免可聽見的嘯叫噪音。頻率越高，每個週期越短，硬體對占空比的解析精度也會隨之下降。

能量等效：平均電壓原理

理解 PWM 如何模擬類比電壓的關鍵在於平均電壓公式。當 PWM 頻率夠高時，負載感受到的等效直流電壓可以用以下公式精確計算：

\[V_{avg} = V_{max} \times \text{Duty Cycle}\]

其中 \(V_{max}\) 是 GPIO 的高電位電壓（ESP32 為 3.3V），Duty Cycle 是占空比（0 到 1 之間的小數）。例如，當占空比為 60% 時：

\[V_{avg} = 3.3V \times 0.6 = 1.98V\]

這個公式的物理含義非常直觀：在一個完整週期 \(T\) 中，高電位持續時間為 \(T_{on}\)，那麼負載接收到的能量正比於 \(T_{on}/T\)，也就是占空比本身。這就是為什麼 PWM 常被稱為「能量調變」——我們透過控制時間寬度來控制能量傳遞。

為什麼看不到閃爍？

你可能會好奇：既然 LED 不斷在亮與滅之間切換，為什麼我們看不到閃爍？答案在於兩個物理現象：

  • 視覺暫留（Persistence of Vision）：人眼對光的變化反應時間約為 16ms（約 60Hz），當 PWM 頻率超過此閾值，大腦會自動將脈衝「平滑化」為連續亮度。
  • 物理慣性：馬達轉子具有機械慣性，即使供電瞬間中斷，轉子不會立刻停止。只要脈衝間隔夠短（頻率夠高），轉速便能維持穩定。
警告 如果將 PWM 頻率設得太低（例如 1Hz），LED 會像閃光燈一樣明顯閃爍，馬達則會出現「抽搐」現象——轉一下停一下，不僅無法穩定運作，還可能因為頻繁啟停產生過大電流而損壞驅動電路。在作業 4-1 中你將親眼觀察這個現象。

占空比 (Duty %) 計算式 等效電壓 (V) LED 表現
0% 3.3 × 0.00 0 V 完全熄滅
25% 3.3 × 0.25 0.825 V 微弱亮光
50% 3.3 × 0.50 1.65 V 中等亮度

ESP32 LEDC 控制核心

ESP32 內建的 LEDC（LED Control）模組是專為 PWM 輸出設計的硬體外設。與 Arduino 的 analogWrite() 不同，LEDC 提供完整的硬體 Timer 支援，讓你能精確控制頻率與解析度，並同時管理最多 16 路獨立的 PWM 頻道。

LEDC 的核心運作可以用三個層級來理解：Timer（定時器）負責產生時鐘基準，決定 PWM 頻率；Channel（頻道）負責設定各路輸出的占空比；GPIO（腳位）則是最終輸出方波的物理引腳。一個 Timer 可以服務多個 Channel，但共用 Timer 的所有 Channel 必須使用相同的頻率。

解析度（Resolution）：精細度的代價

解析度決定了占空比能被分割成多少個離散階層。ESP32 支援 1-bit 到 20-bit 的解析度：

解析度 階層數 最大值 適用場景
8-bit 256 階 255 一般 LED 調光、簡單馬達控制
10-bit 1024 階 1023 精細馬達速度控制
12-bit 4096 階 4095 伺服馬達、高精度定位
16-bit 65536 階 65535 專業照明、科學儀器
注意 解析度與頻率之間存在硬體限制的權衡關係（Trade-off）。ESP32 的 PWM 時鐘源為 80MHz，最大可用頻率公式為：freq_max = 80,000,000 / 2^resolution。例如 8-bit 解析度下最高支援約 312.5kHz，而 16-bit 解析度下最高僅約 1.2kHz。選擇解析度時需同時考慮目標頻率的需求。

PWM 頻率的選擇策略

頻率的選擇取決於被控制的負載類型。以下是常見的工程實踐建議：

  • LED 調光：建議 5kHz 以上。人眼對 100Hz 以下的閃爍最為敏感；相機拍攝時建議提高到 25kHz 以避免「捲簾快門條紋」。
  • 直流馬達：建議 15–25kHz。低於 15kHz 會產生可聽見的「嘯叫聲」（因為馬達線圈在音頻範圍內振動）；高於 25kHz 則完全靜音。
  • 蜂鳴器/音效：根據目標音高調整頻率（如 A4 音 = 440Hz），占空比固定 50% 以獲得最大音量。
// ESP32 LEDC 初始化三步驟（預覽）
const int PWM_CHANNEL = 0;
const int PWM_FREQ = 5000; // 5kHz
const int PWM_RESOLUTION = 8; // 8-bit (0~255)

void setup() {
  ledcSetup(PWM_CHANNEL, PWM_FREQ, PWM_RESOLUTION);
  ledcAttachPin(14, PWM_CHANNEL);
}

void loop() {
  ledcWrite(PWM_CHANNEL, 128); // 50% 占空比
}
提示 在下一個單元中，你將深入學習 ledcSetup、ledcAttachPin、ledcWrite 這三個 API 的完整用法，以及如何規劃多路 PWM 資源分配。此處先建立概念即可。
