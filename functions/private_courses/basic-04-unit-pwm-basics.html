基礎 4 - 能量調變控制

脈衝的藝術：
解密 PWM 占空比魔法

理解如何透過極速開關切換，讓數位腳位模擬出連續的類比電壓感，驅動光線與動能。

🌉 1. 數位轉類比的橋樑：什麼是 PWM？

電腦腳位本質上是數位的，只能輸出 0V (OFF) 或 3.3V (ON)。但透過 PWM (Pulse Width Modulation)，我們可以模擬中間值。

這是一種利用極高頻率切換開關，讓外部設備（LED 或馬達）因為物理慣性感受到一個「平均能量」的技術。

占空比 (Duty Cycle) 的奧秘

25% Duty Cycle

輸出端感覺像是 3.3V 的 1/4 強度。

75% Duty Cycle

高電位占多數，光線明亮且馬達高速。

Width = Energy

⚡ 2. 能量等效：平均電壓原理

∑ 電壓等效公式

\[V_{avg} = V_{max} \times \text{Duty Cycle}\]

為什麼我們看不到 LED 在閃爍？因為切換頻率通常高達數千赫茲 (Hz)，超越了人眼的「視覺暫留 (POV)」與機械轉子的慣性極限。

Oscilloscope Visual

方波的寬度精準定義了輸出能量的大小

🛠️ 3. ESP32 LEDC 控制核心

解析度 (Resolution)

定義占空比的精細程度。若是 8-bit 解析度，代表你可以將動力切成 256 階 (0-255)。

PWM 頻率 (Frequency)

LED 控制建議 5000Hz 以避開啟閃爍；馬達控制則依據電感特性通常在 1k~20kHz 之間選擇。

Efficiency Hint

頻率太高會增加開關損耗，太低則會產生物理共振與噪音。尋找你的驅動「甜蜜點」。

實作任務：掌控能量的規律

將抽象的數位訊號轉化為真實的物理動能與感官體驗。

「階梯式能量觀測」

【作業 4-2】

「非線性感知優化」

🏎️ 60 min

【作業 4-3】

「smartDrive 補償算法」

尋找馬達啟動的「死區」閥值。撰寫自動加上 Offset 的驅動函式，讓無人車在低速指令下也能流暢起步。

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 占空比步進開發：編寫程式，將 PWM 占空比從 10% 階梯式增加到 90%，觀察並記錄 LED 亮度的階梯對比。
  3. 低頻物理測試：將頻率調降至 10Hz，觀察馬達轉子或 LED 的不連續跳動（抽搐）現象。
  4. 結果報告：在 README.md 中以此實驗為例，列出由 Duty Cycle 計算出的「等效平均電壓」數據。

任務 2：「視覺平衡」：Gamma 校正呼吸燈 (Lighting: Gamma Correction)

角色扮演：UI/UX 燈光設計師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 線性呼吸實作：在 8-bit 解析度下實作一個純線性的遞增遞減呼吸燈。
  2. Gamma 映射開發：引入平方公式 (Duty = i*i/255) 重新映射 PWM 數值。
  3. 視覺差異分析：對比兩種方式在亮、暗轉換瞬間的平滑度差異，分析人眼感知的感知非線性。
  4. 最終提交：整併包含平方映射補償的專案並 Push。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 代碼邏輯嚴謹性：在進行平方運算時，是否正確處理了數據類型 (float/int) 與邊界限制？

任務 3：「啟動突破」：克服馬達靜摩擦力 (Motor: Static Friction Workaround)

角色扮演：動力系統工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 死區閥值量測：從零開始緩慢增加 PWM，紀錄在多少數值以上馬達才能成功克服靜摩擦力開始轉動。
  2. 補償函式封裝：撰寫 smartDrive(value) 函式。若指令不為零，則自動加上量測到的死區 Offset。
  3. 平滑起步驗證：實測在極低速指令下，馬達能否立即轉動而非悶哼或暴衝。
  4. 最終 Commit：封裝動力補償模組並 Push，附上測試日誌。
