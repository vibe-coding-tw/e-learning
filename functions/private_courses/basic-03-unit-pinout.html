🗺️ 1. 認識引腳地圖 (Pinout Map)

ESP32 就像「硬體界的瑞士軍刀」，每一條引腳在不同的軟體配置下，都能變幻出截然不同的身分。

GPIO (General Purpose)

你的數位開關，負責讀取按鈕或點亮 LED。

ADC & DAC

類比訊號的魔法門。負責處理電壓大小的變化。

Capacitive Touch

內建電容感應。不用實體按鈕，隔著塑膠殼也能操作。

Hardware Bus

硬體的高速公路：UART、I2C 與 SPI 通訊協議。

💣 2. 絕對不能忽視的「地雷引腳」

🚫

僅限輸入 (Input Only)

GPIO 34, 35, 36, 39 沒有輸出能力。它們適合感測器，但絕不能用來推動 LED 或馬達。

⚙️

啟動配置 (Strapping Pins)

GPIO 0, 2, 5, 12, 15 掌握著開機瞬間的「生殺大權」。若接錯電阻，板子就會進入死鎖或燒錄模式。

⚠️

硬體生存守則 (Case Study)

「為什麼我的專案一接上外部電源就燒錄不進去？

通常是因為你在 GPIO 0 接了一個強力的上拉按鈕，導致 SoC 開機時誤以為你要手工進入 Boot 模式。」

* 專業建議：優先使用 GPIO 12 以後的通用引腳，將地雷區留給最輕量的訊號。

⚡ 3. 供電腳位與安全電流

ESP32 不是電源供應器。過度汲取電流會直接燒毀內部的 I/O 閘道。

Vin

Voltage Input (5V)

外部電源的總入口，通常由 USB 或 5V 變壓器提供。

3V3

LDO Output (3.3V)

降壓後的穩定輸出，專供精密的感測器使用。

任務 1：「數位探針」：驗證引腳限制 (Digital Probe: Pin Validation)

角色扮演：硬體測試工程師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地開發環境。
  2. Input Only 實測：編寫程式嘗試將 GPIO 34 設為 OUTPUT 並輸出高電位，使用 LED 驗證其失敗現象。
  3. Strapping 重啟實驗：將 GPIO 12 接地後按下 Reset 鈕，觀察串口日誌輸出的 Flash 電壓 (MTDI) 錯誤。
  4. 結果記錄：在 README 中對比「正常啟動」與「引腳鎖定」的行為差異，並記錄各引腳的物理特性。

任務 2：「無人車引腳資源大規劃」 (Architecture: Pin Planning)

角色扮演：系統架構師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 配置分析：針對雙馬達、編碼器與 I2C 感測器需求，繪製一份完整的引腳配置 Markdown 表格。
  2. 衝突解耦：確保規劃成功避開 Wi-Fi 開啟時會失效的 ADC2 區域。
  3. 代碼映射：在 main.cpp 中建立清楚的 #define 引腳映射，並包含技術備註。
  4. 文檔提交：將配置圖與理由 Push，展現出對系統資源分配的全局觀。

任務 3：「類比掃描儀」：ADC 精度與底噪 (Signal: ADC Precision)

角色扮演：訊號處理專家 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 雜訊採樣：在 12-bit 解析度下讀取 ADC 數據，觀察懸空狀態下的隨機跳動幅度。
  2. 濾波實作：撰寫「滑動平均濾波 (Moving Average)」邏輯，將連續採樣數據進行平滑處理。
  3. 效能分析：使用 Serial Plotter 繪製濾波前後對比曲線，證明數據穩定度有所提升。
  4. 最終 Commit：整理包含濾波演算法的專案碼並 Push，提交最終實驗報告。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 採樣策略意識：是否理解在高頻控制場景中，採樣次數對系統反應時間 (Response Time) 的影響？

GitHub Classroom 詳細操作指南

本單元教學重點：在撰寫程式碼之前，先建立對「物理實體限制」的絕對敬畏與認識。

  1. Core 1：認識 Swiss Army Knife - 強調 ESP32 引腳的高度複用性，但也伴隨著配置上的複雜度。
  2. Core 2：規避硬體地雷 - 透過 Strapping Pins 實驗，讓學員體會到「僅僅是接錯一根線」就可能導致整個軟體系統無法引導。
  3. Core 3：數據的物理性 - 經由 ADC 雜訊實驗，引導學員理解實體世界與數位邏輯之間的「模糊地帶」與濾波必要性。
