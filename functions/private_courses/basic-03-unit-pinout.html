簡介

ESP32 有超過 30 支 GPIO 腳位，但它們並非全部等價——有些只能輸入、有些在開機時有特殊用途（Strapping Pins）、有些與內部 Flash 共用而禁止使用。 如果你隨意選腳接線，可能會遇到開機失敗、ADC 讀值異常、或 Wi-Fi 功能被干擾等難以除錯的問題。 本單元幫你建立完整的腳位認知地圖，讓每次接線都能精準選擇正確的 GPIO。

情境
你的遙控車專案需要同時接 2 顆馬達驅動器、1 個超音波感測器和 1 顆按鈕。你隨手把按鈕接到 GPIO 0——結果每次上電，ESP32 都進入下載模式而非正常運行。 查了半天才發現 GPIO 0 是 Strapping Pin，開機時若被拉低就會進入燒錄模式。 如果一開始就查閱腳位功能表，選擇安全的通用 GPIO，這個問題根本不會發生。

認識引腳地圖 (Pinout Map)

ESP32 擁有超過 30 支 GPIO（General Purpose Input/Output，通用輸入輸出）引腳，但它們並非全都一樣。每支引腳除了基本的數位輸入/輸出功能外，還可能身兼數職——有些具備 ADC 能力、有些能輸出類比訊號（DAC）、有些是通訊匯流排的專屬腳位。ESP32 就像一把「瑞士軍刀」，引腳在不同的配置下，能變幻出截然不同的身分。

要正確地使用 ESP32 開發複雜的物聯網專案，第一步就是學會閱讀引腳配置圖（Pinout Diagram）。這張圖記載了每支引腳的編號、功能分類與使用限制，是硬體設計的基本依據。

ESP32 的四大引腳功能類別

功能類別 說明 常用引腳
GPIO（數位 I/O） 最基本的功能。讀取按鈕（HIGH/LOW）或驅動 LED、繼電器。支援 PWM 輸出以控制馬達轉速或 LED 亮度。 GPIO 2, 4, 5, 13–33
ADC & DAC（類比轉換） ADC 將類比電壓轉為數位值；DAC 則反向輸出類比電壓。ESP32 有 2 組 ADC（共 18 通道）與 2 組 8-bit DAC。 ADC1: GPIO 32–39
DAC: GPIO 25, 26
電容觸控（Touch Sensor） ESP32 內建 10 組電容感測器，可偵測手指觸碰導電表面。不需按鈕即可實現觸控操作，甚至隔著薄塑膠殼也能感應。 Touch0–Touch9 (GPIO 4, 0, 2, 15, 13, 12, 14, 27, 33, 32)
通訊匯流排 硬體層級的高速通訊介面。UART（序列通訊）、I2C（感測器/螢幕）、SPI（高速裝置如 SD 卡）。ESP32 的匯流排引腳可透過 GPIO 矩陣自由映射。 預設 I2C: SDA=21, SCL=22
預設 SPI: MOSI=23, MISO=19, CLK=18
備註 ESP32 的一大優勢是引腳多工（Pin Multiplexing）。大多數周邊功能（UART、I2C、SPI）都可以透過內部的 GPIO 矩陣映射到任意引腳，不像 Arduino Uno 那樣固定死。但 ADC、DAC 和觸控感測器是硬體綁定的，只能使用特定引腳。

如何閱讀引腳配置圖

當你拿到一塊 ESP32 開發板（如 DevKitC 或 NodeMCU-32S），板子上印刷的引腳標記通常是 GPIO 編號。但你會發現不同廠商的開發板，引腳排列可能不同。以下是閱讀引腳圖的基本原則：

  • GPIO 編號是程式碼中使用的數字，例如 pinMode(14, OUTPUT) 代表操作 GPIO 14。
  • 同一支 GPIO 可能同時具備多種功能（如 GPIO 32 既是數位 I/O，也是 ADC1_CH4 與 Touch9），但同一時間只能使用一種功能。
  • 部分引腳有使用限制（僅限輸入、啟動配置等），下一個單元會詳細說明。
提示 建議將你手邊的 ESP32 開發板型號搭配官方引腳圖一起查閱。不同開發板（30-pin vs 38-pin）的引腳佈局不同，且部分引腳可能未引出。養成查閱 Datasheet 的習慣，是從業餘走向專業的重要一步。

絕對不能忽視的「地雷引腳」

並非所有 GPIO 都是平等的。ESP32 有一些引腳帶有硬體層級的限制——如果你不了解它們就隨意使用，輕則功能異常，重則整塊板子鎖死無法燒錄。這些引腳在 ESP32 社群中被形象地稱為「地雷引腳」。

僅限輸入引腳（Input-Only Pins）

GPIO 34、35、36（VP）、39（VN）這四支引腳在硬體設計上只有輸入能力，無法輸出任何訊號。它們內部缺少輸出驅動電路，因此即使你在程式中寫了 pinMode(34, OUTPUT)，然後執行 digitalWrite(34, HIGH)，引腳上也不會有任何電壓變化。

這些引腳的優勢在於它們屬於 ADC1 群組，且在 Wi-Fi 開啟時仍可正常進行類比讀取。因此，它們非常適合連接不需要供電控制的被動感測器，如光敏電阻、熱敏電阻或分壓偵測電路。

注意 GPIO 34–39 也沒有內部上拉 / 下拉電阻。如果你要在這些引腳上連接按鈕或開關，必須外接實體的上拉（或下拉）電阻，否則引腳會處於浮空狀態。

啟動配置引腳（Strapping Pins）

ESP32 在每次上電或重啟時，會讀取特定引腳的電位來決定啟動模式（正常執行 or 燒錄模式）以及 Flash 電壓等系統級設定。這些引腳被稱為 Strapping Pins：

引腳 啟動時的角色 需要的電位 誤用後果
GPIO 0 選擇啟動模式 HIGH = 正常執行 若被拉低 → 進入下載（燒錄）模式，無法正常運行
GPIO 2 搭配 GPIO 0 決定啟動模式 LOW 或浮空 在某些開發板上，開機時若為 HIGH 可能導致燒錄失敗
GPIO 5 控制啟動時序列輸出 HIGH = 啟動時印出 log 影響不大，但會產生開機雜訊訊息
GPIO 12 設定 Flash 工作電壓 LOW = 3.3V Flash 若被拉高 → Flash 讀取失敗，系統完全鎖死
GPIO 15 控制啟動 log 靜默 HIGH = 正常 log 若被拉低 → 開機時不印出除錯訊息
重要 GPIO 0 和 GPIO 12 是最危險的地雷。GPIO 0 被外部電路拉低會導致每次上電都進入燒錄模式；GPIO 12 被拉高會導致 Flash 電壓配置錯誤，整塊板子完全無法啟動。初學者應優先避開這五支引腳，改用 GPIO 13、14、16–33 等安全的通用引腳。

內部 Flash 佔用引腳

GPIO 6–11 已經被 ESP32 內部用來連接 SPI Flash 記憶體（存放你的程式碼）。這些引腳絕對不能用於任何外部用途，否則會導致 Flash 讀寫錯誤、程式崩潰甚至板子變磚。大多數開發板也不會將這些引腳引出到排針上。

提示 記住一條簡單的安全規則：GPIO 13、14、16–19、21–23、25–27、32、33 是最安全、限制最少的通用引腳。當你設計新專案時，優先從這些引腳開始配置。

供電腳位與安全電流

ESP32 開發板上不只有 GPIO 引腳，還有幾組電源相關引腳。理解它們的功能與限制，是避免燒毀晶片或周邊元件的基本功。記住一個核心觀念：ESP32 不是電源供應器——它的 GPIO 引腳能提供的電流非常有限，過度汲取電流會直接損壞內部的 I/O 驅動電路。

開發板上的電源引腳

引腳名稱 電壓 功能說明 典型用途
Vin (5V) 5V 外部電源輸入，連接到板載 LDO 穩壓器的輸入端。也可反向用來從 USB 取出 5V 電源。 供電給需要 5V 的外部模組（如部分馬達驅動板、繼電器模組）
3V3 3.3V 板載 LDO 穩壓器的輸出端，提供穩定的 3.3V。也可以從這裡為外部低功耗感測器供電。 供電給 I2C 感測器、小型 OLED 螢幕、電位計
GND 0V 接地腳。所有電路的共同參考點。ESP32 開發板通常提供多個 GND 引腳。 必須與所有外部元件共地
EN — 晶片致能引腳（Enable）。拉低此引腳可以將 ESP32 強制關機（進入深度睡眠前的硬體級別斷電）。 外部看門狗、省電控制

GPIO 的電流承載限制

每一支 GPIO 引腳能安全灌入（Sink）或驅出（Source）的電流上限約為 12mA（ESP32 官方建議），絕對最大值為 40mA。超過此限制會造成晶片內部的 MOSFET 過熱甚至永久損壞。

一顆標準 LED 通常需要 10~20mA 的驅動電流。這意味著你可以直接用一支 GPIO 點亮一顆 LED（搭配 220Ω~330Ω 限流電阻），但絕對不能直接用 GPIO 驅動馬達、繼電器線圈或多顆高亮度 LED。這些高功率負載必須透過電晶體（MOSFET）或馬達驅動 IC 來間接控制。

注意 ESP32 所有 GPIO 的總輸出電流也有上限（約 1200mA 由 3.3V LDO 供應，但分配給 GPIO 的部分遠低於此）。即使每支引腳都在安全範圍內，同時驅動太多 LED 仍可能超過總負載能力，導致電壓下降或系統不穩定。

安全供電設計原則

1

LED 必須搭配限流電阻

計算公式：R = (3.3V − VLED) ÷ ILED。以紅色 LED（VLED ≈ 2V, I = 10mA）為例：R = (3.3 − 2) ÷ 0.01 = 130Ω，實務上常用 220Ω。

2

高功率負載使用驅動電路

馬達、繼電器、伺服機等元件的工作電流遠超 GPIO 能力。使用 MOSFET（如 IRLZ44N）或專用驅動 IC（如 L298N、TB6612）來放大控制訊號。

3

獨立供電，共同接地

高功率元件應使用獨立電源（如 12V 電池）供電，而非從 ESP32 的 3V3 引腳取電。但所有電源的 GND 必須互相連接，否則訊號參考點不同，邏輯判讀會出錯。

// 正確做法：用 GPIO 控制 MOSFET 來驅動馬達
// GPIO → MOSFET Gate → 馬達由外部電源驅動

const int MOTOR_PIN = 16;
pinMode(MOTOR_PIN, OUTPUT);
analogWrite(MOTOR_PIN, 200); // PWM 訊號控制 MOSFET
提示 在設計無人車或機器人專案時，建議先計算所有元件的總電流需求，再選擇合適的電源方案。ESP32 本身在 Wi-Fi 傳輸時的峰值電流就可能達到 300~500mA，這已經佔用了 USB 供電的很大一部分容量。

本單元旨在學習如何從系統設計的角度，科學地規劃 ESP32 的硬體引腳資源。你將學習避開「地雷引腳」（如 Strapping Pins），處理僅限輸入的引腳（Input-only Pins），並實作基礎的信號濾波演算法。

🎯 學習目標

  1. 硬體物理邊界認知：了解 ESP32 不同引腳的物理限制。

針對 basic-03-unit-pinout (ESP32 引腳地圖與功能) 單元，這是從「隨便接線」演進到「系統性設計」的必經之路。它要求學員具備 硬體資源分配 (Hardware Resource Allocation) 的能力，學會避開可能會鎖死系統的「地雷引腳」，並能根據專案需求（如無人車、感測器網路）規劃出最穩定、最科學的引腳配置圖。

以下是在 GitHub Classroom 部署其作業 (Assignments) 的具體建議：

1. 範本程式庫 (Template 程式庫) 配置建議

引腳作業的核心在於「邊界測試」與「架構規劃」，範本應包含實驗性的檢測代碼，建議包含：

  • 📂 include/pins.h：核心規劃區。提供一個空的標頭檔，並附上註釋引導學員為「電機 A」、「電機 B」、「超音波感測器」等功能配置適當的引腳編號，並區分 Input/Output 與通訊協定。
  • 📂 src/main.cpp：診斷實作區。內置一套診斷代碼，學員需填入特定引腳編號，用來自動檢測該引腳是否具備輸出能力（驗證 Input-only 引腳）。
  • 📂 docs/PINOUT_PLAN.md：系統藍圖表。提供 Markdown 表格範本，學員需填入最終規劃的「功能-引腳-特性」對應表，並標註為何選擇該引腳（避讓策略）。
  • 📂 README.md：實驗日誌。學員需在此紀錄「Strapping Pin 鎖死實驗」的心得，描述當 GPIO 12 被強制下拉時，系統在重啟過程中發生了什麼錯誤。

2. 作業任務部署細節

任務 1：「數位探針」驗證引腳物理限制 (Digital Probe Lab)

  • 目標：透過實驗親自撞牆，建立對 Input-only 與 Strapping pins 的深刻肌肉記憶。
  • Classroom 部署建議：
    • 輸出失效驗證：要求學員將 GPIO 34 或 35 設為輸出並點亮 LED。學員需在 README 中紀錄「為何失敗」的物理原因。
    • 啟動鎖死實驗：引導學員短接特定引腳（如 GPIO 12）後按下 Reset，記錄序列監視器輸出的 Flash 讀取錯誤日誌。
    • 驗證要點：README 中需包含錯誤日誌的文字截圖與簡要的原理解析。

任務 2：「無人車系統」資源總規劃 (Rover Architecture Planning Lab)

  • 目標：學會面對複雜硬體需求（4 台馬達、1 組 I2C 螢幕、3 組 ADC 感測器）時，如何有條理地分配引腳資源。
  • Classroom 部署建議：
    • 規避衝突策略：學員需在 PINOUT_PLAN.md 中規避掉 ADC2 區域（若 Wi-Fi 同時開啟）以及所有的開機啟動引腳。
    • 代碼映射實作：將所有規劃的引腳定義為自定義宏（#define MOTOR_PWM_FL 32 等），並在 pins.h 中呈現。
    • PR 提交內容：提交一份清晰的硬體架構表格，展現對硬體資源全局分配的掌握。

任務 3：「類比掃描儀」信號質量處理 (Signal Precision Lab)

  • 目標：基於引腳對應，引入變量處理，提升感測器讀值的穩定。
  • Classroom 部署建議：
    • 雜訊採樣觀測：引導學員讀取懸空引腳的 12-bit ADC 值，體會環境底噪。
    • 平滑濾波算法：在 main.cpp 中實作一個「滑動平均（Sliding Window Average）」算法。
    • 驗證要點：展示 Serial Plotter 對比圖。要求：濾波後的曲線在靜態時必須是一條接近直線的平穩數據。

3. 硬體架構專家導師點評標準 (Hardware Reliability & Planning Standards)

此單元的價值在於 「對硬體物理邊界的尊重與衝突預判能力」：

  • [ ] 引腳選擇合理性：是否將 LED 或馬達 PWM 接在了 Input-only 引腳上？
  • [ ] 開機穩定性：配置中是否使用了 GPIO 0 或 12 作為常時上拉/下拉的輸出，導致重啟困難？
  • [ ] 文檔嚴謹度：PINOUT_PLAN.md 是否清晰且具備邏輯性？是否有考慮到 Wi-Fi 開啟後對 ADC2 引腳的排擠？
