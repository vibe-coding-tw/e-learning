🏎️ 1. 為什麼兩顆輪子就能轉向？ (15 min)

在機器人學中，差速驅動（Differential Drive）是最常見且最具教學價值的移動機構之一。 它僅需兩顆獨立驅動的輪子，就能實現前進、後退、轉彎甚至原地旋轉等所有平面運動。 理解其運作原理，需要從剛體運動學（Rigid Body Kinematics）的角度切入。

剛體約束與旋轉中心

車體是一個剛體（Rigid Body），這意味著車上任意兩點之間的距離在運動過程中始終保持不變。 當左右兩輪的瞬時線速度不同時（即 \(V_L \neq V_R\)），剛體約束迫使整台車繞著某一個固定點旋轉。 這個點就是接下來我們要深入探討的 ICC（Instantaneous Center of Curvature，即時曲率中心）。

想像你和朋友手拉手向前走，但你走得比較快——你們兩個人會自然地繞著某個「中心」旋轉。 差速驅動車的原理完全相同：速度差產生曲率，曲率決定軌跡。

備註 差速驅動的核心直覺：左右輪的速度差決定了車體的轉彎半徑， 而左右輪的速度和決定了車體的前進速度。掌握這個直覺，就掌握了差速驅動控制的一半。

差速驅動 vs. 其他轉向機構

在機器人設計中，存在多種轉向機構。理解差速驅動的定位，有助於判斷何時該選用它。

轉向機構 原理 最小轉彎半徑 控制複雜度 典型應用
差速驅動 左右輪速差 0（可原地旋轉） 中 室內機器人、教育平台
阿克曼轉向（Ackermann） 前輪偏角 受限於最大轉向角 低 汽車、戶外車輛
全向輪（Mecanum） 斜向滾輪組合 0 高 倉儲機器人、競賽
履帶式（Skid-Steer） 左右履帶速差 0 中 軍用載具、崎嶇地形

速度差如何產生轉向

假設左輪在單位時間內行進了 \(d_L\) 的距離，右輪行進了 \(d_R\) 的距離。 由於兩輪固定在車軸的兩端（間距為輪距 \(L\)），當 \(d_L > d_R\) 時， 車體的左側「走得多」、右側「走得少」——車體自然向右偏轉。 這就是差速驅動的物理本質：不對稱的移動距離產生旋轉。

從運動學角度，我們可以直接由左右輪速推導出車體的線速度 \(v\) 和角速度 \(\omega\)：

\[v = \frac{V_L + V_R}{2}, \quad \omega = \frac{V_L - V_R}{L}\]

// 差速驅動基礎運動學（C++ / ESP32）
const float L = 0.15; // 輪距 15 cm = 0.15 m
float V_L = 0.20; // 左輪線速度 (m/s)
float V_R = 0.10; // 右輪線速度 (m/s)

// 車體線速度與角速度
float v = (V_L + V_R) / 2.0; // = 0.15 m/s
float omega = (V_L - V_R) / L; // = 0.667 rad/s（向右轉）
提示 在 ESP32 上，「速度」對應的是 PWM 值（0–255）。當你設定 analogWrite(LEFT_MOTOR, 200) 和 analogWrite(RIGHT_MOTOR, 100) 時， 你其實已經在執行差速驅動——只是還沒用數學來精確預測車體會走出什麼軌跡。接下來的 ICC 公式推導，正是為了填補這個缺口。

🌐 2. 核心主角：ICC (Instantaneous Center of Curvature)

ICC（Instantaneous Center of Curvature，即時曲率中心）是差速驅動幾何學中最核心的概念。 在任何一個瞬間，車體上的所有點都繞著 ICC 進行圓周運動。ICC 的位置完全由左右輪的速度比決定， 它可能在車體外側很遠的地方（大半徑轉彎），也可能恰好落在車體中心（原地自轉）。

從剛體約束推導 ICC 公式

設車軸中心到 ICC 的距離為 \(R\)（轉彎半徑），輪距為 \(L\)，車體角速度為 \(\omega\)。 由於左右輪都繞同一個 ICC 旋轉，但它們到 ICC 的距離不同，因此線速度也不同：

  • 左輪到 ICC 的距離為 \(R + L/2\)，所以 \(V_L = \omega \cdot (R + L/2)\)
  • 右輪到 ICC 的距離為 \(R - L/2\)，所以 \(V_R = \omega \cdot (R - L/2)\)

將兩式相除消去 \(\omega\)，即可解出 \(R\)：

\[\frac{V_L}{V_R} = \frac{R + L/2}{R - L/2} \quad \Longrightarrow \quad R = \frac{L}{2} \cdot \frac{V_L + V_R}{V_L - V_R}\]

重要 這個公式是整個差速驅動控制的基石。它告訴我們：轉彎半徑完全由左右輪的速度比決定，與絕對速度無關。 無論車體以 10 cm/s 還是 100 cm/s 行駛，只要速度比相同，走出的圓弧半徑就相同。

公式中各變數的物理意義

符號 名稱 單位 說明
\(V_L\) 左輪線速度 m/s 左輪接觸地面的瞬時速度
\(V_R\) 右輪線速度 m/s 右輪接觸地面的瞬時速度
\(L\) 輪距（Wheel Base） m 兩輪接地點之間的水平距離
\(R\) 轉彎半徑 m 車軸中心到 ICC 的距離（正值向左轉）
\(\omega\) 角速度 rad/s 車體繞 ICC 旋轉的角速率

逆向運動學：從目標半徑求輪速

在實際控制中，我們通常已知「想要的轉彎半徑 \(R\)」和「想要的整體速度 \(V\)」，需要反推出左右輪速。 這就是逆向運動學（Inverse Kinematics）：

\[V_L = V \cdot \frac{R + L/2}{R}, \quad V_R = V \cdot \frac{R - L/2}{R}\]

// 逆向運動學：給定 R 和 V，計算左右輪速
struct WheelSpeed { float left; float right; };

WheelSpeed calcIK(float R, float V, float L) {
WheelSpeed ws;
ws.left = V * (R + L/2.0) / R;
ws.right = V * (R - L/2.0) / R;
return ws;
}

// 範例：半徑 0.5m、速度 0.2 m/s、輪距 0.15m
WheelSpeed s = calcIK(0.5, 0.2, 0.15);
// s.left = 0.230 m/s, s.right = 0.170 m/s
提示 注意公式中 \(L/2\) 的幾何意義：它代表車軸中心到每顆輪子的距離。 如果你的機器人輪距量測不精確，所有的運動學計算都會出現系統性偏差。 建議用捲尺精確量測兩輪接地點的中心距離，誤差控制在 ±1 mm 以內。

🚦 3. 特殊運動狀態

掌握了 ICC 公式 \(R = \frac{L}{2} \cdot \frac{V_L + V_R}{V_L - V_R}\) 之後， 我們可以分析三種邊界條件（Boundary Conditions）—— 這些極端情況不僅在理論上揭示了公式的行為特性，更是在程式實作中必須正確處理的例外邏輯， 否則可能引發除以零錯誤或無窮大溢位。

三種特殊運動狀態

運動狀態 條件 \(R\) 值 ICC 位置 車體行為
直線行駛 \(V_L = V_R\) \(R \to \infty\) 無限遠處 車體沿直線前進，無旋轉
原地自轉 \(V_L = -V_R\) \(R = 0\) 車軸中心 車體繞中心點原地旋轉
單輪旋轉 \(V_L = 0\) 或 \(V_R = 0\) \(R = \pm L/2\) 靜止輪的接地點 車體繞靜止輪旋轉

狀態 1：直線行駛（\(V_L = V_R\)）

當左右輪速度完全相等時，公式分母 \(V_L - V_R = 0\)，導致 \(R \to \infty\)。 物理上，這代表 ICC 在無限遠處——車體上所有點都在做半徑無限大的圓周運動，即直線運動。 在程式中，必須在計算 \(R\) 之前攔截此情況，否則會產生除以零錯誤。

狀態 2：原地自轉（\(V_L = -V_R\)）

當左右輪等速反轉時，代入公式可得 \(R = \frac{L}{2} \cdot \frac{0}{2V_L} = 0\)。 ICC 恰好落在車軸的正中心，車體繞自身中心原地旋轉。 這是差速驅動相較於阿克曼轉向的獨特優勢——它可以在零空間內完成轉向。

狀態 3：單輪旋轉（Pivot Turn）

當一側輪子速度為零（例如 \(V_R = 0\)），代入公式得 \(R = L/2\)。 ICC 恰好落在靜止輪的接地點上，車體繞該輪旋轉。此模式的轉彎半徑固定為 \(L/2\)， 是介於直線行駛和原地自轉之間的折衷方案。

警告 在程式碼中，永遠不要讓分母 \(V_L - V_R\) 直接為零時還去做除法。 使用一個極小的閾值（如 abs(V_L - V_R) < 0.001）來偵測直線行駛狀態， 並切換到專用的直線邏輯分支。

邊界條件的程式實作

以下程式碼展示如何在運動學計算中正確處理所有三種邊界條件：

WheelSpeed calculateIK(float R, float V, float L) {
WheelSpeed ws;

// 狀態 2：原地自轉（R ≈ 0）
if (abs(R) < 0.001) {
ws.left = V;
ws.right = -V;
return ws;
}

// 狀態 1：直線行駛（R 極大）
if (abs(R) > 1000) {
ws.left = V;
ws.right = V;
return ws;
}

// 一般情況：ICC 圓弧運動
ws.left = V * (R + L/2.0) / R;
ws.right = V * (R - L/2.0) / R;
return ws;
}
提示 閾值 0.001 和 1000 並非隨意選取，而是根據你的機器人尺寸和精度需求決定的。 對於輪距 15 cm 的小型教育機器人，半徑小於 1 mm 在物理上沒有意義（因為輪胎的接地面積就已經超過這個值）， 而半徑超過 10 m 的圓弧在實際場地中與直線無異。

進階 10 單元：ICC 幾何學 (Kinematics & ICC Geometry)

歡迎來到本單元的程式庫！這是機器人運動學中最核心的理論單元。在這裡，你將學習如何將思維從「憑感覺轉向」提升到「幾何精確控制」，利用 ICC (即時旋轉中心) 的數學模型，精確預測車體在平面上的每一公分位移。

🎯 學習目標

透過幾何建模，讓車體能完美遵循物理約束，實現精確的圓周與軌跡控制。

🛠️ 作業任務與要求

任務 1：精準圓周控制 (Precision Circle Lab)

  • 目標：透過幾何模型實現「精準圓周線」繞行。
  • 實作要求：
    • 完成程式庫中的核心運算邏輯（如 setSteering(R, V)），將目標半徑轉化為馬達 PWM 指令。
  • 繳交項目：
    • 在地面標記一個 50cm 半徑的圓（直徑 100cm）。
    • 錄製一段車體精準沿著標記線運行的影片，並提交或附在報告中。

任務 2：ICC 漂移量測與補償 (ICC Calibration Lab)

  • 目標：修正硬體不對稱產生的「旋轉中心偏移」。
  • 實作要求：
    • 執行原地自轉 ($V_L = -V_R$)。觀察車體中心是否有在地面發生平移（即 ICC 漂移）。
    • 實作補償係數，例如補償左馬達或右馬達較弱的情況。
  • 繳交項目：
    • 在下方 實驗報告區 中，記錄並分析不同材質表面（例如：磁磚 vs. 地毯）對 ICC 穩定度的影響。

任務 3：變半徑螺旋線路徑規劃 (Spiral Path Lab)

  • 目標：實現動態半徑控制，模擬複雜的導航軌跡。
  • 實作要求：
    • 實作動態演算法，讓目標半徑 $R$ 隨時間或距離線性增加（例如從 20cm 逐漸擴大到 100cm）。
    • 邊界條件處理：當 $R$ 超過 200cm 時，程式應自動平滑切換為「直線行駛」邏輯，防止除以零或無窮大的錯誤。
    • 注意「螺旋線」的平滑度，避免車體在增加半徑時產生劇烈抖動（提示：可考慮加入 Throttling 流控）。

針對 adv-10-unit-icc-geometry (ICC 幾何學) 單元，這是機器人運動學中最核心的理論單元。它將學員的思維從「憑感覺轉向」提升到「幾何精確控制」，利用 ICC (即時旋轉中心) 的數學模型，精確預測車體在平面上的每一公分位移。

以下是在 GitHub Classroom 部署其作業 (Assignments) 的具體建議：

1. 範本程式庫 (Template Repo) 配置建議

幾何學作業高度依賴公式推導與物理參數校準，建議範本包含：

  • 📂 include/kinematics.h：幾何常數定義檔。預設輪距 $L$、輪徑等物理參數，留下從半徑 $R$ 換算為輪速 $(V_L, V_R)$ 的介面。
  • 📂 src/motion_model.js：核心運算邏輯。引導學員實作 $R = \frac{L}{2} \cdot \frac{V_L + V_R}{V_L - V_R}$ 的逆向運算，即：給定目標半徑，輸出馬達指令。
  • 📂 LAB_DATA.md：實驗數據紀錄表。預置表格與座標轴模板，供學員記錄「理論半徑」與「實測半徑」的誤差值。
  • 📂 tests/geometric_consistency.test.js：自動化評分脚本。專門測試「速比一致性」——輸入不同的目標半徑，檢核產出的 $(V_L, V_R)$ 比例是否符合剛體旋轉約束。

2. 作業任務部署細節

任務 1：精準圓周控制 (Precision Circle Lab)

  • 目標：透過幾何模型實現「精準圓周線」繞行。
  • Classroom 部署建議：
    • 代碼數學檢核：學員需實作 setSteering(R, V)，將目標半徑轉化為馬達 PWM。
    • 實測數據提交：學員需在地面標記一個 50cm 半徑的圓（直徑 100cm），錄製一段車體沿線運行的影片。
    • Autograding：餵入半徑 $R=50$ 與 $V=100$，檢核左右輪速的輸出差值是否精確對應。

任務 2：ICC 漂移量測與補償 (ICC Calibration Lab)

  • 目標：修正硬體不對稱產生的「旋轉中心偏移」。
  • Classroom 部署建議：
    • 校準流程要求：執行原地自轉 ($V_L = -V_R$)。學員需觀察車體中心是否有在地面發生平移（即 ICC 漂移）。
    • 補償係數實作： 
      // 學生需實作：馬達出力補償
this.left_comp = 1.05; // 補償左馬達較弱的情況
    • 數據分析回報：在 README 中匯報不同材質表面（例：磁磚 vs. 地毯）對 ICC 穩定度的影響。

任務 3：變半徑螺旋線路徑規劃 (Spiral Path Lab)

  • 目標：實現動態半徑控制，模擬複雜的導航軌跡。
  • Classroom 部署建議：
    • 動態演算法實作：在 loop 中實施一個變量 $R$，讓其隨時間或距離線性增加（例：從 20cm 逐漸擴大到 100cm）。
    • 邊界路徑處理：要求學員實作：當 $R$ 超過 200cm 時，代碼應自動平滑切換為「直線行駛」邏輯，防止除以零或無窮大的錯誤。
    • PR 深度核核：點評其對「螺旋線」平滑度的處理。車體在增加半徑時是否產生了劇烈抖動？（提示：需加入 Throttling 流控）。

3. 幾何建模導師點評標準 (Kinematic Benchmarks)

此單元的價值在於 「對物理約束的幾何化建模能力」：

  • [ ] 幾何關係推導：是否能正確理解公式中 $L/2$ 的幾何意義？
  • [ ] 誤差敏感度：學員是否能指出馬達死區對低速轉彎半徑的巨大影響？
  • [ ] 算法穩健性：在處理直線 ($R \to \infty$) 與自轉 ($R=0$) 的邊界條件時，代碼是否優雅且不崩潰？
