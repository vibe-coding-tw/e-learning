基礎 6 - 網路通訊基礎

數位世界的溝通橋樑：
HTTP 協議與 Fetch API

掌握讓網頁、伺服器與物聯網設備互相「對話」的核心技術。

🍽️ 1. 理解 HTTP 對話

交換數據的標準格式。最直觀的理解方式就是「餐廳點餐」流程：

🙋‍♂️ 請求 (Request)

點餐。包含網址、動作與參數。

🍛 回應 (Response)

送餐。包含狀態碼與具體內文。

🧠 Stateless

伺服器很健忘！連線獨立，不記名。

💡 Why Stateless?

服務大量請求。大規模擴展關鍵。

✉️ 2. 方法、標頭、正文

一封標準 Request 信：

GET
讀取菜單

請求數據。參數在網址後面。

POST
送交訂單

提交資料。封裝 Body，安全。

Headers
信封標註

備註。如宣告、Token。

URL ANATOMY 
http://192.168.4.1/move?dir=forward&speed=200


// GET 參數解析：
{ dir: "forward", speed: 200 }

🏷️ 3. 狀態碼

200 OK

成功！資料送達。

404 Found

找不到。網址錯了。

500 Error

崩潰。邏輯有誤。

課程單元 基礎 6 - 網路通訊基礎

HTTP 協議與數據交換實驗室：請求解構與強韌通訊防錯實戰

任務 1：「數位點餐員」：Postman GET 測試 (Digital Waiter: GET Lab)

角色扮演：測試工程師 | 難度：⭐️ | 估計時長：30 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫。
  2. 端點請求：發送 GET 請求並解析 JSON。
  3. 報告內容：彙整 Headers 與 Status Code。
  4. 安全性分析：說明 GET 安全隱憂。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 解讀準確：識別標頭與正文功能？
  • [ ] 勘查完整：JSON 欄位對齊規範？

任務 2：「數據整合師」：實作 POST 發送 JSON (Data Integrator: POST/JSON)

角色扮演：工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 封裝：ESP32 封裝傳感器 JSON。
  2. 通訊：配置 Content-Type 執行 POST。
  3. 驗證：證明伺服器正確解析。
  4. 提交：Push 穩定代碼。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 規格正確：JSON 符合 RFC 規範？
  • [ ] 配置準確：Header 設定正確 Content-Type？

任務 3：「通訊守門員」：狀態碼攔截與防錯 (Communication Sentry)

角色扮演：工程師 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 狀態攔截：實作碼分流邏輯。
  2. 壓力測試：模擬 404/500。
  3. 自動修復：實作重試補救機制。
  4. 資源考量：說明非阻塞重試策略。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 異常涵蓋：涵蓋 4xx/5xx/Timeout？
  • [ ] 非阻塞：重試期間不卡死設備主線程？
