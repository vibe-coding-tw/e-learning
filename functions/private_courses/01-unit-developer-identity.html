📗 實戰指南：建立你的雲端身分

Google 帳號申請

要申請新的 Google 帳號，最直接的方法是前往 Google 帳戶建立頁面。這將是你進入雲端世界的第一把鑰匙。

步驟教學：

  1. 前往 accounts.google.com/signup 或 Gmail 點選「建立帳戶」。
  2. 選擇「個人用途」。
  3. 輸入姓名、生日、性別與想要的使用者名稱。
  4. 建立一個高強度密碼 (建議包含大小寫字母、數字與符號)。
  5. (建議) 設定備援信箱或電話，方便日後救援。

💡 小提示

  • 手機設定：可至 設定 > 帳戶 > 新增帳戶 > Google 進行。
  • 不想用手機驗證？有些情況下可點選「跳過」。
  • 申請完成後，你將同時擁有 Gmail、雲端硬碟 (Drive) 等完整服務。

GitHub 帳號申請

Step 1: 註冊 (Sign up)

  • 前往 GitHub 官網 點擊右上角 "Sign up"。
  • Email: 建議使用常用的 Google 帳號。
  • 密碼: 需具備高強度。
  • Username: 這是你的開發者身分 (例如 github.com/RoverChen)，建議使用英文+數字，避免過於隨意。

Step 2: 驗證 (Verify)

  • 完成圖片旋轉或拼圖驗證 (證明是人類)。
  • 輸入 Email 收到的 8 位數驗證碼。
  • 填寫基礎問卷 (學生/開發者)，可快速勾選。

Step 3: 安全強化 (2FA)

⚠️ 極重要：為了保護你的程式碼，請務必開啟兩階段驗證。

  1. 進入 Settings -> Password and security
  2. 點擊 Enable Two-factor authentication
  3. 使用 GitHub Mobile App 或 Google Authenticator 掃描 QR Code。
學習重點： • 2FA (兩步驟驗證)：保護帳號不被盜用。 • Git 核心價值：為什麼我們需要版本控制？(多人協作、時光機)

開發者身分與數位通行證 (Developer Identity)

歡迎來到 Vibe Coding 課程的第一步！在這個單元中，你將建立屬於自己的開發者身分，並在 GitHub 上發射你的第一個訊號。這不只是關於技術操作，更是關於如何在這個開源世界中展示你自己。

🎯 學習目標

  • 掌握 Markdown 語法，撰寫專業的技術文檔。
  • 理解 GitHub 個人主頁與程式庫 (Repository) 的運作機制。
  • 強化帳戶安全性（啟用 2FA），保護你的數位資產。

📝 作業任務說明

本單元包含兩個主要任務，請依照以下步驟完成：

任務 1：打造數位身分通行證 (Digital Profile)

你的 my_profile.md 是你在開發者社群的「名片」。

  1. 編輯檔案：打開 my_profile.md 檔案。
  2. 更新內容：
    • 將標題改為你的名字。
    • 在 ## 🎯 About Me 寫下簡短的個人介紹。
    • 列出你的技術技能與感興趣的專案（必須包含 Markdown 清單語法）。
    • 放入你的 GitHub 或 LinkedIn 連結（必須包含 Markdown 超連結語法）。
  3. 上傳圖片：將你的頭像或代表圖片放入 /assets/ 資料夾中（此資料夾不可為空），並在 Markdown 中正確引用路徑。
  4. 安全切結：在檔案末尾確認你已完成 2FA（必須包含「2FA」或「兩步驟驗證」關鍵字）。

任務 2：發射第一個訊號 (Hello World)

練習建立並發布你的第一個公開程式庫。

  1. 建立程式庫：前往你的 GitHub 個人首頁，建立一個名為 hello-world 的 Public 程式庫。
  2. 初始化：在建立時勾選 "Add a README file"。
  3. 提交連結：將該程式庫的連結貼回本程式庫的 submission.md 檔案中。
    • 格式必須為：https://github.com/[你的帳號]/hello-world
    • 重要：請確保該程式庫設為 Public，否則自動評分系統將無法存取（404 錯誤）。

🛡️ 安全性提醒 (Security First)

在開發者世界，帳戶安全至關重要。請務必：

  • 進入 GitHub 設定，啟用 Two-Factor Authentication (2FA)。
  • 妥善保存你的 Recovery Codes。
  • 在 my_profile.md 的底部勾選確認你已完成此安全性設置。

🎓 導師指南：單元 01 開發者身分 (Tutor Guide)

針對 01-unit-developer-identity (開發者身分) 的作業部署與管理，本指南旨在協助導師有效利用 GitHub Classroom 建立學習流，並結合自動化檢測提升教學效率。

📌 單元教學核心

本單元不只是 Markdown 語法練習，更核心的目標是讓學生理解：

  • 數位身分 (Digital Identity)：如何在技術社群呈現專業形象。
  • 開源流程 (OSS Workflow)：建立、公開並提交一個真實的程式庫。
  • 安全意識 (Security)：實踐開發者必備的帳戶防護 (2FA)。

🛠️ 作業設計與部署建議

任務 1：數位身分通行證 (Digital Profile)

  • 部署方式：使用 Classroom Template 提供 my_profile.md 與 /assets/ 資料夾。
  • 教學邏輯：讓學生在受控環境（Classroom）練習，完成後引導其將內容同步至個人的 username/username 特殊程式庫，達成「學以致用」。
  • 評分重點：
    • [ ] Markdown 語法是否標籤正確（標題、清單、連結）。
    • [ ] 圖片資源路徑引用是否正確。
    • [ ] 個人介紹內容的完整度。

任務 2：發射第一個訊號 (Hello World)

  • 部署技巧：要求學生「手動建立」一個獨立於 Classroom 之外的公共程式庫，並將 URL 貼回 submission.md。
  • 教學目標：跳脫範本框架，讓學生練習真實的 GitHub 建立流程與權限設定（Public vs Private）。

🤖 自動化檢測機制 (Autograding)

本單元已整合 .github/scripts/check_assignment.sh 進行自動化評分，主要檢核指標如下：

檢查項目 檢測方式 失敗原因舉例
Markdown 語法 grep 檢查 my_profile.md 漏寫連結、未使用清單符號
資源完整性 檢查 assets/ 資料夾是否非空 忘記上傳圖片檔案
安全性切結 檢查關鍵字「2FA」或「兩步驟驗證」 未在檔案底部進行確認
連結有效性 curl -I 檢查 submission.md 內的 URL 連結錯誤或程式庫設為 Private (404)

💡 教學與反饋心法 (Tutor Tips)

  1. 回饋機制 (Feedback Pull Request)： 強烈建議開啟 GitHub Classroom 的 "Feedback Pull Request" 功能。導師可直接在學生的 Markdown 代碼處留言，點評其選圖或排版細節，建立「代碼審查 (Code Review)」的初步體驗。
  2. 關於 2FA 的驗證： 由於 Action 無法直接存取學生帳戶設定，因此採用「口頭切結」方式。建議導師在課堂上抽查學生的 GitHub 設定頁面，確保其不僅是填寫關鍵字，而是真正啟用了保護。
