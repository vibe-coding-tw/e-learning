📗 實戰指南：建立你的雲端身分

Google 帳號申請

要申請新的 Google 帳號，最直接的方法是前往 Google 帳戶建立頁面。這將是你進入雲端世界的第一把鑰匙。

步驟教學：

  1. 前往 accounts.google.com/signup 或 Gmail 點選「建立帳戶」。
  2. 選擇「個人用途」。
  3. 輸入姓名、生日、性別與想要的使用者名稱。
  4. 建立一個高強度密碼 (建議包含大小寫字母、數字與符號)。
  5. (建議) 設定備援信箱或電話，方便日後救援。

💡 小提示

  • 手機設定：可至 設定 > 帳戶 > 新增帳戶 > Google 進行。
  • 不想用手機驗證？有些情況下可點選「跳過」。
  • 申請完成後，你將同時擁有 Gmail、雲端硬碟 (Drive) 等完整服務。

GitHub 帳號申請

Step 1: 註冊 (Sign up)

  • 前往 GitHub 官網 點擊右上角 "Sign up"。
  • Email: 建議使用常用的 Google 帳號。
  • 密碼: 需具備高強度。
  • Username: 這是你的開發者身分 (例如 github.com/RoverChen)，建議使用英文+數字，避免過於隨意。

Step 2: 驗證 (Verify)

  • 完成圖片旋轉或拼圖驗證 (證明是人類)。
  • 輸入 Email 收到的 8 位數驗證碼。
  • 填寫基礎問卷 (學生/開發者)，可快速勾選。

Step 3: 安全強化 (2FA)

⚠️ 極重要：為了保護你的程式碼，請務必開啟兩階段驗證。

  1. 進入 Settings -> Password and security
  2. 點擊 Enable Two-factor authentication
  3. 使用 GitHub Mobile App 或 Google Authenticator 掃描 QR Code。
學習重點： • 2FA (兩步驟驗證)：保護帳號不被盜用。 • Git 核心價值：為什麼我們需要版本控制？(多人協作、時光機)

📉 進階閱讀：身分與安全的原理

為什麼需要 2FA？(安全對照表)

安全手段 本質 風險
密碼 (Password) 你知道的東西 容易被猜測、側錄或資料庫外洩。
2FA (Authenticator) 你擁有的東西 手機在身邊才能登入，駭客難以竊取。

任務 1：打造數位身分通行證 (Digital Profile)

角色扮演：開源冒險者 | 難度：⭐️⭐️ | 估計時長：30 min

🛠️ 執行技術流程 (Git Workflow)

  1. 接受挑戰：點擊老師提供的 GitHub Classroom 連結，建立專屬 Profile 倉庫。
  2. 建立本機副本：使用 git clone 將倉庫下載至電腦。
  3. 編輯 README：運用 Markdown 語法編寫自我介紹（包含標題、列表、技能標籤）。
  4. 同步雲端：完成後執行 git add, commit 並 push 回 GitHub。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 排版完整性：是否正確使用 Markdown 各級標題與列表？
  • [ ] 身分識別：是否已上傳清晰的 Avatar 頭像？
  • [ ] Profile 規則：倉庫名稱是否與使用者名稱一致（啟用 Special Repo 功能）？

任務 2：發射第一個訊號 (Hello World)

角色扮演：專案管理員 | 難度：⭐️ | 估計時長：15 min

🛠️ 執行技術流程 (Git Workflow)

  1. 手動建庫：在 GitHub 點選 "New Repository"。
  2. 公開權限：確保 Repository 設為 "Public"（這是開源精神的核心）。
  3. 初始化：勾選 "Add a README file"。
  4. 首發 Commit：在網頁介面或本機修改 README 並完成第一次提交。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 可見度檢核：確認倉庫為 Public 狀態。
  • [ ] 紀錄完整：README 是否包含單元名稱與基本說明？

GitHub Classroom 詳細操作指南

本單元教學重點：打造開發者的「第一張名片」。引導學生理解 Git 版本控制、公開倉庫與 Markdown 排版的實戰意義。

  1. 重點 1：數位身分認同 - 透過 2FA 設置與 Profile 頁面撰寫，讓學生感受到身為「開發者」的專業儀式感。
  2. 重點 2：從雲端到本機 - 掌握 Repo 與 Clone 的基本邏輯，這是後續所有代碼同步的基石。
  3. 重點 3：內容即代碼 - 介紹 Markdown 作為技術文件的標準格式，理解「純文字排版」的高效與易讀性。
