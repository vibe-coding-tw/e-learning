通訊協議與硬體實戰

📢 I2C：省線的「廣播系統」

  • ● 僅需兩根線：SDA（數據）與 SCL（時鐘）。
  • ● 定址機制：主機透過廣播「地址」來決定與哪個設備通訊。
  • ● Open-Drain 架構：理解為什麼 I2C 總線需要上拉電阻。

⚡ SPI：高速的「專線系統」

  • ● 通常四根線：SCK、MOSI、MISO、CS（片選）。
  • ● 全雙工傳輸：數據可同時雙向移動，速度極快 (MHz)。
  • ● 點對點機制：主機利用專屬的 CS 線啟動特定設備。

開始位元 (Start Condition)

I2C 的獨家通訊協議：當 SCL 為高電平時，SDA 從高變低，標誌著對話的開始。

時鐘極性與相位 (CPOL/CPHA)

SPI 的四種模式。決定數據是在時鐘的「上升沿」還是「下降沿」被固定並讀取。

💡 Datasheet 尋寶：嘗試找出 MPU6050 的預設 I2C 地址，以及它在通訊時的最高時鐘頻率限制。

進階課程 4 - 單元 1 通訊協議與硬體實務整合

硬體通訊實驗室：I2C/SPI 協議深論與整合

任務 1：I2C 設備大掃描 (I2C Scanner Lab)

角色扮演：韌體除錯工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 掃描實作：在 ESP32 上實作地址掃描迴圈（1-127），對每個地址發送 Start Signal。
  3. 設備對比：記錄所有回傳 ACK 的地址，並翻閱 Datasheet 確認與現有硬體（如 MPU6050, OLED）是否吻合。
  4. Push Final：提交具備「硬體自動偵測」能力的總線掃描工具。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 地址準確性：是否能正確識別出 OLED (0x3C) 與 MPU6050 (0x68) 的實體通訊地址？
  • [ ] 除錯完備度：若掃描不到設備，是否在 README 中列出了可能的硬體故障對策（如接線、上拉電阻）？

任務 2：SPI 刷新率挑戰 (SPI Speed Benchmark)

角色扮演：顯示驅動開發者 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 極限壓測：步進式提升 SPI 時鐘頻率（從 1MHz 直至硬體極限），觀察 OLED 螢幕刷新動態。
  2. 異常捕捉：紀錄在高頻（如 40MHz）通訊下是否出現影像雜訊、雪花或系統不穩定的現象。
  3. 性能分析：計算不同頻率下的每一幀像素寫入時間，並繪製成對比表格。
  4. Push Final：提交具備「極致顯示效能」且穩定的 SPI 驅動代碼。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 性能提升顯著性：最終實踐的 FPS 是否相較於預設配置有顯著提升（> 50%）？
  • [ ] 專業實驗報告：README 中是否解釋了「導線長度」對高頻訊號完整性的具體影響？

任務 3：協議翻譯官：Raw to JSON (Protocol Converter)

角色扮演：嵌入式系統架構師 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 字節流解析：從 I2C/SPI 感測器讀取 Raw Binary Data，使用位元與或結構體進行解算。
  2. 數據封裝：整合 ArduinoJson，將解算後的物理量（如加速度、角速度）轉換為標準 JSON 格式。
  3. 通訊整合：將產出的 JSON 透過串列埠或藍牙發送，實現底層硬體與高層 Web 的無縫對接。
  4. Push Final：提交具備「跨平台數據解耦」能力的通訊橋接包。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 數據換算精確度：轉換後的數值是否與 Datasheet 給出的 LSB/unit 比例完全吻合？
  • [ ] Payload 優化：生成的 JSON 結構是否簡潔高效，具備良好的開發者可讀性？

GitHub Classroom 詳細操作指南

本單元教學重點：通訊協議的「硬漢實力」。掌握 I2C/SPI 的時序物理意義與現代化數據封裝流程。

  1. 重點 1：匯流排定址與專線選擇 - 透過 1-1 掃描任務，強化學生對 Master-Slave 架構中設備辨識的底層認知。
  2. 重點 2：物理層的極限探索 - 利用 1-2 任務探討時脈頻率對通訊質量的影響，理解數位訊號在高速下的物理變形與雜訊。
  3. 重點 3：二進制與語義化轉換 - 教導學生如何將「冷冰冰的 Hex」轉換為「易懂的 JSON」，這是 IoT 全棧開發的關鍵能力。
