HTML5 Canvas 影像繪製與處理

🎨 1. Canvas 畫布：數位繪圖的物理基礎 (20 min)

座標系統：理解 Canvas 的原點 (0, 0) 位在左上角。$x$ 軸向右增加，$y$ 軸向下增加（這與傳統笛卡兒座標系不同）。

🖌️ Context (2D)

它是你的「畫筆」，所有的繪圖指令（如 fillRect, drawImage）都必須透過它發送。

📷 影像來源

學習如何將 <img> 標籤或 <video> 標籤的內容「捕捉」到 Canvas 中實施即時處理。

座標對比：

  • Web Canvas: y 軸向下為正
  • 常見數學: y 軸向上為正

🔍 學習任務

在紙上畫出一個 $320 \times 240$ 的網格，標註出 $(160, 120)$ 的位置，並思考為什麼高頻率重繪（如 30 FPS）需要 requestAnimationFrame。

🧬 2. ImageData：揭開像素的秘密 (25 min)

RGBA 陣列：當我們執行 getImageData() 時，會得到一個超長的一維陣列 Uint8ClampedArray。

儲存規律：每 4 個數字代表一個像素

R G B A ...
  • R, G, B: 紅綠藍 (0-255)
  • A: 透明度 (Alpha, 0-255)

📐 索引公式 (Index Formula)

學習如何在二維影像中定位一維陣列的索引：

$Index = (y \times Width + x) \times 4$

* 乘以 4 是因為每個像素包含 R, G, B, A 四個分量。

🧮 3. 濾波與變換的數學邏輯 (15 min)

影像處理核心概念

  • 灰階化：將 R、G、B 取平均值。
  • 水平翻轉 (Mirroring)：座標變換的應用。
  • 二進位影像 (Blob)：學習如何將 Blob 繪製到畫布。

效能與優化

<img> 標籤與 <canvas> 繪圖的效能差異。

線上任務：

撰寫 JS，實作在影像上方疊加「十字準星」或「網格線」。

任務 2：視覺準星與動態標註 (Visual Overlay System)

任務 3：鏡像修正與 ROI 提取 (Mirroring & ROI)

角色扮演：視覺導航工程師 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 水平鏡像：解決自拍鏡頭帶來的左右相反問題，利用 $x_{new} = Width - 1 - x_{old}$ 進行像素搬移。

  3. 數據封裝：將 ROI 數據轉換為 Base64 格式並模擬上傳流程。
  4. Push Final：交付「具備感知前處理能力」的視覺處理器。

GitHub Classroom 詳細操作指南

本單元教學重點：掌握「影像主權」。讓學生明白網頁端如何接管影像數據並加上 UI 回饋。

  1. 重點 1：Blob 到 ImageBitmap - 強調 MJPEG 只針對一張圖，而 H.264 針對「時間軸上的差異」。
  2. 重點 2：ROI (Region of Interest) 的座標映射 - 實驗 2-2 是最有感的，讓學生看見演算法「預測」需要代價（延測）。
  3. 重點 3：低延遲標記 - 引導學生不只看技術參數，要從「使用者需求（遙控 vs 存擋）」出發做技術選型。
