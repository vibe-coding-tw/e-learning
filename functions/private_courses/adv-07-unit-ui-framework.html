🧱 1. 什麼是元件化設計 (Component-Based Design)？ (25 min)

將複雜的網頁視為樂高積木。每個元件（如「馬達狀態列」）應完全封裝自己的 HTML 結構、CSS 樣式與狀態邏輯，實現「高內聚、低耦合」。

Design Pattern

  • 原子元件 (Atoms)：按鈕、輸入框、標籤。
  • 組件 (Organisms)：導覽欄、遙測儀表盤、通訊面板。

🔌 2. 資料傳遞：Props 與 Events (25 min)

元件之間通訊的橋樑。父元件透過 Props 注入數據（如藍牙連線狀態），而子元件透過 Events 向外發出通知（如搖桿座標變更）。

⚡ 3. 響應式佈局與遙測視覺化

使用 CSS Grid 與 Flexbox 打造「數據驅動 (Data-Driven)」的儀表板。當資料狀態 (State) 變更時，介面應自動響應，無需手動操作 DOM 節點。

Layout Efficiency

  • 單向更新流：告別 document.getElementById，擁抱狀態同步。
  • 響應式網格 (Responsive Grid)：在大螢幕顯示全儀表，在手機端自動切換為縱向控制欄。
  • 元件生命週期：理解元件何時「掛載 (Mount)」與「卸載」，防止藍牙回調造成的內存洩漏。

「通用原子元件」

設計一個接收 label, value, color 的數據組件。實作當電量低於 20% 時背景自動變紅的邏輯。學會建立可多次調用但表現不同的基礎元件。

單元課程 進階 7 - UI 框架

架構實驗室：前端元件化與狀態通訊

任務 1：通用原子組件 (Atomic Data Component)

角色扮演：UI 系統設計師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 類別/函式宣告：在 components/Gauge.js 中定義組件類別。
  3. 屬性 (Props) 處理：實作 render() 函式，根據傳入的 this.props.value 動態改變 CSS 類名（例如：數值 > 100 報警紅）。
  4. 多實例化測試：在 index.html 中同時呼叫三次組件，分別顯示「電池」、「轉速」、「溫度」。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 封裝完整度：組件的樣式是否會意外「溢出」影響到頁面其他按鈕？
  • [ ] 代碼複用力：是否只需透過修改參數即可產生截然不同的顯示效果？

任務 2：事件發射器 (Joystick Event Emitter)

角色扮演：交互開發專家 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. Canvas 封裝：將原本寫在全域的搖桿代碼移入 Joystick 類別中。
  2. 自定義事件 (Custom Events)：當 pointermove 觸發時，發射包含 {x, y} 的 'joystick:change' 事件。
  3. 父子通訊測試：在 app.js 監聽該事件，並呼叫藍牙發送模組。
  4. Push Final：提交具備「關注點分離」架構的控制核心。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 架構層次感：搖桿組件內部是否真的「不含有」任何關於藍牙發送的代碼？
  • [ ] 事件解耦力：更換父層邏輯後（例如改為儲存數據而非發送），搖桿組件是否能不經修改直接運行？

任務 3：響應式佈局重構 (Responsive Dashboard Refactor)

角色扮演：佈局工程師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. Grid 系統建立：設計 .dashboard-grid，在手機版使用 1fr，電腦版使用 repeat(3, 1fr)。
  2. 狀態驅動樣式：當連線成功時，給予 一個 .is-connected 類名，全局 CSS 自動切換背景色調。
  3. 提交截圖報告：在 README 中貼上橫屏與豎屏的對比截圖。
  4. Push Final：完成具備生產環境等級的遙控儀表板。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 適配精確度：在不同寬度下，組件是否會發生嚴重的重疊或破圖？
  • [ ] 視覺專業感：配色與間距是否符合 Vibe Coding 的 Premium 審美標準？

GitHub Classroom 詳細操作指南

本單元教學重點：讓學生體驗「宣告式 UI」相對於「指令式 DOM 操作」的維護優勢。

  1. 重點 1：封裝思維 - 學生常習慣把所有代碼寫在 app.js。必須強制練習將介面拆分為獨立的組件檔案。
  2. 重點 2：資料流向 - 明確區分 Data (Props) 進與 Signaling (Events) 出的單向循環。
  3. 重點 3：響應式基礎 - 雖然有 Tailwind，但理解原生 CSS Grid 與 Flexbox 的原理對未來掌握 React 元件庫至關重要。
