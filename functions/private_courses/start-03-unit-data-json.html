為什麼我們需要 JSON？學習如何將搖桿座標與按鈕狀態，轉換為 ESP32 能聽懂的通用語言。

🧪
1. 課前體驗：為什麼是 JSON？

想像你要傳送「向左轉，速度 100」給無人車。如果只是單純拼接字串，接收端會很難解析。 JSON 因應而生，它像是一個標準化的包裹，讓發送端 (Web) 與接收端 (ESP32) 都能精準理解。

⛔ 如果用字串拼接...

"L-100-MOVE" ← 順序錯了就完了！

✅ 使用 JSON 標準格式

{"dir": "L", "speed": 100} ← 清晰明確

2. 核心原理：JSON.stringify 與 ESP32 解析

JavaScript 端：序列化

JSON.stringify() 是我們最強大的助手。但要注意，它會過濾掉它不認識的東西！

console.js 🔴 Live Demo
const
data
= {
  speed: 100,
  func: () => {}, // 🙈 會消失
  val: undefined // 🙈 會消失
};

JSON.stringify(data)

↳ "{"speed":100}"

ESP32 端：ArduinoJson

微控制器沒有無限的記憶體。使用 StaticJsonDocument 來預先配置固定的記憶體空間，避免記憶體碎片化。

C++ / Arduino
String input ➔ "{\"cmd\":\"move\"}"
Deserialization ➔
StaticJsonDocument<200> doc;
deserializeJson(doc, input);
const char* cmd = doc["cmd"];

💡 key-Value 設計思維

為什麼不直接傳送 "前進"？因為中文編碼複雜且佔用頻寬。使用 {"cmd": "move", "val": 255} 能兼顧人類可讀性與機器解析效率。

數據格式化：從物件到 JSON

任務 1：基礎封裝與序列化 (Serializing Objects)

角色扮演：物聯網開發者 | 難度：⭐️⭐️ | 估計時長：40 min

🛠️ 執行技術流程 (Git Workflow)

  1. 物件構建：在 JavaScript 中定義包含 direction, speed 與 timestamp 的指令物件。
  2. 標準化轉換：使用 JSON.stringify() 將物件轉換為字串，並輸出至 Console 驗證格式是否為標準雙引號。
  3. 異常處理：測試當物件包含 undefined 或 function 時，序列化後的結果會如何變化。
  4. 初步提交：將基礎序列化函式提交至 GitHub。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 語法正確性：輸出的字串是否符合標準 JSON 規範（雙引號、大括號成對）？
  • [ ] 時間戳記：物件是否包含動態產生的 Date.now() 確保封包唯一性？

任務 2：封包瘦身與通訊頻寬優化

角色扮演：韌體通訊工程師 | 難度：⭐️⭐️ | 估計時長：40 min

🛠️ 執行技術流程 (Git Workflow)

  1. Key Name 縮寫：將 command 簡化為 c, velocity 簡化為 v，以節省字元空間。
  2. 長度對照：計算簡化前後的字節數 (Bytes)，確保封包長度縮減 50% 以上。
  3. 精簡驗證：確保在縮寫後，接收端 (ESP32) 仍能透過約定的 Key 取得正確數值。
  4. 技術提交：提交優化後的短碼協議設計。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 瘦身成效：最終 JSON 字串長度是否成功壓低在 20 bytes 以內？
  • [ ] 邏輯定義：README 中是否清晰說明了縮寫與原始含義的映射關係？

任務 3：狀態機巢狀資料結構

角色扮演：嵌入式系統架構師 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 層級設計：構建一個巢狀物件，分離「搖桿座標 (joy)」與「按鈕狀態 (btn)」。
  2. 結構對齊：確保資料層級不超過三層，平衡資料清晰度與解析難度。
  3. 自動化測試：撰寫簡單的測試案例，模擬不同操作組合下的資料正確性。
  4. 正式提交：將完整的資料協議方案 Push 至 GitHub。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 結構合理性：巢狀結構是否邏輯分明，易於在 C++ 端（ArduinoJson）進行解析？
  • [ ] 擴展性預留：在不破壞現有結構的前提下，是否能輕易加入新的感測器數據欄位？

B. 參考範例代碼 (Reference Example Code)

protocol-helper.js (封裝與縮寫)

// 基礎封裝 (Task 1)
function createPacket(btnId) {
    const data = {
        btn: btnId,
        ts: Date.now()
    };
    return JSON.stringify(data);
}

// 封包瘦身 (Task 2)
// 使用單字元 Key 可節省約 50% 頻寬
function createOptimizedPacket(cmd, val) {
    return JSON.stringify({ c: cmd, v: val });
}

// 巢狀結構 (Task 3)
const joyData = {
    joy: { x: 10, y: -5 },
    btn: { a: 1 }
};

C. 自動評分 (Autograding) 的設計

  • 字節檢查： 使用 Actions 檢查產生的 JSON 字串長度（優化挑戰要求 < 20 bytes）。
  • 語法檢查： 驗證學生產出的 JSON 是否包含正確的嵌套 Key (joy/btn)。
