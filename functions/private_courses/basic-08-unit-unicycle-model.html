🚲 1. 什麼是 Unicycle Model？

將機器人簡化為一個運動點。我們控制兩個宏觀參數：

v 線速度 (Linear)

代表整體前進速度。搖桿 Y 軸。

ω 角速度 (Angular)

代表轉向速度。搖桿 X 軸。

Velocity Decomposition

建立直覺操控基礎：將「去哪」與「怎麼轉」解耦。

MotorMix = f(v, ω)

⚙️ 2. 差速驅動與混控公式

左右兩輪獨立驅動。轉向必須在輪間產生「速度差」。

\[L = v - \omega\]
Left Motor
\[R = v + \omega\]
Right Motor
MIXER LOGIC (C++) 
void applyMix(float v, float w) {
  float left = v - w;
  float right = v + w;
  motorL.setSpeed(left * 255);
  motorR.setSpeed(right * 255);
}

🛑 3. 飽和處理：限幅思維

若馬達已最大，直接裁切 (Clipping) 導至轉向失靈。

Normalizing Mixers

  • 等比例縮放：溢出時同步縮小兩輪，保留轉向比。
  • 防裁切機制：全速行駛仍能順滑蛇行。
  • 飽和度判定：使用 max(|L|, |R|, 1.0) 作分母。

Robust Control

"比例大於絕對值。在極限保留轉向意圖。"

課程單元 基礎 8 - 運動學轉換

差速轉換數學 (Unicycle Model)

本單元的目標是學習如何將機器人的抽象運動指令（線速度 $v$ 與角速度 $\omega$）轉換為實體的左、右馬達輸出。這是一個關鍵的技術里程碑，你將學習如何處理硬體限制並守護操縱意圖。

任務目標

任務 1：基礎差速混合器實作 (Kinematics Mixer)

  • 目標：實作 Unicycle Model 到差速傳動的幾何映射。
  • 實作檔案：src/mix_engine.cpp
  • 公式：
    • $L = v - \omega$
    • $R = v + \omega$
  • 驗證：確保當 $\omega > 0$（向右轉）時，右輪轉速降低，左輪轉速增加。

任務 2：飽和度處理與比例縮放 (Saturation Scaling)

  • 目標：在馬達輸出達到上限時，透過等比例縮放來保護轉向意圖。
  • 挑戰：當 $v + \omega > 1.0$ 時，直接裁切 (Clipping) 會導致轉向失效。
  • 解決方案：實作 output = target / max(|L|, |R|, 1.0)。

任務 3：轉向敏感度調教 (Dynamic Sensitivity)

  • 目標：引入轉向係數 $K$，根據物理環境調整操縱感。
  • 公式：$\omega' = K \cdot \omega$

如何開始

  1. 克隆此 程式庫。
  2. 打開 src/mix_engine.cpp。
  3. 根據 include/mixer_def.h 中的定義完成 mix_unicycle 函式。
  4. 運行測試以驗證你的實作： 
    pio test -e native

交付清單

  • [ ] 通過所有的單元測試。
  • [ ] (選修) 在不同的環境下測試並調整 $K$ 值。

2. 作業任務部署細節

任務 1：基礎差速混合器實作 (Kinematics Mixer Lab)

  • 目標：掌握 Unicycle Model 到差速傳動的幾何映射，確保指令（要去哪）與執行（輪子轉速）的精確對齊。
  • Classroom 部署建議：
    • 座標對齊：實作基礎混控。技術要領：當 $\omega > 0$（向右轉）時，右輪轉速應降低，左輪轉速應增加。
    • 原地自旋驗證：輸入 $v=0$ 且 $\omega$ 不為零，確認機器人是否能繞幾何中心點進行完美的「坦克轉向」。
    • 驗證要點：馬達轉向方向是否與搖桿邏輯一致？原地自旋時左右輪轉速是否絕對對稱？

任務 2：飽和度處理與比例縮放 (Saturation Scaling Lab)

  • 目標：掌握「意圖導向」的控制策略，學習如何在硬體輸出溢出時，透過等比例縮小速度來保護轉向意圖。
  • Classroom 部署建議：
    • Clipping 現象觀察：在全速 $v=1.0$ 下輸入轉向指令，觀察單純裁切邏輯為何會造成「轉不動」的控制鎖死。
    • 縮放演算法實作：實作 output = target / max(|L|, |R|, 1.0)。
    • 極限操控實測：確保在馬達全開的情況下，機器人依然能執行流暢的「大轉彎」動作。
    • 驗證要點：關鍵技術審核。學員是否有處理分母為零的異常？當 $v, \omega$ 總和超過 1.0 時，機器人是否能維持轉彎弧度的比例而非直行？

任務 3：轉向敏感度調教 (Dynamic Sensitivity Lab)

  • 目標：建立「環境感知」的調校思維，理解不同摩擦係數的地表需配合不同的 $K$ 值，以達成最佳操控感。
  • Classroom 部署建議：
    • 引入係數 K：將角速度指令變更為 $K \cdot \omega$，並測試不同 $K$ 值對繞錐效率的影響。
    • 地表適配報告：分別在「地毯」、「磁磚」與「粗糙水泥地」測試，尋找各環境下的黃金 $K$ 值。
    • 交付報告：在 README 中說明：高摩擦力地表為何需要更大的轉向係數？
    • 驗證要點：代碼架構是否支持動態調整 $K$ 值？調校後的機器人是否在該地表下反應更加靈敏而不失控？

3. 動態混合與控制韌性點評標準 (Resilient Control & Mixer Standards)

此單元的價值在於 「對硬體極限的數學預見力與對操縱意圖的物理保護力」：

  • [ ] 模型轉化準確度：馬達行為是否完美呼應幾何運動學定義？
