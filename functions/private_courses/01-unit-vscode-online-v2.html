一鍵啟動：github.dev 的祕密

GitHub 內建了一個幾乎沒有人在官方文件首頁強調、卻極為好用的功能： 只要在任何 Repository 頁面按下 .（句點鍵）， 就能立刻在瀏覽器中開啟 VS Code 介面。

網址的變化

這個快捷鍵的背後邏輯非常簡單：

狀態網址說明
按下 . 前 github.com/user/repo GitHub 標準 Repository 頁面
按下 . 後 github.dev/user/repo VS Code 網頁版介面，直接載入同一個 Repo
小技巧 你也可以直接在網址列把 .com 改成 .dev 達到同樣效果， 或手動輸入 vscode.dev 使用獨立版本（不綁定特定 Repository）。

限制說明

github.dev 是個輕量工具，有幾個重要限制需要了解：

  • 無法執行程式：沒有終端機，無法跑 python main.py、npm start 等指令。
  • 擴充套件受限：只有支援 Web 的擴充套件才能安裝，許多功能強大的套件無法使用。
  • 讀取本機檔案受限：無法存取本機硬碟（除非透過瀏覽器的 File System Access API）。
最佳使用場景 github.dev 最適合「閱讀程式碼」和「輕量文字編輯」。 需要執行程式時，請改用 GitHub Codespaces。

GitHub Codespaces：完整雲端開發環境

當你需要真正執行程式時，GitHub Codespaces 是答案。 它在 GitHub 的雲端伺服器上啟動一個完整的 Linux 虛擬機， 讓你在瀏覽器中擁有和本機開發一樣完整的 VS Code 體驗——包含終端機、套件安裝和程式執行。

情境
小華拿到了一個 ESP32 Arduino 專案，需要跑 npm install 安裝前端套件， 並確認程式碼邏輯沒有問題。他的 MacBook 帶去修理了， 但用 Codespaces，他在圖書館的公用電腦上完成了這一切， 程式碼直接 commit 回 GitHub，完全沒有「環境問題」。

啟動 Codespace

1

前往 GitHub Repository

開啟你想開發的 Repository 頁面。

2

點擊 Code 按鈕

點擊綠色的 Code ▾ 按鈕，切換到 Codespaces 分頁。

3

建立 Codespace

點擊 Create codespace on main，等待雲端環境啟動（約 30–60 秒）。

4

開始開發

環境就緒後，你會看到和桌面版幾乎完全相同的 VS Code 介面， 包含左側的檔案樹、下方的終端機（Terminal），以及完整的 Git 整合。

Codespaces 終端機示範

以下是在 Codespaces 終端機中常用的指令：

TERMINAL

免費額度說明 GitHub 免費帳號每月提供 120 核心小時的 Codespaces 使用時間（以最小 2 核心計算，約 60 小時）。 超出後需付費。 用完後請記得停用（Stop codespace）而非直接關閉瀏覽器，以避免不必要的計費。

管理你的 Codespace

操作方式說明
停用左下角 ⚡ → Stop Current Codespace停止計費，資料保留
重新啟動github.com/codespaces在管理頁面重新啟動
刪除github.com/codespaces → ⋯ → Delete釋放儲存空間

自動儲存 Codespaces 中的所有修改都會自動同步到 GitHub 的暫存空間， 即使停用後也不會消失，直到你手動刪除或 30 天後自動清除（未推送的變更）。

實作練習：雲端開發者認證

理論已經就緒，現在動手驗證你的學習成果。 本練習共三個任務，難度遞增，完成後你將具備在任何裝置上進行雲端開發的能力。

  作業 1
約 10 分鐘

Codespaces 一鍵啟動與測試驗證

完成從啟動 Codespace 到修正測試錯誤的完整循環，體驗雲端開發的工作流程。

  • 在指定 Repository 頁面，透過 Code → Codespaces → Create codespace on main 啟動雲端環境
  • 等待環境就緒後，在終端機執行測試指令（如 npm test 或 python -m pytest）
  • 找出一個失敗的測試項目，修正對應的程式碼
  • 再次執行測試，確認通過
  • 將修正過程的截圖或說明寫入 commit message，並提交
繳交方式 在 commit message 中說明你修正了什麼，並把 commit 連結傳給助教。
  作業 2
約 15 分鐘

Codespaces 雲端安裝套件

體驗在瀏覽器中執行 Command Line 的感覺——這是許多人第一次「不用本機」就完成的開發任務。

  • 在 Codespaces 終端機執行 npm install，等待套件安裝完成
  • 觀察安裝過程的輸出訊息，確認安裝成功
  • 完成後，執行 npm start 或對應的啟動指令，確認專案可以執行
  • 截圖終端機的執行結果，作為繳交依據
  • 完成後使用 Stop Current Codespace 正確停用環境
  作業 3
約 10 分鐘

跨裝置同步：用手機編輯程式碼

拿出你的手機或平板，驗證「任何裝置都能開發」的核心概念。

  • 用手機瀏覽器開啟 github.com，前往你的 Repository
  • 在網址列手動將 .com 改為 .dev，或按下頁面上的句點按鈕
  • 在手機的 VS Code 介面中，開啟任一程式碼檔案並修改一行內容（如 README 的文字）
  • 使用左側 Source Control（Git 圖示）填寫 commit message 並提交
  • 切換回電腦，確認 GitHub Repository 上已出現剛才的 commit
驗證成功標準 在電腦上的 GitHub Repository 頁面，能看到來自手機的 commit 記錄， 即代表跨裝置同步成功。
