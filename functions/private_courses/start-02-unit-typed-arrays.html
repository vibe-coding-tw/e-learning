Module 2: Core Principles

👓 類型化陣列是你的眼鏡

ArrayBuffer 是一塊原始的記憶體空間，裡面只有 0 和 1。我們無法直接操作它，必須戴上不同度數的「眼鏡」——也就是 Typed Array (如 Uint8Array, Int16Array) 才能解讀其中的意義。

每個格子被視為獨立的數字 (0-255)。

⇄ 位元組順序 (Endianness)

當數字超過 8 bits (例如 1000 = 0x03E8) 時，它需要佔用兩個位置。先放高位 (0x03) 還是低位 (0xE8)？這就是 Big-Endian 與 Little-Endian 的差異。

任務 2：還原感測器的 16-bit 數值 (Data Decoder)

角色扮演：數據逆向工程師 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立測試 Buffer：先準備一組 2-byte 範例資料，明確標出高位與低位。
  2. 使用 DataView 解碼：用 getUint16() 還原數值，並比較 Big-Endian 與 Little-Endian 的差異。
  3. 寫下觀察：在註解或 README 說明為什麼同一組位元組在不同端序下會得到不同結果。
  4. 同步到雲端：提交解碼函式、測試案例與說明文件。

// 3-1: 馬達指令封裝 (Uint8Array)
function encodeMotorSpeed(left, right) {
  const buffer = new Uint8Array(2);
  buffer[0] = Math.min(255, Math.max(0, left));
  buffer[1] = Math.min(255, Math.max(0, right));
  return buffer;
}

// 3-2: 感測器數據解碼 (DataView)
function decodeSensorData(buffer) {
  const view = new DataView(buffer);
  // 使用 Big-Endian (預設) 或 Little-Endian
  // return view.getUint16(0, false); 
  return view.getUint16(0, true); 
}

C. 自動評分 (Autograding) 的設計

  • Buffer 長度檢查： 檢查 3-1 產出的 ArrayBuffer 長度是否精確為 2。
  • 數值測試： 傳入 0x03E8 緩衝區，檢查對應 Endianness 下的解碼結果是否符合預期。
