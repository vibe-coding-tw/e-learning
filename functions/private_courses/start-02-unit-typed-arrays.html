Module 2: Core Principles

透視 ArrayBuffer 與位元組順序 (Endianness)

👓 類型化陣列是你的眼鏡

ArrayBuffer 是一塊原始的記憶體空間，裡面只有 0 和 1。我們無法直接操作它，必須戴上不同度數的「眼鏡」——也就是 Typed Array (如 Uint8Array, Int16Array) 才能解讀其中的意義。

Memory (ArrayBuffer) 0x00 - 0x03
11111111
11111111
00000000
00000000
每個格子被視為獨立的數字 (0-255)。

⇄ 位元組順序 (Endianness)

當數字超過 8 bits (例如 1000 = 0x03E8) 時，它需要佔用兩個位置。先放高位 (0x03) 還是低位 (0xE8)？這就是 Big-Endian 與 Little-Endian 的差異。

數據解析與類型化陣列 (Typed Arrays) 作業指南

歡迎來到 start-02-unit-typed-arrays 單元！

在這個單元中，我們將從「標籤化數據 (JSON)」邁向「位元級操作 (Binary Data Manipulation)」，這是一個前端工程師邁向底層與硬體通訊的重要里程碑。你將學習如何利用 ArrayBuffer 與 TypedArray 進行記憶體級別的操作，並掌握「位元組順序 (Endianness)」這項跨平台通訊的必備知識。

🎯 作業目標

本次作業包含兩個核心實驗，你需要完成通訊解析器框架的實作與邊界測試：

任務 1：雙通道馬達指令封裝 (Binary Encoder Lab)

將抽象的速度指令封裝為硬體可理解的 2-byte Payload，並掌握 Uint8 範圍限制與索引順序。

  • 實作封包邏輯：在 parser.js 建立函式，將左右馬達速度 (0-255) 依序寫入一個 Uint8Array(2)。
  • 數值範圍保護：實作「鉗位 (Clamping)」邏輯，處理輸入負數或大於 255 的例外狀況（如強制降為 0 或升為 255），確保系統安全。

任務 2：還原感測器的 16-bit 數值 (Data Decoder & Endianness Lab)

破解跨平台通訊的核心陷阱，學會利用 DataView 處理高低位元組並應對不同的端序 (Endianness)。

  • 解碼器實作挑戰：將一個 16-bit 的十六進位值（如 0x03E8）還原成十進位整數。
  • 端序切換練習：分別讀取大端序 (Big-Endian) 與小端序 (Little-Endian) 下的數值，並觀察結果的差異。

3. 位元組精準度與封包效率點評標準 (Binary Precision & Efficiency Standards)

此單元的價值在於 「對數值位元的極致簡潔與對底層兼容性的判斷力」：

  • [ ] 封包穩定性：產出的 ArrayBuffer 格式是否穩定，沒有出現不必要的動態長度變化？
  • [ ] 邊界安全性：面對異常的輸入數據，封裝器是否展現出預期的健壯性（Clamping 或是報錯提示）？
  • [ ] 端序邏輯清晰度：學員對於「高位在前」還是「低位在前」是否有明確的設計思維而非猜測？
