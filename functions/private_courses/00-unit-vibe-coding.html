主流 Vibe Coding 工具旗艦店

Replit Agent

最強一條龍：自動建立資料庫、後端與部署，連環境都不用看。

Lovable.dev

審美天花板：UI 極其精緻，擅長處理複雜前端交互與 Supabase 整合。

Cursor / Windsurf

本地 IDE 王者：理解整個 Repo 的架構，處理複雜邏輯重構的標竿。

Claude Code

終端機極速 Agent：直接在 CLI 運行，自主掃描修復 Bug 並提交 Commit。

ORCHESTRATION

Antigravity：指揮多位 Agent

在 Antigravity 中，handle 多個 agents 的核心在於利用其獨特的「雙視窗架構」。你可以同時讓 Agent A 重構後端，Agent B 設計前端。

  • 任務指揮中心 (Mission Control) 批次核准 Artifacts
  • rules.md 定義「協作 DNA」防止檔案衝突
  • Git Worktrees 實體隔離不同的開發嘗試

@Architect { role: "Planning", scope: "Structure" }

@Coder { role: "Implementation", scope: "src/" }

為什麼 Agent 無法直接部署？

安全沙盒 (Security Sandbox)

部署涉及基礎設施權限與敏感密鑰 (API Keys)。為確保安全，Agent 運行在隔離環境，無權直接存取你的正式伺服器。

最佳實踐：AI 寫 Code, CI/CD 部署

讓 Agent 撰寫 GitHub Actions YAML 腳本。你審核並合併 PR 後，由自動化流水線完成部署，這才是最專業的流程。

作業指引 (Assignment Guide)

1. GitHub Classroom 設定

請將 vibe-coding-action-lab 連結至課程儀表板。學生需透過此分發倉庫上傳他們使用 Vibe 工具完成的作品截圖與 Prompt 紀錄。

2. 翻轉課堂策略

  • 課前預習：註冊 Lovable, Replit, 與 v0.dev，並體驗其中一個工具產出基本的 UI 元件。
  • 課中實作：進行「Speed Coding」對決，使用不同工具在限定時間內實作同一個功能的 UI/UX。

3. 單元任務 (Unit Tasks)

任務說明 配分
【作業 3-1】展示一個由 v0 或 Lovable 生成並部署的完整頁面連結 35 pts
【作業 3-2】建立一個含資料庫讀取的 Replit 全棧應用 35 pts
【作業 3-3】提供一段經過 Claude Code CLI 修復過後的 Debug Log 紀錄 30 pts

Vibe Coding 實戰教學指引

輔導建議：

Vibe Coding 強調「快而準」。老師在批改時應重點關注學生是否能描述出代碼背後的「Prompt 邏輯」，而非單純的代碼正確性。

專案託管：

建議引導學生將成品部署至 Netlify 或 Vercel，展現完整的開發閉環。
