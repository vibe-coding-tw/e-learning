流暢自駕的秘密：
預覽控制 (Look-ahead)

為何機器人必須向前看？學習如何透過預覽遠方目標點，實現平滑且高速的自動導航。

🔭 1. 為什麼要「往遠處看」？ (20 min)

感測器擷取、AI 計算到馬達轉向都有延遲。如果只看車頭正下方，高速時會衝出彎道。預覽遠方可以讓轉向動作更提早、更柔和。

Control Benefits

  • 震盪抑制：解決蛇行震盪問題。
  • 預判過彎：在入彎前就開始切換轉向舵角。

🔄 3. 純追蹤 (Pure Pursuit) 概念與切彎問題

預覽過遠會導致「截彎取直 (Cutting Corners)」現象。學習如何控制預覽長度，在「過早轉彎」與「來不及轉彎」之間取得平衡。

Path Geometry Control

  • 切西瓜問題：視線過遠，車輛會提早切入彎道內側。
  • 追蹤延遲：視線過近，高速下會產生嚴重的超調 (Overshoot)。
  • 幾何補償：透過 $K_p$ 係數與 Y 軸高度的連動優化。

Steering Correction

$\delta = \arctan(\frac{2L \sin \alpha}{l_d})$

"預防勝於治療：看遠方就是為了提早應對。"

⚡ 60 min

【作業 4-2】

「自適應預覽」

撰寫動態高度函式 $Y_{target} = f(Speed)$。速度愈快，掃描線愈往上移（看愈遠）。測試此機制對高速直線路徑震盪的抑制效果。

🔧 45 min

【作業 4-3】

「預覽整合除錯」

解決「截彎取直」問題。調校比例增益與預覽高度，繪製車頭中心與預覽點的連線，優化出具備專業自駕質感的入彎軌跡。

單元課程 進階 6 - 預覽控制

實體實驗室：預覽控制系統調校

任務 1：實作「多層次 ROI」檢測器 (Multi-level ROI Detector)

角色扮演：自駕視覺工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 掃描線開發：在 Tracker.js 中定義三個掃描高度（如 Y=200, 150, 80）。
  3. 視覺化回饋：在 Canvas 渲染器中，用不同顏色畫出這三層的質心 Target 標註，觀測入彎時「遠處先偏、近處後偏」的物理特性。
  4. 數據擷取：紀錄一組入彎過程中的三點 X 座標變動，作為後續控制補償的依據。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 標註準確度：三層掃描點是否皆能正確對齊賽道中心線？
  • [ ] 渲染層次感：Canvas 上是否具備清晰的輔助虛線，輔助工程師除錯預覽行為？

任務 2：自適應預覽距離 (Adaptive Look-ahead)

角色扮演：控制系統優化師 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 適應性算法實作：撰寫 getDynamicLookHeight(speed) 函式。確保在 Speed 增加時，Y 座標線性（或非線性）減小（影像座標系中減小代表往上看）。
  2. 穩定性測試：在直道測試高速行駛下，適應性預覽是否能平滑微小的像素震盪。
  3. 分析報告：在 README 中記錄對比「固定低位」、「固定高位」與「動態位」下的循跡震盪頻率分析。
  4. Push PR：提交具備動態適應能力的導航引擎。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 控制流暢度：系統在大範圍提速時，轉向動作是否依然能維持柔和而不產生突發性抖動？
  • [ ] 邊界數值合理性：預覽高度是否設定在合理範圍（不至於看天空、也不至於只看車頭皮）？

任務 3：預覽點與 P-Control 整合除錯 (Look-ahead Integration)

角色扮演：系統整合專家 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. Error 鏈路串接：將遠方預覽點的偏差 $E_{dist} = \bar{x}_{far} - 160$ 作為 P-Control 的主要輸入。
  2. 截彎取直修正：觀察車輛是否太早入彎。調整 $K_p$ 係數與「預覽寬度放大倍率」，直到循跡中心能完美貼合賽道弧度。
  3. 完整路徑錄影：錄製一段從直線平穩加速進入 90 度彎道並平穩出彎的完整過程，證明預覽控制的優越性。
  4. Push Final：提交具備專業自駕感的最終代碼專案。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 物理路徑優雅度：無人車過彎時的圓弧是否自然？是否具備「提早進彎、平順修正」的特徵？
  • [ ] 系统響應可靠性：控制指令發送頻率是否能穩定維持在 15Hz 以上？

GitHub Classroom 詳細操作指南

本單元教學重點：破解「循跡震盪」的魔咒。讓學生理解預測性控制 (Predictive Control) 的核心價值。

  1. 重點 1：空間換時間 - 預覽控制本質上是用「空間上的遠方數據」來換取「處理上的時間餘裕」。
  2. 重點 2：動態權衡力 - 引導學員思考看太遠的副作用（截彎取直）與看太近的缺點（超調），培養工程權衡思維。
  3. 重點 3：視覺化除錯 - 強調 Canvas 標註的重要性。看不見預覽點，就無法除錯導航行為。
