使用 Copilot Agent Mode (自主代理模式)

這是最直接的結合方式。在 VS Code 中，你可以將 Copilot 切換到 Agent Mode。它是一個「自主程式設計師」，可以跨多個檔案進行修改、執行終端機命令、讀取編譯錯誤並自動修復。

操作流程

  1. 1 進入 Chat 視窗： 打開 Copilot Chat (Ctrl+Alt+I)。
  2. 2 切換模式： 在 Chat 頂部的下拉選單中選擇 "Agent" 模式。
  3. 3 下達任務： 輸入高階指令。例如：「幫我把這個專案的 API 從 REST 改成 GraphQL」。

自主迴圈 (Agentic Loop)

Agent 會先列出計畫 (Plan)，獲得你同意後開始修改檔案、跑測試、看到錯誤後自動修正，直到任務完成。這就是所謂的「自主回饋機制」。

結合 MCP (Model Context Protocol)

你可以透過 MCP 協議，讓 GitHub 的 AI Agent 擁有「手和腳」，去存取外部工具或私有數據。

應用場景

  • 自動查詢 Jira / GitHub Issues
  • 搜尋 Google 即時資訊
  • 讀取電腦本地的特定資料庫

設定方式

// 在 VS Code 設定中開啟

"chat.mcp.enabled": true

// 連接 MCP 伺服器，賦予 API 呼叫能力

自定義 Agent 指令 (AGENTS.md)

如果你希望 Agent 在你的 Workspace 中遵循特定的架構或風格，你可以「調教」它。在專案根目錄建立一個 AGENTS.md。

範例指示 (Instruction Example)

  • - 所有 API 必須使用 TypeScript 定義介面
  • - 所有組件必須使用 Tailwind CSS 進行樣式設定
  • - 測試必須使用 Jest 且覆蓋率需大於 80%

Agent 時代與模式 - AI 代理人專題

任務 1：跨檔案手術 - 全自動代碼重構

角色扮演：資深系統架構師 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 啟動模式：在 VS Code 中開啟 Copilot Chat 並切換至 Agent Mode。
  2. 下達指令：要求 Agent 將一個現有的功能（例如：User Login）從原本的分散邏輯重構為單一 Service 模式。
  3. 監控執行：觀察 Agent 如何自動讀取相關檔案、產生計畫並執行修改。
  4. 自動校正：如果編譯失敗，讓 Agent 讀取 Terminal 報錯並自行修正。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 計畫完整性：Agent 提出的 Plan 是否涵蓋了所有受影響的檔案？
  • [ ] 邏輯一致性：重構後的代碼是否維持原有功能並符合架構規範？

任務 2：協作 DNA - AGENTS.md 指令調教

角色扮演：技術負責人 (Tech Lead) | 難度：⭐️⭐️ | 估計時長：30 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立規則：在專案根目錄建立一個 AGENTS.md。
  2. 定義規範：在裡面明確寫下至少 3 條 Agent 必須遵循的規範（例如：所有變數名使用 camelCase、API 必須包含 JavaDoc）。
  3. 測試執行：讓 Agent 產生一段新代碼，觀察它是否嚴格遵守你設定的規則。
  4. 提交紀錄：將 AGENTS.md 與實作成果一併提交至 GitHub。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 約束有效性：Agent 是否能正確讀取並落實 AGENTS.md 中的指令？
  • [ ] 規範嚴謹度：設定的規範是否能有效提昇專案的維護品質？

任務 3：打破邊界 - MCP 外部工具整合

角色扮演：DevOps 工程師 | 難度：⭐️⭐️⭐️ | 估計時長：40 min

🛠️ 執行技術流程 (Git Workflow)

  1. 協議開啟：在 VS Code 設定中啟用 chat.mcp.enabled。
  2. 連接 Server：串接一個外部 MCP Server（例如：Google Search 或 GitHub Issue Tracker）。
  3. 執行指令：透過 @copilot 要求它搜尋外部資訊或自動讀取遠端 Issue 狀態。
  4. 結果驗證：確保 Agent 能成功獲取其訓練資料庫以外的即時外部數據。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 工具運用能力：是否能讓 Agent 透過 MCP 獲取正確的上下文 (Context)？
  • [ ] 解決方案品質：整合外部資料後的開發建議是否更具實踐性？

Agent 模式操作指南

教學重點：

本單元的核心在於「掌控力」。教師應引導學生理解 Agent 不是黑盒子，而是可以透過 AGENTS.md 和 MCP 進行深度調優的協作夥伴。評分時應著重於學生如何「引導」Agent，而非程式碼本身。
