CORS 與硬體防禦策略

掌握 CORS 運作機制。為物聯網建立安全屏障。

🛑 1. 什麼是 CORS？

決定是否允許網頁讀取跨網域資料防禦機制。

Same-Origin Policy

防止網站偷隱私。禁止網域 A 讀取 B。

Origin Matching

vibe.car/a.js
vibe.car/b.js Same
https://vibe.car
http://vibe.car Cross

🙋‍♂️ 2. 預檢請求：敲門機制

複雜請求時，發送 OPTIONS 敲門。

The Knock

OPTIONS 方法

詢問：「准許入內？」

The Ticket

Allow-Origin

伺服器回傳合規標頭。

ESP32 CONFIG 
DefaultHeaders::Instance().addHeader(
  "Access-Control-Allow-Origin", "*"
);
DefaultHeaders::Instance().addHeader(
  "Access-Control-Allow-Methods", "GET,POST,OPTIONS"
);

🔐 3. 硬體安全防禦

合規通訊。防禦性編程。

Defense Strategy

  • 權限最小化：不使用萬用符號。只開放必要。
  • 指令啟動：雜湊效驗防止竄改。
  • 資料遮蔽：敏感資訊封裝 Body。

Warning Zone

"CORS 保護使用者，後端仍需驗證。"
