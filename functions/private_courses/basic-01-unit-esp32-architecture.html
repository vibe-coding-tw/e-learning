⚡ 為什麼選擇 ESP32？

相較於傳統 Micro-controllers，ESP32 是一個質的飛躍。

🧠

雙核心 (Dual-Core)

運算速度高達 240MHz。一核處理通訊，一核專注執行邏輯。

📡

無線連線能力

內建 Wi-Fi 與 雙模藍牙。這是物聯網神器的核心。

🔌

豐富的外設

內建觸控、感應、多組序列接口。輕鬆驅動馬達與讀取感測器。

實戰任務：架構導覽與 DNA 提取

請動手完成以下三項實作任務，這是驗證本單元學習成果的核心挑戰。 每項任務完成後，請點擊卡片繳交作業。

  作業 1-1
約 45 分鐘

摩斯密碼閃爍

目標：利用板載 LED 實作 SOS 閃爍，掌握基礎 GPIO 控制與標準時序。

  • 將開發板透過 USB 連接並設定好 VS Code/PlatformIO 專案
  • 在 src/main.cpp 中編寫 LED 控制腳位初始化
  • 實作 SOS 閃爍時序（Dot 點與 Dash 橫，且 Dash 時間為 Dot 的 3 倍長）
  • 上傳 SOS 閃爍的演示影片或 GIF 到 assets/ 資料夾中
提示說明 標準摩斯密碼中，Dot 點與 Dash 橫的間隔應為 1 個 Dot 的長度，字母間隔為 3 個 Dot 長度。完成後請點擊此卡片繳交。
  作業 1-2
約 60 分鐘

晶片身分證偵查

目標：撰寫 API 查詢代碼，提取 CPU 核心數與唯一的 MAC 網卡地址。

  • 拍攝開發板實體照片，標註出 RESET 與 BOOT 鍵位置，存入 assets/board_map.jpg
  • 使用 esp_chip_info() 讀取晶片型號與 CPU 核心數量
  • 使用 esp_read_mac() 讀取硬體唯一 MAC 地址，並格式化輸出至 Serial Monitor
  • 將 Serial Monitor 輸出的 DNA 報告複製填寫至 tutor-guide.md 或學習筆記中
晶片資訊 API ESP-IDF 提供了內建 API 可以隨時詢問系統當前狀態。利用這些硬體參數可以為你的設備建立唯一識別碼（如 DNA 標籤）。
  作業 1-3
約 45 分鐘

啟動日誌診斷與發熱量監測

目標：分析 Bootloader 啟動時的初期日誌，並測量晶片在 WiFi 掃描時的熱溫升。

  • 按下 RESET 鍵重置 ESP32，捕獲 Serial Monitor 最早期的 Bootloader 啟動日誌
  • 識別並記錄 Flash Mode (例如 DIO/QIO) 與 Crystal Frequency (晶振頻率)
  • 將分析結果與數據填寫至 docs/BootAnalysis.md
  • 啟動 WiFi 掃描循環 5 分鐘，進行熱傳導觀測，摸晶片屏蔽蓋體感溫度
安全性與功耗注意 晶片發熱量是評估硬體功耗的重要依據。如果在掃描 WiFi 時感到過度發熱，應檢查是否有短路或供電不穩的現象。

實驗單元：ESP32 架構導覽與 DNA 提取

本單元是進入 ESP32 開發世界的第一步。你將從硬體觀察開始，深入了解這顆「電子大腦」的實體佈局，並透過程式碼提取其獨一無二的「數位 DNA」（晶片資訊），最後完成經典的摩斯密碼閃爍測試。

🎯 學習目標

  1. 掌握 ESP32 硬體佈局與核心按鍵（RESET/BOOT）的功能。
  2. 學習使用 ESP-IDF API 提取晶片型號、核心數及 MAC 地址。
  3. 實作基礎 GPIO 控制，透過摩斯密碼進行狀態反饋。
  4. 解析 Bootloader 啟動日誌，理解系統啟動參數。

🛠️ 任務說明

任務 1：硬體觀察 (Hardware Observation)

請觀察你的開發板，並完成以下動作：

  • 拍攝一張你的開發板照片，存放在 assets/ 資料夾中。
  • 在照片中標註出 RESET 與 BOOT 鍵的位置。
  • 思考：在什麼情況下你需要手動按住 BOOT 鍵？

任務 2：數位 DNA 提取 (DNA Extraction)

在 src/main.cpp 中實作晶片資訊提取邏輯。

  • 使用 esp_chip_info() 獲取晶片型號與核心數量。
  • 使用 esp_read_mac() 讀取唯一的硬體 MAC 地址。
  • 將結果格式化輸出至串口監控器 (Serial Monitor)。

任務 3：摩斯密碼測試 (Morse Code Lab)

利用板載 LED 實作 SOS 閃爍。

  • . (Dot) 代表短閃，- (Dash) 代表長閃。
  • 確保閃爍時序符合標準（例如：Dash 是 Dot 的 3 倍長）。
  • 上傳一段演示影片或 GIF 到 assets/。

任務 4：啟動日誌分析 (Bootloader Analysis)

重置設備並觀察 Serial Monitor 最開始輸出的訊息。

  • 記錄 Flash Mode (DIO/QIO) 與 Crystal Frequency。
  • 將分析結果填寫至 docs/BootAnalysis.md。
