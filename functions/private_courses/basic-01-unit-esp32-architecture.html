簡介

ESP32 是物聯網開發的瑞士刀——雙核處理器、內建 Wi-Fi 與藍牙、豐富的外設接口，全部整合在一顆不到百元的晶片上。 然而，如果你不了解開發板上每個元件的角色與電壓限制，就可能在接線時燒毀晶片，或在除錯時找不到問題根源。 本單元帶你從硬體層面認識這顆「電子大腦」，為後續所有實作建立穩固的認知地圖。

為什麼選擇 ESP32？

在嵌入式系統的世界中，微控制器（Microcontroller, MCU）是一切裝置的核心大腦。從家用電器到工業自動化，MCU 無處不在。然而，傳統的 MCU（如 Arduino UNO 上的 ATmega328P）在處理能力、無線連線與外設整合方面存在明顯的侷限性。ESP32 的出現，徹底改變了這個格局。

ESP32 由樂鑫科技（Espressif Systems）設計，是一款整合了 Xtensa 雙核心 32 位元處理器、Wi-Fi、藍牙（Classic + BLE）的系統單晶片（SoC）。它的時脈頻率可達 240 MHz，搭配 520 KB SRAM，足以處理複雜的即時運算與多工任務——而這一切的價格不到一杯咖啡。

對物聯網（IoT）開發者而言，ESP32 最令人興奮的特點在於它的「All-in-One」設計哲學。傳統方案中，你可能需要一塊 MCU 主板加上一塊獨立的 Wi-Fi 模組（如 ESP8266），再加上藍牙轉接器，光是接線就容易出錯。ESP32 把這些通通整合到一顆晶片中，大幅降低了系統複雜度與功耗。

ESP32 vs 傳統微控制器

以下表格比較了 ESP32 與常見入門級 MCU 的關鍵規格差異：

特性 Arduino UNO (ATmega328P) ESP32
處理器核心 單核 8-bit AVR 雙核 32-bit Xtensa LX6
時脈頻率 16 MHz 最高 240 MHz
SRAM 2 KB 520 KB
Flash 32 KB 4 MB（外掛）
Wi-Fi 無 802.11 b/g/n
藍牙 無 Classic + BLE 4.2
ADC 通道 6 個（10-bit） 18 個（12-bit）
觸控感測 無 10 組電容觸控
參考售價 約 NT$200~350 約 NT$80~150

雙核心架構的威力

ESP32 擁有兩顆獨立的 CPU 核心（稱為 PRO_CPU 與 APP_CPU），它們可以同時執行不同的任務。在 FreeRTOS 即時作業系統的調度下，一顆核心可以專門處理 Wi-Fi/藍牙通訊協定堆疊，另一顆核心則專注於你的應用程式邏輯（例如讀取感測器、控制馬達）。這種分工模式確保了通訊的穩定性，同時不犧牲應用程式的反應速度。

小技巧 在 Arduino 框架下，你的 setup() 與 loop() 預設運行在 APP_CPU（Core 1）上。如果需要將特定任務固定到 PRO_CPU（Core 0），可以使用 FreeRTOS 的 xTaskCreatePinnedToCore() API 來指定核心。

豐富的外設接口

除了無線通訊能力，ESP32 還內建了極為豐富的硬體外設，讓你幾乎不需要額外的擴充晶片就能完成各種應用：

  • GPIO：多達 34 個通用輸入/輸出腳位，支援中斷與 PWM 輸出
  • ADC：18 個 12-bit 類比數位轉換通道，可讀取感測器的類比訊號
  • DAC：2 個 8-bit 數位類比轉換通道，可輸出真正的類比電壓
  • I2C / SPI / UART：多組序列通訊介面，輕鬆連接各種感測器與顯示模組
  • 電容觸控：10 組觸控感測引腳，無需外接按鈕即可實現觸控互動
  • 霍爾感測器：內建磁場偵測功能
注意 並非所有 GPIO 都能自由使用。部分引腳（如 GPIO 6~11）連接到內部 Flash，不可作為一般 I/O 使用；GPIO 34~39 僅支援輸入模式，無法設為輸出。在設計電路前，務必參考官方的引腳對照表。

開發板外觀大解剖

拿到一塊 ESP32 開發板（如常見的 ESP32 DevKit V1），第一件事不是急著寫程式，而是先認識板子上每個元件的角色。理解硬體佈局能幫助你快速定位問題、避免接線錯誤，並且在除錯時知道該從哪裡下手。

一塊典型的 ESP32 開發板由以下幾個關鍵區域組成：頂部的 ESP-WROOM-32 模組（銀色金屬屏蔽蓋）、底部的 USB 連接埠與橋接晶片，以及兩側排列整齊的 GPIO 排針。此外，板上還有 LDO 穩壓器、電源指示燈、以及兩顆關鍵按鍵。

核心元件詳解

ESP-WROOM-32 模組

這是開發板的「心臟」。銀色金屬屏蔽蓋下方封裝了 ESP32 SoC 晶片與外掛的 Flash 記憶體（通常為 4 MB）。金屬蓋的作用是提供電磁屏蔽（EMI Shielding），防止高頻無線訊號干擾內部電路或被外部雜訊影響。模組側邊的蛇形走線是板載 PCB 天線，負責收發 Wi-Fi 與藍牙訊號。

天線區域注意 模組頂端的天線區域不應被金屬物遮蔽或被手指長時間觸碰，否則會嚴重影響無線訊號品質。在設計外殼時，天線區域應盡量外露或使用非金屬材質覆蓋。

USB-to-UART 橋接晶片

位於 USB 連接埠旁的小型 IC，常見型號為 CP2102（Silicon Labs）或 CH340（南京沁恒）。它的職責是在 USB 協定與 UART 序列協定之間進行轉換，讓電腦能夠透過 USB 線與 ESP32 通訊。不同品牌的開發板使用的橋接晶片可能不同，這也是為什麼有些板子需要額外安裝驅動程式的原因。

LDO 穩壓器

USB 供電電壓為 5V，但 ESP32 的核心工作電壓僅為 3.3V。LDO（Low Dropout Regulator）線性穩壓器負責將 5V 降壓至穩定的 3.3V，為晶片與 GPIO 提供工作電源。這意味著 ESP32 的所有 GPIO 輸出電壓皆為 3.3V，而非 Arduino UNO 的 5V。

電壓限制 — 非常重要 ESP32 的 GPIO 腳位最高只能承受 3.3V 的輸入電壓。如果直接接入 5V 訊號（例如某些 5V 感測器的輸出），將可能永久損壞晶片。連接 5V 設備時，必須使用電位轉換器（Level Shifter）或分壓電路。

兩顆關鍵按鍵

按鍵 標示 功能 使用時機
RESET EN / RST 重置晶片，等同斷電重開 程式卡死或需要重新觀察啟動日誌時按下
BOOT BOOT / IO0 將晶片切換至下載模式 燒錄失敗時，按住 BOOT 再按 RESET 可強制進入燒錄模式
小技巧 大多數新款開發板已內建自動下載電路（Auto-Download Circuit），在燒錄時會自動觸發 BOOT 模式，不需要手動按鍵。但如果遇到燒錄超時的問題，手動操作「按住 BOOT → 按一下 RESET → 放開 BOOT」仍然是最可靠的解法。

GPIO 排針佈局

開發板兩側各有一排排針，總共提供約 30 個可用腳位。每個腳位旁邊通常印有編號（如 GPIO2、GPIO4），並非所有編號都是連續的。規劃接線前，建議先列印一份開發板的 Pinout Diagram（引腳圖），貼在工作桌旁隨時參考。

軟體環境預熱

硬體已經認識完畢，接下來要確保你的電腦能「看見」並「對話」這塊開發板。這個步驟看似簡單，卻是許多初學者卡關最久的地方——因為它涉及到驅動程式安裝、通訊埠識別、以及開發工具的正確配置。

整個軟體環境的建置可以分為三大層次：IDE 開發環境（你撰寫與編譯程式碼的地方）、驅動程式（讓作業系統識別開發板的橋接晶片）、以及通訊埠設定（確保電腦知道要透過哪個「門」與開發板對話）。

步驟一：安裝 IDE 開發環境

我們推薦使用 VS Code + PlatformIO 作為主要開發環境。相較於傳統的 Arduino IDE，PlatformIO 提供了更專業的功能：IntelliSense 智慧補全能減少拼寫錯誤、內建的專案管理讓多檔案結構井然有序、而 platformio.ini 配置檔則讓環境設定可重複、可分享。

1

下載並安裝 VS Code

前往 code.visualstudio.com 下載適合你作業系統的版本（Windows / macOS / Linux）。

2

安裝 PlatformIO 擴充套件

開啟 VS Code 後，點選左側的「Extensions」圖示，搜尋 PlatformIO IDE 並安裝。安裝完成後，左側工具列會出現一個外星人圖示。

3

等待初始化完成

首次安裝後，PlatformIO 會自動下載必要的工具鏈（Toolchain），這可能需要 5~10 分鐘。下方狀態列會顯示安裝進度。

步驟二：安裝橋接晶片驅動程式

開發板透過 USB 線連接電腦時，作業系統需要正確的驅動程式才能與板上的橋接晶片（CP2102 或 CH340）通訊。Windows 10/11 與 macOS 通常會自動安裝 CP2102 的驅動，但 CH340 則幾乎總是需要手動安裝。

如何判斷橋接晶片型號？ 翻到開發板背面或 USB 連接埠附近，找到一顆小型 IC 晶片。上面的刻字通常會標示 CP2102 或 CH340G。若看不清楚，可以搜尋你購買的開發板型號查詢規格表。

步驟三：驗證通訊埠

驅動安裝成功後，插上開發板，你應該能在系統中看到新出現的通訊埠（COM Port）。驗證方法因作業系統而異：

作業系統 查看方法 典型埠名稱
Windows 裝置管理員 → 連接埠 (COM 和 LPT) COM3、COM4
macOS 終端機：ls /dev/tty.* /dev/tty.usbserial-0001
Linux 終端機：ls /dev/ttyUSB* /dev/ttyUSB0
找不到埠號？ 如果插上開發板後完全看不到任何新裝置出現，請依序檢查：(1) USB 線是否為「數據線」而非僅能充電的線；(2) 橋接晶片驅動是否已正確安裝；(3) USB 埠是否正常運作（換一個埠試試）。

實驗單元：ESP32 架構導覽與 DNA 提取

本單元是進入 ESP32 開發世界的第一步。你將從硬體觀察開始，深入了解這顆「電子大腦」的實體佈局，並透過程式碼提取其獨一無二的「數位 DNA」（晶片資訊），最後完成經典的摩斯密碼閃爍測試。

🎯 學習目標

  1. 掌握 ESP32 硬體佈局與核心按鍵（RESET/BOOT）的功能。
  2. 學習使用 ESP-IDF API 提取晶片型號、核心數及 MAC 地址。
  3. 實作基礎 GPIO 控制，透過摩斯密碼進行狀態反饋。
  4. 解析 Bootloader 啟動日誌，理解系統啟動參數。

🛠️ 任務說明

任務 1：硬體觀察 (Hardware Observation)

請觀察你的開發板，並完成以下動作：

  • 拍攝一張你的開發板照片，存放在 assets/ 資料夾中。
  • 在照片中標註出 RESET 與 BOOT 鍵的位置。
  • 思考：在什麼情況下你需要手動按住 BOOT 鍵？

任務 2：數位 DNA 提取 (DNA Extraction)

在 src/main.cpp 中實作晶片資訊提取邏輯。

  • 使用 esp_chip_info() 獲取晶片型號與核心數量。
  • 使用 esp_read_mac() 讀取唯一的硬體 MAC 地址。
  • 將結果格式化輸出至串口監控器 (Serial Monitor)。

任務 3：摩斯密碼測試 (Morse Code Lab)

利用板載 LED 實作 SOS 閃爍。

  • . (Dot) 代表短閃，- (Dash) 代表長閃。
  • 確保閃爍時序符合標準（例如：Dash 是 Dot 的 3 倍長）。
  • 上傳一段演示影片或 GIF 到 assets/。

任務 4：啟動日誌分析 (Bootloader Analysis)

重置設備並觀察 Serial Monitor 最開始輸出的訊息。

  • 記錄 Flash Mode (DIO/QIO) 與 Crystal Frequency。
  • 將分析結果填寫至 docs/BootAnalysis.md。

ESP32 架構導覽與 DNA 提取報告

這是你與 ESP32 「電子大腦」的第一次親密接觸。本單元要求你從觀察硬體佈局開始，進而透過程式碼提取晶片的「數位 DNA」，並完成第一次摩斯密碼閃爍測試。

1. 硬體觀察 (Hardware Observation)

請在 assets/ 資料夾中存放一張你的開發板照片，並在此處展示。

任務：請在照片中標註出 RESET 與 BOOT 鍵的位置。

燒錄自檢：

  • 在燒錄過程中，你是否需要手動按住 BOOT 鍵？ [ 是 / 否 ]
  • 你的開發板型號是：________________

2. 數位 DNA 提取 (DNA Extraction)

請將 src/main.cpp 執行後的串口輸出截圖或文字貼在此處。

// Paste your DNA Report here

驗證點：

  • 你的 MAC 地址與開發板標籤（如有）是否一致？
  • 你的 ESP32 是單核 (Single-core) 還是雙核 (Dual-core)？

3. 摩斯密碼測試 (Morse Code Lab)

請上傳一段 SOS 閃爍的影片或 GIF 到 assets/，並在此連結。
