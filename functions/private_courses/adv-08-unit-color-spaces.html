數位影像處理基礎：
顏色空間 (Color Spaces)

理解 RGB、HSV 與 YUV 的本質差異。學習在不同的場景下選擇最適合的「色彩語言」。

🌈 1. RGB：光的加法與顯示器的邏輯 (20 min)

RGB (Red, Green, Blue) 是最直覺的色彩空間，直接對應顯示器硬體。然而，它最大的限制是「亮度與顏色耦合」，這對電腦視覺識別來說非常致命。

Coupling Limitation

  • 環境變數干擾：當燈光變暗，R、G、B 數值會同時等比例下降。
  • 硬體依賴：不同攝像頭感測到的 RGB 原色分布可能完全不同。

Pixel Intensity Projection

🎯 2. HSV/HSL：色彩處理的利器 (30 min)

透過解除亮度耦合，HSV 讓我們能鎖定 **H (Hue, 色相)**。即使在陰影下，只要色相不變，系統就能穩定地定位目標物體。

Cylindrical Coordinate System

H: 0-360°
S: 0-100%
V: 0-100%

Object Tracking Range

Robust Detection Architecture

🎞️ 3. YUV/YCbCr：影像傳輸與壓縮

YUV 將影像拆分為亮度 (Y) 與色度 (UV)。由於人類視覺對明暗極其敏感，但對顏色細節相對遲鈍，這成為 JPEG 壓縮與 ESP32 攝像頭輸出的核心協議。

Compression Logistics

  • Chroma Subsampling：捨棄部分 UV 數據（如 4:2:2），節省 33% 頻寬而不失真感官體驗。
  • Y-Channel Focus：利用亮度頻道進行高效率的邊緣檢測 (Edge Detection)。
  • RGB565 Optimization：理解 16-bit 像素如何在 MCU 記憶體中節省每一分 Byte。

Bandwidth Dial

3:2:1 Ratio

任務 2：RGB565 位元壓縮 (Memory Optimization Lab)

角色扮演：嵌入式優化師 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

  3. 視覺品質評估：截圖並說明顏色斷層 (Banding) 在深色陰影區是否明顯。

任務 3：YUV 亮度自適應 (Auto-Exposure Leveling)

角色扮演：控制演算法工程師 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 亮度提取：從模擬 YUV 流中僅提取 Y 頻道，繪製亮度直方圖。
  2. 動態增益補償：實作邏輯：若 Average(Y) < 50，則自動對所有像素進行 V-Value 的線性補償。
  3. 壓力測試：錄製影片展示從明亮區快速移入陰影區時，物體識別輪廓是否能瞬間「追回」穩定度。
  4. Push Final：提交具備光影魯棒性的環境感知方案。

GitHub Classroom 詳細操作指南

本單元教學重點：破解「影像只是圖片」的迷思。讓學生理解影像本質上是數組陣列，而顏色空間是觀察數組的「濾鏡」。

  1. 重點 1：HSV 的神性 - 在電腦視覺中，HSV 是識別的「救星」。必須讓學生親手調校滑桿，感受它在光影下比 RGB 穩定得多的特性。
  2. 重點 2：位元級思維 - 嵌入式系統不再是無限容量。學習 RGB565 能讓學生對 0 與 1 的排列組合更有感。
  3. 重點 3：感知優先 - 透過 YUV 的案例，教導學員「有選擇地捨棄數據」在大型通訊系統中的戰略價值。
