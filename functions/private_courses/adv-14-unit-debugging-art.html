🦆 1. 除錯心理學與科學方法

除錯不僅是技術，更是嚴謹的心理戰。我們常被「剛才明明還是好的」這種經驗主義蒙蔽。學會利用科學檢驗流程與「橡皮鴨除錯法」來系統化定位問題。

Scientific Cycle

  • 現象記錄：不帶成見地觀察 Bug 觸發的精確硬體與軟體條件。
  • 建立假說：邏輯推演可能的原因（如：整數溢位、反向映射、競態條件）。
  • 隔離實驗：縮小代碼範圍，驗證假說是否正確，直到鎖定「犯罪現場」。

🛡️ 3. 防禦性重構：寫出「不壞」的代碼

除錯的最高境界是「不需要除錯」。透過邊界驗證、異常捕獲 (Try-Catch) 與非同步流程管理，建立代碼的免疫能力。

Robustness Archetypes

  • 邊界防禦 (Edge Case Guard)：主動預測並攔截 NaN、Null 與負值溢位。
  • 競態保護 (Race Buffer)：確保非同步指令依序執行，防止任務衝突。
  • 感知型報錯 (Verbose UX)：故障發生時給予診斷引導，而非靜默當機。

單元課程 進階 14 - 除錯藝術

除錯劇場：命案現場與防禦性設計

任務 1：犯罪現場偵查實錄 (Crime Scene Investigation Lab)

角色扮演：軟體鑑識偵探 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 故障重現：運行專案，記錄：在何種極端參數下馬達會發生反轉或死機？
  3. 假設定位：利用 console.log() 或斷點追蹤追蹤變數。判斷是「型別轉換錯誤」還是「變數作用域污染」。
  4. Push Final：提交包含「詳細偵調報告」的 README。記錄從發現、假說、證偽到最終修正的全過程。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 邏輯洞察力：是否正確找出了專案中預埋的 3 個邏輯陷阱（如負值映射溢位）？
  • [ ] 排查完整性：日誌是否呈現了找到答案之前的「證偽」過程？（證明使用了科學方法而非盲目猜測）。

任務 2：通訊黑盒診斷 (Communication Blackbox Analysis Lab)

角色扮演：全鏈路通訊專家 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 數據抓包：利用 DevTools Network 面板捕捉點擊按鈕時的 fetch / ws 請求包。
  2. 封包校驗：比對 Web 發出的 JSON 欄位與 ESP32 解析器預期的變數名，找出拼寫或大小寫不一致的「隱形錯誤」。
  3. 故障隔離：撰寫診斷，判斷問題發生在「前端發送」、「傳輸協議」還是「後端解析」層次。
  4. Push Final：提交包含「抓包數據比對圖」與「隔離修復代碼」的 PR。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 工具熟練度：能否從 Network 面板中準確提取出傳輸 Payload 並解讀其結構？
  • [ ] 推論嚴謹性：給出的故障定位是否具備充分的「數據證據」支持？

任務 3：防禦性重構實戰 (Defensive Refactoring Lab)

角色扮演：系統健壯性工程師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 脆弱性掃描：對 PID 運算模組進行單元測試：輸入 NaN 或極大值時，系統是否會崩潰？
  2. 加固代碼：部署 Clamping 防護、型別強制校驗與逾時自動復位機制。
  3. 回報機制實作：在前端實作 Toast 警告通知，當捕獲到異常數據時，主動告知操作者具體原因而非沈默失靈。
  4. Push Final：提交具備「自我防護、感知異常」的最終版高階控制庫。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 防護廣度：代碼是否考慮了所有可能的非法輸入（負值、空值、非法字串）？
  • [ ] 用戶告知力：當系統進入失效保護 (Fail-Safe) 狀態時，介面是否提供了清晰的診斷訊息引導？

GitHub Classroom 詳細操作指南

本單元教學重點：將「Debug」視為一種嚴謹的科學實驗。由「隨機修改」進化為「邏輯觀察」。

  1. 重點 1：重現是修復的前題 - 強調沒有被重現的 Bug 是無法被徹底消滅的。指導學生建立重現步驟紀錄。
  2. 重點 2：工具是延伸的感官 - 指導學生使用 Breakpoints 與 Network 面板代替盲目的 console.log，獲取更完整的「時空上下文」。
  3. 重點 3：免疫比治療更划算 - 透過「防禦性重構」任務，讓學員體認到預防錯誤代碼比排查錯誤更快。
