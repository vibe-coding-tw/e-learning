Module 2: Circular Clamping

限位邏輯：半徑的限制

為什麼不能直接讓搖桿頭跟著手指跑？如果手指滑太遠，搖桿頭會飛出底座！我們需要實作圓形限制 (Circular Clamping)。

關鍵邏輯：

  1. 設定搖桿底座半徑為 \(R\) (例如 50px)。
  2. 計算目前距離 \(d = \sqrt{\Delta x^2 + \Delta y^2}\)。
  3. 判斷：如果 \(d > R\)，則必須縮放位置。
// 比例縮放公式
if (distance > maxRadius) { const ratio = maxRadius / distance; newX = startX + deltaX * ratio; newY = startY + deltaY * ratio; }

Module 3: Normalization

向量標準化與實作任務

預習：速度百分比計算

硬體或 API 通常需要 -1.0 到 1.0 或 0 到 100% 的數值，而不是像素值。我們需要進行歸一化 (Normalization)。

Power (推力):
\(\text{Power} = \min(\frac{d}{R}, 1.0)\)
最大值鎖定在 100%。
Angle (角度):
\(\theta = \text{atan2}(\Delta y, \Delta x)\)
計算方向弧度。

搖桿數學實驗室

任務 1：搖桿邊界鎖定 (Circular Clamping)

角色扮演：物理模擬工程師 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫。
  2. 距離運算：應用勾股定理 \(d = \sqrt{dx^2 + dy^2}\) 計算觸控點與中心的距離。
  3. 限位實作：當 \(d > R\) 時，利用比例關係 ratio = R / d 將搖桿頭鎖定在圓周邊界。
  4. 同步提交：撰寫 README 紀錄如何處理 d = 0 的例外狀況，並 push 成果。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 數學嚴謹度：是否正確處理了隱藏的「除以零」風險？
  • [ ] 手感順暢度：觸發限位時，搖桿頭是否會發生跳變或抖動？
  • [ ] 代碼結構：限位邏輯是否封裝成獨立的可重用函數？

任務 2：動態數據歸一化 (Normalization)

角色扮演：系統架構師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 定義區間：將像素座標映射至 \([-100, 100]\) 或 \([0, 1]\) 的標準區間。
  2. 數據輸出：在網頁上即時顯示 Power (0-100%) 與 Angle (弧度/角度)。
  3. 精確度優化：使用 Math.round 處理輸出數值，確保數據穩定。
  4. 提交報告：將不同位置的數值截圖存入 README 後 push。

🎯 教師點評標準 (Tutor Benchmarks)

任務 3：搖桿自動回彈 (Auto-Centering)

角色扮演：交互設計師 | 難度：⭐️⭐️ | 估計時長：30 min

🛠️ 執行技術流程 (Git Workflow)

  1. 事件監聽：捕捉 touchend 或 mouseup 事件。
  2. 狀態重置：當手指放開時，將所有位置數據歸零，並發送「停止」指令。
  3. 進階挑戰：實作帶有動畫過場的回彈效果，而非瞬間跳回中心。
  4. 版本提交：紀錄重置邏輯的心得，推送到 GitHub。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 例外處理：在拖曳中突然跳出網頁，回彈邏輯是否依然能正確執行？
