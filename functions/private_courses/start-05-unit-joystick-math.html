Module 2: Circular Clamping

限位邏輯：半徑的限制

為什麼不能直接讓搖桿頭跟著手指跑？如果手指滑太遠，搖桿頭會飛出底座！我們需要實作圓形限制 (Circular Clamping)。

關鍵邏輯：

  1. 設定搖桿底座半徑為 \(R\) (例如 50px)。
  2. 計算目前距離 \(d = \sqrt{\Delta x^2 + \Delta y^2}\)。
  3. 判斷：如果 \(d > R\)，則必須縮放位置。
// 比例縮放公式
if (distance > maxRadius) { const ratio = maxRadius / distance; newX = startX + deltaX * ratio; newY = startY + deltaY * ratio; }

Module 3: Normalization

向量標準化與實作任務

預習：速度百分比計算

硬體或 API 通常需要 -1.0 到 1.0 或 0 到 100% 的數值，而不是像素值。我們需要進行歸一化 (Normalization)。

Power (推力):
\(\text{Power} = \min(\frac{d}{R}, 1.0)\)
最大值鎖定在 100%。
Angle (角度):
\(\theta = \text{atan2}(\Delta y, \Delta x)\)
計算方向弧度。

Joystick Math - 搖桿限位數學與數據歸一化

在本練習中，你將學習如何利用數學邏輯為網頁搖桿建立「物理邊界」，並將觸碰座標轉化為精確的控制參數。

🎯 學習目標

  1. 勾股定理的實戰應用：計算搖桿頭與中心點的直線距離。
  2. 圓形限位邏輯 (Circular Clamping)：防止搖桿頭超出底座範圍。
  3. 數據歸一化 (Normalization)：將像素位移轉換為百分比 (%) 與角度 (Degrees)。
  4. 自動回彈 (Auto-Centering)：實作放手後搖桿自動歸零的邏輯。

🚀 任務說明

請開啟 math-logic.js 檔案，並尋找 // TODO 標記完成以下功能：

任務 1：計算距離與限位

  • 使用勾股定理 $d = \sqrt{dx^2 + dy^2}$ 計算當前手指位置與中心點的距離。
  • 當距離超過底座半徑時，計算縮放比例 (Ratio)，確保搖桿頭停留在圓周邊緣。

任務 2：數據轉換與輸出

  • 計算 Power (%)：將位移距離轉換為 $0%$ 到 $100%$ 的數值。
  • 計算 Angle (Deg)：利用 Math.atan2 計算搖桿的方向角度。

任務 3：安全重置

  • 確保在 touchend 或 mouseup 事件發生時，搖桿會自動回到中心 $(0,0)$，且推力數據歸零。

🛠️ 開發建議

  • 即時遙測：善用頁面上方的「數據看板」，觀察計算出的數值是否符合預期。
  • 例外處理：思考當手指剛好點在中心點 ($d=0$) 時，程式是否會因為除以零而崩潰？
  • 程式碼結構：保持邏輯清晰，優先處理數學運算，再更新 UI 顯示。

📂 程式庫結構 (Project Structure)

  • index.html: 搖桿介面與數據看板。
  • math-logic.js: 核心開發區，你需要在這裡實作數學邏輯。
  • style.css: 介面樣式定義。
  • assets/: 相關參考資源。

2. 作業任務部署細節

任務 1：搖桿圓形限位實作 (Circular Clamping Lab)

  • 目標：打造「有物理邊界」的互動。應用勾股定理 $d = \sqrt{dx^2 + dy^2}$ 判斷距離，並利用比例關係將搖桿頭限制在圓周內，避免操作失控。
  • Classroom 部署建議：
    • 極限值處理實作：編寫一個判斷函式，當距離 $d > R$ 時，將 $dx, dy$ 等比例縮小。
    • 除以零例外處理：思考並寫下：如果手指剛好點在圓心 $(d=0)$，你的縮放公式是否會崩潰？該如何防護？
    • 驗證要點：邊界健壯性審核。學員是否能讓搖桿頭平滑地在圓周邊緣滑動而不會跳變？公式是否簡潔（避免了複雜的三角函數，改用比例運算優先）？

任務 2：動態數據歸一化與輸出 (Data Normalization Lab)

  • 目標：將「像素」轉化為「指令」。學會將畫布座標映射至 $[-100, 100]$ 或 $[0, 1]$ 的標準區間，並計算方向角度（利用 Math.atan2）。
  • Classroom 部署建議：
    • 推力百分比計算：實作 Power = Math.min(distance / R, 1.0)，確保輸出資料始終在 100% 以內。
    • 象限感知角度運算：使用 Math.atan2(dy, dx) 正確判斷 360 度的方向，並轉換為角度制（Degree）。
    • 驗證要點：遙測準確性審核。顯示的數值是否能隨著手指移動精準更新？當向「左後方」推時，角度與推力是否符合預期？

任務 3：搖桿自動回彈與安全重置 (Auto-Centering Lab)

  • 目標：實作「失效安全」。確保使用者放開手指時，搖桿頭能自動歸零 $(0,0)$，同時機器人接收到明確的「停止」指令。
  • Classroom 部署建議：
    • 釋放事件重置：在 touchend 中將全域狀態清零。
    • 回彈動畫 (進階)：挑戰利用 requestAnimationFrame 製作一個具備物理摩擦感的彈回中心效果。
    • 驗證要點：操控安全性審核。放手後，發送出的 JSON 封包中 Power 是否確實變為 0？是否有任何狀態殘留導致機器人停不下來？
