Module 2: Core Principles

HTML5 Canvas 繪圖基礎

2D 渲染上下文

Canvas 就像一張空白的畫布，我們需要透過 JavaScript 取得 getContext('2d') 這支「畫筆」才能開始作畫。

JavaScript 
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

function draw() {
    // 1. 清除畫布 (必備！)
    ctx.clearRect(0, 0, canvas.width, canvas.height);

    // 2. 開始繪製路徑
    ctx.beginPath();
    ctx.arc(x, y, radius, 0, Math.PI * 2);
    ctx.fillStyle = 'blue';
    ctx.fill();
    
    // 3. 動畫循環
    requestAnimationFrame(draw);
}
  • 1
    座標系：

    原點 $(0,0)$ 位於畫布的左上角，X軸向右，Y軸向下。

  • 2
    路徑繪製：

    所有的繪圖都從 beginPath() 開始，這告訴瀏覽器我們要畫一個新的形狀。

Canvas Coordinate System

試著移動滑鼠：
X: 0, Y: 0

Module 3: Practical Challenge

向量計算與實戰作業

預習：向量與角度 (Trigonometry)

要讓搖桿跟隨手指，我們需要計算兩點之間的距離與角度。

1. 角度計算 (Angle)

使用 Math.atan2(y, x) 計算滑鼠位置相對於圓心的角度。

\(\theta = \text{atan2}(dy, dx)\)

2. 距離計算 (Distance)

使用畢氏定理計算距離，確保搖桿頭不會超出底座範圍。

\(d = \sqrt{dx^2 + dy^2}\)

HTML5 Canvas 與 幾何運算 - 虛擬搖桿與儀表板

課程目標

本單元將帶領你從「網頁排版工程師」晉升為「圖形渲染工程師」。你將學習如何使用 HTML5 Canvas API 進行像素級繪圖，並運用三角函數與座標轉換技術，打造一個精密且動態的虛擬搖桿與儀表系統。

任務說明

任務 1：座標映射與事件處理 (View Transformation)

  • 目標：掌握滑鼠/觸控座標到 Canvas 內部座標的精確轉換。
  • 重點：
    • 使用 getBoundingClientRect() 計算偏移。
    • 實作「清除畫布 (clearRect) -> 更新狀態 -> 重新繪製」的循環邏輯。
    • 確保渲染乾淨度，避免產生殘影。

任務 2：動態指針儀表板 (Dynamic Gauge)

  • 目標：使用 Canvas 狀態棧 (State Stack) 實作物理旋轉效果。
  • 重點：
    • 熟練運用 save()、translate()、rotate() 與 restore()。
    • 將數據（如 0-100）映射到旋轉弧度（如 0-π）。
    • 確保狀態隔離，避免指針旋轉影響其他 UI 元素。

任務 3：視覺回饋特效 (Visual Feedback FX)

  • 目標：運用過渡動畫提升使用者體驗。
  • 重點：
    • 利用 Math.sin() 實作平滑的「呼吸燈」閃爍效果。
    • 透過 globalAlpha 或 rgba() 調整透明度。
    • 維持 60 FPS 的流暢渲染效能。

如何開始

  1. 檢視結構：
    • index.html：包含畫布容器與基本的佈局。
    • main.js：主要的程式邏輯區，你將在此實作繪圖循環。
    • style.css：定義介面外觀與圓形佈局。
  2. 實作細節：
    • 請根據 main.js 中的 // TODO 標記完成各項繪圖功能。
  3. 提交作業：
    • 完成後請將程式碼提交並 push 到你的遠端程式庫。

評分標準

2. 作業任務部署細節

任務 1：從 DOM 到 Canvas 的座標映射 (View Transformation Lab)

  • 目標：掌握「座標系切換」。學習如何透過 getBoundingClientRect() 精確計算滑鼠在畫布內部的相對位置，並解決「小畫家式」的殘影 Bug。
  • Classroom 部署建議：
    • 小球追蹤實作：實作一個隨手指移動的小圓，並確保背景能被正確 clearRect() 刷新。
    • 防偏移校正：手動計算 Canvas 的 CSS 尺寸與實際繪圖尺寸 (width/height 屬性) 的比例差異。
    • 驗證要點：渲染乾淨度審核。小球中心是否精確對準指尖？在快速移動時是否會產生斷續或殘影？

任務 2：打造動態指針儀表板 (Dynamic Gauge Lab)

  • 目標：掌握「畫布狀態保存與轉換」。學習利用 save()、translate()、rotate() 與 restore() 四部曲，製作一個隨數據變化的物理指針。
  • Classroom 部署建議：
    • 指針旋轉實作：將一個 0100 的速度變數映射到 0180 度的旋轉弧度。
    • 狀態隔離挑戰：確保儀表指針旋轉時，不會拖累畫面上的其他文字或 UI 元素跟著轉動（藉此導入 save/restore 概念）。
    • 驗證要點：幾何演算法審核。指針的旋轉中心點是否正確位於圓心？角度變換是否平滑且符合映射規則？

任務 3：全方位視覺回饋特效 (Visual Feedback FX Lab)

  • 目標：掌握「過渡動畫與透明度」。製作一個「預警呼吸燈」特效，當系統偵測到特定狀態時，在 Canvas 上產生紅色閃爍的視覺警示。
  • Classroom 部署建議：
    • 透明度動畫實作：利用 Math.sin() 函數配合 globalAlpha 或 rgba()，製作一個平滑漸變的紅色警示遮罩。
    • 分層渲染概念：確保警示層繪製在所有介面的最上方。
    • 驗證要點：動畫流暢度審核。呼吸燈的頻率是否穩定？在高頻率更新下是否依然維持 60 FPS 的渲染效能？
