🌊 2026 程式教育的典範轉移

隨著生成式 AI 從輔助工具進化為具備自主決策能力的代理 (AI Agents)，程式設計的本質已發生結構性改變。Andrej Karpathy 提出的「Vibe Coding」理念已演變為教育界必須正視的新教學範式。

從打字員到導演

開發者從逐行輸入語法的「打字員」，轉變為透過高品質 Prompt 定義架構與使用者體驗的「導演」。自然語言成為了主要的開發媒介。

多代理協作系統

引入「架構師代理」、「編碼代理」與「測試代理」的分工模式。開發者描述需求後，系統在幾秒鐘內完成生成、測試與部署，實現「所思即所得」。

🏛️ GitHub Classroom 基礎建設

在 Vibe Coding 時代，GitHub Classroom 不再僅是繳交作業的平台，更是整合了環境配置、持續整合 (CI) 與 AI 協作治理的「工程化學習空間」。

1

範本儲存庫與雲端 IDE

教師提供配置好的 Devcontainer。學生點擊一鍵啟動 GitHub Codespaces，即可在瀏覽器獲得統一的開發環境，免除本地配置困擾。

2

微提交與 Feedback PR

系統自動建立 Feedback PR。學生需進行原子化提交 (Atomic Commits)，教師可在此進行行內評論，實踐深度的 Code Review。

3

Autograding 自動評分

結合 GitHub Actions 執行輸入比對、單元測試或檔案檢查。每次 Push 後，即時回饋綠燈 ✅ 或紅燈 ❌，驅動學生修正程式碼。

🎯 Vibe Coding 教學五大策略

1. 以「評論」代替「撰寫」

不再要求從零寫程式，而是讓學生在 GitHub PR 中對 AI 生成的有瑕疵代碼進行逐行評審 (Code Review)。

2. 強化「過程透明」

提交完整的 Prompt Log。評價學生如何將複雜業務需求拆解為 AI 可理解的子任務。

3. 測試驅動開發 (TDD) 的極致應用

學生必須先手動撰寫單元測試。只有當 AI 生成的程式碼通過了自己寫的測試集，才算完成任務。

03-unit-github-classroom: Professional Standard Vibe Coding

歡迎來到「專業級 Vibe Coding」實務單元！這個專案旨在模擬真實的 AI 輔助開發工作流，你將學習如何從定義願景開始，透過微粒子提交與自動化測試，最終建立出高品質、安全的軟體。

🚀 快速開始

  1. 複製程式庫：點擊 GitHub Classroom 生成的連結，將此程式庫 Clone 到本地。
  2. 安裝依賴： 
    npm install
  3. 執行測試（目前預期會失敗）： 
    npm test

📋 任務說明

本作業分為三個模組，請依照順序完成：

🟦 Module 1：定義與願景 (Definition & Vision)

  • Task 1-1: 專案願景：
    • 在 docs/VIBE.md 中定義你的專案願景。
  • Task 1-2: 糾正幻覺 (Audit)：
    • 檢查 src/buggy_code.js，找出 AI 產生的邏輯漏洞或 Bug。
    • 在 docs/AUDIT.md 的「幻覺審計」章節記錄你的發現與修正用的 Prompt。
  • Task 1-3: 微粒子提交 (Micro-commits)：
    • 修正 buggy_code.js。
    • 要求：至少包含 3 個獨立的 Commit（例如：修正邏輯、重構代碼、添加註釋）。

🟪 Module 2：工作流實踐 (Workflow Practice)

  • Task 2-3: 紅燈轉綠燈 (Autograding)：
    • 目前的測試 tests/lab.test.js 預期是「失敗」的（紅燈）。
    • 修正代碼後，確保本地執行 npm test 通過（綠燈）。
    • Push 代碼到 GitHub，確認 GitHub Actions 上的 Autograding 變為綠色勾勾。

🟩 Module 3：工程思維 (Engineering Mindset)

  • Task 3-1: 規格說明書 (Spec-First)：
    • 在 docs/SPEC.md 中填寫你的功能規格說明書。
  • Task 3-2: 安全性審計 (Security)：
    • 檢查 src/buggy_code.js 是否包含敏感資訊（如 Hardcoded API Key）。
    • 將敏感資訊移除，並記錄於 docs/AUDIT.md 的「安全性審計」章節。

📤 提交方式

  1. 完成所有任務並確保留下清晰的 Git Commit 紀錄。
  2. 將所有更改 Push 到你的 GitHub 程式庫。
  3. 前往 GitHub 程式庫的 Actions 分頁，確認自動評分腳本執行成功。

⚖️ 評核標準

  • 定義力：VIBE.md 與 SPEC.md 是否清晰且具備結構。
  • 糾錯力：AUDIT.md 是否詳實記錄了對 AI 幻覺與安全性的處理。
  • 流程力：Git Commit 是否符合「微粒子提交」原則，歷史紀錄是否展現了思考過程。

祝你 Coding 愉快！

導師指南 (Tutor Guide) - 03-unit-github-classroom

針對 03-unit-github-classroom (Classroom 與 Vibe Coding 實務) 單元，由於這是銜接入門與進階開發的「關鍵橋樑」，其作業 (Assignments) 涵蓋了從 AI 提示詞到工程自動化的全維度。

以下是針對這 9 個核心任務 在 GitHub Classroom 部署的整合性建議：

1. 部署模式建議：單一大型專案 (Monolith Portfolio)

建議將這 9 個任務整合在 同一個 GitHub Classroom 作業 中。

  • 作業名稱：vibe-coding-professional-standard
  • 優點：學員在同一個程式庫中能完整體現從「定義 (Vibe)」到「實作 (Commit)」再到「自動化 (CI/CD)」的思考進化過程。

2. 各模組部署策略與技術細節

🟦 Module 1：定義與願景 (AI 素養)

  • 任務 1-2 糾正幻覺 (Hallucination Audit)：
    • 範本部署：提供一個 src/buggy_code.js（內容可以是一個有 Bug 的陣列排序或 API 呼叫函數）。
    • 驗證方式：要求學生將「修正用 Prompt」寫在 docs/AUDIT.md 並提交修復後的代碼。
  • 任務 1-3 微粒子提交 (Micro-commits)：
    • 要求：明確限制此任務必須由至少 3 個獨立的 Commit 組成（例如：UI 結構、CSS 樣式、邏輯綁定）。
    • 導師評核：在 Classroom 儀表板檢查 Commit 節奏，這能確認學生不是直接從 AI 複製整大塊代碼。

🟪 Module 2：工作流實踐 (Git Flow)

  • 任務 2-3 紅燈轉綠燈 (Autograding Fix)：
    • 部署技巧：範本程式庫中的測試檔案 (tests/lab.test.js) 預設應為 「失敗」 狀態。
    • 過關標準：學生必須經歷「Push -> 紅燈 -> 線上查報錯 -> 修正 -> Push -> 綠燈」的過程。
    • 驗證：查看 Actions 歷史，必須有至少一次失敗（紅叉）後轉成功的紀錄。

🟩 Module 3：自動化與矩陣 (工程思維)

  • 任務 3-1 規格說明書 (Spec-First)：
    • 範本部署：提供一個結構良好的 docs/SPEC.md 空範本，引導學生練習 Markdown 的表格與代碼塊語法。
  • 任務 3-2 安全性審計：
    • 部署技巧：在範本代碼中故意寫入一個 Hardcoded API Key。
    • 驗證：檢查學生是否將該 Key 移至 .env 或 secrets 中，並在審計報告中提出說明。

3. 推薦範本程式庫結構 (GitHub Classroom Template)

.
├── .github/
│   ├── workflows/        # 預置的自動評分腳本
│   └── tutor_guide.md    # 導師指南 (本檔案)
├── src/                  
│   └── buggy_code.js     # 任務 1-2 的偵錯對象
├── docs/                 
│   ├── VIBE.md           # 任務 1-1 填寫處
│   ├── SPEC.md           # 任務 3-1 填寫處
│   └── AUDIT.md          # 任務 3-2 填寫處
├── tests/                
│   └── lab.test.js       # 任務 2-3 的紅燈測試腳本
└── package.json          # 預設開發依賴

💡 導師評核建議：

本單元不應只看「最後有沒有 ✅」，更要看「Git 歷史紀錄」。一個專業的 Vibe Coder 會留下清晰的對話/修正足跡。建議導師在 Feedback Pull Request 中，針對學生的 docs/AUDIT.md（修正 AI 幻覺的邏輯）給予具體回饋，這對提升學生的「定義力」非常有幫助。
