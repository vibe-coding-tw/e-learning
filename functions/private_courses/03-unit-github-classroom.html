版本控制作為學習足跡

每一筆 Git Commit 都是你的思考軌跡。這能幫助導師區分哪些是「AI 生成」，哪些是你基於理解所做的「人類修正」。

Autograding 即時回饋

在快速迭代中，自動化評分系統提供了客觀的指標（測試案例），確保你的 Vibe 沒有偏離正確的邏輯軌道。

學員操作：5 步驟輕鬆交作業

1

接受邀請

點擊專屬連結並授權 GitHub。

2

Clone 程式庫

將影印好的作業下載至本機。

3

開始實作

在代碼中透過 Vibe 引導 AI。

4

Push 同步

上傳代碼，啟動自動化測試。

5

確認結果

查看 ✅ 指標，確認任務達成。

Vibe 時代的提交規範

在 AI 協助開發的時代，一次性提交（複製貼上）會導致學習過程不對稱。我們建議採用 「微提交 (Micro-commits)」 策略：

範例提交訊息：
• git commit -m "[AI-Generated] 實作 BLE 連線基礎邏輯"
• git commit -m "[Human-Modified] 修正 MTU 限制下的分段傳輸 Bug"
• git commit -m "[Review-Only] 完成單元測試驗證"

💡 每一筆 Commit 都是你的開發思考痕跡。

自動評分 (Autograding) 邏輯

系統會模擬真實的產品驗收，對你的程式碼進行功能測試。

✅
通過測試
邏輯符合預期，滿足產品規格。
❌
測試失敗
點擊紅叉查看報錯，修正後再次 Push。

核心教學策略：從「寫代碼」轉向「架構設計」

Prompt 審查作業

除了提交代碼，你也需要記錄與 AI 的對話過程。我們關注的是你如何精確描述需求，以及如何糾正 AI 的「幻覺」。這才是 Vibe Coding 真正的基本功。

規格說明書 (Spec) 的定義權

訓練撰寫高品質的 Markdown 文檔，定義介面、資料結構與流程。在 2026 年，定義問題的能力遠比寫出語法更重要。

👨‍🏫 未來教師技能組合

技能類別 具體能力要求 教學應用場景
工作流架構力 熟悉 Git Branch/Merge、YAML 腳本、Organization 權限配置。 構建大規模自動化評分系統，確保 300+ 學生作業的穩定執行。
Prompt 審計力 識別高效與低效 Prompt 的特徵，具備引導 AI 進行深層推理的能力。 在碼審課中，指導學生如何糾正 AI 的「幻覺」。
規格說明書撰寫 能用嚴謹的 Markdown 語言定義專案需求、介面協議與驗收標準。 撰寫讓學生和 AI 都能理解的作業要求，減少理解歧義。
安全與倫理意識 識別 AI 生成程式碼中的漏洞、隱私風險及版權隱患。 引導學生建立「數位品德」，正確使用技術而非濫用。
