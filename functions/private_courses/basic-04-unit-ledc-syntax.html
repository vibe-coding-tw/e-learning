🎼 1. 硬體 PWM 三部曲

在 ESP32 中，PWM 不再是簡單的 analogWrite，而是具備層級架構的精密模組。

⏱️ 定時器 (Timer)

決定 PWM 的「心跳頻率」。所有共用定時器的頻道，其頻率與解析度必須保持一致。

🔢 頻道 (Channel)

ESP32 擁有 16 個獨立頻道 (0-15)，每個頻道可獨立設定其「占空比」來控制輸出強度。

🧩 解析度 (Resolution)

定義控制的精細程度。8-bit 對應 256 階；13-bit 對應 8192 階。

🔗 腳位綁定 (Attach)

最具彈性的特點：任何頻道都可以靈活地映射到幾乎具何一個物理 GPIO 腳位上。

💡 Resource Tip

  • ● 定時器資源是稀缺的，優先將同頻率設備放在一起。
  • ● 共用 Timer 下，不同頻道仍可獨立控制 Duty 值。
  • 精確管理資源是開發高效能系統的第一步。

⚙️ 2. 關鍵語法與核心 API

ledcSetup(ch, freq, res);

初始化設定：指定那個頻道、跳動頻率、以及多少位元的解析度。

ledcAttachPin(gpio, ch);

腳位映射：將實體接腳導通至對應的邏輯頻道。

ledcWrite(ch, val);

輸出能量：更新該頻道的占空比。數值上限取決於 Resolution。

思考題

如果我需要用馬達播放「簡單的音階」，改變 Freq 還是 Duty？為什麼？

C++ FRAMEWORK 
// 步驟：設定 -> 綁定 -> 驅動
void setup() {
  ledcSetup(0, 5000, 8);  // 5kHz, 8bit
  ledcAttachPin(14, 0);   // GPIO14 -> CH0
}

void loop() {
  ledcWrite(0, 128);      // Output 50% Power
  delay(1000);
}

⚡ 3. 為什麼不直接用 analogWrite？

LEDC 提供的是「硬體層級」的精確資源控制。你可以在程式運行中，零延遲地修改頻率（動態變頻）或更改映射，這對於動態調節馬達效率或生成複雜音效是必備的能力。

動態變頻

即時調整載波頻率，優化馬達在不同轉速下的發熱與噪音。

位元對比

12-bit 下 4096 階的精微控制，讓你的無人車能緩步穩健起步，不暴衝。
