🎼 1. 硬體 PWM 三部曲

在 ESP32 中，PWM 是具備層級架構的精密模組。

⏱️ 定時器 (Timer)

決定頻率。共用定時器的頻道，頻率必須一致。

🔢 頻道 (Channel)

16 個獨立頻道 (0-15)，獨立控制強度。

🧩 解析度 (Resolution)

控制精細度。8-bit 對應 256 階；13-bit 對應 8192 階。

🔗 腳位綁定 (Attach)

頻道可映射到幾乎任何物理 GPIO 腳位。

💡 Resource Tip

  • ● 定時器資源稀缺，優先將同頻設備放在一起。
  • ● 共用 Timer 下，不同頻道仍可獨立控制 Duty 值。
  • 精確管理資源是開發第一步。

⚙️ 2. 關鍵語法與核心 API

ledcSetup(ch, freq, res);

初始化：指定頻道、頻率、解析度。

ledcAttachPin(gpio, ch);

映射：腳位導通至邏輯頻道。

ledcWrite(ch, val);

輸出：更新占空比。上限取決於 Resolution。

思考題

播放「音階」時，改變 Freq 還是 Duty？為什麼？

C++ FRAMEWORK 
// 步驟：設定 -> 綁定 -> 驅動
void setup() {
  ledcSetup(0, 5000, 8);  // 5kHz, 8bit
  ledcAttachPin(14, 0);   // GPIO14 -> CH0
}

void loop() {
  ledcWrite(0, 128);      // 50% Power
  delay(1000);
}

⚡ 3. 為什麼不直接用 analogWrite？

LEDC 提供「硬體層級」精確資源控制。零延遲修改頻率（變頻）或更改映射，是運動控制必備能力。

動態變頻

優化馬達在不同轉速下的發熱與噪音。

位元對比

12-bit 下 4096 階控制，讓起步更穩健不暴衝。
