簡介

ESP32 內建的 LEDC 模組提供 16 個獨立硬體 PWM 頻道，讓你精確分配定時器、頻率與解析度。 本單元將帶你掌握 ledcSetup、ledcAttachPin、ledcWrite 三步驟語法， 並理解為何專業韌體開發不使用 analogWrite，而是透過 LEDC 實現動態變頻與多路資源管理。

情境
你的無人車需要同時驅動兩顆馬達（20kHz）和一組車頭燈（5kHz），但發現把它們設在同一個定時器上時， 頻率被強制統一，燈光開始閃爍、馬達發出刺耳嘯叫。你意識到需要正確規劃 LEDC 的硬體資源分配。

硬體 PWM 三部曲

ESP32 的 LEDC 模組不是一個簡單的「輸出 PWM」指令，而是一個具備三層硬體架構的精密資源管理系統。理解這個層級結構，是正確使用 LEDC 的前提——如果你隨意分配資源，將會遇到頻率衝突、頻道佔用等難以除錯的問題。

第一層：Timer（定時器）

ESP32 擁有 4 個高速定時器（Timer 0–3，使用 80MHz 時鐘源）和 4 個低速定時器（Timer 0–3，使用 8MHz 時鐘源），總計 8 個獨立時鐘基準。每個 Timer 負責產生一個特定頻率的計數時鐘，所有綁定到同一 Timer 的頻道（Channel）將共享相同的 PWM 頻率與解析度。

這意味著：如果你的馬達需要 20kHz 而 LED 需要 5kHz，它們絕對不能共用同一個 Timer。你必須將它們分配到不同的 Timer 上。

第二層：Channel（頻道）

ESP32 提供 16 個 PWM 頻道（高速 8 個 + 低速 8 個）。每個頻道獨立維護自己的占空比數值，但其頻率由所綁定的 Timer 決定。你可以將頻道想像成「音量旋鈕」——所有旋鈕播放的音調（頻率）由 Timer 決定，但每個旋鈕的音量（占空比）可以獨立調整。

第三層：GPIO（腳位綁定）

最後一步是將邏輯上的 Channel 映射到物理上的 GPIO 腳位。ESP32 的 GPIO 矩陣非常靈活，幾乎所有 GPIO 腳位都可以被指派為 PWM 輸出（少數例外如 GPIO 34–39 為純輸入腳位）。一個 Channel 通常綁定一個 GPIO，但技術上你可以將同一個 Channel 映射到多個 GPIO 以獲得同步輸出。

注意 LEDC 架構的資源分配邏輯為：Timer → Channel → GPIO。設定時必須按照此順序進行——先配置 Timer（頻率 + 解析度），再將 Channel 綁定到 Timer，最後將 GPIO 映射到 Channel。

層級架構一覽

層級 數量 決定的參數 共享規則
Timer 8 個（高速 4 + 低速 4） 頻率、解析度 同 Timer 下所有 Channel 頻率相同
Channel 16 個（高速 8 + 低速 8） 占空比（Duty） 各 Channel 的占空比獨立控制
GPIO 幾乎所有輸出腳位 物理輸出位置 每個 GPIO 僅能綁定一個 Channel
提示 實務上，Arduino-ESP32 框架的 ledcSetup(channel, freq, resolution) 會自動為你分配 Timer。Channel 0–1 共用 Timer 0，Channel 2–3 共用 Timer 1，以此類推。因此，如果你需要不同頻率，請選擇不屬於同一 Timer 群組的 Channel 編號。

關鍵語法與核心 API

使用 ESP32 LEDC 模組控制 PWM 輸出只需要三個核心函式。這三個函式對應了 LEDC 架構的三個配置步驟——初始化 Timer、綁定 GPIO、設定輸出值。讓我們逐一深入了解每個 API 的參數意義與使用細節。

步驟一：ledcSetup() — 初始化頻道

ledcSetup(channel, freq, resolution_bits) 負責配置指定頻道的 PWM 參數。此函式會自動將頻道綁定到對應的 Timer，並回傳實際設定成功的頻率值（可能與請求值略有差異，因為硬體除頻器只能產生特定頻率）。

// 語法
double ledcSetup(
  uint8_t channel,      // 頻道編號 (0–15)
  double  freq,         // 目標頻率 (Hz)
  uint8_t resolution_bits // 解析度位元數 (1–20)
);

步驟二：ledcAttachPin() — 綁定腳位

ledcAttachPin(pin, channel) 將物理 GPIO 腳位連接到邏輯頻道。呼叫後，該 GPIO 的輸出權將完全由 LEDC 硬體接管，你不能再用 digitalWrite() 控制它（除非先呼叫 ledcDetachPin() 解除綁定）。

// 語法
void ledcAttachPin(
  uint8_t pin,    // GPIO 腳位編號
  uint8_t channel // 頻道編號 (0–15)
);

步驟三：ledcWrite() — 輸出占空比

ledcWrite(channel, duty) 更新指定頻道的占空比。duty 參數的有效範圍取決於初始化時設定的解析度：8-bit 為 0–255，10-bit 為 0–1023，以此類推。設為 0 代表持續低電位（0% 占空比），設為最大值代表持續高電位（100% 占空比）。

// 語法
void ledcWrite(
  uint8_t  channel, // 頻道編號 (0–15)
  uint32_t duty    // 占空比值 (0 ~ 2^resolution - 1)
);

完整範例：LED 呼吸燈

#define LED_PIN    14
#define PWM_CHANNEL 0
#define PWM_FREQ   5000  // 5 kHz
#define PWM_RES    8     // 8-bit → 0~255

void setup() {
  ledcSetup(PWM_CHANNEL, PWM_FREQ, PWM_RES);
  ledcAttachPin(LED_PIN, PWM_CHANNEL);
}

void loop() {
  for(int duty = 0; duty <= 255; duty++) {
    ledcWrite(PWM_CHANNEL, duty);
    delay(10);
  }
  for(int duty = 255; duty >= 0; duty--) {
    ledcWrite(PWM_CHANNEL, duty);
    delay(10);
  }
}
警告 ledcWrite() 的 duty 值不能超過 2^resolution - 1。例如 8-bit 解析度下寫入 256 會導致溢位歸零，輸出瞬間從最亮變成全暗。務必使用 constrain() 或手動檢查邊界。
思考題 如果你想用蜂鳴器播放「Do Re Mi」音階，應該改變的是 freq（頻率）還是 duty（占空比）？答案是頻率——因為音高由振動頻率決定（A4 = 440Hz），占空比只影響音量大小。可使用 ledcWriteTone(channel, freq) 來動態切換頻率。

為什麼不直接用 analogWrite？

如果你有 Arduino UNO 的使用經驗，可能習慣了 analogWrite(pin, value) 這個簡潔的函式。它在 Arduino 框架中確實好用，但在 ESP32 上，直接使用 LEDC API 有著不可替代的優勢。讓我們來比較兩者的差異，理解為什麼專業的 ESP32 韌體開發應該選擇 LEDC。

analogWrite 的限制

在傳統 Arduino 中，analogWrite() 有以下根本性限制：

  • 頻率固定：Arduino UNO 的 PWM 頻率固定為 490Hz 或 980Hz（取決於腳位），無法自訂。ESP32 早期版本雖然也移植了 analogWrite()，但它本質上只是 LEDC 的簡化包裝。
  • 解析度固定：一律為 8-bit（0–255），無法提升精細度。
  • 無法動態變頻：一旦初始化就無法在運行時改變頻率，無法實現蜂鳴器音效或馬達噪音優化。
  • 資源不可控：無法指定哪個 Timer 和 Channel，多路 PWM 容易產生衝突。

LEDC 的核心優勢

能力 analogWrite LEDC API
頻率控制 固定，無法修改 1Hz – 40MHz，可動態調整
解析度 固定 8-bit (256 階) 1-bit 至 20-bit 可選
頻道管理 自動分配，不可控 16 頻道手動規劃
動態變頻 不支援 ledcChangeFrequency()
音調產生 需手動計算 ledcWriteTone(ch, freq)
腳位解綁 不支援 ledcDetachPin(pin)

實際應用場景

以下是幾個只有 LEDC 能做到、analogWrite() 無法實現的進階操作：

場景一：動態變頻降噪

馬達在低轉速時使用較低的 PWM 頻率以獲得更好的扭力響應，高轉速時切換到 20kHz 以上以消除嘯叫噪音。這需要在運行時即時修改頻率：

// 根據轉速動態調整 PWM 頻率
if (speed < 30) {
  ledcChangeFrequency(MOTOR_CH, 1000, 10); // 低速: 1kHz
} else {
  ledcChangeFrequency(MOTOR_CH, 20000, 10); // 高速: 20kHz
}

場景二：多路異頻共存

同時驅動 5kHz 的車頭 LED 與 20kHz 的馬達，各自使用獨立 Timer，互不干擾：

// LED 使用 Channel 0 (Timer 0) — 5kHz, 8-bit
ledcSetup(0, 5000, 8);
ledcAttachPin(LED_PIN, 0);

// 馬達使用 Channel 2 (Timer 1) — 20kHz, 10-bit
ledcSetup(2, 20000, 10);
ledcAttachPin(MOTOR_PIN, 2);
注意 從 ESP32 Arduino Core 3.x 版本開始，analogWrite() 已被官方移除，所有 PWM 操作必須透過 LEDC API 完成。即便在舊版本中，analogWrite() 內部也只是對 LEDC 的薄包裝。直接使用 LEDC 不僅功能更強，也避免了版本升級後的相容性問題。

  • 紀錄：將你的觀察與工程決策紀錄在 assets/sweep_log.csv 中。

  • 目標：體會頻率對應硬體效率與噪訊的影嚮，培養工程直覺。
  • Classroom 部署建議：
    • 自動掃描算法：撰寫一段程式碼，每隔 5 秒改變一次 PWM 頻率（從 100Hz 漸進到 40kHz）。
    • 感官實測紀錄：學員需在 README 中紀錄在哪些頻段會有明顯的高頻噪音（尖叫聲），以及在哪些頻段馬達轉動最為平穩。
    • 工程決策：解釋為何最終選擇了該特定的工作頻率（例如：選用 20kHz 是為了避開人耳聽覺範圍）。

3. LEDC 專業與資源管理點評標準 (Peripheral Design & Optimization Standards)

此單元的價值在於 「對硬體資源的精細規劃與軟體抽象的深度」：

  • [ ] 資源利用率：Timer 與 Channel 的分配是否考慮了未來擴充性？
  • [ ] API 魯棒性：setMotorSpeed() 是否處理了各種極限輸入？數學轉換公式是否簡潔高效？
  • [ ] 調優意識：是否理解解析度 (Resolution) 越高、可控頻率越低的硬體權衡 (Trade-off)？
