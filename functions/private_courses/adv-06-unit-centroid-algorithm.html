⚖️ 1. 什麼是影像質心？ (25 min)

影像質心 (Centroid) 是描述物體位置的最核心指標。它將成千上萬的像素簡化為一個單一的座標點。

Mathematical Definition

$$\bar{x} = \frac{\sum_{i=1}^{n} x_i}{n}, \quad \bar{y} = \frac{\sum_{i=1}^{n} y_i}{n}$$
  • 幾何平衡點：質心是物體的平均位置座標。
  • 降維打擊：將 2D 矩陣數據轉化為 1D 點訊息。

🔭 3. ROI、誤差與控制連動

誤差是控制系統的靈魂。將影像中心與質心的偏差轉化為轉向角指令。透過 ROI (感興趣區域) 排除雜訊干擾。

Control Pipeline

  • ROI 裁切：只關注賽道範圍，無視天空或背景。
  • 誤差計算 (Error)：$e = \bar{x} - Center$。
  • 回授控制：將 $e$ 映射到 Servo 轉向 PWM。

ADV-06：視覺質心算法 (Unit Centroid Algorithm)

🎯 作業目標

本單元的核心任務是實作一套完整的「視覺質心追蹤系統」，從低階像素分析出發，逐步建構出能驅動實體硬體的穩定控制訊號。

完成本作業後，你將能夠：

  • 對影像像素進行遍歷與累加，計算出目標物件的質心座標
  • 使用數位低通濾波器消除感測器雜訊，讓質心軌跡更加平滑
  • 將質心誤差線性映射為轉向控制角度，完成視覺回饋控制迴路

📋 作業任務

任務 1：質心追蹤器 — 從像素到座標

目標：在 js/accumulators.js 中實作像素遍歷與質心計算邏輯。

需求：

  • 對影像所有像素進行雙重迴圈遍歷
  • 累加所有黑色像素的 X、Y 座標總和與計數
  • 計算平均質心座標 $(\bar{x}, \bar{y})$
  • 必須實作除以零的保護邏輯（當畫面無目標時，不得讓程式崩潰）

評分重點：自動測試將送入空白影像，驗證你的除以零保護是否正確運作。

任務 2：質心穩定化過濾器

目標：在 js/smooth_filters.js 中實作低通濾波，消除質心座標的抖動。

需求：

  • 實作加權移動平均公式：filteredX = (newX * 0.7) + (prevX * 0.3)
  • 設定 MIN_PIXELS 面積門檻，過濾散點雜訊
  • 在 README 末尾附上截圖，對比加入濾波前後質心軌跡的差異

評分重點：測試將送入含雜訊影像，驗證質心是否能鎖定主要目標而不被雜訊拉偏。

任務 3：質心誤差與轉向角度映射

目標：在 js/servo_mapper.js 中將質心位置轉換為舵機控制角度。

需求：

  • 計算質心與畫面中心的誤差值：error = centroidX - 160
  • 將誤差線性映射至轉向角度區間 $[60°, 120°]$
  • 必須實作數值限幅 (Clipping)，確保輸出角度永遠不超出安全範圍

評分重點：給定位於畫面最左側的質心，輸出角度必須精確等於 $60°$。

1. 範本程式庫 (Template Repo) 配置建議

演算法的學習重點在於「邏輯的嚴密性」與「數據的流暢度」，建議範本包含：

  • 📂 js/tracker.js：核心追蹤引擎，提供 ImageData 的解碼框架，留下雙重迴圈遍歷與累加器更新的 TODO。
  • 📂 js/filter_logic.js：訊號過濾模組。預置低通濾波 (Low-pass Filter) 函式骨架，讓學員練習定義 alpha 權重。
  • 📂 assets/simulation/：一組預錄的「二值化數據流 (JSON)」，模擬車輛在賽道上前進時像素的動態變化，讓學員在沒有實體車時也能調試演算法。
  • 📂 tests/centroid_precision.test.js：自動化評分脚本。餵入標準幾何圖形（如：正方形、長方形），檢核學員計算出的質心座標 $( \bar{x}, \bar{y} )$ 是否精確命中幾何中心。

2. 作業任務部署細節

任務 1：「質心追蹤器」：從像素到座標 (Build Centroid Tracker)

  • 目標：透過加權平均法 (Weighted Average) 實現空間維度的降維提取。
  • Classroom 部署建議：
    • 核心代碼檢核： 
      // 學生需實作：累加與除法
for (let y = 0; y < height; y++) {
    for (let x = 0; x < width; x++) {
        if (isBlack(x, y)) { sumX += x; sumY += y; count++; }
    }
}
if (count > 0) { centroidX = sumX / count; }
    • Autograding：餵入一個完全空白的影像，檢核學員是否實作了「除以零保護」。這能有效區分「只會寫公式」與「具備防禦性編程意識」的學生。

任務 2：「質心穩定化」過濾器 (Centroid Stabilization)

  • 目標：引入數位濾波概念，解決物理感測器常見的「數據抖動 (Jitter)」問題。
  • Classroom 部署建議：
    • 濾波演算法檢核：要求學員實作加權移動平均： filteredX = (newX * 0.7) + (prevX * 0.3)。
    • 面積門檻測試：設定 MIN_PIXELS 過濾器。餵入帶有少量散點雜訊的影像，檢核質心準星是否能保持在主體目標上而不被雜訊「拉走」。
    • 實驗紀錄：學員需在 README 中對比加入濾波前後，質心座標隨時間變化的波形曲線（截圖）。

任務 3：質心誤差與轉向控制連動 (Centroid-to-Control Linkage)

  • 目標：完成視覺回饋控制 (Visual Feedback Control) 的最後一哩路。
  • Classroom 部署建議：
    • 地圖映射邏輯： 
      // 學生需實作：線性映射
let error = centroidX - 160; 
let steerAngle = map(error, -160, 160, 60, 120);
    • Autograding：給定一個位於畫面最左側的質心，檢核輸出的轉向角是否精確等於 $60^\circ$。
    • Tutor View (PR 審核)：導師点評其對「數值限幅 (Clipping)」的處理。若算出的角度超過 $[60, 120]$，代碼是否能自動截斷，以防止舵機過載燒毀？

3. 演算法導師點評標準 (Algorithm Benchmarks)

此單元的價值在於 「數據的純淨度與控制的平滑度」：

  • [ ] 計算魯棒性 (Robustness)：代碼是否能處理「多目標並存」或「目標消失」的邊界情況？
  • [ ] 訊號延遲平衡 (Lag-Stability Trade-off)：在實驗報告中，學員是否理解了過濾強度強（穩定）與回應延遲（遲鈍）之間的取捨？
  • [ ] 物理安全意識：映射邏輯中是否有確保控制指令永遠處於硬體安全區間內的防護代碼？
