CSS Flexbox：遙控器排版的神器

模組 5 個單元 入門 前端開發 排版佈局

告別 float 與 position 的噩夢，用彈性盒子打造完美的響應式遙控介面。 在這個單元中，你將學會 Flexbox 的核心概念、主軸與交錯軸的軸線思維、對齊三劍客（justify-content、align-items、flex-wrap）的分配邏輯， 以及如何在實際專案中，以巢狀 Flexbox 實作十字方向鍵與響應式遙控面板。

簡介

在過去的 CSS 排版中，要將多個按鈕水平並排、垂直居中，或者在不同尺寸的手機螢幕上自動拉伸， 需要使用非常複雜的 float、position: absolute 或者 display: inline-block。 這常導致「跑版」或「排版碎裂」。

情境
小明想設計一個遙控車的十字方向鍵，結果左、右按鈕和上、下按鈕重疊在一起； 他又想做一個即時監控面板，但在手機直向時按鈕擠成一團，橫向時卻又太扁。 老師說：「你需要的是 Flexbox 的軸線控制，以及 Media Query 響應式切換。」

Flexbox (Flexible Box Layout) 彈性盒子佈局，是現代 Web 排版中最強大也最通用的工具。 它可以讓我們輕鬆搞定對齊、等寬分配、主軸方向翻轉等任務，特別適合用於建構需要精準觸控與多裝置適應的物聯網遙控面板。

核心概念：容器與軸線

只要在父容器設定 display: flex;，它就變成了 Flex Container (彈性容器)， 裡面的子元素立刻變成乖巧的 Flex Items (彈性項目)。

雙軸系統 (Two-Axis System) Flexbox 的對齊和空間分配是基於兩條看不見的軸線：主軸 (Main Axis) 和 交錯軸 (Cross Axis)。 理解這兩條軸的變換規律，是掌握 Flexbox 的核心要訣。
屬性 主軸 (Main Axis) 交錯軸 (Cross Axis)
預設方向 水平向右 (當 flex-direction: row) 垂直向下 (與主軸正交)
控制對齊 justify-content align-items
主要用途 決定按鈕排隊方向與排列順序 決定元素的高度對齊、居中或拉伸

對齊三劍客

在 Flexbox 排版中，最常用的三個排版屬性被稱為「對齊三劍客」。 它們決定了子元素在主軸上的分配、在交錯軸上的對齊，以及在空間不足時的折行策略。

1. justify-content
主軸上的空間分配

決定子項目沿主軸的對齊方式與間距分配。如靠左、靠右、居中，或平分剩餘空間。

space-between
2. align-items
交錯軸上的對齊方式

決定子項目在垂直於主軸之交錯軸上的對齊方式。如頂部對齊、底部對齊，或居中對齊。

center
3. flex-wrap
子項目換行策略

決定當子項目總寬度超出容器時，要強制壓縮並擠在同一行，還是自動折行到下一排。

wrap

十字方向鍵佈局

目標：達成完美的幾何對稱。無論螢幕多寬，按鈕組永遠在正中央。禁止使用 position: absolute 或 float 等傳統屬性。

  • 外層容器使用 flex-direction: column 處理上下按鈕
  • 中間層嵌入 flex-direction: row 容器處理左右按鈕
  • 活用 align-items: center 與 justify-content: center 確保垂直水平居中
  • 確認無論視窗如何縮放，十字方向鍵皆能保持對稱置中
  作業 2-2
約 60 分鐘

響應式儀表板

目標：讓遙控面板根據手機持握方向自動調整。直向螢幕時上下堆疊，橫向螢幕時左右並排。

  • 使用 Media Queries，設定螢幕寬度 < 600px 為直向斷點
  • 直向時外層容器設定 flex-direction: column
  • 橫向/電腦版設定 flex-direction: row
  • 使用 flex-grow: 1 讓 FPV 影像區自動拉伸填滿剩餘空間
  作業 2-3
約 30 分鐘

間距與觸控優化

目標：設計合適的手指點擊熱區與防誤觸安全間距。

  • 使用 gap 屬性（建議 1rem / 16px 以上）精確控制按鈕間距
  • 對「低速/中速/高速」按鈕群組設定 flex: 1 平分寬度
  • 確保所有點擊按鈕大小均不小於 48x48 像素以利觸控

作業：CSS Flexbox - 遙控器排版的神器

歡迎來到 start-01-unit-flexbox-layout 單元！在這個作業中，你將學習如何使用 CSS Flexbox 來建立現代且響應式的網頁佈局，透過實作一個控制面板來掌握容器嵌套與彈性排版。

🎯 學習目標

  1. 掌握軸線思維：理解 Flexbox 的主軸與交錯軸。
  2. 容器嵌套實戰：透過嵌套的 Flex 容器實現複雜佈局。
  3. 響應式設計：配合 Media Queries 讓介面適應不同螢幕大小。
  4. 提升觸控體驗：運用間距控制與空間分配優化操作介面。

📝 任務說明

任務 1：十字方向鍵幾何對稱挑戰 (D-Pad Geometric Symmetry Lab)

請在 style.css 中實作方向鍵的排版：

  • 目標：透過巢狀 Flexbox 實現完美的幾何中心對齊。
  • 要求：禁止使用 position: absolute 或 float。請使用 display: flex 與 flex-direction: column 管理上下層，並嵌套一個 row 管理左右按鈕。

任務 2：響應式儀表板切換 (Responsive Dashboard Toggle Lab)

請加入響應式設計，讓控制台能根據裝置方向自動切換：

  • 手機直向（寬度 < 600px）：控制台與影像區應呈現上下堆疊 (column)。
  • 電腦或橫向：切換為左右並列 (row)。
  • 提示：活用 flex-grow: 1 讓顯示區自動填滿剩餘空間。

任務 3：間距與觸控優化 (Spacing & Touch Optimization Lab)

優化按鈕的間距與佈局：

  • 間距控制：使用 gap 屬性（建議 1rem 以上）精確設定按鈕間距，避免誤觸。
  • 等寬分配：對特定的按鈕群組（如速度控制）使用 flex: 1，確保它們在不同螢幕寬度下都能平分空間。

📂 專案結構

  • index.html：遙控器結構框架。已經為你準備好基礎的 DOM 結構。
  • style.css：佈局實作區。請在此檔案中撰寫你的 CSS 程式碼，完成上述任務。
  • README.md：設計邏輯日誌。完成作業後，請在此處或作業繳交區簡單說明：

    2. 你設定的響應式斷點 (Breakpoint) 是多少？為什麼？

2. 作業任務部署細節

任務 1：十字方向鍵幾何對稱挑戰 (D-Pad Geometric Symmetry Lab)

  • 目標：透過巢狀 Flexbox 實現完美的幾何中心對齊，杜絕使用絕對定位造成的排版破碎。
  • Classroom 部署建議：
    • 容器層級實作：將方向鍵拆分為上、中（左、右）、下三層。
    • 軸線對齊要求：強制要求使用 display: flex 與 flex-direction: column 管理上下，再嵌套一個 row 管理左右。
    • 驗證要點：關鍵技術審核。檢查 CSS 是否包含 position: absolute 或 float？若有，即視為未達成「彈性佈局」的教學目標。

任務 2：響應式儀表板切換 (Responsive Dashboard Toggle Lab)

  • 目標：掌握 flex-direction 與 Media Queries 的配合，讓介面能根據裝置方向自動最佳化。
  • Classroom 部署建議：
    • 自適應切換：規定在手機直向（寬度 < 600px）時，控制台與影像區為上下堆疊 (column)；在電腦或橫向時切換為左右並列 (row)。
    • 空間拉伸練習：活用 flex-grow: 1 讓中央影像區自動填滿剩餘空間。
    • 驗證要點：拖動視窗寬度時，佈局是否能無縫「翻轉」？按鈕與顯示區的比例是否依然和諧？

任務 3：間距與觸控優化 (Spacing & Touch Optimization Lab)

  • 目標：運用現代 CSS 屬性提升操作容錯率，並學習等寬分配的進階技巧。
  • Classroom 部署建議：
    • 精密間距控制：使用 gap 屬性精確設定按鈕間距（建議 1rem 以上），而非單獨設定每個按鈕的 margin。
    • 等寬權重分配：對「低/中/高」速度按鈕組實作 flex: 1，確保它們在不同手機寬度下都能平分空間。
    • 驗證要點：在開發者工具模擬不同手機，確認按鈕是否會互相重疊或有怪異間隙？gap 是否正確提供了防誤觸的空間？
