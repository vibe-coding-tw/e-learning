作業：CSS Flexbox - 遙控器排版的神器

歡迎來到 start-01-unit-flexbox-layout 單元！在這個作業中，你將學習如何使用 CSS Flexbox 來建立現代且響應式的網頁佈局，透過實作一個控制面板來掌握容器嵌套與彈性排版。

🎯 學習目標

  1. 掌握軸線思維：理解 Flexbox 的主軸與交錯軸。
  2. 容器嵌套實戰：透過嵌套的 Flex 容器實現複雜佈局。
  3. 響應式設計：配合 Media Queries 讓介面適應不同螢幕大小。
  4. 提升觸控體驗：運用間距控制與空間分配優化操作介面。

📝 任務說明

任務 1：十字方向鍵幾何對稱挑戰 (D-Pad Geometric Symmetry Lab)

請在 style.css 中實作方向鍵的排版：

  • 目標：透過巢狀 Flexbox 實現完美的幾何中心對齊。
  • 要求：禁止使用 position: absolute 或 float。請使用 display: flex 與 flex-direction: column 管理上下層，並嵌套一個 row 管理左右按鈕。

任務 2：響應式儀表板切換 (Responsive Dashboard Toggle Lab)

請加入響應式設計，讓控制台能根據裝置方向自動切換：

  • 手機直向（寬度 < 600px）：控制台與影像區應呈現上下堆疊 (column)。
  • 電腦或橫向：切換為左右並列 (row)。
  • 提示：活用 flex-grow: 1 讓顯示區自動填滿剩餘空間。

任務 3：間距與觸控優化 (Spacing & Touch Optimization Lab)

優化按鈕的間距與佈局：

  • 間距控制：使用 gap 屬性（建議 1rem 以上）精確設定按鈕間距，避免誤觸。
  • 等寬分配：對特定的按鈕群組（如速度控制）使用 flex: 1，確保它們在不同螢幕寬度下都能平分空間。

📂 專案結構

  • index.html：遙控器結構框架。已經為你準備好基礎的 DOM 結構。
  • style.css：佈局實作區。請在此檔案中撰寫你的 CSS 程式碼，完成上述任務。
  • README.md：設計邏輯日誌。完成作業後，請在此處或作業繳交區簡單說明：
    1. 你為什麼在十字方向鍵中使用嵌套 Flexbox？
    2. 你設定的響應式斷點 (Breakpoint) 是多少？為什麼？

2. 作業任務部署細節

任務 1：十字方向鍵幾何對稱挑戰 (D-Pad Geometric Symmetry Lab)

  • 目標：透過巢狀 Flexbox 實現完美的幾何中心對齊，杜絕使用絕對定位造成的排版破碎。
  • Classroom 部署建議：
    • 容器層級實作：將方向鍵拆分為上、中（左、右）、下三層。
    • 軸線對齊要求：強制要求使用 display: flex 與 flex-direction: column 管理上下，再嵌套一個 row 管理左右。
    • 驗證要點：關鍵技術審核。檢查 CSS 是否包含 position: absolute 或 float？若有，即視為未達成「彈性佈局」的教學目標。

任務 2：響應式儀表板切換 (Responsive Dashboard Toggle Lab)

  • 目標：掌握 flex-direction 與 Media Queries 的配合，讓介面能根據裝置方向自動最佳化。
  • Classroom 部署建議：
    • 自適應切換：規定在手機直向（寬度 < 600px）時，控制台與影像區為上下堆疊 (column)；在電腦或橫向時切換為左右並列 (row)。
    • 空間拉伸練習：活用 flex-grow: 1 讓中央影像區自動填滿剩餘空間。
    • 驗證要點：拖動視窗寬度時，佈局是否能無縫「翻轉」？按鈕與顯示區的比例是否依然和諧？

任務 3：間距與觸控優化 (Spacing & Touch Optimization Lab)

  • 目標：運用現代 CSS 屬性提升操作容錯率，並學習等寬分配的進階技巧。
  • Classroom 部署建議：
    • 精密間距控制：使用 gap 屬性精確設定按鈕間距（建議 1rem 以上），而非單獨設定每個按鈕的 margin。
    • 等寬權重分配：對「低/中/高」速度按鈕組實作 flex: 1，確保它們在不同手機寬度下都能平分空間。
    • 驗證要點：在開發者工具模擬不同手機，確認按鈕是否會互相重疊或有怪異間隙？gap 是否正確提供了防誤觸的空間？

3. 佈局架構與 CSS 規範點評標準 (Layout Architecture & CSS Standards)

此單元的價值在於 「對彈性空間分配的直覺掌控與對多裝置適應性的工程預見」：

  • [ ] 容器層級合理性：是否透過合理的嵌套實現了複雜排版，而非靠「 magic numbers 」湊位置？
  • [ ] 屬性運用準確性：是否深刻理解了主軸與交錯軸在不同 direction 下的變換關係？
  • [ ] 人機工程意識：間距與按鈕大小是否真正考慮到了移動端使用者的觸控體驗？
