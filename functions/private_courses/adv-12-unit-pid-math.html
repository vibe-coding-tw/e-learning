比例 (P)：關注當下的距離

比例項 (Proportional Term) 是 PID 控制器中最直觀的一環。它的邏輯極其簡單：誤差越大，出力越猛；誤差越小，出力越輕。就像你開車接近停車場入口——還差很遠時全速前進，快到了就慢慢滑行。

數學公式

比例項的輸出完全由當前誤差決定：

\[Output_P = K_p \times e(t)\]

其中：

  • \(K_p\) — 比例增益 (Proportional Gain)，決定系統的反應靈敏度
  • \(e(t) = \text{Target} - \text{Measured}\) — 當前時刻的誤差
// C++ 離散實作
float error = targetPos - currentPos;
float output_P = Kp * error;

// 範例數值：
// targetPos = 150 cm, currentPos = 80 cm
// error = 70 cm
// Kp = 2.0
// output_P = 2.0 × 70 = 140（PWM 輸出值）

Kp 的效果：太小 vs. 太大

選擇正確的 \(K_p\) 值是控制工程中最基本的調校任務。以下表格說明不同 \(K_p\) 的影響：

\(K_p\) 大小 系統行為 物理現象
太小 (如 0.5) 反應遲鈍，需很長時間才接近目標 機器人像烏龜爬行，停在目標前很遠處
適中 (如 2.0) 快速接近目標，最終穩定（但仍有殘留誤差） 機器人平穩靠近目標，但最後差幾公分
太大 (如 10.0) 反應過激，在目標附近劇烈震盪 機器人像醉漢一樣左右搖擺，無法停穩
穩定性邊界 當 \(K_p\) 超過某個臨界值時，系統會從「收斂」變成「發散」——誤差不但不減小，反而越來越大。這個臨界點稱為穩定性邊界 (Stability Margin)，是控制工程中必須嚴格避免的危險區域。

P 控制的固有缺陷：靜態誤差

純 P 控制有一個無法迴避的物理侷限——靜態誤差 (Steady-State Error)。當誤差足夠小時，\(K_p \times e\) 產生的出力會低於馬達的最小啟動電壓（死區電壓），系統就卡住了。

用數學表達這個平衡：

\[K_p \times e_{ss} = F_{friction} \quad \Rightarrow \quad e_{ss} = \frac{F_{friction}}{K_p}\]

即使增大 \(K_p\)，殘留誤差 \(e_{ss}\) 只會變小但永遠不會歸零（除非摩擦力為零，但這在物理世界中不存在）。這就是為什麼我們需要積分項 (I) 來徹底消除靜態誤差。

學習重點 P 項是 PID 控制器的「脊梁骨」——它決定了系統的基本反應速度。但它只看「此刻」，對過去的累積和未來的趨勢視而不見。接下來的 I 項和 D 項將分別彌補這兩個盲點。

積分 (I)：記仇的歷史紀錄

積分項 (Integral Term) 是 PID 中的「長期記憶」。它不看此刻的誤差大小，而是關注誤差存在了多久。如果系統卡在目標前方太長時間，積分值會不斷累積，像慢慢填滿的水桶一樣，持續增加推力直到最終突破阻力、消滅殘留誤差。

數學公式

積分項對歷史誤差做時間累積：

\[Output_I = K_i \times \int_0^t e(\tau) \, d\tau\]

離散化為程式碼時，積分變成了累加：

// 離散積分實作
integral += error * dt;    // 每個迴圈週期累加
float output_I = Ki * integral;

// 範例：error = 3 cm（很小，P 項已無力）
// 經過 100 個週期 (dt = 0.01s)：
// integral = 3 × 0.01 × 100 = 3.0
// Ki = 5.0 → output_I = 5.0 × 3.0 = 15.0
// 時間越長，推力越大，最終克服死區！

為什麼 I 項能消除靜態誤差？

回顧 P 項的侷限：當 \(K_p \times e\) 小於馬達死區電壓時，系統卡住。但 I 項不會因為誤差小就放棄——只要誤差持續存在，積分值就會持續增長。最終，\(K_i \times \text{integral}\) 會大到足以克服任何靜摩擦力，把系統推到精確的目標點。

直覺理解 想像你推一扇很重的門。P 項是你施加的即時力量——門很重，你的力氣不夠推動它。但 I 項就像你「越來越生氣」——推了 5 秒沒開，你使出 120% 的力；推了 10 秒還沒開，你怒氣值爆表，最終一定能推開。

積分飽和 (Integral Windup) 問題

I 項的最大副作用是積分飽和 (Windup)。當系統長時間無法到達目標（例如被障礙物卡住），積分值會累積到極大數字。一旦障礙物移除，這個巨大的積分值會導致系統猛烈過衝 (Overshoot)。

階段 積分值狀態 系統行為
正常運行 小幅度累積，穩步消除殘餘誤差 平穩收斂至目標
受阻期間 持續瘋狂累積（因為誤差一直存在） 輸出飽和，馬達全速但車動不了
障礙移除後 巨大的積分值需要很長時間才能「洩掉」 嚴重過衝，可能撞毀

Anti-Windup 解法：積分限幅

// Anti-Windup：限制積分值的上下界
integral += error * dt;
integral = constrain(integral, -MAX_INTEGRAL, MAX_INTEGRAL);

// 或者：當誤差改變方向時，清除積分
if ((error > 0 && lastError < 0) ||
    (error < 0 && lastError > 0)) {
  integral = 0;  // 過零點清除
}
工程實務 在嵌入式系統中，永遠不要讓積分值無限增長。constrain() 是你的第一道防線。專業的控制庫（如 Arduino PID Library）內建 Anti-Windup 機制，但理解其原理才能在極端場景下正確調校。

微分 (D)：預測未來的預警師

微分項 (Derivative Term) 是 PID 中的「先知」。它不關心你離目標多遠（那是 P 的事），也不關心你累積了多少未償誤差（那是 I 的事）。它只關注一件事：誤差正在以多快的速度變化。如果誤差急速減小（代表你正在高速衝向目標），D 項會提前踩下煞車，防止過衝。

數學公式

微分項計算的是誤差的變化率（斜率）：

\[Output_D = K_d \times \frac{de(t)}{dt}\]

離散化實作：

// 離散微分實作
float derivative = (error - lastError) / dt;
float output_D = Kd * derivative;
lastError = error;  // 儲存本次誤差供下次使用

// 範例：dt = 0.01s
// 上一次 error = 50 cm, 這次 error = 30 cm
// derivative = (30 - 50) / 0.01 = -2000 cm/s
// 負值！代表誤差正在快速減小 → 正在接近目標
// Kd = 0.02 → output_D = 0.02 × (-2000) = -40
// D 項產生反向力，踩煞車防止過衝！

D 項的物理直覺：汽車避震器

想像一輛沒有避震器的車開過減速帶——車身會上下劇烈彈跳。避震器的作用是阻尼：它不關心車身在哪個高度（那是彈簧的事），它只關心車身移動的速度。移動越快，阻尼力越大。D 項在控制系統中扮演的就是避震器的角色。

誤差變化趨勢 derivative 值 D 項的行為
誤差快速減小（高速逼近目標） 大負值 產生強力反向煞車，提前減速
誤差緩慢減小（慢速接近） 小負值 輕微阻尼，不影響前進
誤差不變（系統卡住） 零 D 項完全不作用
誤差增大（正在遠離目標） 正值 加速推向目標方向

D 項的致命弱點：雜訊放大

微分運算本質上是對信號做「差分」——這會將高頻雜訊放大到災難性的程度。即使感測器只有 ±1 的微小抖動，經過 \(\div dt\) 運算後也會變成巨大的尖峰信號，導致馬達瘋狂抖動。

雜訊放大效應 假設感測器雜訊 = ±1 cm，dt = 0.01s。
微分值 = 1 / 0.01 = 100 cm/s 的假信號！
若 Kd = 0.5，D 項會輸出 ±50 的隨機抖動——馬達直接瘋掉。

解法：低通濾波器

在計算微分之前，先對誤差信號做移動平均或一階低通濾波，濾除高頻雜訊：

// 一階低通濾波 (Exponential Moving Average)
const float alpha = 0.1;  // 濾波係數 (0~1, 越小越平滑)
filteredError = alpha * error + (1 - alpha) * filteredError;

// 用濾波後的信號計算微分
float derivative = (filteredError - lastFilteredError) / dt;
float output_D = Kd * derivative;
工程取捨 濾波強度是一把雙刃劍：濾太多會引入相位延遲 (Phase Lag)，讓 D 項的「預判」變成「後知後覺」；濾太少則雜訊依然猖狂。找到平衡點是系統調校的藝術。建議從 \(\alpha = 0.1\) 開始，逐步增大直到抖動可接受。

完整 PID 公式彙整

將三項合併，完整的離散 PID 控制器輸出為：

// 完整離散 PID 控制器
error = target - measured;
integral += error * dt;
integral = constrain(integral, -MAX_I, MAX_I);
derivative = (error - lastError) / dt;

output = Kp*error + Ki*integral + Kd*derivative;
output = constrain(output, -255, 255);

lastError = error;

adv-12-unit-pid-math：PID 數學直覺與控制實作

歡迎來到 PID 控制系統的核心單元！在這個程式庫中，你將深入理解 比例 (P)、積分 (I)、微分 (D) 三者如何協同工作，賦予機器人如生物般的直覺反應。

🎯 學習目標

  1. 直覺理解：掌握 P、I、D 在物理系統中的互補角色。
  2. 演算法實作：將離散 PID 公式轉化為高效且魯棒的 C++ 代碼。
  3. 系統優化：解決積分飽和 (Windup) 與微分雜訊 (Noise) 等實際工程挑戰。
  4. 精準調校：透過實驗找出硬體系統的最佳控制參數。

📂 程式庫結構

  • src/main.ino：主要的程式邏輯入口，你將在此實作 PID 迴路。
  • include/pid_engine.h：PID 核心結構與參數定義。
  • lib/SimpleFilter.h：數位濾波器庫，用於處理傳感器雜訊。
  • TUNING_REPORT.md：你的實驗紀錄表，請在此記錄調校過程。
  • tests/：包含自動化測試指令碼，用於驗證你的邏輯正確性。

🚀 任務指南

任務 1：離散 PID 公式實作

你需要實作核心的 PID 運算邏輯：

  • P (Proportional)：根據目前誤差立即反應。
  • I (Integral)：累積歷史誤差，消除靜態偏差（需實作 Anti-windup）。
  • D (Derivative)：預判未來趨勢，抑制震盪（需搭配濾波）。

任務 2：抗飽和與抗雜訊挑戰

  • 實作 積分限幅 (Anti-windup) 邏輯，防止系統因長時間受阻而產生瘋狂過衝。
  • 對微分項進行 數位濾波，避免雜訊放大導致馬達劇烈抖動。

任務 3：性能調校與實驗報告

  1. 使用 Serial Plotter 觀察階躍響應。
  2. 目標：在 1 秒內進入穩定區間（誤差 < 5%），且過衝不得超過 10%。
  3. 將調校過程記錄在 TUNING_REPORT.md 中。

1. 範本程式庫 配置建議

PID 數學實作需要嚴謹的演算邏輯與數據紀錄，建議範本包含：

  • 📂 src/PID_Controller.cpp：PID 核心類別。定義 compute(target, measured, dt) 方法，留下 P、I、D 三項運算的 TODO。
  • 📂 include/control_params.h：參數定義檔。讓學員定義 $K_p, K_i, K_d$ 以及積分限幅（Anti-windup）的閾值。
  • 📂 lib/SimpleFilter.h：數位濾波器庫。提供移動平均濾波的原型，引導學員解決微分項的雜訊放大問題。
  • 📂 TUNING_REPORT.md：調校實驗報告。提供標準化表格，記錄從「只有 P」到「PI」再到「PID」的波形變化歷程。

2. 作業任務部署細節

任務 1：人肉 PID 模擬實驗 (Human-in-the-loop Simulation)

  • 目標：透過體感建立對「累積誤差 (I)」與「預判動量 (D)」的直覺理解。
  • Classroom 部署建議：
    • 行為實驗紀錄：學員需兩人一組（一人當感測器讀數，一人當處理器踩油門）。
    • 決策路徑分析：在 README 中說明：當你發現自己「衝過頭」時，是大腦的哪一個邏輯（P、I 還是 D）告訴你要提前減速？
    • 報告提交：上傳一份關於「物理控制直覺」的思維導圖或文字報告。

任務 2：離散 PID 公式實作與波形最佳化 (Discrete PID Tuning Lab)

  • 目標：將數學積分與微分轉化為代碼，並尋找硬體系統的黃金參數。
  • Classroom 部署建議：
    • 公式精準實作： 
      // 學生需實作：離散 PID
integral += error * dt;            // 累積歷史
derivative = (error - lastError) / dt; // 預判趨勢
output = Kp*error + Ki*integral + Kd*derivative;
    • 積分限幅 (Anti-windup)：學員必須加入限制 integral 最大值的邏輯，防止系統在受阻後產生瘋狂的過衝。
    • 波形截圖驗證：在 Serial Plotter 中記錄「階躍響應 (Step Response)」。目標：在 1 秒內進入穩定區間（誤差 < 5%）且過衝 (Overshoot) 不得超過 10%。

任務 3：微分項抗噪挑戰 (Derivative Noise Resilience)

  • 目標：觀察 D 項對雜訊的放大效應，並實作數位濾波進行補償。
  • Classroom 部署建議：
    • 雜訊放大實驗：在反饋訊號中注入微小干擾，觀測高增益 $K_d$ 如何導致馬達劇烈抖動。
    • 數位濾波實作：學員需在計算微分項前，對 error 進行簡單的移動平均。
    • 性能對比測試：在 README 中展示兩張 Serial Plotter 對比圖：Raw D-Value (極吵) vs. Filtered D-Value (穩定)。
    • PR 深度核核：點評其對「相位延遲」的理解。過強的濾波會導致控制落後，學員是否在濾波強度與穩定度之間找到了平衡？

3. 計算工程導師點評標準 (Computational Engineering Benchmarks)

此單元的價值在於 「對離散數學在即時系統中實踐的精密程度」：

  • [ ] 時間完整性：$\Delta t$ (dt) 是否正確參與了積分與微分的運算？
  • [ ] 代碼魯棒性：是否處理了 dt = 0 的零除錯誤？是否考慮了數據溢位？
  • [ ] 物理還原度：調校出的極速曲線是否符合該馬達與車體的物理運動學規律？
