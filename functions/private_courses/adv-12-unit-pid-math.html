PID 三兄弟的
控制直覺

為什麼機器人會抖？深入 P、I、D 的物理意涵，從直覺掌握公式。

P 1. 比例 (P)：關注當下的距離

比例項就像是你的腳踩油門的力度。離目標越遠，踩得越重；快到了，就慢慢放開。

P Output Formula

\[Output_p = K_p \times Error\]

Error = Target - Current

Static Error Barrier

❌ 靜態誤差

純 P 控制永遠差那麼一點。
因為推力與誤差平衡時，
系統會卡在終點前。

I 2. 積分 (I)：記仇的歷史紀錄

積分項會記住過去「沒到達目標」的時間。如果卡在終點前太久，它會越來越「不爽」，慢慢增加推力直到剷平誤差。

I Output Formula

\[Output_i = K_i \times \sum (Error \cdot \Delta t)\]

消除靜態誤差的最後一哩路

Accumulation Effect

Side Effect: Overshoot Risk

D 3. 微分 (D)：預測未來的預警師

微分項關注的是誤差變化的「斜率」。當系統衝得太快，它會提前踩下阻尼，防止系統衝過頭 (Overshoot)。

D Output Formula

\[Output_d = K_d \times \frac{Error - Last\_Error}{\Delta t}\]
  • 抑制震盪：像避震器一樣讓系統平滑且安分。
  • 對雜訊敏感：但也因為關注微小變化，感測器雜訊也會被成倍放大。

PID 控制直覺與調校實驗室

任務 1：「人肉 PID」平衡實驗 (Human-in-the-loop PID)

角色扮演：模擬控制器 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 策略測試：手動模擬純 P 控制：只根據距離決定推力。觀察系統是否能在靜態摩擦前停下。
  3. 策略再升級：手動模擬 P+I：若卡住不動，則逐漸加壓。體會積分累積的感覺。
  4. 感悟紀錄：在 README 中紀錄在手動模式下，抑制「過度反應」最困難的物理環節是什麼。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 直覺體悟力：能否說明 P、I、D 三者分別對應了人的哪種行為反應（如：急性子、記恨、預判）？
  • [ ] 實驗數據呈現：README 是否描述了不同手動策略下「目標與實際」的差距變化。

任務 2：「參數調校馬拉松」：尋找黃金 Kp/Ki/Kd (Optimal Parameter Hunting)

角色扮演：系統調校工程師 | 難度：⭐️⭐️⭐️ | 估計時長：75 min

🛠️ 執行技術流程 (Git Workflow)

  1. 代碼實作：在 pid_calculate() 中實作離散運算邏輯，包含誤差累積與變化率計算。
  2. 階躍響應測試：設定目標速度突然改變，觀察 Serial Plotter 繪出的響應波形。
  3. 參數閉迴路優化：重複「調大 Kp -> 加入 Ki 消除殘差 -> 加入 Kd 減少過衝」的標準調校步驟。
  4. 數據備存：將最終測得的最佳參數組與波形圖，截圖並 Commit 到專案的 docs/ 目錄。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 波形優良度：最終系統是否能在 1.5 秒內穩定進入目標帶且 Overshoot 低於 10%？
  • [ ] 調校邏輯嚴格性：開發日誌是否清楚說明了每一項參數調整後對波形的具體改善效果？

任務 3：「雜訊大挑戰」：微分項的副作用觀測 (Derivative Noise Stress Test)

角色扮演：訊號處理師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 雜訊注入：在代碼中刻意為反饋值加上隨機噪聲，模擬不穩定的實體訊號。
  2. 壓力觀察：在有雜訊的情況下瘋狂增大 Kd，記錄馬達輸出的抖動強度（或聲音變化）。
  3. 解決方案提案：在 README 中提出一種軟體過濾雜訊的構想，或解釋為何在雜訊大的環境要降低 D 項。
  4. PR 提交：將實驗心得並入 master 分支出並發起 Pull Request。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 物理限制理解：能否分析出感測器雜訊如何在高 \(K_d\) 下導致輸出指令「瘋狂跳火」？
  • [ ] 對策思考力：提出的抗噪建議是否符合嵌入式系統的運算邏輯（如：簡單平均濾波）？

GitHub Classroom 詳細操作指南
