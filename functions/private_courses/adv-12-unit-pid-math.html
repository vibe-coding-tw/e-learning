PID 三兄弟的
控制直覺

為什麼機器人會抖？深入 P、I、D 的物理意涵，從直覺掌握控制公式。

P 1. 比例 (P)：關注當下的距離

比例項就像是你的腳踩油門的力度。離目標越遠，踩得越重；快到了，就慢慢放開。這是最直觀的控制邏輯。

P Output Formula

\[Output_p = K_p \times Error\]

Error = Target - Current

Static Error Barrier

❌ 靜態誤差 (Steady state error)

純 P 控制永遠差那麼一點。
因為推力與誤差平衡時，
系統會卡在終點前不前進。

I 2. 積分 (I)：記仇的歷史紀錄

積分項會記住過去「沒到達目標」的時間。如果卡在終點前太久，它會越來越「不爽」，慢慢增加推力直到剷平誤差。

I Output Formula

\[Output_i = K_i \times \sum (Error \cdot \Delta t)\]

消除靜態誤差的最後一哩路

Accumulation Effect

Side Effect: Overshoot Risk

D 3. 微分 (D)：預測未來的預警師

微分項關注的是誤差變化的「斜率」。當系統衝得太快，它會提前踩下阻尼，防止系統衝過頭 (Overshoot)。

D Output Formula

\[Output_d = K_d \times \frac{Error - Last\_Error}{\Delta t}\]
  • 抑制震盪：像避震器一樣讓系統平滑且安分地到達目標。
  • 預判阻尼：提前感應速度變化，防止機械結構因慣性受損。

Future Prediction Engine

"微分讓我們看見變動的趨勢，而非僅僅是數字。"

單元課程 進階 12 - 數學直覺

adv-12-unit-pid-math：PID 數學直覺與控制實作

歡迎來到 PID 控制系統的核心單元！在這個程式庫中，你將深入理解 比例 (P)、積分 (I)、微分 (D) 三者如何協同工作，賦予機器人如生物般的直覺反應。

🎯 學習目標

  1. 直覺理解：掌握 P、I、D 在物理系統中的互補角色。
  2. 演算法實作：將離散 PID 公式轉化為高效且魯棒的 C++ 代碼。
  3. 系統優化：解決積分飽和 (Windup) 與微分雜訊 (Noise) 等實際工程挑戰。
  4. 精準調校：透過實驗找出硬體系統的最佳控制參數。

📂 程式庫結構

  • src/main.ino：主要的程式邏輯入口，你將在此實作 PID 迴路。
  • include/pid_engine.h：PID 核心結構與參數定義。
  • lib/SimpleFilter.h：數位濾波器庫，用於處理傳感器雜訊。
  • TUNING_REPORT.md：你的實驗紀錄表，請在此記錄調校過程。
  • tests/：包含自動化測試指令碼，用於驗證你的邏輯正確性。

🚀 任務指南

任務 1：離散 PID 公式實作

你需要實作核心的 PID 運算邏輯：

  • P (Proportional)：根據目前誤差立即反應。
  • I (Integral)：累積歷史誤差，消除靜態偏差（需實作 Anti-windup）。
  • D (Derivative)：預判未來趨勢，抑制震盪（需搭配濾波）。

任務 2：抗飽和與抗雜訊挑戰

  • 實作 積分限幅 (Anti-windup) 邏輯，防止系統因長時間受阻而產生瘋狂過衝。
  • 對微分項進行 數位濾波，避免雜訊放大導致馬達劇烈抖動。

任務 3：性能調校與實驗報告

  1. 使用 Serial Plotter 觀察階躍響應。
  2. 目標：在 1 秒內進入穩定區間（誤差 < 5%），且過衝不得超過 10%。
  3. 將調校過程記錄在 TUNING_REPORT.md 中。

🛠️ 如何測試

本程式庫配置了自動化測試，你可以執行以下命令來驗證你的邏輯：

npm install
npm test

1. 範本程式庫 配置建議

PID 數學實作需要嚴謹的演算邏輯與數據紀錄，建議範本包含：

  • 📂 src/PID_Controller.cpp：PID 核心類別。定義 compute(target, measured, dt) 方法，留下 P、I、D 三項運算的 TODO。
  • 📂 include/control_params.h：參數定義檔。讓學員定義 $K_p, K_i, K_d$ 以及積分限幅（Anti-windup）的閾值。
  • 📂 lib/SimpleFilter.h：數位濾波器庫。提供移動平均濾波的原型，引導學員解決微分項的雜訊放大問題。
  • 📂 TUNING_REPORT.md：調校實驗報告。提供標準化表格，記錄從「只有 P」到「PI」再到「PID」的波形變化歷程。

2. 作業任務部署細節

任務 1：人肉 PID 模擬實驗 (Human-in-the-loop Simulation)

  • 目標：透過體感建立對「累積誤差 (I)」與「預判動量 (D)」的直覺理解。
  • Classroom 部署建議：
    • 行為實驗紀錄：學員需兩人一組（一人當感測器讀數，一人當處理器踩油門）。
    • 決策路徑分析：在 README 中說明：當你發現自己「衝過頭」時，是大腦的哪一個邏輯（P、I 還是 D）告訴你要提前減速？
    • 報告提交：上傳一份關於「物理控制直覺」的思維導圖或文字報告。

任務 2：離散 PID 公式實作與波形最佳化 (Discrete PID Tuning Lab)

  • 目標：將數學積分與微分轉化為代碼，並尋找硬體系統的黃金參數。
  • Classroom 部署建議：
    • 公式精準實作： 
      // 學生需實作：離散 PID
integral += error * dt;            // 累積歷史
derivative = (error - lastError) / dt; // 預判趨勢
output = Kp*error + Ki*integral + Kd*derivative;
    • 積分限幅 (Anti-windup)：學員必須加入限制 integral 最大值的邏輯，防止系統在受阻後產生瘋狂的過衝。
    • 波形截圖驗證：在 Serial Plotter 中記錄「階躍響應 (Step Response)」。目標：在 1 秒內進入穩定區間（誤差 < 5%）且過衝 (Overshoot) 不得超過 10%。

任務 3：微分項抗噪挑戰 (Derivative Noise Resilience)

  • 目標：觀察 D 項對雜訊的放大效應，並實作數位濾波進行補償。
  • Classroom 部署建議：
    • 雜訊放大實驗：在反饋訊號中注入微小干擾，觀測高增益 $K_d$ 如何導致馬達劇烈抖動。
    • 數位濾波實作：學員需在計算微分項前，對 error 進行簡單的移動平均。
    • 性能對比測試：在 README 中展示兩張 Serial Plotter 對比圖：Raw D-Value (極吵) vs. Filtered D-Value (穩定)。
    • PR 深度核核：點評其對「相位延遲」的理解。過強的濾波會導致控制落後，學員是否在濾波強度與穩定度之間找到了平衡？

3. 計算工程導師點評標準 (Computational Engineering Benchmarks)

此單元的價值在於 「對離散數學在即時系統中實踐的精密程度」：

  • [ ] 時間完整性：$\Delta t$ (dt) 是否正確參與了積分與微分的運算？
  • [ ] 代碼魯棒性：是否處理了 dt = 0 的零除錯誤？是否考慮了數據溢位？
  • [ ] 物理還原度：調校出的極速曲線是否符合該馬達與車體的物理運動學規律？
