基礎 4 - 馬達驅動架構

H 橋驅動原理：
控制馬達的精密開關

掌握馬達正反轉的核心電路架構，實現精準的動力控制與具有物理慣性的動態煞車。

🏗️ 1. H 橋的構造：電流的幾何學

拓撲原理： 想像一個「H」字型電路，中間的橫桿是馬達，四個支柱各由一個開關（如 MOSFET）控制。

透過對角線導通，我們可以精準引導電流「從左向右」或「從右向左」流經馬達，從而控制馬達的旋轉方向。

構造核心要素

  • ● 四個獨立電子開關 (S1-S4) 構成控流矩陣。
  • ● 對角線開關一對導通，代表一個動力向量。

H-Bridge Topology

🕹️ 2. 四種核心邏輯狀態

正轉 (Forward)

一組對角線開導通，通正向電流。

反轉 (Reverse)

另一組對角線導通，通反向電流。

煞車 (Brake)

低側開關同時導通，短接馬達線圈。

空轉 (Coast)

所有開關切斷，全靠慣性滑行。

⚠️ 禁忌狀態：直通短路 (Shoot-through)

絕對不能同時導通同一側的上下開關！這會導致電源直接對地短路，瞬間產生極大電流燒毀驅動晶片。

驅動邏輯真值表

IN1 IN2 PWM (EN) 動作結果
HIGH LOW 255 正轉 (全速)
LOW HIGH 128 反轉 (半速)
HIGH HIGH ANY 急煞 (Brake)

⚡ 3. 速度與制動的整合

H 橋不只是「開」或「關」。透過在導通期間疊加 PWM 訊號，我們可以控制馬達獲得的平均功率，實現精細的加減速與定位。

Professional API Logic

在程式碼中封裝 drive(int speed) 函式：正數驅動、負數倒車、零則觸發煞車。這將是建構複雜導航演算法的基石。

Brake vs. Coast

「急煞」透過馬達自身的感應電流 (Back EMF) 產生阻力，「空轉」則讓其隨物理慣性耗盡動能。選擇哪種模式，決定了無人車的閃避靈敏度。

任務 3：「全域控制」：雙向動力 API 封裝 (Omni-Control: Bi-directional API)

角色扮演：高級動力系統工程師 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 核心 API 封裝：實作 drive(int speed)，處理 -255~255 的映射轉向邏輯。
  2. 壓力演習實作：撰寫 Sequence「緩起步 -> 全速 -> 持續一秒 -> 急煞 -> 緩慢倒車 -> 靜止」。
  3. 邊界測試：測試當輸入 0 或極小數值時，系統是否能穩定觸發煞車或克服死區起點。

GitHub Classroom 詳細操作指南

本單元教學重點：從單純的數位輸出，跨入「能量橋樑 (Bridge)」控制，理解馬達驅動的工業級架構。

  1. Point 1：不再直連 GPIO - 強申高功率負載必須透過驅動晶片，並解釋橋式電路的電流方向控制。
  2. Point 2：制動的物理意義 - 讓學員從電子邏輯（H,H）連結到物理效能（煞車感），建立工程直覺。
  3. Point 3：API 的魯棒性 - 強調方向切換時的邏輯乾淨度，這對未來寫 PID 或自動巡航算法至關重要。
