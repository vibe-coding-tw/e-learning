簡介

H 橋是控制直流馬達正反轉的核心電路——四個開關排成「H」字型，透過對角線導通改變電流方向。 本單元將帶你理解正轉、反轉、急煞與空轉四種邏輯狀態的物理意義， 並學會將複雜的引腳操作封裝為一個專業的 drive(speed) API。

H 橋的構造：電流的幾何學

要讓直流馬達正轉和反轉，我們需要能夠改變流經馬達的電流方向。這聽起來簡單，但僅靠一個 GPIO 腳位的 HIGH/LOW 切換是做不到的——GPIO 只能決定「有沒有電流」，不能決定「電流往哪個方向走」。H 橋（H-Bridge）電路正是為了解決這個問題而誕生的經典拓撲。

H 型拓撲結構

H 橋由四個電子開關（可以是 MOSFET 或 BJT 電晶體）排列成英文字母「H」的形狀：兩個開關在左側（上臂 S1、下臂 S3），兩個開關在右側（上臂 S2、下臂 S4），馬達橫跨在 H 的中間橫槓上。電源（V+）連接在 H 的頂部，接地（GND）連接在底部。

透過選擇性地導通不同的開關對，電流就能從不同方向流經馬達：

  • S1 + S4 導通（左上 + 右下）：電流從左流向右，馬達正轉。
  • S2 + S3 導通（右上 + 左下）：電流從右流向左，馬達反轉。
  • S3 + S4 導通（左下 + 右下）：馬達兩端同時接地，形成短路煞車。
  • 全部斷開：馬達與電源完全隔離，靠慣性自由空轉。
禁忌：直通短路（Shoot-through） 絕對不能同時導通同一側的上臂與下臂開關（例如 S1 + S3 或 S2 + S4）！這會造成電源直接對地短路，瞬間產生極大電流，可能燒毀開關元件甚至整塊電路板。專業的 H 橋驅動 IC（如 TB6612FNG）內建了防直通邏輯，但理解這個禁忌是工程師的基本素養。

為什麼不直接用繼電器？

機械繼電器雖然也能切換電流方向，但它的切換速度僅約 10–20ms，無法承受 PWM 所需的每秒數千次高頻切換。電子式 H 橋使用 MOSFET 或 BJT 作為開關元件，切換速度可達奈秒等級，完美適配 PWM 的高頻需求。這也是為什麼在需要「速度控制」的場景中，H 橋 + PWM 是唯一的正確選擇。

在教學與原型開發中，我們通常不會自己用離散元件搭建 H 橋，而是使用整合好的驅動 IC。以下是常見的選擇：

四種核心邏輯狀態

H 橋驅動 IC（如 TB6612FNG）透過兩個邏輯輸入腳位（IN1、IN2）的 HIGH/LOW 組合，加上一個 PWM 使能腳位（EN/PWM），可以產生四種完全不同的物理動作。理解這四種狀態是正確使用 H 橋的基礎——每一種都有其獨特的電氣行為與應用場景。

狀態一：正轉（Forward）

當 IN1 = HIGH、IN2 = LOW 時，H 橋內部的 S1（左上）與 S4（右下）導通，電流從馬達的 A 端流入、B 端流出。馬達以 PWM 占空比決定的功率正向旋轉。PWM 值越大，轉速越快。

狀態二：反轉（Reverse）

當 IN1 = LOW、IN2 = HIGH 時，S2（右上）與 S3（左下）導通，電流方向反轉——從 B 端流入、A 端流出。馬達反向旋轉，速度同樣由 PWM 占空比控制。

狀態三：短路煞車（Brake）

當 IN1 = HIGH、IN2 = HIGH（或兩者皆 LOW，取決於 IC 型號）時，馬達兩端被短接到同一電位。此時馬達的旋轉慣性會在線圈中產生反向電動勢（Back-EMF），由於線圈兩端短路，這個反向電壓會驅動一個與旋轉方向相反的電流，形成強大的電磁制動力。馬達會快速減速並停止。

注意 短路煞車的制動力來自電磁感應，而非機械摩擦。馬達轉速越快，產生的 Back-EMF 越大，制動力也越強。這意味著煞車是「自適應」的——高速時制動力強，接近停止時制動力趨近於零。

狀態四：空轉（Coast）

當所有開關斷開（或 STBY 引腳拉低）時，馬達與驅動電路完全隔離。馬達純粹依靠機械慣性繼續旋轉，直到摩擦力自然使其停下。相比短路煞車，空轉的減速過程要緩慢得多。

速度與制動的整合

到目前為止，我們已經學會了如何透過 IN1/IN2 控制方向，也知道了四種邏輯狀態。但真實的馬達控制不只是「全速前進」或「全速後退」——我們需要精細的速度調節。這就是 PWM 與 H 橋結合的威力所在：H 橋決定方向，PWM 決定功率。

PWM + H 橋 = 完整動力控制

在 TB6612FNG 模組上，每組馬達驅動有三個輸入：IN1、IN2 控制旋轉方向，PWM（EN 腳位）控制功率大小。PWM 腳位接收的占空比決定了馬達接收到的平均電壓——50% 占空比約等於半速運轉，25% 約等於四分之一速，以此類推。

這種架構的優雅之處在於：方向邏輯（IN1/IN2）是靜態設定，而速度控制（PWM）是動態調變。你可以先設好「前進」方向，然後只透過調整 PWM 值實現從怠速到全速的平滑過渡。

封裝 drive() API

在專業的韌體開發中，我們不會在主程式中直接操作 IN1、IN2 和 PWM 腳位。正確的做法是將這些底層邏輯封裝成一個簡潔的 API——drive(int speed)：

  • 正數（1 ~ 255）：設定 IN1=HIGH, IN2=LOW，PWM 輸出 speed 值。馬達正轉。
  • 負數（-255 ~ -1）：設定 IN1=LOW, IN2=HIGH，PWM 輸出 abs(speed) 值。馬達反轉。
  • 零（0）：設定 IN1=HIGH, IN2=HIGH，執行短路煞車。馬達立即制動。
void drive(int speed) {
  speed = constrain(speed, -255, 255); // 邊界保護

  if (speed > 0) { // 正轉
    digitalWrite(AIN1, HIGH);
    digitalWrite(AIN2, LOW);
    ledcWrite(MOTOR_CH, speed);
  } else if (speed < 0) { // 反轉
    digitalWrite(AIN1, LOW);
    digitalWrite(AIN2, HIGH);
    ledcWrite(MOTOR_CH, abs(speed));
  } else { // 煞車
    digitalWrite(AIN1, HIGH);
    digitalWrite(AIN2, HIGH);
    ledcWrite(MOTOR_CH, 0);
  }
}
警告 constrain(speed, -255, 255) 是不可省略的安全防護。如果上層程式意外傳入超範圍的值（例如導航算法計算出 300），沒有邊界保護可能導致數值溢位，使馬達產生不可預期的行為。所有對外的 API 都必須在入口處進行輸入驗證。

動作序列（Sequencer）設計模式

有了 drive() API，你就可以用極簡的程式碼撰寫複雜的動作序列。以下是一個典型的功能驗證序列：

void loop() {
  drive(200);   // 全速前進
  delay(2000);  // 持續 2 秒

  drive(0);     // 急煞
  delay(500);   // 等待 0.5 秒

  drive(-128);  // 半速後退
  delay(1000);  // 持續 1 秒

  drive(0);     // 停止
  delay(3000);  // 休息 3 秒後重複
}
提示 在作業 4-3 中，你將實作完整的 drive() API 並設計一組測試序列來驗證所有運動狀態。好的 API 設計讓使用者只需思考「我要馬達怎麼動」，而不需要記住底層的引腳邏輯組合。這就是軟體抽象的價值。

  • 📂 docs/BRAKE_TEST.md：動態阻尼實驗室。要求學員記錄：在全速轉動時，切換至「急煞 (Brake)」與「空轉 (Coast)」模式下，馬達停止旋轉所需的時間或慣性滑動距離。

2. 作業任務部署細節

任務 1：「邏輯審計」驅動真值表實測 (Logic Audit Lab)

  • 目標：透過實測掌握 H 橋的四種基本組態，建立「控制位元」與「物理動作」的直覺。
  • Classroom 部署建議：
    • 遍歷真值表：要求學員撰寫 code 遍歷 (H, L), (L, H), (H, H), (L, L) 四種組合。
    • 記錄動作：在 README 中彙整每種組合對應的馬達動作（正轉、反轉、煞車、空轉）。
    • 驗證要點：README 中必須清楚標註 (H, H) 產生的物理現象，探討為何這被稱為「短路煞車」。

任務 2：「動態阻尼」急煞 vs. 緩停實驗 (Dynamic Damping Lab)

  • 目標：理解反向電動勢 (Back-EMF) 產生的制動力，這對未來「精準停靠」任務至關重要。
  • Classroom 部署建議：
    • 慣性量化量測：設定馬達全速轉動 2 秒後突然停止。A 組使用 Coast (IN1/IN2 全 LOW)，B 組使用 Brake (IN1/IN2 全 HIGH)。
    • 分析現象：觀察並描述兩者「停止瞬間」的差異。
    • 驗證要點：在報告中解釋：為何 Brake 模式能有效縮短停止後的滑行距離？（引用電磁感應原理）。

任務 3：「動力總控」雙向動力 API 封裝 (Omni-Control API Lab)

  • 目標：將複雜的 H 橋引腳操作抽象化為單一 API，建立具備保護邏輯的專業動力接口。
  • Classroom 部署建議：
    • 驅動 API 實作：實作 drive(int speed)，正數表示前進，負數表示後退，0 執行 Brake 邏輯。
    • 邊界條件處理：處理當輸入值大於 255 或小於 -255 的限制保護 (Constrain)。
    • 序列上演習：撰寫一個 Sequencer，執行：全速前進 2s -> 急煞 0.5s -> 半速後退 1s -> 空轉。
    • 最終 Commit：封裝完整的馬達控制類別並 Push。
