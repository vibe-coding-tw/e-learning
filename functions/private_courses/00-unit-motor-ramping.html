Ramping 原理示意

為什麼需要 Ramping？

當您在操控無人車時，若直接將 PWM 從 0 瞬間拉到 255，會發生以下問題：

  • 電流突波：瞬間大電流可能導致 ESP32 重啟。
  • 機械應力：齒輪箱承受瞬間撞擊，容易損壞。
  • 打滑失控：輪胎失去抓地力，導致車輛打轉。
rampAccelStep 就像是「虛擬彈簧」，數值越小，彈簧越軟（加速越平滑）；數值越大，反應越直接。

為什麼使用二進位傳輸？

1. 極致效能：28 Bytes vs JSON 字串 (~150 Bytes)。

2. 零解析開銷：嵌入式端只需 memcpy，無需 JSON Parser。

錯誤排查

❌ 無法掃描到裝置

請重新插拔 ESP32 電源，並確認藍牙權限已開啟。

⚠️ 數據長度錯誤

若讀取時發生 Error，代表網頁端定義的 28 Bytes 與韌體端不一致，請檢查 Struct 對齊。

❌ 寫入後沒反應

請確認數值是否為整數 (Integer)。DataView 對型別非常敏感。

Motor Control Logic (Ramping 狀態機)

void applyRamping() {
  if (targetPWM > currentPWM) {
    currentPWM = min(targetPWM, currentPWM + rampAccelStepT);
  } else if (targetPWM < currentPWM) {
    currentPWM = max(targetPWM, currentPWM - rampAccelStepT);
  }
  ledcWrite(CHANNEL, currentPWM);
}

C. 自動評分 (Autograding) 的設計
