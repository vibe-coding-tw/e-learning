⚖️ 1. 質心的數學本質 (20 min)

物理直覺：想像影像中所有被偵測到的「目標像素（如：黑色循跡線）」都是等重的小球，質心就是這群小球的平均位置。

計算公式

$\bar{x} = \frac{\sum x_i}{N}$

$\bar{y} = \frac{\sum y_i}{N}$

$N$ 是目標像素總點數

質心計算與視覺誤差 (Centroid Calculation & Vision Error)

本專案是一個關於自動駕駛視覺系統核心算法的練習。你將學習如何從原始像素數據中提取「質心」(Centroid)，並將其轉換為導航系統可以使用的「誤差」(Error) 訊號。

學習目標

  1. 實作空間加權平均算法來計算物體質心。
  2. 實作誤差正規化 (Normalization) 與死區 (Deadzone) 邏輯。
  3. 處理空視野與雜訊等例外狀況，提高系統魯棒性。

任務說明

任務 1：質心提取器 (Centroid Extractor)

在 src/vision_math.js 中實作 calculateCentroid 函式：

  • 遍歷影像像素。
  • 計算所有「開啟」像素 (像素值 > 128) 的 X 與 Y 座標總和。
  • 回傳質心的平均座標。
  • 重要：必須處理 totalPixels === 0 的情況（例如回傳畫面中心點），以避免除以零導致的系統崩潰。

任務 2：誤差量化與正規化 (Error Normalization & Deadzone)

在 src/vision_math.js 中實作 normalizeError 函式：

  • 將質心的 X 座標轉換為相對於中心點的偏移量。
  • 將偏移量縮放至 [-1.0, 1.0] 的範圍（左側為負，右側為正）。
  • 實作「死區」邏輯：當偏移量絕對值小於門檻值（預設為 5）時，回傳 0，以避免在中心附近的微小抖動。

任務 3：環境干擾下的魯棒性

確保你的算法能處理 data/ 中的各種模擬情況，包括空視野與帶有雜訊的畫面。

如何開始

  1. 複製此程式庫到你的本地環境。
  2. 使用瀏覽器開啟 ui/dashboard.html 以查看視覺化儀表板。
  3. 在 src/vision_math.js 中撰寫你的實作邏輯。
  4. 執行測試以驗證你的程式碼。

測試指令

你可以分別執行以下測試腳本：

node tests/coordinate.test.js
node tests/deadzone.test.js

程式庫結構

  • src/vision_math.js: 核心數學邏輯（你主要工作的檔案）。
  • ui/dashboard.html: 視覺化偵錯工具。
  • tests/: 自動化測試腳本。
  • data/: 模擬影像數據。

針對 adv-05-unit-centroid-error (質心計算與視覺誤差) 單元，這是一個將「混亂像素」轉化為「精確座標」的數學核心課，對於開發穩定、精準的自駕循跡系統具有決定性意義。

以下是在 GitHub Classroom 部署其作業 (Assignments) 的具體建議：

1. 範本程式庫 (程式庫範本) 配置建議

質心運算需要穩定的數據來源來測試「邊界與極端情況」，建議範本包含：

  • 📂 src/centroid.js：核心運算檔案，提供影像遍歷的 Boilerplate，留下 $\sum x_i$ 與 $\bar{x}$ 計算的 TODO。
  • 📂 data/mock_frames.json：提供一組 $320 \times 240$ 的模擬二值化影像。包含：
    • 正中心圓點（驗證 Error 為 0）。
    • 極右側線條（驗證正規化後 Error 為 1.0）。
    • 空視野（驗證「除以零」保護）。
    • 帶雜訊點的視野（驗證抗干擾能力）。
  • 📂 ui/dashboard.html：視覺化儀表板。學員寫好邏輯後，網頁端會顯示一個「誤差指針」隨著物體位置擺動。
  • 📂 tests/math_accuracy.test.js：自動化評分腳本，精確比對座標計算值與理論值的誤差。

2. 作業任務部署細節

任務 1：「質心提取器」：從像素到座標 (Centroid Extractor Lab)

  • 目標：掌握空間加權平均的基本算法，並落實安全編程（例外處理）。
  • Classroom 部署建議：
    • 核心代碼檢核： 
      // 學生需實作：遍歷像素並計算質心
if (totalPixels === 0) return { x: centerImageX, y: centerImageY, error: 0 };
let centroidX = sumX / totalPixels;
    • Autograding：餵入一個 $4 \times 4$ 的像素陣列測試案例，檢核回傳的 $(\bar{x}, \bar{y})$ 是否與手算結果一致。

任務 2：誤差量化與正規化 (Error Normalization & Deadzone)

  • 目標：將物理位移轉化為通用的導航參數（誤差係數）。
  • Classroom 部署建議：
    • 正規化公式檢核： 
      // 學生需實作：將誤差值縮放至 [-1.0, 1.0]
let error = (centroidX - imageMidPoint) / imageMidPoint;
// 加入死區 (Deadzone) 邏輯
if (Math.abs(centroidX - imageMidPoint) < 5) error = 0;
    • 驗證方式：在儀表板上模擬物體在中心微小擺動，檢核「死區」是否能有效抑制指針抖動。這展現了軟體對硬體抖動的控制策略。

任務 3：環境干擾下的抗性測試 (Noise Robustness Challenge)

  • 目標：設計過濾機制，確保質心不會被背景雜訊「拉走」。
  • Classroom 部署建議：
    • 魯棒性實作：要求學員實作一個「區域權重」或「最小像素門檻」。例如：若偵測到的點數 $N < 10$，則視為雜訊並維持上一幀的質心。
    • 實驗報告：學員需在 README 中記錄：當背景出現「散點雜訊」時，其質心計算值偏離主體的最大百分比。
    • Tutor View (PR 審核)：導師点評學員是否考慮到了 ROI（感興趣區域）的概念，例如只計算畫面下半部的質心以提高循跡穩定性。

3. 量化感知導師點評標準 (Vision Quantization Benchmarks)

此單元的價值在於 「對底層數據處理的嚴謹度」：

  • [ ] 例外防護意識：代碼中是否包含了對「全黑視野（$N=0$）」的顯式判斷，以防止 NaN 導致系統連鎖崩潰？
  • [ ] 數學轉換能力：輸出的 Error 是否具備線性特徵？（即：位移一倍，Error 是否也隨之翻倍）。
  • [ ] 控制靈敏度權衡：設定的「死區大小」是否兼顧了「抑制抖動」與「及時回正」？
