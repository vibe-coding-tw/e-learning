質心計算與視覺誤差

⚖️ 1. 質心的數學本質 (20 min)

物理直覺：想像影像中所有被偵測到的「目標像素（如：黑色循跡線）」都是等重的小球，質心就是這群小球的平均位置。

計算公式

$\bar{x} = \frac{\sum x_i}{N}$

$\bar{y} = \frac{\sum y_i}{N}$

$N$ 是目標像素總點數

  • ● 誤差公式：$Error = \bar{x} - Center_{image}$

🔄 3. 數據正規化 (Normalization) (20 min)

核心概念：將誤差縮放到 $[-1, 1]$ 區間。

"這樣無論影像解析度是 160 還是 1280，控制邏輯都能通用，這就是演算法的抽象化。"

進階課程 5 - 單元 2 質心計算與視覺誤差

視覺感知實驗室：質心運算與導航誤差量化

任務 1：「質心提取器」：從像素到座標 (Centroid Extractor Lab)

角色扮演：影像處理工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 演算法實作：在 centroid.js 中撰寫遍歷邏輯，統計目標像素的 $X, Y$ 分佈與總點數 $N$。
  3. 除零保護：特別實作針對 $N=0$（無目標）情境的保護邏輯，確保程式在空視野下不會發生崩潰。
  4. Push Final：提交具備「高度精確」且運作穩定的質心提取模組。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 座標換算精度：輸出的 $(\bar{x}, \bar{y})$ 是否能隨物體移動即時更新且無明顯殘影？
  • [ ] 程式魯棒性：面對「斷線」或「物體消失」情境時，後端代碼是否能平穩處理例外？

任務 2：誤差量化與動態儀表板 (Visual Error Dashboard)

角色扮演：UI/UX 數據工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 誤差定義：實作偏差值公式 $Error = \bar{x} - Center_{image}$，並完成 $[-1, 1]$ 區間的正規化。
  2. 儀表板開發：建立具備即時指針或進度條的監控頁面，將偏差量轉化為直觀的視覺反饋。
  3. 死區設計：加入軟體死區 (Deadzone) 邏輯，減少物體在中心微小晃動時引發的過度修正。
  4. Push Final：提交具備「極低時延」且具備直觀反饋的誤差監控介面。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 正規化準確度：當物體位於影像最右緣時，輸出的 Error 是否穩定顯示為 $1.0$？
  • [ ] 回應平滑度：視覺化組件是否能流暢呈現誤差變化，且標識出正確的轉向修正方向？

任務 3：環境干擾下的抗性測試 (Noise Robustness Challenge)

角色扮演：系統魯棒性測試員 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 雜訊注入：在測試環境中刻意加入隨機的小亮點雜訊，觀察其對質心計算的干擾規律。
  2. 閾值過濾：實作基於「連通點數」或「區域權重」的過濾方案，剔除無意義的環境背景噪聲。
  3. 實驗報告：在 README 中對比過濾前後的誤差值穩定度，並分析不同雜訊強度下的失效臨界點。
  4. Push Final：提交具備「環境適應能力」且抗干擾的高階質心測量方案。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 抗干擾成效：在背景存在雜訊點的情境下，質心輸出是否仍能穩定鎖定在主體目標上？
  • [ ] 邏輯分區思考：是否考慮到了將影像切割為不同 ROI（關注區域）來降低非必要區域的噪音影響？

GitHub Classroom 詳細操作指南

本單元教學重點：從「看見」到「量化」。理解質心的數學物理本質，並掌握導航誤差的正交化定義。

  1. 重點 1：座標系與參考點 - 讓學生體會「中心點」作為相對參考的重要性，這是閉迴路控制的核心起點。
  2. 重點 2：正規化的價值 - 強調將像素距離轉換為 $[-1, 1]$ 比例，是為了實現跨解析度的通用控制器邏輯。
  3. 重點 3：除以零的災難與預防 - 透過 2-1 任務中的空視野情境，培養學生在寫入物理感測代碼時必須具備的例外處理思維。
