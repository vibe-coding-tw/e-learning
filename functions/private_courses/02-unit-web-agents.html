GitHub Copilot Workspace

原生 Web 代理環境：以「任務」為中心的平台

規劃 (Plan)

代理人會讀取整個程式庫，針對你的需求寫出一份實施計畫。

執行 (Implement)

確認計畫後，它會自動在雲端環境中修改多個檔案。

驗證 (Verify)

自動執行測試，如果失敗，代理人會自行修復錯誤。

核心工作流：From Issue to PR

一旦任務完成並通過驗證，Agent 會直接幫你開啟 Pull Request。它是目前自動化程度最高的 Web 編程環境。

GitHub Agentic Workflows (自動化)

讓 AI Agent 變成一個自動運行的後台機器人。例如：直接指派 Issue 給 @copilot，它會自動開分支、寫程式碼並提交 PR。

.github/workflows/agent.yml
# 範例：當 Issue 標記為 'ai-fix' 時啟動 Agent on: issues: types: [labeled] jobs: agentic-fix: if: github.event.label.name == 'ai-fix' runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Run Copilot Agent run: copilot-agent --issue ${{ github.event.issue.number }}

02-Unit-Web-Agents: 原生 Web 代理環境實戰

歡迎來到第二單元！在這個單元中，我們將離開本地開發環境，將開發戰場移向雲端，並實踐 「以任務為中心 (Task-Centric)」 的現代開發流程。你將學會如何指揮 AI 代理人（AI Agents）在雲端完成從需求描述到代碼部署的完整流程。

🎯 學習目標

  • 掌握 Copilot Workspace 的完整開發閉環。
  • 學會配置並觸發 GitHub Actions 中的 AI 自動化工作流。
  • 實踐 Zero-Install 開發模式，利用瀏覽器即時修復與重構代碼。

🚀 作業任務

任務 1：從需求到 PR - Copilot Workspace 實戰

在本任務中，你將練習如何透過精確的需求描述，引導 AI 生成解決方案並開啟 Pull Request (PR)。

  1. 在程式庫中開啟一個新的 Issue，描述一個功能需求或修復任務。
  2. 點擊 "Open in Copilot Workspace"。
  3. 在 Workspace 介面中審查 AI 產生的計畫 (Plan)。
  4. 重要：根據實際需求微調計畫，體現你作為「指揮官」的決策價值。
  5. 執行計畫並開啟 Pull Request。
  6. 將產生的 PR 連結填寫至本程式庫的 PR_LINK.md 檔案中。

任務 2：幽靈編程師 - GitHub Actions 自動化

本任務要求你將 AI 代理人嵌入 CI/CD 流程中，實現標籤觸發的自動代碼檢查與修復。

  1. 查看 .github/workflows/ai-agent.yml，確保其觸發條件設定為 types: [labeled]。
  2. 在程式庫中針對一個現有的 Bug 或改進點建立 Issue。
  3. 手動為該 Issue 貼上 ai-fix 標籤。
  4. 前往程式庫的 "Actions" 頁籤，確認自動化工作流已成功啟動並開始執行任務。

任務 3：雲端極簡主義 - Zero-Install 開發實踐

利用瀏覽器快捷鍵快速進入輕量化編程環境，完成代碼重構。

  1. 在程式庫首頁按下鍵盤上的 . (句點) 鍵，進入 github.dev 編輯環境。
  2. 針對 tasks/style-refactor.css 進行跨檔案的代碼重構（例如：修改 CSS 變數或統一風格）。
  3. 完成後進行 Commit，並在 Commit Message 中註明：Refactored via github.dev Web Agent。

📝 提交清單

  • [ ] 已將 Task 1 的 PR 連結填入 PR_LINK.md。
  • [ ] 已在 Task 2 的 Actions 頁面確認執行紀錄。
  • [ ] 已完成 Task 3 的網頁版 Commit。

📂 程式庫結構說明

  • .github/workflows/：存放自動化腳本，包含 AI 代理人的調度邏輯。
  • tasks/：包含待重構或修復的練習檔案。
  • src/：本單元的核心程式碼區。
  • PR_LINK.md：請在此記錄你的任務成果連結。

提示：本單元強調「瀏覽器即是最強 IDE」。請盡可能在雲端環境完成所有操作。

針對 02-unit-web-agents (原生 Web 代理環境) 單元，這個單元將開發戰場從本地 IDE 移向了雲端，強調 「以任務為中心 (Task-Centric)」 的開發流程。

以下是在 GitHub Classroom 部署其作業 (Assignments) 的具體建議：

1. 範本程式庫 (Template 程式庫) 配置建議

由於本單元高度依賴 GitHub 的原生功能（如 Workspace, Actions, Issues），範本程式庫應包含：

  • 📂 .github/workflows/ai-agent.yml (範本)：預置 GitHub Actions 結構，但在 steps 中留下 run: TODO 區塊，讓學員練習配置 AI 代理人指令。
  • 📂 tasks/：放入幾個帶有明確 Bug 或需重構的檔案，作為學員在 Workspace 中練習的對象。
  • 📂 .github/ISSUE_TEMPLATE/：預置一個「Feature Request」模板，引導學員寫出能讓 Agent 理解的高品質需求描述。

2. 作業任務部署細節

任務 1：從需求到 PR - Copilot Workspace 實戰 (Product Engineering)

  • 目標：練習「需求描述 -> AI 規劃 -> 代碼生成 -> PR 開啟」的完整雲端閉環。
  • Classroom 部署建議：
    • 成果提交：本任務的最終產出是一個 Pull Request (PR)。
    • 核核重點：導師需查看該 PR 的描述。一個標準的 Workspace PR 會包含 Agent 的原始計畫。導師應檢查學員是否有在 Workspace 視窗中對計畫進行過人為微調 (Plan Tuning)，這體現了「指揮官」的介入價值。

任務 2：幽靈編程師 - GitHub Actions 自動化 (Workflow Automation)

  • 目標：將 AI 代理人嵌入 CI/CD 流程，實現標籤觸發的自動代碼檢查。
  • Classroom 部署建議：
    • 驗證方式：學員需在程式庫的 "Actions" 頁籤 展示一次成功的執行紀錄。
    • 操作要求：學員必須手動為一個 Issue 貼上 ai-fix 標籤，並截圖證明自動化工作流已啟動並對相關代碼進行了掃描與回報。
    • Autograding：檢查 .github/workflows/ai-agent.yml 是否正確設定了 types: [labeled] 觸發條件。

任務 3：雲端極簡主義 - Zero-Install 開發實踐 (Cloud Native)

  • 目標：熟練使用瀏覽器快捷鍵 . (句點) 進入輕量化網頁編程環境。
  • Classroom 部署建議：
    • 成果提交：學員需在 github.dev (或是網頁版 VS Code) 中完成一次跨檔案的代碼重構（例如：修改 CSS 變數並同步至所有 HTML 模板）。
    • 核核重點：查看提交日誌 (Commit Log)。網頁版編輯器的 Commit 通常會帶有特定的元數據標籤。要求學員在 Commit Message 註明「Refactored via github.dev Web Agent」，以核對其操作環境。

3. 給導師的評估與導引重點 (Tutor Benchmarks)

本單元的評分重點在於學員對 「雲端開發流程 (Cloud Flow)」 的適應度：

  • [ ] 規劃能力：在 Workspace 生成計畫前，學員是否能寫出具備 Context 的 Issue 描述？
  • [ ] 自動化邏輯：是否理解 YAML 腳本中如何調度 AI 權限與觸發時機？
  • [ ] 應急修復力：是否學會了在無開發環境的電腦上，利用電腦瀏覽器快速修復線上 Bug 並 Push 的技巧？
