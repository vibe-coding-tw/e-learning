Module 1: Intuitive Lab

滑鼠 vs. 手指：速度對決

為什麼手機上 click 比較慢？

在觸控螢幕剛誕生時，為了偵測「雙擊縮放 (Double Tap to Zoom)」，瀏覽器在使用者點擊後會等待約 300ms，確認沒有第二次點擊才會觸發 `click` 事件。

這對於一般網頁瀏覽沒問題，但對於即時遙控車來說，300ms 的延遲就像是一世紀那麼長！

⚠️ 實驗觀察重點

  • 使用滑鼠點擊方塊，觀察 log。
  • 使用手機模式 (DevTools Cmd+Shift+M) 輕觸方塊。
  • 注意 touchstart 與 click 的時間差。

Module 2: Core Principles

事件結構與監聽陷阱

特性 滑鼠事件 (Mouse) 觸控事件 (Touch)
關鍵事件 mousedown, mousemove, mouseup touchstart, touchmove, touchend
座標取得 e.clientX, e.clientY e.touches[0].clientX (多層結構)
多點支援 無 (單游標) 有 (TouchList 陣列)
預設行為 選取文字、拖曳圖片 捲動頁面、雙擊縮放、長按選單
JavaScript - Universal Coordinate Helper 
function getPosition(e) {
  let x, y;
  
  if (e.type.includes('mouse')) {
    // 滑鼠直接取座標
    x = e.clientX;
    y = e.clientY;
  } else {
    // 觸控要判斷事件類型
    // touchmove/start 看 touches
    // touchend 看 changedTouches
    let touch = e.touches[0] || e.changedTouches[0];
    x = touch.clientX;
    y = touch.clientY;
  }
  
  return { x, y };
}

! 被動監聽器 (Passive Listener)

為了提升捲動效能，現代瀏覽器預設將 `touchstart` 和 `touchmove` 視為 { passive: true }。 這意味著你無法在這些事件中呼叫 preventDefault() 來阻止網頁捲動。

若要製作搖桿或拖曳功能，必須明確設定 { passive: false }：

element.addEventListener('touchmove', handler, { passive: false });

任務 2：防抖滑桿 (Interaction Locking)

角色扮演：用戶體驗研究員 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 定義容器：製作一個橫向滑桿 UI，並監聽其 touchmove 事件。
  2. 鎖定視窗：手動設定 { passive: false }，並在滑動時呼叫 e.preventDefault()。
  3. 精確對位：實作 getPosition(e) 函數，確保滑塊中心點精確跟隨手指位置。
  4. 提交報告：將滑桿操作的錄影或截圖存入 README 後 push。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 視窗鎖定：操作滑桿時，網頁背景是否完全靜止（不發生上下捲動）？
  • [ ] 座標穩定度：手指快速滑動時，滑塊有無發生劇烈跳動或偏移？
  • [ ] 代碼穩健性：是否正確區分了 e.touches 與 e.changedTouches 的使用時機？

GitHub Classroom 詳細操作指南

本單元教學重點：掌握跨平台事件生成的順序與攔截機制。

  1. 第一步：範本設置 - 提供一個會產生「數據污染」的壞代碼範例（即重複觸發點擊），讓學生自行診斷。
  2. 第二步：發布作業 - 在 Classroom 啟用專用測試環境，重點檢查對 { passive: false } 的應用。
  3. 第三步：開發引導 - 確保學生理解 e.preventDefault() 的真正作用：它切斷了「觸控」向「模擬滑鼠」轉換的鍊條。
  4. 第四步：驗收回饋 - 檢查 getPosition 函數是否能跨環境穩定運行，這是未來所有複雜 UI 的基礎。
