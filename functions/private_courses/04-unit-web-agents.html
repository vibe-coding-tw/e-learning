傳統 IDE 是給人寫的，現代 Web Agent 是給「任務」執行的。掌握未來軟體開發的新常態。

GitHub Copilot Workspace

原生 Web 代理環境：以「任務」為中心的平台

規劃 (Plan)

代理人會讀取整個倉庫，針對你的需求寫出一份實施計畫。

執行 (Implement)

確認計畫後，它會自動在雲端環境中修改多個檔案。

驗證 (Verify)

自動執行測試，如果失敗，代理人會自行修復錯誤。

核心工作流：From Issue to PR

一旦任務完成並通過驗證，Agent 會直接幫你開啟 Pull Request。它是目前自動化程度最高的 Web 編程環境。

GitHub.com Copilot Chat

直接在倉庫網頁使用對話。最新功能支援 Tool Calls，這意味著 Agent 可以調用工具來搜尋程式碼、分析檔案結構。

按下 Ctrl+B 啟動

在倉庫頁按 "." (句點) 進入

GitHub Agentic Workflows (自動化)

讓 AI Agent 變成一個自動運行的後台機器人。例如：直接指派 Issue 給 @copilot，它會自動開分支、寫程式碼並提交 PR。

.github/workflows/agent.yml
# 範例：當 Issue 標記為 'ai-fix' 時啟動 Agent on: issues: types: [labeled] jobs: agentic-fix: if: github.event.label.name == 'ai-fix' runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Run Copilot Agent run: copilot-agent --issue ${{ github.event.issue.number }}

功能 GitHub.com (Web Chat) Copilot Workspace VS Code (Desktop)

原生 Web 代理環境 - AI 代理人專題

任務 1：從需求到 PR - Copilot Workspace 實戰

角色扮演：產品工程師 (Product Engineer) | 難度：⭐️⭐️ | 估計時長：30 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立任務：在 GitHub 建立一個 Issue，描述一個明確的功能需求（例如：新增一個 Footer 組件）。
  2. 啟動 Workspace：點擊 "Open in Copilot Workspace"。
  3. 審核計畫：讓 Agent 產生實施計畫，並針對不滿意的地方進行對話調整。
  4. 執行並提交：核准代碼變動，讓 Agent 自動開啟 Pull Request。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 需求轉譯：Issue 的描述是否足以讓 Agent 產生正確且完整的實施計畫？
  • [ ] 雲端驗證：是否成功在 Workspace 的雲端測試環境中驗證了功能？

任務 2：幽靈編程師 - GitHub Actions 自動化

角色扮演：自動化專家 (Automation Specialist) | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 配置工作流：建立 .github/workflows/ai-agent.yml。
  2. 觸發設定：設定當 Issue 被貼上 ai-fix 標籤時，自動觸發 Copilot Agent 掃描。
  3. 實測運行：對一個帶有 Bug 的檔案貼上標籤，觀察 GitHub Actions 是否自動啟動並產出分析報告。
  4. 同步分析：將 Action 的運行截圖與設定檔一併繳交。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 自動化邏輯：Workflow 觸發條件是否設定正確？
  • [ ] 整合深度：是否理解如何將 AI 代理人嵌入既有的 CI/CD 流程中？

任務 3：雲端極簡主義 - Zero-Install 開發實踐

角色扮演：雲端開發者 (Cloud Developer) | 難度：⭐️⭐️ | 估計時長：20 min

🛠️ 執行技術流程 (Git Workflow)

  1. 秒級啟動：在瀏覽器倉庫頁面按下 . (句點) 進入 github.dev。
  2. 協作對話：使用內建的 Copilot Chat 進行一段跨檔案的功能重構。
  3. 即時同步：在不離開瀏覽器的情況下完成 Commit 並 Push 回 Origin 分支。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 雲端操作熟練度：是否能流暢使用網頁版 IDE 與 Agent 進行協作？
  • [ ] 開發思維轉換：是否體會到「免安裝、即開即寫」對緊急線上修復的價值？
