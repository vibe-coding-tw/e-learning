Browser-Native Agent

網頁版代理人環境

你不一定需要安裝桌機版 VS Code，GitHub 的 AI Agent 能力已經全面覆蓋 Web 端。

GitHub Copilot Workspace

原生 Web 代理環境：以「任務」為中心的平台

規劃 (Plan)

代理人會讀取整個倉庫，針對你的需求寫出一份實施計畫。

執行 (Implement)

確認計畫後，它會自動在雲端環境中修改多個檔案。

驗證 (Verify)

自動執行測試，如果失敗，代理人會自行修復錯誤。

核心工作流：From Issue to PR

一旦任務完成並通過驗證，Agent 會直接幫你開啟 Pull Request。它是目前自動化程度最高的 Web 編程環境。

立即前往

GitHub.com Copilot Chat

直接在倉庫網頁使用對話。最新功能支援 Tool Calls，這意味著 Agent 可以調用工具來搜尋程式碼、分析檔案結構。

按下 Ctrl+B 啟動

github.dev / Codespaces

這是運行在雲端的「完整開發環境」。你可以在 Terminal 中安裝套件、執行程式碼，並使用完整的 Agent Mode。

在倉庫頁按 "." (句點) 進入

GitHub Agentic Workflows (自動化)

讓 AI Agent 變成一個自動運行的後台機器人。例如：直接指派 Issue 給 @copilot，它會自動開分支、寫程式碼並提交 PR。

.github/workflows/agent.yml
# 範例：當 Issue 標記為 'ai-fix' 時啟動 Agent on: issues: types: [labeled] jobs: agentic-fix: if: github.event.label.name == 'ai-fix' runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Run Copilot Agent run: copilot-agent --issue ${{ github.event.issue.number }}

Web vs. Desktop 功能對照

功能 GitHub.com (Web Chat) Copilot Workspace VS Code (Desktop)
跨檔案編輯 僅限建議 自動規劃與執行 Agent Mode 自助修改
執行指令 (Terminal) 不支援 雲端執行與驗證 本地終端機執行

作業指引 (Assignment Guide)

1. GitHub Classroom 設定

請將 web-agent-workspace-lab 連結至課程儀表板。學生需透過此分發倉庫實作 GitHub Copilot Workspace 的 Plan 描述與 PR 產生流程。

2. 翻轉課堂策略

  • 課前預習：確保 GitHub 帳號具備 Copilot 權限，並能在瀏覽器直接開啟 Workspace 頁面。
  • 課中實作：引導學生針對同一個 Issue，嘗試不同的自然語言描述，觀察 Agent 產生的程式碼差異。

3. 單元任務 (Unit Tasks)

任務說明 配分
【作業 2-1】透過 Workspace 成功產生一個包含 3 個以上檔案修改的 PR 40 pts
【作業 2-2】配置 GitHub Actions 並成功觸發 Agent 掃描紀錄 30 pts
【作業 2-3】在 github.dev 中完成一段完整的代碼重構與 Commit 30 pts
