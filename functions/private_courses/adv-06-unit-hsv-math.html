進階 6 - 單元 3

為何電腦看不懂紅色：
HSV 的故事

深入探討 HSV 相比於 RGB 在變動光影環境下的穩定性，學習如何定義精準的顏色區間。

💔 1. RGB 的致命傷：光影耦合 (20 min)

在 RGB 空間中，如果你調暗燈光，$R, G, B$ 三個數值會同時下降。這導致針對「紅色」的門檻值會隨著環境光變動而頻繁失效。

Coupling Problem

  • 數值漂移：同一物體在陽光與陰影下的特徵不一致。
  • 非互斥性：R 分量包含亮度與顏色訊息。

RGB Space Observation

Light: (255, 60, 60) Bright Red
Dark: (110, 20, 20) Deep Red

📐 2. HSV：幾何上的「分而治之」 (30 min)

將亮度 (Value) 與顏色 (Hue) 徹底分離。在 HSV 中，只要顏色不變，即使燈光強弱改變，Hue (色相) 依然保持高度穩定。

⭕ 3. 色環的連續性與斷點

紅色在色環的「斷點」處 ($0^{\circ}$ 與 $360^{\circ}$)。學習如何處理數值上的跨區過濾，讓追蹤不因為微小的角度跳變而失效。

Binary Mask Strategy

  • 雙重門檻 (Dual Threshold)：同時檢查 $H > 350$ 或 $H < 10$。
  • 飽和過濾 (S-Filter)：排除灰度背景雜訊。
  • 多區間合併 (Union)：將多段色值結果整合成最終 Mask。

單元課程 進階 6 - HSV 數學

色彩實驗室：HSV 數值分析與遮罩實作

任務 1：色彩壓力測試 (Color Stress Test)

角色扮演：光學感測工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 環境變數紀錄：對同一目標（紅球）在陽光直射與陰影處分別截取 RGB 像素值與換算後的 HSV 值。
  3. 變異係數計算：在 README 中列出表格。計算 $\sigma / \mu$ (標準差/均值)，量化對比 RGB 的混亂度與 HSV 的穩定度。
  4. 診斷提交：在 PR 中論述為何在高動態光照下（如賽道反光）必須捨棄 RGB 模型。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 實驗數據嚴謹度：報告中是否明確包含不同照度下的數據對照？
  • [ ] 邏輯洞察力：能否指認出 RGB 中哪個通道受亮度 (Intensity) 影響最劇烈？

任務 2：動態 HSV 遮罩產生器 (HSV Mask Generator)

角色扮演：視覺演算法開發者 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 數學轉換實作：在 JS 中實作 rgbToHsv() 函式。
  2. 互動控制建立：建立滑桿 (Sliders) 以定義 H_min, H_max, S_min 等參數。當 $H_{min} > H_{max}$ 時，自動轉換為 OR 邏輯以應對紅色斷點。
  3. Mask 即時渲染：將畫面上符合條件的像素塗白，不符合的塗黑，產出純淨的二值化遮罩。
  4. Push PR：提交具備「紅色斷點處理能力」的強大遮罩產生器。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 算法邊界處理：當滑桿拉至 350-10 度區間（跨越 0 度）時，遮罩是否能正確顯現紅色區域而不消失？
  • [ ] 物件封裝性：rgbToHsv 函式是否足夠獨立且能被其他模組輕鬆調用？

任務 3：紅綠燈行為優先權判定 (Decision Logic)

角色扮演：智慧系統架構師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 多色彩遮罩配置：定義紅燈 (H~0) 與 綠燈 (H~120) 兩組 HSV 設定。
  2. 決策引擎實作：撰寫 DecisionMaker()。邏輯：IF red_count > threshold THEN mode = STOP; ELSE IF green_count > threshold THEN mode = FOLLOW;。
  3. 連通測試：在畫面中同時移動紅、綠板，錄製影片證明系統在「衝突情境」下依然能做出符合安全優先順序的判斷。
  4. Push Final：提交具備視覺邏輯決策能力的專案。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 邏輯完備性：當畫面上同時出現紅綠顏色時，是否能依照「安全優先」原則輸出停止指令？
  • [ ] 虛警率 (False Positives)：透過 S (飽和度) 過濾，是否能有效排除淺色背景引起的誤判？

GitHub Classroom 詳細操作指南

本單元教學重點：破解「為什麼這顏色追不到」的迷思。引導學生從幾何學角度理解色彩空間的穩定性。

  1. 重點 1：RGB 的光影陷阱 - 引導學生動手實驗，讓他們親眼看見數值在影子下如何劇烈崩塌。
  2. 重點 2：色環的環狀性質 - 強調 H 值是「角度」而非線性數值，解釋為什麼紅色追蹤通常最難（因為斷在 0/360）。
  3. 重點 3：決策者的思維 - 將色彩偵測與「行為樹 (Behavior Tree)」初步掛鉤。顏色只是輸入，決策才是 AI。
