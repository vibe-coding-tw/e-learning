🖱️ 1. 模式切換的 UI 邏輯

控制組件的選擇決定了介面的直覺程度。我們需要明確區分「互斥模式」與「功能開關」。

Segmented Control

適合互斥模式。如：手控 vs. 自動。當前選中項必須具備高對比高亮。

Toggle Switch

適合單項功能的啟用。如：避障功能 ON/OFF。直覺且省空間。

📱 2. 響應式佈局與觸控安全

機器人介面需要在晃動的手機上操作，按鈕必須夠大。同時在自動駕駛時，必須隔離手動搖桿防止誤觸。

Safety Guidelines

  • ❌ 避免在搖桿附近放置關鍵切換鈕。
  • ✅ 自動模式下，將搖桿設為 pointer-events: none。
  • ✅ 按鈕最小觸控尺寸需大於 48dp x 48dp。

UX Engineering

"介面的權限隔離設計，是軟體層級對物理安全的第一道防禦。"

Authority Shield: Active

🚨 3. 雙向同步與 E-Stop 警報

介面必須反映硬體的真實物理狀態。E-Stop 按鈕應整合視覺與觸覺 (Haptic) 回饋，確保高度注意力鎖定。

Haptic & Vision

  • 狀態自動校正：若硬體自動退出某模式，網頁按鈕必須同步自動跳回，徹底消除假象。
  • 紅色遮罩 (Red Overlay)：最高優先權全螢幕覆蓋，強制阻止所有非安全操作。
  • 震動 API：利用 navigator.vibrate() 在緊急事件發生時提供物理脈衝警告。

任務 1：戰術面板：CSS 狀態切換實作 (Tactical Panel & Authority)

角色扮演：介面設計工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並引入基礎樣式表。
  2. 面板佈構設計：實作包含 Manual (手控)、Auto-Pilot (自律) 與 Eco-Mode (節能) 的側邊切換選單。
  3. 動態反饋開發：利用 JavaScript 在切換時切換 .active 類別，賦予選中項發光、陰影或微縮放的視覺反饋。
  4. 權限隔離實施：設計核心逻辑：當模式切換至 Auto-Pilot 時，原本的手動搖桿區應自動設定為 pointer-events: none 並降低不透明度。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 視覺層級清晰度：當前激活模式是否具备極高的視覺對比度，讓使用者能瞬间判斷系統控制權歸屬？
  • [ ] 操作安全機制：切換至自動模式後，原本的手動觸發事件是否能被確實拦截，防止震動或意外誤觸干預行駛？

任務 2：雙向狀態同步與校正 (Bi-directional State Sync)

角色扮演：全端開發工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 心跳監聽層實作：在 App 端實作數據解析層，監聽來自 ESP32 每秒回傳的 current_state 心跳報文。
  2. 真值校正邏輯：若回傳狀態與 UI 現狀不符（如硬體因低壓自動退回 IDLE），介面必須強制跳轉至正確狀態，不允許「假性成功」。
  3. 重連一致性驗證：測試機器人斷電重啟後，網頁介面是否能根據連線後收到的首筆數據包自動恢復正確的模式按鈕。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 狀態真實性：網頁面板顯示的選項是否 100% 代表「硬體真實現狀」而非僅是「使用者最後點擊的紀錄」？
  • [ ] 解析効能優化：狀態同步的報文解析邏輯是否簡潔，有無造成前端 UI 渲染主執行緒的顯著卡頓？

任務 3：「E-Stop」緊急停機與多感官警告 (Haptic & Visual E-Stop)

角色扮演：安全互動設計師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 緊急遮罩設計：按下實體或虛擬 E-Stop 後，立即呼叫一組帶有最高 z-index 的紅色全覆蓋遮罩，強行封鎖所有操作。
  2. 觸覺回饋接入：正確實作 navigator.vibrate() API，在緊急觸發或異常報警時提供同步的物理脈衝震動感。
  3. 安全重置序列：設計具備高注意力的「長按 3s」或「特定滑動解鎖」機制，才能解除紅色遮罩並復歸至待機狀態。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 安全機制搶佔性：緊急停機遮罩是否能優先於任何 UI 層級觸發，並成功鎖定全域輸入權限？
  • [ ] 多感官警示實效：是否充分利用了行動裝置特性（震動）來顯著提升緊急事件的知覺強度？

GitHub Classroom 詳細操作指南

本單元教學重點：從「展示型 UI」進化為具備工業安全標準的「控制面板」。

  1. 重點 1：交互的權限感 - 引導學員理解 UI 不只是按鈕，更是賦予或限制機器動作权力的樞紐（活用 `pointer-events`）。
  2. 重點 2：對抗認知假象 - 強調「雙向同步」的物理性意義：若硬體壞了，UI 絕不准顯示「運行中」。
  3. 重點 3：多維警訊告知 - 演示紅色遮罩（視覺）與震動（觸覺）如何共同作用，確保操作者在極端突發狀況下能瞬間察覺系統停擺。
