任務 1：戰術面板：CSS 狀態切換實作 (Tactical Panel & Authority)

角色扮演：設計工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫。
  2. 設計：實作三模式切換選單。
  3. 動態反饋：利用 JS 切換發光及陰影。
  4. 隔離實施：Pilot 模式鎖死搖桿互動。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 層級清晰：激活模式具備極高對比？
  • [ ] 安全機制：自動模式確實攔截誤觸？

任務 2：雙向狀態同步與校正 (Bi-directional State Sync)

角色扮演：全端工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 監聽層：監聽 current_state 心跳。
  2. 真值校正：回傳不符時，強制跳轉 UI。
  3. 一致性驗證：驗證重啟後自動恢復正確模式。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 真實版本：面板代表物理現狀而非使用紀錄？
  • [ ] 效能優化：解析邏輯未造成主線程卡頓？

任務 3：「E-Stop」緊急停機與警報 (Haptic & Visual E-Stop)

角色扮演：安全設計師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 緊急遮罩：按下即封鎖全域權限。
  2. 震動接入：實作異常物理脈衝。
  3. 安全序列：設計長按解鎖機制。

GitHub Classroom 詳細操作指南

本單元教學重點：從「展示型 UI」進化為控制面板。

  1. 重點 1：交互權限感 - UI 是限制或賦予動作權力的樞紐。
  2. 重點 2：對抗認知假象 - 物理同步勝過使用者意圖。
  3. 重點 3：多維警訊 - 演示視覺與觸覺共同作用。
