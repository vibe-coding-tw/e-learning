簡介

線性映射雖然簡單，但人類對搖桿的操控感受並不是線性的——低速區需要更細膩的控制，高速區則希望一推就到位。 指數曲線（或二次曲線）能讓低速段「放大」、高速段「壓縮」，大幅提升操控手感。 本單元比較不同響應曲線的數學與實際體驗差異，幫你找到最適合自己車輛的映射函數。

情境
你的遙控車在低速微調時總是「頓頓的」，稍微推一點搖桿就衝太快； 但全速行駛時又覺得推到 80% 跟 100% 沒什麼差別。 這就是線性映射的先天限制——套上一條指數曲線後，低速更絲滑、高速更分明，操控體驗瞬間升級。

線性響應 (Linear Response)

線性響應是最簡單、最直覺的搖桿映射方式：輸出與輸入成正比， 數學表達為 \(f(x) = x\)。搖桿推了 30%，馬達就輸出 30% 的動力；推了 80%，就輸出 80%。 斜率在整個範圍內恆定為 1.0，沒有任何加速或減速的區段。

這種映射的最大優勢是可預測性——操控者可以精確地知道搖桿位置對應多少出力， 不需要適應任何非線性的「手感曲線」。對於初學者或需要快速部署的場景，線性響應通常是第一選擇。

數學特性

線性響應函數的完整定義：

\[f(x) = x, \quad x \in [-1.0,\; 1.0]\]

其關鍵特性如下：

  • 恆定斜率：\(f'(x) = 1.0\)，在中心和邊緣的敏感度完全相同
  • 零截距：\(f(0) = 0\)，搖桿中心精確對應零輸出
  • 端點閉合：\(f(1) = 1\)，全推對應全力
  • 符號保留：\(f(-x) = -f(x)\)，正反方向對稱

優缺點分析

面向 評價 說明
直覺性 優 輸入與輸出 1:1 對應，無需學習或適應
實作複雜度 優 無需額外計算，直接使用歸一化值
低速微調 劣 中心區域斜率過陡，微小位移就產生明顯出力變化
高速辨識度 劣 80% 到 100% 的出力差異不明顯，缺乏「爆發」層次
備註 線性響應的「中心過敏」問題在實際操控中非常明顯：當你嘗試讓遙控車慢慢前進時， 稍微推一下搖桿就會「衝」出去。這是因為人類手指的精度約為 5%–10%， 而線性映射在這個區間的輸出變化已經足以讓馬達產生可感知的動力。

程式碼實作

線性響應的實作極為簡單——歸一化後的值本身就是線性映射的結果：

// 線性響應：直接回傳歸一化值
function linearResponse(x) {
  return x;
}

// 在控制管線中使用
const raw = normalize(joystickY, maxRadius);
const output = linearResponse(raw); // output === raw

雖然線性響應看似「什麼都不用做」，但把它封裝成獨立函式仍然很重要—— 這為後續替換成指數曲線或其他映射函數提供了統一的介面，是良好的軟體架構實踐。

指數響應 (Exponential Response)

線性響應的先天限制在於：人類的手指肌肉控制精度有限，中心區域需要更大的「容錯空間」。 指數響應（或稱 Expo 曲線）透過數學方式重新分配敏感度—— 讓搖桿中心附近的斜率變平緩（低速更容易微調），邊緣附近的斜率變陡峭（高速動力更集中）。 這就是遙控車與無人機領域中所謂的「軟中心 (Soft Center)」效果。

混合映射公式

最常用的指數響應公式是線性項與三次方項的加權混合：

\[f(x) = (1 - k) \cdot x + k \cdot x^3\]

其中 k 是靈敏度係數（範圍 0.0 到 1.0）。 它控制了曲線的「彎曲程度」：

  • \(k = 0\)：退化為純線性 \(f(x) = x\)，無任何 Expo 效果
  • \(k = 0.5\)：適中的軟中心，適合一般場景
  • \(k = 0.8\)：明顯的軟中心，適合需要精密微調的低速操作
  • \(k = 1.0\)：完全三次方 \(f(x) = x^3\)，最大的中心緩衝區

不同 k 值的數值比較

輸入 \(x\) \(k = 0\) (線性) \(k = 0.5\) \(k = 0.8\) \(k = 1.0\) (三次方)
0.1 0.100 0.051 0.021 0.001
0.3 0.300 0.164 0.081 0.027
0.5 0.500 0.313 0.200 0.125
0.8 0.800 0.656 0.570 0.512
1.0 1.000 1.000 1.000 1.000
備註 注意表中最後一列：不論 k 取什麼值，f(1.0) 永遠等於 1.0。 這是混合公式的數學性質——端點閉合保證了全推搖桿時，馬達一定能達到 100% 出力。

程式碼實作

以下 JavaScript 函式實作了帶符號保留的指數響應曲線：

// 混合指數響應：(1-k)·x + k·x³
function expoResponse(x, k) {
  return (1.0 - k) * x + k * Math.pow(x, 3);
}

// 使用範例
expoResponse(0.5, 0.7);   // → 0.238 (遠小於線性的 0.5)
expoResponse(-0.5, 0.7);  // → -0.238 (符號自動保留)
expoResponse(1.0, 0.7);   // → 1.0 (端點閉合)
提示 這個公式使用的是奇數次冪（三次方），因此天然保留了輸入的正負號—— \((-x)^3 = -(x^3)\)。如果使用偶數次冪（如平方），必須額外用 Math.sign(x) 恢復符號，否則後退操作會失效。

掌握了指數響應的數學原理後，下一步是學習如何在實際操控中選擇最佳的 k 值， 以及如何實作即時模式切換功能。

調校實務

沒有一條曲線能適用所有場景：在寬闘賽道上全速衝刺時，線性響應的直覺反應可能更好； 在狹窄障礙物之間精密穿梭時，指數響應的軟中心更能避免碰撞。 專業的遙控系統應該支援即時模式切換，讓操控者根據任務需求在不同的響應曲線之間熱切換。

模式切換架構

實作模式切換的關鍵在於將映射函式抽象為可替換的策略。以下是推薦的架構設計： 

// 定義可切換的響應模式
let currentMode = 'expo';
let expoK = 0.7;

function applyResponse(x) {
  if (currentMode === 'linear') return x;
  if (currentMode === 'expo')
    return (1 - expoK) * x + expoK * Math.pow(x, 3);
  return x;
}

// 模式切換（由 UI 按鈕觸發）
function toggleMode() {
  currentMode = currentMode === 'linear' ? 'expo' : 'linear';
  updateModeUI();
}
警告 模式切換的瞬間可能產生控制跳動 (Control Glitch)：假設搖桿正停在 0.5， 從線性模式（輸出 0.5）切換到 Expo 模式（輸出 0.238）， 馬達會突然減速。在高速場景下，這種跳動可能造成失控。建議在搖桿接近中心時才允許切換， 或加入平滑過渡 (interpolation) 機制。

視覺回饋設計

模式切換必須搭配清晰的視覺回饋，避免操控者不知道目前處於哪種模式。 以下是推薦的 UI 設計原則：

模式 建議配色 標籤文字 使用場景
直覺模式 (Linear) 暖色系（橙、紅） DIRECT 開闘區域、高速巡航
精準模式 (Expo) 冷色系（藍、紫） PRECISION 狹窄空間、精密定位

符號保留的重要性

在實作非線性曲線時，一個常見的陷阱是偶數次冪破壞符號。 例如如果使用平方映射 \(f(x) = x^2\)，當 \(x = -0.5\) 時輸出是 \(0.25\)（正數）， 後退操作就失效了。正確的做法有兩種：

  • 使用奇數次冪：如三次方 \(x^3\)，天然保留正負號（推薦）
  • 手動補償符號：Math.sign(x) * Math.pow(Math.abs(x), n)
// 安全的冪映射（適用於任意指數）
function safePow(x, n) {
  return Math.sign(x) * Math.pow(Math.abs(x), n);
}

safePow(-0.5, 2);  // → -0.25 (符號正確保留)
safePow(-0.5, 3);  // → -0.125
提示 在實際的繞錐測試中，先用線性模式建立對賽道的空間感， 再切換到 Expo 模式嘗試精密走位。比較兩種模式下的碰撞次數與完成時間， 你將親身體會到「曲線優化能彌補人類肌肉震顫」的工程洞見。
