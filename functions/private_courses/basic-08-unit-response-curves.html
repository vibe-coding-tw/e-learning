基礎 8 - 控制數學

曲線優化：
線性 vs. 指數響應

掌握演算法如何彌補人類肌肉控制的不足，打造完美手感的「人機合一」。

📏 1. 線性響應 (Linear Response)

定義：輸出與輸入成正比（\(y = x\)）。搖桿推多少，馬達就出多少力。

👍 優點

極度直觀、好預測且完全沒有反應延遲。

👎 缺點

中心點過於敏感。對於需要精密繞錐或微幅轉向的場景，操作感極難調教。

Linear Logic

\[f(x) = x\]

Constant Slope: 1.0

📈 2. 指數響應 (Exponential Response)

利用冪函數 (Power Function) 讓輸出隨輸入非線性增長。這模擬了人類對精細動作的心理需求。

Soft Center (軟中心)

中心區域斜率平緩，容許大幅度的「手感」微調而不失控。

Punchy End (暴力尾段)

當搖桿推至邊緣時，數值急劇攀升，保留賽車所需的爆發動力。

PRO MIXER (JS) 
// 經典混合公式 (1-k)x + kx^3
function getExpoResponse(x, k) {
  return (1.0 - k) * x + k * Math.pow(x, 3);
}

// k=0: 純線性 / k=1: 純指數

🔘 3. 調校實務：熱切換與視覺回饋

沒有一套曲線是萬能的。在不同的賽道、不同的駕駛心理下，我們需要即時切換控制特性的能力。

Control Architecture

  • 模式熱切換：在網頁上實作 Toggle，一鍵切換精準模式 (Expo) 與直覺模式 (Linear)。
  • 視覺連動：當 K 係數改變時，UI 的配色或圖示應有顯著回應，防止操作誤判。
  • 極性補償：處理 \(x^2\) 等偶次方時，務必以 Math.sign(x) 保留原本的方向資訊。

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 混合模型編寫：在 JS 邏輯中實作複合公式：Output = (1 - k) * x + k * x^3。
  3. 參數可定義化：透過調校變量 k (0.0 ~ 1.0)，實現輸出的控制靈敏度線性/指數平滑切換。
  4. 正確性檢錄：記錄在 \(k=0.7\) 時，輸入 \(0.5\) 對應的輸出值，確保運算邏輯在 Math.pow 介入後仍能精確計算。
