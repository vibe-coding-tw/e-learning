🙈 1. 開迴路控制：盲目的信任

系統僅根據事先設定的指令做事，完全不觀察輸出的結果。就像蒙著眼睛走路。

System Characteristics

  • 無反饋 (No Feedback)：輸出不影響輸入。
  • 抗干擾力弱：電壓下降、摩擦力增加會直接導致任務失敗。
  • 範例：微波爐定時、舊式洗衣機。

👁️ 2. 閉迴路控制：觀察與修正的智慧

系統會將目前的輸出（如速度、位置）反饋回來與目標值對比，算出「誤差 (Error)」，並動態調整出力。

Feedback Equation

\[Error = Target - Measured\]

🎛️ 3. 進化：為什麼需要 PID？

單純的比例修正 (P) 常因物理阻力無法到達精確點。PID 結合成熟的歷史累積 (I) 與未來預測 (D)，達成絲滑控制。

Control Matrix

  • P (Now)：誤差越大出力越大。負責系統的主反應。
  • I (Past)：解決靜態誤差。對抗摩擦力與長時間偏移。
  • D (Future)：預判煞車。防止系統衝過頭 (Overshoot)。

開迴路與比例控制實驗室

任務 1：「盲人領航員」：開迴路實驗 (Blind Navigator: Open-Loop Test)

角色扮演：盲測工程師 | 難度：⭐️ | 估計時長：40 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 排除反饋：在 setup() 中刻意不初始化編碼器，僅憑 delay() 控制行進。
  3. 跨路面壓力測試：在阻力極低的磁磚與阻力高的地毯上執行同一命令（如 PWM 150, 2000ms）。
  4. 誤差紀錄分析：測量與目標(1公尺)的實際差距。將路面差異導致的距離差紀錄在 README 中。

  • [ ] 物理直覺證實：能否透過實驗結果具體描述開迴路系統在「變動環境」下的低度強韌性？
  • [ ] 實驗數據嚴謹度：README 是否包含多種路面下的測量值對比曲線或表格？

任務 2：「比例的力量」：首個 P 控制器實作 (The Power of P Control)

角色扮演：控制工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. Error 計算實作：在主迴圈內持續讀取編碼器，並實作 Error = Target - CurrentPos。
  2. 比例輸出映射：定義 \(K_p\)，並將 \(Output = K_p \times Error\) 限制在馬達可接受的 PWM 範圍（0-255）。
  3. 臨界震盪觀測：逐漸增加 \(K_p\)，直到機器人抵達終點時發生劇烈的前後晃動。在 README 紀錄此臨界 \(K_p\)。
  4. 成果 Push：將具備基礎 P 回饋邏輯的代碼提交至 GitHub 倉庫。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 閉環邏輯正確性：當 Error 變小時，出力是否也跟著線性減小，而非保持恆定？
  • [ ] 現象描述力：能否說明 \(K_p\) 的物理意義與系統穩定性（穩態 vs 震盪）之間的關係？

任務 3：「靜態誤差挑戰」：對抗物理阻力 (Fighting Steady-State Error)

角色扮演：系統優質化專家 | 難度：⭐️⭐️ | 估計時長：50 min

🛠️ 執行技術流程 (Git Workflow)

  1. 阻力場景重現：在機器人背部增加重物，並使用任務 2 調校出的最佳 \(K_p\) 運行。
  2. 死區現象紀錄：觀測並記錄機器人最終停止的位置（通常會早於 1 公尺）。
  3. 靜態誤差量化：測量此時的「殘餘誤差」，並解釋為什麼 P 項無法獨自克服此阻力。
  4. 對策提案：在 README 中提出：除了加大 Kp，還有什麼數學方法能對抗「長時間累積」的阻力？

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 殘餘誤差概念掌握：學員是否理解當出力小於靜摩擦力時，單純依賴當前 Error 是無法收斂的？
  • [ ] 邏輯推演力：提出的對策（如積分項概念）是否具備邏輯連貫性，為下一單元的 I 做準備？

GitHub Classroom 詳細操作指南

本單元教學重點：從「蒙眼行軍」到「睜眼修正」的範式轉移。

  1. 重點 1：開迴路的代價 - 透過跨路面實驗，讓學員親手證實缺乏反饋的系統在真實世界中有多脆弱。
  2. 重點 2：負回饋的初探 - 手把手指導學員實作 Error 與 Kp，建立「偏差越大力度越大」的直覺物理關聯。
  3. 重點 3：工程死區 (Dead-zone) - 關鍵在於讓學員觀察到 P 項的侷限性，從而對下一單元的積分 (I) 產生強烈求知慾。
