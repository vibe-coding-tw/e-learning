Open vs. Closed Loop

從盲目預測到精確修正：掌握自動駕駛最核心的反饋控制邏輯。

🙈 1. 開迴路控制：盲目的信任

系統僅根據事先設定的指令做事，完全不觀察輸出的結果。就像蒙著眼睛走路。雖然簡單，但在物理環境變動時（如路面摩擦、電壓波動）極易失控。

System Characteristics

  • 無反饋 (No Feedback)：輸出不影響輸入，僅憑時序控制。
  • 抗干擾力弱：電壓下降、摩擦力增加會直接導致任務偏差。
  • 常見範例：舊式微波爐定時器、傳統洗衣機、無感測馬達驅動。

👁️ 2. 閉迴路控制：觀察與修正的智慧

系統具備感測器，能將目前的輸出（如速度、位置）反饋回來與目標值對比，算出「誤差 (Error)」，並隨時調整出力以導正軌跡。

🎛️ 3. 進化：為什麼需要 PID？

單純的比例修正 (P) 常因物理阻力無法到達精確點。PID 結合成熟的歷史累積 (I) 與未來預測 (D)，達成全自動且絲滑的運動路徑。

Control Evolution

  • P (Proportional)：當下的衝勁。誤差越大出力越猛。
  • I (Integral)：過去的記性。剷除累積的殘留誤差與阻力。
  • D (Derivative)：未來的預感。提前減速防止慣性衝過頭。

Full Autonomy

"讓機器人具備像人類淋浴調溫般的感知與修正能力，是控制工程的最高境界。"

【作業 12-2】

「比例神經網路」

恢復感測器。實作 Error 計算與 \(K_p\) 比例映射。觀測機器人在不同 \(K_p\) 下的反應速度，並記錄產生「蛇行」震盪的臨界點。

🪜 50 min

【作業 12-3】

「死區抗爭者」

在機器人上增加重物，觀察 P 控制為何永遠停在目標前方的某一點。分析定標誤差與靜摩擦力的平衡邏輯，並提出改進構想。

感知控制實驗室：開迴路 vs. 閉迴路

任務 1：開迴路盲測實驗 (Blind Open-Loop Task)

角色扮演：系統可靠性檢測員 | 難度：⭐️ | 估計時長：40 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 盲目指令：在代碼中移除編碼器呼叫，僅憑 delay(2000) 控制行進。
  3. 物理擾動測試：在不同摩擦力的路面上執行同一次任務。
  4. 誤差紀錄：量化並在 README 記錄兩種環境下的「停車位置差距」。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 原理描述：能否準確分析「為什麼路面變平滑後誤差會急遽增大」的物理原因？
  • [ ] 數據完整性：README 中是否包含至少 3 組以上的實驗誤差對比。

任務 2：P 比例控制器實作 (Proportional Feedback Design)

角色扮演：初級控制工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 反饋實作：在主迴圈內持續監測 Error = Target - CurrentPos。
  2. 線性映射：實作 Output = Kp * Error。
  3. 極限調校：挑戰將 \(K_p\) 加到極致而不產生持續震盪。
  4. Push Final：提交具備「誤差感應」能力的閉迴路代碼。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 邏輯自洽性：當 Error 接近零時，馬達 PWM 是否隨之平滑下降？
  • [ ] 關鍵參數標註：README 中是否標註了系統進入震盪狀態的臨界 \(K_p\) 值？

任務 3：靜態誤差與阻力分析 (Static Error Case Study)

角色扮演：誤差診斷專家 | 難度：⭐️⭐️ | 估計時長：50 min

🛠️ 執行技術流程 (Git Workflow)

  1. 障礙重現：在負重或小坡度環境下執行任務 2 的控制器。
  2. 現象監測：記錄機器人最終「差一公分停下」的靜態誤差狀態。
  3. 瓶頸分析：寫下為何純 P 控制在出力低於「啟動電壓」後無法自動修正的理論分析。
  4. Push Final：提交具備「物理瓶頸洞察」的控制優化提案報告。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 物理死區理解：學員是否能理解馬達靜摩擦力與比例出力之間的平衡關係？
  • [ ] 未來對策：是否能在報告中正確引導出「需要將時間累加 (I)」的需求背景？

GitHub Classroom 詳細操作指南

本單元教學重點：從「程式時序控制」進化到「物理結果控制」。學會如何量化誤差並進行負回饋修正。

  1. 重點 1：信任是危險的 - 透過開迴路盲測，打破學員「代碼正確 = 物理正確」的迷思。
  2. 重點 2：負回饋的簡單美 - 帶領學員理解 \(Error \rightarrow Output\) 的線性魔法。
  3. 重點 3：殘留誤差的挫折感 - 故意讓學員感受到純 P 控制的不足，這是導入後續 PID 課程最強的動機。
