Open vs. Closed Loop

從盲目預測到精確修正：掌握自動駕駛最核心的反饋控制邏輯。

🙈 1. 開迴路控制：盲目的信任

系統僅根據事先設定的指令做事，完全不觀察輸出的結果。就像蒙著眼睛走路。雖然簡單，但在物理環境變動時（如路面摩擦、電壓波動）極易失控。

System Characteristics

  • 無反饋 (No Feedback)：輸出不影響輸入，僅憑時序控制。
  • 抗干擾力弱：電壓下降、摩擦力增加會直接導致任務偏差。
  • 常見範例：舊式微波爐定時器、傳統洗衣機、無感測馬達驅動。

👁️ 2. 閉迴路控制：觀察與修正的智慧

系統具備感測器，能將目前的輸出（如速度、位置）反饋回來與目標值對比，算出「誤差 (Error)」，並隨時調整出力以導正軌跡。

The Core Equation

\[Error = Target - Measured\]

🎛️ 3. 進化：為什麼需要 PID？

單純的比例修正 (P) 常因物理阻力無法到達精確點。PID 結合成熟的歷史累積 (I) 與未來預測 (D)，達成全自動且絲滑的運動路徑。

Control Evolution

  • P (Proportional)：當下的衝勁。誤差越大出力越猛。
  • I (Integral)：過去的記性。剷除累積的殘留誤差與阻力。
  • D (Derivative)：未來的預感。提前減速防止慣性衝過頭。

Full Autonomy

"讓機器人具備像人類淋浴調溫般的感知與修正能力，是控制工程的最高境界。"

單元課程 進階 12 - 控制原理

PID 控制原理作業 (adv-12-unit-pid-control)

歡迎來到 PID 控制原理 單元！在這個作業中，你將從「盲目駕駛」進化到「有感反饋」，親身體會如何透過比例控制 (P Control) 讓機器人更精準地到達目標位置。

🎯 學習目標

  1. 理解 開迴路 (Open-Loop) 與 閉迴路 (Closed-Loop) 系統的差異。
  2. 實作 比例控制器 (P-Regulator) 並理解其運作邏輯。
  3. 分析 靜態誤差 (Steady-State Error) 的成因與物理侷限。

🛠️ 作業任務

任務 1：開迴路盲測實驗 (Open-Loop Task)

  • 檔案：src/open_loop_drive.ino
  • 說明：執行該指令碼，觀察機器人在不同表面（如磁磚、地毯、桌面）上移動相同的時間後，停下的位置是否有差異。
  • 紀錄：請將觀測結果填寫至 docs/exp_analysis.md 的 Task 1 表格中。

任務 2：P 比例控制器實作 (Proportional Feedback Design)

  • 檔案：src/main.ino
  • 說明：
    1. 開啟 src/main.ino，找到 TODO: Task 2 區塊。
    2. 計算誤差：error = targetPos - currentPos。
    3. 計算輸出：output = KP_GAIN * error。
    4. 將計算結果傳遞給馬達驅動函數 motorDrive(output)。
  • 調校：嘗試不同的 KP_GAIN 值（在 include/control_gain.h 中定義），尋找能最快到達目標且不產生劇烈震盪的「黃金 $K_p$」。
  • 紀錄：將不同 $K_p$ 值的測試數據填寫至 docs/exp_analysis.md 的 Task 2 表格。

任務 3：靜態誤差與阻力分析 (Steady-State Error Analysis)

  • 說明：在機器人上增加負荷（例如放一瓶水），觀察 P 控制為何會「差一點點」停在目標點前。
  • 紀錄：在 docs/exp_analysis.md 中回答為什麼當誤差極小時，馬達會發出聲音卻不動。

📂 程式庫結構

  • src/main.ino: 主程式實作區（Task 2 & 3）。
  • src/open_loop_drive.ino: 開迴路測試指令碼（Task 1）。
  • include/control_gain.h: 參數定義區（目標位置、增益值）。
  • docs/exp_analysis.md: 你需要填寫的實驗日誌。
  • tests/: 自動化邏輯檢核腳本。

2. 作業任務部署細節

任務 1：開迴路盲測實驗 (Blind Open-Loop Task)

  • 目標：打破「程式寫好就一定會跑準」的幻想，理解環境擾動的威力。
  • Classroom 部署建議：
    • 故障模擬檢測：學員需在磁磚與地毯上執行同一個 delay(2000) 指令。
    • 誤差量化紀錄：在 README 中上傳照片，展示在兩種環境下停車點的巨大落差（通常會有 10-30 公分不等的偏差）。
    • 原因分析報告：說明為什麼純時序控制無法應對物理世界的變動。

任務 2：P 比例控制器實作 (Proportional Feedback Design)

  • 目標：賦予機器人「視覺（反饋）」，實現自動導向目標。
  • Classroom 部署建議：
    • 反饋邏輯實作： 
      // 學生需實作：負回饋計算
float error = targetPos - currentPos;
float output = Kp * error;
motorDrive(output);
    • 穩定性調校：挑戰尋找能讓機器人「最快到達、且不發生劇烈震盪」的黃金 $K_p$ 值。
    • 震盪觀測驗證：要求學員錄影展示：當 $K_p$ 太大時，機器人是如何像沒對準的舵機一樣瘋狂抖動。

任務 3：靜態誤差與阻力分析 (Steady-State Error Analysis)

  • 目標：發現 P 控制的阿基里斯之踵——無法克服靜摩擦力的「最後一哩路」。
  • Classroom 部署建議：
    • 負重挑戰：在機器人上放一瓶水，觀察 P 控制為何會「差一點點」停在目標點前，且馬達只發出嗡鳴聲而不動。
    • 物理死區分析：要求學員解釋「為什麼當誤差極小時，比例出力會低於馬達啟動電壓？」。
    • PR 深度核核：點評其對解決方案的預見。是否能正確推導出「我們需要一個能隨著時間不斷累積力量的項（積分 I）」來解決這個問題？
    • 數據趨勢圖：在 Serial Plotter 中截圖展示 Error 曲線最終停在一條水平線上（非零）的現象。

3. 控制工程導師點評標準 (Control Engineering Benchmarks)

此單元的價值在於 「對系統行為的量化診斷能力」：

  • [ ] 誤差敏感度：代碼是否能隨時根據位置變化調整驅動力？
  • [ ] 穩定性邊界意識：學員是否理解了增益過大會導致系統發散（震盪）？
  • [ ] 物理侷限洞察：是否能清楚描述「靜態誤差 (Steady-state error)」的成因？
