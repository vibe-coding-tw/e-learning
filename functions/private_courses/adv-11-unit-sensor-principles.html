👁️ 1. 光電遮斷器：數位化的眼睛

光電遮斷器（Photo-Interrupter）是最常見的數位測速感測器。它由一顆紅外線 LED 與一顆光電晶體 (Phototransistor) 對向安裝組成「光閘」。 當編碼盤（Encoder Disk）的格柵旋轉通過光閘時，紅外線交替被遮斷與通過，產生對應的高低電位數位訊號。

注意：光電遮斷器是數位感測器，輸出只有 HIGH (3.3V) 和 LOW (0V) 兩種狀態。 它不像類比感測器能輸出連續值。遮斷（Blocked）時輸出 HIGH；通過（Clear）時輸出 LOW（以 NPN 型為例，實際極性依電路設計而定）。
光電遮斷器硬體規格
元件 功能 典型規格
紅外線 LED 發射端：持續發射 940nm 紅外線 正向電流 20mA，需串聯限流電阻
光電晶體 接收端：有光時導通，無光時截止 回應時間 < 10μs
編碼盤 固定在馬達軸上，隨馬達旋轉 常見格數：10、20、40 格/圈
輸出訊號 數位方波，頻率正比於轉速 0V – 3.3V（或 5V）

編碼盤的格數決定了感測器的解析度 (PPR, Pulses Per Revolution)。 格數越多，每一格代表的角度越小，能感知的最小旋轉量也越精細。 但格數並非越多越好——格柵太密會導致紅外線無法完全被遮斷，訊號邊緣變模糊。

光電遮斷器接線步驟

1
將紅外線 LED 端串聯限流電阻（220Ω – 330Ω），連接至 3.3V 與 GND。
2
將光電晶體端的集極接 3.3V（經上拉電阻 10kΩ），射極接 GND。集極同時接至 ESP32 的 GPIO 輸入腳位。
3
將編碼盤安裝在馬達軸上，確保格柵能順暢通過光閘的縫隙。
4
用 digitalRead() 或示波器驗證：手動旋轉時，GPIO 應交替讀到 HIGH 與 LOW。
提示：許多市售光電遮斷器模組已內建限流電阻與上拉電阻，可直接接 VCC、GND、OUT 三線。 但自製電路時，請務必確認限流電阻的阻值，避免燒毀紅外線 LED。

📉 2. 方波訊號與物理換算

光電遮斷器的輸出是一連串的方波 (Square Wave)——在示波器上呈現為規則的高低電位交替。 理解方波的三個關鍵參數，是從原始訊號中提取物理資訊的基礎。

方波訊號三大參數
參數 定義 在測速中的意義
振幅 (Amplitude) 高電位與低電位的差值（如 3.3V） 決定 GPIO 能否正確辨識為 HIGH/LOW
頻率 (Frequency) 每秒的脈衝數（Hz） 頻率 = PPR × RPS，與轉速成正比
占空比 (Duty Cycle) HIGH 時間佔一個週期的百分比 理想編碼盤為 50%（格柵寬 = 間隙寬）

從脈衝計數換算轉速的核心公式如下。假設在 1 秒內計數到 N 個脈衝，而編碼盤每圈有 PPR 個格：

核心公式： \[RPM = \frac{N \times 60}{PPR}\] 其中 N 為每秒脈衝數，PPR 為每圈脈衝數 (Pulses Per Revolution)，RPM 為每分鐘轉速。
C++ (Arduino) — 基礎 RPM 計算
const int PPR = 20;           // 編碼盤格數
volatile int ticks = 0;

void IRAM_ATTR countTick() {
    ticks++;
}

void setup() {
    Serial.begin(115200);
    attachInterrupt(digitalPinToInterrupt(SENSOR_PIN),
                    countTick, FALLING);
}

void loop() {
    delay(1000);              // 每秒取樣一次
    noInterrupts();
    int n = ticks;
    ticks = 0;                // 歸零重新計數
    interrupts();

    float rpm = (float)n * 60.0 / PPR;
    Serial.printf("Ticks: %d, RPM: %.1f\n", n, rpm);
}
重要：PPR 的值必須透過實測校準，而非僅依賴規格書。 方法：手動旋轉馬達一整圈，記錄中斷計數器的增量，即為實際 PPR。 不同編碼盤、不同安裝方式可能導致 PPR 偏差 ±1–2 格。

📏 3. 解析度與物理量映射

知道轉速 (RPM) 還不夠——在機器人導航中，我們需要的是線速度 (cm/s) 和行進距離 (cm)。 這需要將感測器的脈衝數映射到實際的物理尺寸，建立完整的脈衝→距離→速度換算管線。

核心映射公式： \[\text{cm/tick} = \frac{\pi \times D}{PPR}\] 其中 D 為輪胎直徑 (cm)，PPR 為每圈脈衝數。 每一個脈衝代表車輪前進了 cm/tick 公分。
物理量換算管線
原始資料 換算公式 結果 (物理量)
脈衝總數 Ntotal 距離 = Ntotal × cm/tick 行進距離 (cm)
每秒脈衝數 N 速度 = N × cm/tick 線速度 (cm/s)
每秒脈衝數 N RPM = N × 60 / PPR 轉速 (RPM)
RPM v = RPM × π × D / 60 線速度 (cm/s)，交叉驗證用
C++ (Arduino) — 完整物理量換算
const int   PPR      = 20;          // 每圈脈衝數（實測值）
const float WHEEL_D  = 6.5;         // 輪胎直徑 cm
const float CM_TICK  = PI * WHEEL_D / PPR;  // ≈ 1.021 cm/tick

volatile int ticks = 0;
long totalTicks = 0;

void IRAM_ATTR countTick() { ticks++; }

void loop() {
    delay(100);  // 100ms 取樣
    noInterrupts();
    int n = ticks;
    ticks = 0;
    interrupts();

    totalTicks += n;
    float speed_cmps = n * CM_TICK * 10;   // ×10 因取樣 100ms
    float distance   = totalTicks * CM_TICK;

    Serial.printf("Speed: %.1f cm/s | Distance: %.1f cm\n",
                  speed_cmps, distance);
}
警告：輪胎直徑 D 必須精確量測——包含橡膠胎的壓縮量。 建議的校準方法：讓機器人直線行駛已知距離（如 100cm），比對脈衝累積數與理論值，反推實際有效直徑。 輪胎磨損、氣壓變化都會影響 D 的準確性。
提示：解析度 (cm/tick) 越小，定位精度越高。若需要更高精度，可以： (1) 增加編碼盤格數、(2) 使用 CHANGE 模式同時偵測上升緣與下降緣（等效 PPR 翻倍）、 (3) 使用減速齒輪組放大脈衝頻率。
