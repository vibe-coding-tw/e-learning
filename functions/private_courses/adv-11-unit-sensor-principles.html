👁️ 1. 光電遮斷器：數位化的眼睛

透過紅外線對管與編碼盤（Encoder Disk）的組合，將旋轉的機械運動轉化為 0 與 1 的數位訊號。這是所有數位測速系統的物理基礎。

Working Principle

  • 遮斷 (Blocked)：輸出高電位 (3.3V)。
  • 通過 (Clear)：輸出低電位 (0V)。
  • 解析度 (PPR)：一圈產生的脈衝數計為格數。

📉 2. 方波訊號與物理換算

我們透過計算「每秒產生的脈衝數」來得知物理速度。重點在於邊緣 (Edge) 的精確捕捉。

RPM Formula

\[RPM = \frac{Ticks_{1s} \times 60}{PPR}\]

📏 3. 解析度與物理量映射

理解「一個脈衝代表前進多少公分」是自動導航的基石。這涉及到輪胎直徑與格數的幾何關係。

Kinematic Calibration

  • 解析度 (Ticks)：格數越多，解析度越高，能感知的最小移動量越小。
  • 遲滯 (Hysteresis)：防止在臨界位置時訊號因微小晃動而瘋狂跳動。
  • 映射邏輯：建立從脈衝到 RPM，再從 RPM 到 cm/s 的穩定轉換管線。

GitHub Classroom 詳細操作指南

本單元教學重點：將「光與電」的連續物理現象轉化為「0 與 1」的高精度數位模型。這是自動導航最底層的感知基石。

  1. 重點 1：訊號不是理想的 - 引導學生觀察波形的上升毛刺或抖動。
  2. 重點 2：中斷的原子性 - 強調在獲取 Ticks 數值時，應注意多執行緒/中斷衝突的可能性。
  3. 重點 3：量綱換算 - 確保學員理解從「次數 (Count)」轉換為「頻率 (Hz)」再到「物理量 (m/s)」的嚴謹性。
