進階 13 - 技術敘事

視覺敘事：
如何展示技術專案

技術再強，沒人看懂也是白搭。學習用視覺與故事為代碼注入靈魂，打造令人驚艷的作品集。

📖 1. 英雄之旅：技術敘事的骨架

別只說 "I made a car."，試著講一個關於「解決挑戰」的故事。Reviewer 只有 30 秒能被你吸引，你需要有邏輯的敘事。

Narrative Structure

  • 挑戰 (The Problem)：為什麼要做這專案？解決了什麼現實痛點。
  • 掙扎 (The Struggle)：開發中遇到了什麼技術障礙（延遲、崩潰）？
  • 突破 (The Solution)：你用了什麼黑科技（PID, FSM, Binary Proto）解決它。

🎬 3. 實機演示與電梯簡報

沒有影片就沒真相 (Pics or it didn't happen)。錄製子母畫面 (PiP) 影片同步展示操作與反應，展現系統的毫秒級實力。

Video Production Tips

  • 子母畫面 (PiP)：左側放手機錄影，右側放車子實際動態。
  • 高光特寫：拍攝馬達精確停靠或高速過彎的慢動作鏡頭。
  • 30 秒原則：別錄太長！在最初的 10 秒內就要讓觀眾看到最亮眼的功能。

技術劇場：作品展示與敘事實驗室

任務 1：「一眼就懂」的架構圖設計 (System Architecture Design)

角色扮演：技術文件作家 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 系統拆解：將專案分為「感測層」、「控制層 (ESP32)」、「通訊層 (BLE)」與「應用層 (Dashboard)」。
  3. 圖表繪製：使用 Mermaid (於 README 內) 或 Figma 繪製數據流動圖。標註出關鍵的指令包 (Payload) 是如何流動的。
  4. 文檔提交：將架構圖嵌入 README 並 Push。確保這張圖能讓新手快速理解目前的代碼結構。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 視圖專業度：架構圖是否能準確反映硬體介面 (GPIO/ADC) 與軟體層級的關聯？
  • [ ] 傳達效率：看完架構圖後，是否能在大腦中概括出數據從「感測器」到「馬達」的完整鏈路？

任務 2：動態數據儀表板展示 (Visualizing Results)

角色扮演：數據視覺化工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 採集性能指標：挑選最亮點的數據（如：每秒處理 100 筆更新、誤差收斂曲線）。
  2. 視覺化證據：錄製 Dashboard 上的即時波形。使用 GIF 錄製軟體擷取系統在「過彎」或「加速」時的波形特寫。
  3. 成效對比：在 README 中呈現「優化前 vs 優化後」的延遲數據對比表。
  4. PR 發起：提交包含視覺化圖表與效能說明的 docs 資料夾。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 數據真實性：展示的圖表是否具備標籤、單位與具體的測試場景描述？
  • [ ] 視覺品質：波形截圖或 GIF 是否清晰，能輔助觀看者理解艱澀的控制理論？

任務 3：「技術電梯簡報」錄製 (Technical Pitch & Demo)

角色扮演：產品經理 (PM) | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 劇本構思：採用「問題 -> 掙扎 -> 突破」框架編寫個人自白。準備一段 30 秒的技術亮點介紹。
  2. PiP 演示錄製：同時錄製車子在地板跑與手機上的控制 UI 面板。同步展示「當我按下煞車，車子即刻停下」的低延遲實力。
  3. 成品發布：將影片連結放入 README 最上方。確保這是讀者點開 Repo 後第一眼看到的亮點。
  4. Push Final：推送所有作品集素材，完成本單元任務。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 敘事連貫性：口述內容是否緊扣技術重點？有無展現出解決問題的工程師思維？
  • [ ] 示範完整性：影片中是否涵蓋了功能正常運行與異常處理 (Fail-safe) 的雙重展示？

GitHub Classroom 詳細操作指南

本單元教學重點：培養「溝通」能力。如何將硬核技術轉化為非技術主管或贊助者也能懂的視覺報告。

  1. 重點 1：架構圖即語言 - 引導學員理解高質量架構圖的價值：它能讓團隊快速同步技術藍圖，減少溝通成本。
  2. 重點 2：數據的視覺說服力 - 教導學員不要只寫「很快」，要展示「縮短了 200ms」並輔以波形圖證據。
  3. 重點 3：作品集意識 (Portfolio Mindset) - 提醒學員這不僅是作業，這是未來職涯或升學的重要技術背書。
