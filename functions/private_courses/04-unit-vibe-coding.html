為什麼 Agent 無法部署？

部署涉及基礎設施權限與敏感密鑰 (API Keys)。為確保安全，Agent 運行在隔離環境，無權直接存取你的正式伺服器。

最佳實踐：CI/CD 部署

讓 Agent 撰寫 GitHub Actions YAML 腳本。你審核並合併 PR 後，由自動化流水線完成部署，這才是最專業的流程。

Vibe Coding 實戰課

Vibe Coding 實戰 - AI 代理人專題

任務 1：審美天花板 - UI/UX 視覺實戰

角色扮演：資深前端工程師 | 難度：⭐️⭐️ | 估計時長：30 min

🛠️ 執行技術流程 (Git Workflow)

  1. 選擇工具：進入 v0.dev 或 Lovable.dev 平台。
  2. 定義 Vibe：輸入一段「Cyberpunk 風格、含有毛玻璃質感與霓虹燈效」的控制面板描述。
  3. 迭代優化：針對產出的 UI 進行至少 3 次的細節微調 (Prompt Tuning)。
  4. 紀錄與繳交：將成果截圖與最終 Prompt 分享至你的 Repository。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 視覺達成度：是否能讓 AI 精確呈現指定的視覺風格？
  • [ ] 對話深度：是否展現出透過迭代優化 UI 細節的能力？

任務 2：10 分鐘全棧挑戰 (Replit Agent)

角色扮演：全端開發者 | 難度：⭐️⭐️⭐️ | 估計時長：20 min

🛠️ 執行技術流程 (Git Workflow)

  1. 開啟 Agent：在 Replit 中啟動 Agent 模式。
  2. 下達指令：要求 Agent 在 10 分鐘內建立「含資料庫、後端 API 與前端頁面」的待辦清單 (Todo app)。
  3. 即時部署：點擊 Deploy 並獲得一個可公開訪問的連結。
  4. 驗證邏輯：測試資料新增、刪除是否在重新整理後依然存在。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 功能完整性：資料庫是否正確連接？CRUD 功能是否齊全？
  • [ ] 速度與效率：是否能有效管理 Agent 的規劃與執行過程？

任務 3：CLI 掃描與安全修復 (Claude Code)

角色扮演：安全工程師 (SecOps) | 難度：⭐️⭐️⭐️ | 估計時長：40 min

🛠️ 執行技術流程 (Git Workflow)

  1. 終端連動：在你的本機專案目錄中執行 claude。
  2. 下達任務：要求 Agent：「掃描此專案中的潛在安全性風險，並自主進行修復」。
  3. 審核變動：逐一核准 Agent 提出的檔案修改計畫。
  4. 同步提交：執行 git push 同步 Agent 產出的安全性補丁。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 風險辨識：是否能引導 Agent 找出常見的 Vulnerability？
  • [ ] 自主性：是否學會如何對 CLI Agent 下達高級指令進行批次處理？

Vibe Coding 實戰教學指南

本單元是「工具合奏」的巔峰。重點不在於語法，而在於學生是否能從中體會到「工程效率」的維度跳躍。導師應多觀察學生對不同工具特長的理解程度（例如：Lovable 強於審美，Cursor 強於架構）。
