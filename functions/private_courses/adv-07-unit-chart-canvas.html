🖌️ 1. 渲染原理：畫布 vs. 物件 (25 min)

Canvas 是一塊即時畫布，你必須手動下指令「抹除再重繪」。而 Chart.js 則是基於 Canvas 的高度封裝，讓你專注於數據本身而非座標計算。

Comparison Architecture

  • 低階 Canvas：極致性能，適合處理數萬個頻繁跳動的像素點。
  • 高階 Chart.js：高度語意化，自動處理座標軸、縮放與適應性佈局。

⚡ 3. 更新頻率與視覺感知

人眼難以感知超過 60Hz 的數值跳變。學習如何使用 requestAnimationFrame 配合螢幕刷新率，並關閉不必要的圖形動畫以減少 CPU 負載。

Efficiency Checklist

  • 停用動畫 (Animation: false)：在即時圖表中，過度動畫會造成渲染卡頓與數據延遲。
  • 分組更新 (Batching)：如果數據爆量，則每 5-10 筆數據才刷新一次 UI。
  • 滾動軸優化：固定 X 軸的顯示跨度，維持視覺上的連續性。

單元課程 進階 7 - 數據視覺化

遙測實驗室：即時資料結構與高頻渲染

任務 1：滑動視窗 (Sliding Window) 實作

角色扮演：資料緩衝工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 隊列 (Queue) 邏輯實作：在 DataStore.js 中定義一個 Buffer 陣列。
  3. 原子性操作：確保每筆新數據進入時，同步執行 push() 與 shift()，並嚴格檢查 buffer.length 是否恆定。
  4. 渲染更新：調用 chart.update('none') 並關閉動畫選項，在 README 中貼上平滑滾動的截圖。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 內存管理：資料陣列是否在長時間運行後依然保持在 50 個點內？
  • [ ] 視覺即時性：圖表移動是否具備「零延遲」的感覺？有無殘影或過渡動畫干擾？

任務 2：多軸同步監控儀 (Multi-axis PID Monitor)

角色扮演：控制系統分析師 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 多曲線配置：在 Chart.js 配置中新增兩個 DataSets，分別標註顏色與 Label。
  2. 座標對齊：設定 Y 軸範圍固定（如 0-255），確保兩組數據在同一比例尺下具備可比性。
  3. 聯通測試：手動擾動馬達，錄製影片上傳至 PR，展示實際速度如何緩步「追回」目標速度的過程。
  4. Push Final：提交具備專業調教介面的監控面板。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 視覺辨識度：兩條曲線是否具備清晰的顏色對比？圖例 (Legend) 是否清楚？
  • [ ] 物理意義正確性：當馬達目標值改變時，圖表是否能即時反映出 Error 的收斂？

任務 3：原生 Canvas 高頻示波器 (Direct Canvas Rendering)

角色扮演：高性能渲染專家 | 難度：⭐️⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 畫布重置邏輯：在每個框架開始時，正確執行 clearRect() 清空畫布。
  2. 路徑運算實作：利用網格映射公式：$y_{pixel} = height - (value / 255 * height)$，實作點對點的連線。
  3. 壓力測試紀錄：測試 100Hz 刷新頻率下瀏覽器的 CPU 佔用率，並記錄在 README 的 Performance Section。
  4. Push Final：提交具備極致效能的裸機 (Bare-metal) 繪圖引擎。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 座標映射精確度：輸入 0 時，波形是否正確位於底部？輸入 255 時，是否對齊頂端？
  • [ ] 穩定性評估：高頻更新時，網頁是否會發生卡頓、閃爍或無響應狀況？

GitHub Classroom 詳細操作指南

本單元教學重點：讓學生掌握「即時系統」的數據流設計。這不只是繪圖，更是內存與效能管理的修煉。

  1. 重點 1：隊列思維 - 學生常犯的錯誤是無限追加入數據導致崩潰。必須強制引導他們建立 Sliding Window。
  2. 重點 2：犧牲華麗換取真實 - 在儀表板開發中，1ms 的動畫過渡可能導致你漏掉 10 筆關鍵數據。學會關閉動畫。
  3. 重點 3：底層渲染機制 - 透過原生 Canvas 任務，讓他們理解圖表庫背後的像素座標變換原理。
