🔍 1. 偵測效能黑洞 (Bottlenecks)

控制系統的速度取決於「最慢」的那一環。即使馬達再快，如果感測器讀取或數據傳輸卡住，控制表現就會像「Lag」一樣遲鈍。

The Three Efficiency Layers

  • 通訊瓶頸 (Latency)：藍牙頻寬不足或封包內容過於肥大導致延遲。
  • 運算瓶頸 (CPU)：主迴圈內含有過多阻塞式 I/O 或複雜數學運算。
  • 渲染瓶頸 (Frontend)：瀏覽器過度重繪與 DOM 更新導致指令響應緩慢。

⚡ 3. 前端極速渲染：告別控制卡頓

瀏覽器在處理海量即時遙測數據時，容易因 Canvas 頻繁刷新而阻塞。透過渲染緩衝 (Buffering) 與 Frame Rate 管理，確保操控手感如絲般順滑。

Advanced Rendering Strategy

  • 離屏渲染 (Offscreen Draw)：將靜態圖層預先繪製，減少重複渲染運算負荷。
  • 請求幀節流 (rAF)：同步螢幕刷新率，防止過度計算造成的 CPU 資源浪費。
  • Buffer 管理：預建循環隊列 Buffer，優化記憶體分配並減少 GC 引起的抖動。

單元課程 進階 13 - 效能優化

效能實驗室：全鏈路通訊與渲染優化

任務 1：系統耗時診斷 (System Profiling Lab)

角色扮演：系統性能架構師 | 難度：⭐️⭐️ | 估計時長：50 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 埋點量測：在程式中利用 micros() 捕捉關鍵演算法（如 PID、Sensor Fussion）的純執行時間。
  3. 負載分析：記錄並在 README 撰寫各模組的時間佔比圖，識別「顯性」與「隱性」延遲來源。
  4. Push Final：提交具備「性能透視能力」的診斷分析報告。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 診斷深度：README 是否能區分「通訊耗時」與「物理計算耗時」，並給出具體毫秒值？
  • [ ] 優化建議力：能否根據數據報告，提出一個有效的「低回報代價」優化方案？

任務 2：通訊協議瘦身 (Protocol Compression Lab)

角色扮演：物聯網協議專家 | 難度：⭐️⭐️⭐️ | 估計時長：70 min

🛠️ 執行技術流程 (Git Workflow)

  1. 二進位映射：實作自定義緊湊協議，將數值打包為 bit-level 或短字串。
  2. 解析器重構：同步更新前端 Javascript，確保能正確解碼瘦身後的傳輸協議。
  3. 頻率增益驗證：在同等 BLE 環境下，對比優化前後數據「實際刷新率」的數倍提升。
  4. Push Final：提交具備「極致傳輸效率」的高性能通訊模組。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 壓縮比達成率：封包體積是否顯著縮小 50% 以上，且不損失原本數據精度？
  • [ ] 協議強韌度：對於截斷或損毀的壓縮封包，前端是否存在對應的校驗與容錯機制？

任務 3：Canvas 優化與渲染加速 (Rendering Optimization Lab)

角色扮演：資深前端工程師 | 難度：⭐️⭐️⭐️ | 估計時長：40 min

🛠️ 執行技術流程 (Git Workflow)

  1. 性能錄製分析：使用 Chrome DevTools 指出導致 Canvas 卡頓的關鍵代碼行。
  2. 非同步繪製部署：實作 requestAnimationFrame 確保渲染不阻塞主控制邏輯的指令發送。
  3. 離屏緩格實作：將靜態軸線與背景異步化，達成核心波形渲染的資源最小化。
  4. Push Final：提交具備「高幀率、低負載」特性的最終前端 DashBoard。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 幀率表現：在每秒 100 筆數據流衝擊下，UI 是否依然能維持穩定、流暢的渲染而不掉幀？
  • [ ] 內存管理：長時間運行後，系統內存（Heap）是否維持穩定，無洩漏現象？

GitHub Classroom 詳細操作指南

本單元教學重點：在 IoT 與控制系統中，如何建立「全鏈路優化」的思維。從嵌入式 CPU 到瀏覽器 GPU 的每一環節壓榨效能。

  1. 重點 1：數據驅動優化 - 強調「先測量、後動刀」。透過 micros() 埋點帶領學生發現程式中的真實效能黑洞。
  2. 重點 2：從文字到位元 (Bits) - 引導學員理解協議設計的藝術：流暢的控制來自於極致的數據精簡。
  3. 重點 3：感知性能 (Perception) - 指導學員理解前端優化的隱性價值：穩定的 UI 就是操控者的信心來源。
