基礎 7 - 網路通訊架構

Wi-Fi 雙模式解析：
AP (自立門戶) vs. STA (加入家族)

掌握 ESP32 的通訊雙重身分，打造能在任何環境下連線的產品級智慧控制系統。

🗼 1. AP (Access Point) 模式：自立門戶

核心概念：ESP32 自己當「基地台」。就像車子背著一台迷你路由器，手機可以直接連到車子發出的 Wi-Fi。

💪 優點

不依賴外部網路。去到哪裡都能連線（適合戶外賽道），建立私有控制網段。

⚠️ 缺點

手機連上後通常無法上網。連線距離受限於發射功率，且連線人數有記憶體上限。

ESP32 Soft-AP Config

WiFi.softAP("VibeCar_01", "12345678");
IPAddress IP = WiFi.softAPIP();

預設 Gateway IP 通常為 192.168.4.1

🏠 2. STA (Station) 模式：加入家族

核心概念：ESP32 像手機或電腦一樣，「加入」現有的路由器環境，成為區域網路的一份子。

The Advantage

  • 長距控制：只要在同個 WiFi 下，距離不再是問題。
  • 同步連網：車子可以同時與雲端伺服器交換數據。
  • 多機協作：多台設備可透過路由器進行數據導向。
EVENT LISTENER 
// 專業的非阻塞連線監測：
WiFi.onEvent([](WiFiEvent_t event, WiFiEventInfo_t info){
  if (event == SYSTEM_EVENT_STA_GOT_IP) {
    Serial.println("已加入大家門！");
  }
});

⚡ 3. 混合備援：產品級配網邏輯

你家裡的智慧燈泡如何得知 WiFi 密碼？這就是 AP + STA 雙模式的經典應用：配網流程 (Provisioning)。

Smart Provisioning Loop

  1. 啟動 AP：設備發出 SSID 讓手機連入進行設定。
  2. 傳輸密條：手機將外部 Router 的密碼 POST 給設備。
  3. 身份切換：設備收件後關閉 AP，啟動 STA 加入指定網路。
  4. Fallback：若連線失敗，自動退回 AP 模式尋求協助。

Self-Healing System

"一個好的產品永遠不應該陷入『斷線後必須拔插頭重開』的困境。"

Auto-Recovery Active

從建立獨立領地到實作具備自我修復能力的產品級通訊骨幹。

【作業 7-1】

「獨立基地台連線基礎」

實作自定義 SSID 與加密。設定子網段網關 (192.168.4.1) 並驗證手機連線後的 DHCP 配發與 Web 首頁連通性。

【作業 7-2】

「穩定性斷線自動修補」

實作 WiFi 事件監聽器。模擬路由器斷電重啟，撰寫背景重連邏輯，並在 Serial 記錄恢復通訊所需的平均秒數。

🛡️ 60 min

【作業 7-3】

「自我修復視覺狀態機」

實作逾時自動切換 AP 的降級邏輯。配合 LED 以不同閃爍頻率標示當前模式，確保設備在任何環境下皆不失聯。

任務 2：「智慧尋路」：STA 模式與斷線重連 (STA: Smart Reconnect)

角色扮演：穩定性測試工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 環境適配：撰寫 WiFi.begin() 令設備加入指定 WiFi，並輸出連線後的分配 IP。
  2. 事件監聽實作：導入 WiFi.onEvent() 處理機制，攔截 WIFI_EVENT_STA_DISCONNECTED 訊號。
  3. 背景修復邏輯：實作非阻塞式的「指數退避」重連邏輯，確保設備在 Router 回歸後自動恢復通訊。
  4. 日誌分析報告：在 README 中截圖 Serial 輸出，記錄設備在多種斷訊情境下的「平均自動修復耗時」。

任務 3：「終極備援」：AP + STA 自動切換 (Provisioning Fallback)

角色扮演：強韌系統架構師 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 優先權鏈結實作：設定開機邏輯：先嘗試 STA (10s 逾時限制)。
  2. 降級機制部署：當 STA 逾時失敗，自動開啟 AP Mode 供使用者連入修改 WiFi 設定。
  3. 狀態視覺機：實行 LED 指示定義：STA=恆亮, AP=慢閃(2Hz), 嘗試中=快閃(5Hz)。
  4. 正式提交：將具備自我修復能力的完整 main.cpp 推送到 GitHub，標註此設計如何避免產品變磚（Bricked）。

GitHub Classroom 詳細操作指南

本單元教學重點：掌握 ESP32 作為通訊端節點的身分切換與強韌設計。

  1. 重點 1：身份的主從關係 - 理解 AP 是「我是主機」，STA 是「我是客戶端」。
