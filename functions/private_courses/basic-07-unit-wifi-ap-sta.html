🗼 1. AP 模式：自立門戶

ESP32 自己當「基地台」。像迷你路由器，手機直接連到車子。

💪 優點

去哪都能連。建立私有控制網段。

⚠️ 缺點

通常無法上網。連線人數有上限。

ESP32 Soft-AP Config

WiFi.softAP("VibeCar_01", "12345678");
IPAddress IP = WiFi.softAPIP();

預設 Gateway IP：192.168.4.1

🏠 2. STA 模式：加入家族

ESP32 「加入」現有路由環境。成為區域網路一份子。

The Advantage

  • 長距控制：同 WiFi 下距離不再是問題。
  • 同步聯網：可同時與雲端交換數據。
  • 多機協作：可透過路由導向。
EVENT LISTENER 
// 非阻塞連線監測：
WiFi.onEvent([](WiFiEvent_t event, WiFiEventInfo_t info){
  if (event == SYSTEM_EVENT_STA_GOT_IP) {
    Serial.println("已連入大家門！");
  }
});

⚡ 3. 混合備援：產品級配網

AP + STA 雙模式經典應用。配網流程 (Provisioning)。

Smart Provisioning Loop

  1. 啟動 AP：發出 SSID 讓手機連入。
  2. 傳輸密條：手機 Post 路由密碼給設備。
  3. 身分切換：關閉 AP，以 STA 加入網路。
  4. Fallback：若失敗，退回 AP 模式。

ESP32 Wi-Fi 雙模式 (AP vs. STA) - 學生作業指南

課程目標

掌握 ESP32 的雙重通訊身分，學習如何處理不穩定的網路環境，並利用 事件驅動 (Event-Driven) 模式實作具備「自我修復能力 (Self-Healing)」與「配網備援 (Provisioning Fallback)」的強韌韌體架構。

作業任務

任務 1：「我的領地」AP 模式與 SSID 配置 (Access Point Lab)

  • 目標：建立私有網路控制段，理解 DHCP IP 分配機制，確保在無外部路由環境下依然具備控制權。
  • 實作要求：
    • 在 src/main.cpp 中實作 WiFi.softAP()，設定個性化的 SSID（如 MyCar_ID）與加密密碼。
    • 驗證：手機能連入並獲取 IP (通常為 192.168.4.x)，且能成功開啟 ESP32 託管的簡單網頁。

任務 2：「智慧尋路」STA 模式與斷線重連 (Smart Reconnect Lab)

  • 目標：掌握非同步事件監讀，學習如何在不干擾機器人主線程運行的情況下，自動修復網路連線。
  • 實作要求：
    • 實作 WiFi.onEvent() 攔截 ARDUINO_EVENT_WIFI_STA_DISCONNECTED。
    • 實作自動重連邏輯（如：每 5 秒嘗試一次）。
    • 關鍵限制：禁止 在 loop() 函數中使用 delay() 等待連線。
    • 數據提交：在 docs/Recovery_Log.md 中記錄關掉路由器後，設備恢復連線所需的平均時間。

任務 3：「終極備援」AP + STA 自動切換 (Provisioning Fallback Lab)

  • 目標：打造「永不失聯」的產品級韌體架構。當 STA 無法加入網路時，自動降級為 AP 模式供使用者重新設定，避免設備「變磚」。
  • 實作要求：
    • 開機先嘗試 STA 連線。若失敗，自動啟動 AP 模式。
    • UI 指示設計：使用 LED 閃爍頻率標示模式（慢閃 = STA 連線中，快閃 = 已降級為 AP 模式）。
    • 恢復機制：在 AP 模式下，若偵測到環境中預設路由出現，應能自動切回 STA 模式。
    • 驗證要點：模擬路由器關機情境，確認設備是否能在 30 秒內自動開啟 AP 訊號供緊急存取。
