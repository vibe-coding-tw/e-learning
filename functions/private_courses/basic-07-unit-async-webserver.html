基礎 7 - 網路通訊架構

ESP32 Web Server：
從「阻塞」到「非同步」的進化

擺脫 loop() 的束縛，實作基於事件驅動的高性能網頁控制伺服器。

🛡️ 1. 認識非同步：網頁守門員

在控制高速移動的無人車時，通訊的「即時性」至關重要。傳統同步方式會與主程式互相搶奪 CPU，而非同步則能各司其職。

🐌 同步 (Sync)

必須在 loop() 呼叫 handleClient()。若 loop 忙碌（如算力不足），網頁通訊即會斷供或延遲。

⚡ 非同步 (Async)

基於事件驅動，背後自動處理網頁請求。不佔用 loop() 行程，最適合賽車即時採樣控制。

Async Logic Analogy

如同百貨公司美食街，點完餐領取號碼牌 (Callback)，不需在櫃檯死等。

"這讓 ESP32 能夠一邊以 50Hz 頻率控制馬達，一邊同時響應前端畫面的資料請求。"

🗺️ 2. 路由 (Routing) 與 Lambda 魔法

利用 C++11 的 Lambda (匿名函式) 語法，我們可以在宣告路徑的當下直接定義處理邏輯，讓代碼結構大幅簡化。

Request Handling

  • 路徑對應：將 /api/control 對準特定的功能區。
  • 非阻塞回應：使用 request->send() 立即釋放連線。
  • 參數捕獲：從網址 query 提取即時控制數值。
LAMBDA SYNTAX 
server.on("/on", HTTP_GET, [](AsyncWebServerRequest *req){
  digitalWrite(LED, HIGH);
  req->send(200, "text/plain", "LED is ON");
});

🎨 3. 範本引擎：動態靜態資源分離

將整個 HTML 寫在 C++ 字串裡會導致維護災難。利用 Processor 佔位符機制，我們可以實現內容與數據的優雅分離。

Server-side Rendering

  • R"rawliteral"：使用原始字串規則，在 C++ 中保留 HTML 縮排與雙引號，不再需要轉義符。
  • %PLACEHOLDER%：於 HTML 埋設標點。ESP32 會在發送前將其替換為即時動態數據。
  • Processor 回呼：集中管理數據替換邏輯，使程式碼結構一目瞭然。

Template Power
