🌓 1. 影像二值化 (Thresholding) 的原理 (25 min)

將 256 階的彩色世界簡化為只有 0 (黑) 與 1 (白) 的數位矩陣。這是機器「看見」具體特徵的基礎。

Mathematical Rule

IF Gray < Threshold THEN Pixel=BLACK (0)
ELSE Pixel = WHITE (1)
  • 雜訊抑制：消除微弱的光影干擾。
  • 特徵提取：鎖定如「賽道線」或「色塊」等關鍵特徵。

🔬 3. 侵蝕與擴張：形態學優化 (10 min)

二值化後的影像不一定完美。利用形態學運算加強特徵：侵蝕去除細碎雜點，擴張填補斷裂線條。

Processing Pipeline

  • 侵蝕 (Erosion)：縮減邊緣，移除孤立的干擾噪聲像素。
  • 擴張 (Dilation)：延伸邊緣，連接斷開的賽道線。
  • 特徵提取：完成「原圖 -> 濾波 -> 二值化 -> 形態學」的標準流程。

針對 adv-06-unit-threshold-filter (二值化與濾波) 單元，這是數位影像處理 (DIP) 的基礎，也是機器視覺的門檻。它要求學生理解如何將雜亂的色彩訊號「降維」成純粹的特徵數據。

以下是在 GitHub Classroom 部署其作業 (Assignments) 的具體建議：

1. 範本倉庫 (Template Repo) 配置建議

二值化與濾波是純粹的「矩陣運算」與「訊號處理」，建議範本包含：

  • 📂 js/image_engine.js：核心影像引擎，提供提取 Canvas 像素 Buffer 的框架，留下灰階轉換與二值化公式的 TODO。
  • 📂 js/convolution_kernels.js：卷積過濾模組。預置 $3 \times 3$ 的滑動視窗迴圈邏輯，讓學員實作均值與中值的算法核心。
  • 📂 assets/image_samples/：包含一組標準影像與一組「刻意加入噪點」的影像，用於檢驗濾波器的實際去噪效能。
  • 📂 tests/filter_accuracy.test.js：自動化評分脚本。餵入一組 1D/2D 像素陣列，檢核學員的二值化邏輯（如：輸入 150 > 門檻 100 是否輸出 255）與濾波數值計算是否正確。

2. 作業任務部署細節

任務 1：互動式二值化工具 (Interactive Threshold Tool)

  • 目標：掌握亮度門檻對特徵提取的決定性影響。
  • Classroom 部署建議：
    • 核心邏輯檢核： 
      // 學生需實作：灰階門檻判定
let gray = (r * 0.299 + g * 0.587 + b * 0.114);
pixel = (gray > threshold) ? 255 : 0;
    • Autograding：給定一個閾值與多個灰階像素值，檢核輸出的二元矩陣是否符合邏輯。
    • 數據提交：要求學員在不同光照下尋找「最優閾值」，並在 README 中記錄為何環境變暗時需要調低門檻量。

任務 2：「噪點消滅者」濾波器實驗 (Noise Killer Filter Lab)

  • 目標：理解卷積運算 (Convolution) 如何平滑訊號，並對比不同濾波器的優缺點。
  • Classroom 部署建議：
    • 中值濾波實作：這是進階考點。學員需在 $3 \times 3$ 視窗中取出 9 個值並進行排序 (Sort)，取中間值。
    • 去噪效果對比：學員需在 README 中對比「均值（模糊）」與「中值（保留邊緣）」對胡椒鹽雜訊處理後的細節差異。
    • Autograding：餵入一個帶有一個極大雜訊點的 9 宮格陣列，檢核學員的中值濾波函式是否能完美「剔除」該點。

任務 3：特徵中心點計算 (Feature Centroid Detection)

  • 目標：將二值化結果轉換為具備導航意義的座標點。
  • Classroom 部署建議：
    • 重心公式檢核： 
      // 學生需實作：座標降維累加
avgX = sumX / totalPixels;
avgY = sumY / totalPixels;
    • 邊界防護測試：餵入一個「全白（無目標）」的影像，檢核學員的代碼是否具備 if (count > 0) 的除以零檢查。
    • 視覺化驗證：在 Canvas 上用 context.fillRect 繪製出計算出的重心中心點，並截圖提交。

3. DIP 工程導師點評標準 (DIP Engineering Benchmarks)

此單元的價值在於 「數據降維的準確性與程式效能」：

  • [ ] 特徵提取純淨度：二值化邊緣是否清晰？有沒有大量的散碎雜點？
  • [ ] 去噪與銳利的權衡：濾波器是否在去除雜訊的同時，過度模糊了賽道線的邊緣導致中心點偏移？
  • [ ] 運算效率意識：在 $3 \times 3$ 像素遍歷中，是否有加入邊界檢查防止 Array Index Out of Bounds 的崩潰？

GitHub Classroom 詳細操作指南

本單元教學重點：引導學生理解「機器視覺」不是在看照片，而是在進行「特徵提取 (Feature Extraction)」。

  1. 重點 1：降維的藝術 - 解釋為什麼要丟棄色彩訊息。將 $320 \times 240 \times 3$ 的複雜數據簡化為一組座標，是提升自駕系統響應速度的關鍵。
  2. 重點 2：噪聲是數學問題 - 讓學生體悟到「中值濾波」如何利用統計學（中位數）來對抗不可避免的物理電噪。
  3. 重點 3：門檻的相對性 - 強調沒有「絕對正確」的 Threshold。這為後續單元引進「自適應門檻」埋下伏筆。
