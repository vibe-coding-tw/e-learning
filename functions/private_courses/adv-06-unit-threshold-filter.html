數位影像特徵提取：
二值化與濾波

影像處理的第一步。學習如何透過數學手段，將複雜的現實世界轉化為機器易於理解的數位矩陣。

🌓 1. 影像二值化 (Thresholding) 的原理 (25 min)

將 256 階的彩色世界簡化為只有 0 (黑) 與 1 (白) 的數位矩陣。這是機器「看見」具體特徵的基礎。

Mathematical Rule

IF Gray < Threshold THEN Pixel = BLACK (0)
ELSE Pixel = WHITE (1)
  • 雜訊抑制：消除微弱的光影干擾。
  • 特徵提取：鎖定如「賽道線」或「色塊」等關鍵特徵。

🔬 3. 侵蝕與擴張：形態學優化 (10 min)

二值化後的影像不一定完美。利用形態學運算加強特徵：侵蝕去除細碎雜點，擴張填補斷裂線條。

Processing Pipeline

  • 侵蝕 (Erosion)：縮減邊緣，移除孤立的干擾噪聲像素。
  • 擴張 (Dilation)：延伸邊緣，連接斷開的賽道線。
  • 特徵提取：完成「原圖 -> 濾波 -> 二值化 -> 形態學」的標準流程。

單元課程 進階 6 - 二值化與濾波

視覺實驗室：二值化門檻調校與數位濾波

任務 1：互動式二值化工具 (Interactive Threshold Tool)

角色扮演：影像處理工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 灰階轉換實作：在 App.js 中撰寫迴圈，利用加權公式將彩圖轉為灰階。
  3. 門檻邏輯串接：監聽 UI 滑桿事件，將灰階值與門檻比對，產出純黑白 (0/255) 的二值化結果。
  4. 截圖提交：在 README 中分別記錄「強光」與「暗處」環境下，取得最純淨黑線的最佳 Threshold 數值。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 轉換正確性：輸出的 Canvas 影像是否確實只有 0 與 255 兩種數值？
  • [ ] UI 反饋一致性：滑桿拖動時，影像是否具備足夠流暢的即時刷新感？

任務 2：「噪點消滅者」濾波器實驗 (Noise Killer Filter Lab)

角色扮演：訊號處理專家 | 難度：⭐️⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 雜訊產生器：實作一個 Function，在影像中隨機撒入 2% 的白點雜訊 (Salt noise)。
  2. 卷積算法開發：撰寫 $3 \times 3$ 的滑動視窗逻辑。
  3. 比較分析：在同一雜訊來源下，對比「均值濾波」與「中值濾波」對黑白雜訊的抹除能力。
  4. 代碼提交：Push 效能最佳化的濾波函式，並附上處理前後的對比截圖。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 去噪質量評估：中值濾波是否能完美拔除「孤立噪點」而不同時模糊特徵邊緣？
  • [ ] 算法效率：在處理 $320 \times 240$ 影像時，是否能維持在 30 幀以上？

任務 3：特徵中心提取 (Feature Centroid Detection)

角色扮演：視覺演算法開發者 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 重心公式應用：統計所有二值化後為「黑」點的 X, Y 總和除以總點數。
  2. 座標標註：在處理後的 Canvas 上，利用 context.strokeRect 標記出計算出的導航重心。
  3. Push Final：提交具備「從雜亂背景中穩定定位黑線中心」能力的最終專案。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 中心點穩定度：在影像光暗微調時，計算出的中心座標是否發生劇烈跳變？
  • [ ] 邏輯嚴謹性：代碼中是否正確處理了「完全沒有目標物 (n=0)」時的除以零邊界情況？

GitHub Classroom 詳細操作指南

本單元教學重點：引導學生理解「機器視覺」不是在看照片，而是在進行「特徵提取 (Feature Extraction)」。

  1. 重點 1：降維的藝術 - 解釋為什麼要丟棄色彩訊息。將 $320 \times 240 \times 3$ 的複雜數據簡化為一組座標，是提升自駕系統響應速度的關鍵。
  2. 重點 2：噪聲是數學問題 - 讓學生體悟到「中值濾波」如何利用統計學（中位數）來對抗不可避免的物理電噪。
  3. 重點 3：門檻的相對性 - 強調沒有「絕對正確」的 Threshold。這為後續單元引進「自適應門檻」埋下伏筆。
