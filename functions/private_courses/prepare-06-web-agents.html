Copilot Workspace：From Issue to PR

Copilot Workspace 是目前自動化程度最高的 Web 編程環境。 它以「任務」為中心，代理人會讀取整個程式庫，針對你的需求自動完成規劃、實作與驗證。

核心工作流

1

規劃 (Plan)

代理人讀取整個程式庫，針對你的 Issue 需求寫出一份實施計畫，列出要修改的檔案與步驟。

2

執行 (Implement)

確認計畫後，Agent 會自動在雲端環境中修改多個檔案，產生完整的程式碼變更。

3

驗證 (Verify)

自動執行測試。如果測試失敗，代理人會自行修復錯誤直到通過。

4

開啟 Pull Request

一旦任務完成並通過驗證，Agent 會自動開啟 PR，附上完整的變更說明與計畫紀錄。

關鍵技巧 在步驟 1 中，你可以手動微調 Agent 的計畫再讓它執行。 這個「Plan Tuning」能力體現了你作為「指揮官」的決策價值——這是 AI 無法替代的。

Web Chat、Codespaces 與 Agentic Workflows

GitHub.com Copilot Chat

直接在程式庫網頁使用 AI 對話。最新功能支援 Tool Calls， 這意味著 Agent 可以調用工具來搜尋程式碼、分析檔案結構，無需離開瀏覽器。

啟動方式 在任何 GitHub 程式庫頁面按下 Ctrl+B（Mac 為 Cmd+B）即可開啟 Copilot Chat 面板。

github.dev / Codespaces

這是運行在雲端的「完整開發環境」。你可以在 Terminal 中安裝套件、執行程式碼， 並使用完整的 Agent Mode。最適合需要編譯、測試的進階開發場景。

快速進入 在程式庫首頁按下鍵盤上的 . (句點) 鍵，即可秒速進入 github.dev 網頁版 VS Code。

GitHub Agentic Workflows (自動化)

讓 AI Agent 變成一個自動運行的後台機器人。例如：直接指派 Issue 給 @copilot， 它會自動開分支、寫程式碼並提交 PR。

# 範例：當 Issue 標記為 'ai-fix' 時啟動 Agent on: issues: types: [labeled] jobs: agentic-fix: if: github.event.label.name == 'ai-fix' runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Run Copilot Agent run: copilot-agent --issue ${{ github.event.issue.number }}

02-Unit-Web-Agents: 原生 Web 代理環境實戰

歡迎來到第二單元！在這個單元中，我們將離開本地開發環境，將開發戰場移向雲端，並實踐 「以任務為中心 (Task-Centric)」 的現代開發流程。

🎯 學習目標

  • 掌握 Copilot Workspace 的完整開發閉環。
  • 學會配置並觸發 GitHub Actions 中的 AI 自動化工作流。
  • 實踐 Zero-Install 開發模式，利用瀏覽器即時修復與重構代碼。

🚀 作業任務

任務 1：從需求到 PR - Copilot Workspace 實戰

  1. 在程式庫中開啟一個新的 Issue。
  2. 點擊 "Open in Copilot Workspace"。
  3. 審查並微調 AI 產生的計畫。
  4. 執行計畫並開啟 Pull Request。
  5. 將 PR 連結填寫至 PR_LINK.md。

任務 2：幽靈編程師 - GitHub Actions 自動化

  1. 查看 .github/workflows/ai-agent.yml。
  2. 建立 Issue 並貼上 ai-fix 標籤。
  3. 確認 Actions 工作流已成功啟動。

任務 3：雲端極簡主義 - Zero-Install 開發

  1. 按下 . 進入 github.dev。
  2. 對 tasks/style-refactor.css 進行重構。
  3. Commit 並註明 Refactored via github.dev Web Agent。

針對 02-unit-web-agents 單元，強調「以任務為中心」的開發流程。

1. 範本程式庫配置

  • .github/workflows/ai-agent.yml：預置 Actions 結構。
  • tasks/：待重構的練習檔案。
  • .github/ISSUE_TEMPLATE/：引導學員撰寫高品質需求。
