🏁 1. 競爭條件 (Race Condition)

無線環境下，指令發送順序不等於抵達順序。當多個操作同時發生，最終結果取決於誰「跑得快」，這會導致嚴重的邏輯錯誤。

Failure Scenario

  1. 1. 網頁發送「進入自動模式」
  2. 2. 車子因電力不足自動退回「待機模式」
  3. 3. 「電力不足」的回報被延遲。網頁先收到「模式確認」，導致顯示錯誤狀態。

🔄 3. 應答機制 (ACK) 與定時輪詢

如何確認指令抵達？「輪詢」主動詢問，「應答 (ACK)」則由硬體執行完畢後主動通知，是最高效的一致性方案。

Protocol Strategy

  • 版本序號 (Sequence)：為封包加上遞增編號，過期序號一律捨棄。
  • 雙向握手 (ACK)：包含指令 ID，確定特定的操作在遠端確實落地。
  • 指數退避 (Backoff)：通訊失敗時自動重試，但間隔逐漸拉長，避免癱瘓網路。
// App 发送具備 ID 的指令
sendCmd({ id: 101, action: "START" });

// 硬體執行後 ACK 回传
{ "ack": true, "cmdId": 101 }

分散式一致性實驗室

任務 1：版本序號同步法 (Sequence Number Logic)

角色扮演：分散式系統工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 序號注入封裝：修改 ESP32 的 JSON 發送邏輯，為每個包含感知狀態或模式的封包新增遞增 seq 序號字段。
  3. 過濾機制作戰：在網頁端接收層實作邏輯：僅接受 seq 大於當前 local 最大儲存值的數據，高效忽略網路亂序包。
  4. 亂序穩定性驗證：手動擾亂數據發送順序（模擬高延遲波動），觀察網頁端 UI 顯示是否能維持穩定，不發生「新舊數據交錯跳動」。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 序號過濾嚴密性：網頁端是否能 100% 擋下因延遲抵達的舊序號數據，保持顯示狀態的「線性前進」？
  • [ ] 極限溢位備案：README 中是否分析並提出針對序號遞增至極限值歸零 (Reset) 後的同步修正策略？

任務 2：應答確認機制 (ACK/NACK Framework)

角色扮演：通訊協定設計師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 指令非同步化：將網頁 App 的發送函式改寫為異步 Promise 框架，點擊後的 UI 鈕進入 Pending 等待視覺。
  2. 確認回軌實作：硬體端在完成邏輯操作後立即回傳含 cmdId 的 ACK 報文；網頁收到匹配的 ACK 後方可確認切換成功。
  3. 自動重傳邏輯：實作超時偵測：若 500ms 內未獲響應，執行最多 3 次的指數退避 (Exponential Backoff) 重傳機制。
  4. 故障路徑驗證：模擬通訊斷路環境，確認 UI 能在重試失敗後正確顯示「同步失敗」警告，而非卡在載入動畫中。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 反饋同步品質：介面是否能透過明顯的視圖變換，讓使用者建立「指令處理中」與「執行已落地」的心理區分？
  • [ ] 傳輸魯棒性指標：在隨機丟包率 20% 的測試環境下，指令是否能最終成功同步且不產生雙端狀態分裂 Bug？

任務 3：真值來源 (SSoT) 重構 (Single Source of Truth Implementation)

角色扮演：系統架構師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 前端主權剝離：徹底移除 App 內所有點擊即切換顏色的「樂觀更新」邏輯，UI 主動權交給通訊層。
  2. 全訂閱制 UI 刷新：強制所有關鍵面板（如模式標籤、解鎖狀態）之變更，必須由接收到的 NOTIFY 指令觸發。
  3. 一致性差異分析：在限速通訊環境下，記錄並分析「SSoT 架構」相較於傳統架構在消除「虚假反饋」上的實質優勢。
  4. 交付物提交：將重構後的單向數據流版本代碼推送到 GitHub PR。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 單向流架構純度：前端 App 是否真正達成 100% 被動刷新？（即必須由硬體事件回傳驅動 UI 變更）
  • [ ] 狀態吻合度實測：網頁面板顯示的數據狀態是否與機器人實體端的物理輸出（馬達、LED）達成了感知層的一致？

GitHub Classroom 詳細操作指南

本單元教學重點：在物理不可靠的通訊介質中，構建邏輯上的絕對可靠。

  1. 重點 1：對抗時間的亂序 - 引導學員理解無線網路中「後發先至」的常態，從而體會 版本序號 (Sequence) 的核心價值。
