Module 1: Intuitive Lab

JS 的心跳：定時器實驗

setInterval 就像一個永不疲倦的節拍器，每隔固定時間就會執行一次任務。而 clearInterval 則是讓它停下來的唯一開關。 試著操作右側的實驗裝置，觀察數字的變化與節奏。

🔑 核心語法

const timer = setInterval(fn, 1000);
// 每 1000ms 執行一次 fn
clearInterval(timer);
// 停止指定的 timer ID

Module 2: Core Principles

長按邏輯的「三部曲」架構

1

Start (按下瞬間)

監聽 touchstart 或 mousedown 事件。 在這裡啟動 setInterval，開始不斷發送指令。

2

Loop (持續長按)

定時器每隔一段時間 (例如 50ms) 執行一次 callback，向 ESP32 發送「前進」指令。

3

Stop (放開手指)

監聽 touchend 或 mouseup 事件。 清除定時器 並發送最後一次「停止」指令。

⚠️ 危險：Timer Stacking (定時器疊加)

如果使用者狂點按鈕，可能會連續觸發多次 Start 而來不及 Stop，導致產生多個「停不下來」的幽靈定時器。

解法： 在每次 setInterval 之前，先執行一次 clearInterval。

long-press.js 
let timer = null;

// 1. Start
function startMove() {
  // 防呆：先清除舊的，避免疊加
  if (timer) clearInterval(timer);

  console.log('Move Forward'); // 立即執行第一次
  
  // 2. Loop
  timer = setInterval(() => {
    console.log('Move Forward...');
    // sendCommand('F');
  }, 50);
}

// 3. Stop
function stopMove() {
  if (timer) {
    clearInterval(timer);
    timer = null;
  }
  console.log('STOP');
  // sendCommand('S');
}

btn.addEventListener('touchstart', startMove);
btn.addEventListener('touchend', stopMove);

長按邏輯的實現 —— 定時器的妙用

歡迎來到第四單元！在本單元中，我們將學習如何從簡單的「點擊」進階到「長按連發」的控制邏輯。這對於開發流暢的機器人控制介面至關重要。

🎯 學習目標

  1. 定時器生命週期管理：學習如何使用 setInterval 與 clearInterval。
  2. 防止定時器疊加：確保每次啟動新定時器前，舊的定時器已被正確清理。
  3. 安全熔斷機制：實作 5 秒自動停止功能，確保操作安全。

🚀 作業任務

任務 1：穩定且不疊加的長按連發

  • 目標：在 timer.js 中實作 startMove 與 stopMove。
  • 要求：按下按鈕時啟動 setInterval，放開時清除。關鍵在於啟動前必須先檢查並清除已存在的定時器，避免「幽靈定時器」導致速度疊加。

任務 2：5 秒強制熔斷的安全機制

  • 目標：實作超時自動煞車。
  • 要求：當長按超過 5 秒時，系統應自動呼叫 stopMove() 強制停止，防止機器人無限衝撞。

任務 3：多按鈕並行測試

  • 目標：確保不同方向的按鈕定時器互不干擾。
  • 要求：測試左轉與前進同時按壓時，兩者的定時器是否能各自獨立運作。

🛠 開發與測試工具

本程式庫提供了一個定時器監控面板（在 index.html 中）。開發時請留意面板上的 Timer ID：

  • 正確行為：按下時出現 ID，放開時 ID 變回 null。
  • 錯誤行為：連續點擊導致 ID 持續跳動或增加，代表發生了定時器疊加。

📝 繳交檢查清單

  • [ ] 按下按鈕時，動作是否流暢連發？
  • [ ] 連續點擊時，Timer ID 是否保持穩定（不會無限累加）？
  • [ ] 按住 5 秒後，動作是否會自動停止？
  • [ ] 切換視窗或分頁時，機器人是否會自動煞車？

2. 作業任務部署細節

任務 1：穩定且不疊加的長按連發 (Stable Press-and-Hold Lab)

  • 目標：打造「零殘留」的控制迴圈。學會管理 setInterval 的回傳 ID，並在每次啟動前執行「防疊加清理」。
  • Classroom 部署建議：

    • 驗證要點：資源管理審核。學員是否理解了為什麼不清理計時器會導致「指令暴衝」？放開按鈕後，控制台是否還有殘留的發送日誌？

任務 2：5 秒強制熔斷的安全熔絲 (5s Safety Kill-switch Lab)

  • 目標：實作「自動停止」保險。防止使用者長時間按壓不放或手機當機導致機器人無限衝撞。
  • Classroom 部署建議：
    • 超時停止實作：在啟動長按的同時，設定一個 setTimeout，在 5 秒後無論是否放開都強制執行 stopMove()。
    • 異常情境處理：額外監聽 window.blur 或 visibilitychange 事件，確保切換 APP 時機器人能自動煞車。
    • 驗證要點：失效安全 (Fail-safe) 審核。超時保護是否真的生效？當學員在長按時「切換分頁」或「手機來電」，系統是否能自動切斷馬達輸出？

任務 3：多按鈕並行不干擾 (Multi-Button Concurrency Lab)

  • 目標：處理「雙手並行操控」。確保左手轉向、右手拉油門時，兩組定時器能協同運作而不互相覆蓋 ID。
  • Classroom 部署建議：
    • 獨立定時器管理：練習將不同按鈕的 Timer ID 分別存儲在物件或陣列中，而非共用單一變數。
    • 數據合併發送：嘗試將兩組狀態打包成單一 JSON 封包發送。
    • 驗證要點：多指操作下是否流暢？當其中一根手指放開時，另一個動作是否依然維持連發？
