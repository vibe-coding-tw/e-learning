基礎 8 - 控制數學

搖桿數據的本質：
歸一化、映射與死區

將混亂的物理像素轉化為精確的硬體指令，掌握操縱感的科學底層。

📏 1. 什麼是歸一化 (Normalization)？

不同手機解析度不同，搖桿半徑可能是 80px 或 150px。如果不歸一化，同樣的滑動距離在不同設備上會產生完全不同的車速。

The Core Concept

將不規則的數值（像素偏移）轉換為標準的數學比例（如 0.0 到 1.0）。這讓你的代碼可以與硬體解析度完美解耦。

Normalization Formula

\[V_{norm} = \frac{Offset}{MaxRadius}\]

Range: [-1.0, 1.0]

🗺️ 2. 數值映射 (Mapping) 的藝術

歸一化後的數值（如 0.75）需要被轉換為硬體能理解的區間。這就是「職人級」操作感誕生的關鍵。

PWM Control

將 0 ~ 1.0 映射為 0 ~ 255 (馬達動力)。

Servo Control

將 -1.0 ~ 1.0 映射為 0 ~ 180 (轉向舵機)。

LINEAR MAP (C++) 
float lerp(float x, float inMax, float outMax) {
  return (x / inMax) * outMax; 
}

🌑 3. 死區 (Deadzone) 與雜訊抑制

搖桿在中央時，讀取值往往不會剛好是 0。如果不設定死區，微小的雜訊會讓馬達微弱共振抖動，造成能量浪費。

Noise Suppression Loop

  1. 採樣數值：獲取目前的 X/Y 歸一化座標。
  2. 閥值判定：檢查 abs(v) 是否大於 0.1 (10% 死區)。
  3. 數值切掉：若小於閥值，一律強制視為 0。
  4. 縮放插值：對剩餘的 0.1~1.0 進行線性補償，確保起點平滑。

Safe System

"一個好的控制系統，在靜止時應該像冰塊一樣安靜。"

控制數據處理實驗室

任務 1：數據標準化實作 (Joystick Normalization)

角色扮演：操縱感設計師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 連結建立個人作業倉庫並 Clone 到本地。
  2. 歸一化演算：實作將物理像素偏移轉換為純數學座標 [-1.0, 1.0] 的演算邏輯。
  3. 死區過濾 (Deadzone)：設定 10% 的閥值，確保搖桿在中心附近的雜訊位移不會觸發任何通訊。
  4. UI 回饋驗證：在畫面顯示目前實時的標準化 (X, Y)，確認各方向極值均能正確觸發。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 極值完整性：不論螢幕解析度如何，搖桿完全推到底時，輸出是否能精準穩定在 1.0？
  • [ ] 中心點穩定度：手指離開搖桿後，輸出是否能立刻恢復為 0.0 且不產生任何脈動數據？

任務 2：非線性映射：指數型油門 (Exponential Mapping Profile)

角色扮演：車輛動力調校工程師 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 動力曲線編寫：在控制邏輯中導入指數映射公式（如 out = in^2 或 pow(in, 2)）。
  2. 微操感實測：比較並記錄線性映射與指數映射在 0% ~ 30% 低速區間的數據增量差異。
  3. 端點閉合校準：驗證非線性處理後，最大輸入值是否依然與硬體滿額輸出 (100% PWM) 完美對應。
  4. 調校報告：在 README 中說明指數映射如何協助駕駛在過彎時進行更精細的速度微調。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 指數特性掌握：輸出的動力曲線是否體現出明顯的「起始溫和、末段爆發」特徵？
  • [ ] 代碼穩健性：處理非線性運算時，是否正確處理了負數值（方向）的符號保留？

任務 3：數據打包與發送節流 (Throttling & Packaging)

角色扮演：通訊協定優化專家 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 報文格式規範：將最終數據封裝進標準 JSON (例：{"x":0.58,"y":-1.0}) 並嚴格限制有效位數。
  2. 發送頻率控制：實作 Throttling (節流) 機制，設定最短發送間隔（如 50ms）或變動門檻判定。
  3. 負擔減輕驗證：觀察 Console logs，確認在靜止狀態下，網路通訊頻寬開銷已被有效抑制。
  4. 正式提交：將具備效能優化意識的完整代碼 Push 提交至作業倉庫 PR。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 優化實效性：控制請求的發送頻率是否處於合理區間，且能隨操作積極度自動調節？
  • [ ] 報文解析度：生成的 JSON 結構是否簡練、標準，符合嵌入式後端低負載解析的需求？

GitHub Classroom 詳細操作指南

本單元教學重點：掌握數據標準化、非線性動力學與高效通訊優化。

  1. 重點 1：歸一化的純粹數學意義 - 解釋為何將「像素」這種物理量轉化為「比率」感官變數是開發跨平台控制器的基石。
  2. 重點 2：感官與數值的映射 - 引導學員理解人類手部操作的非線性特質，以及為何指數曲線能創造「高級」的操作感。
  3. 重點 3：通訊節流的必要性 - 討論在物聯網賽道中，不必要的頻繁 Request 如何導致路由器擁塞與控制延遲。
