🚀 1. 揭祕 DMA：硬體數據搬運助手

CPU 不應浪費在搬運像素點上。DMA (Direct Memory Access) 就像專業的搬運工，讓鏡頭數據自動流向記憶體，釋放極其珍貴的 CPU 核心算力用於避障決策。

Hardware Architecture Advantages

  • 非阻塞傳輸：傳輸過程無需 CPU 參與，可同時執行 PID 運算或 AI 識別。
  • PSRAM 直接映射：數據塊繞過繁重總線，直接進入外部大容量記憶體存取。
  • 中斷回呼驅動：僅在影幀傳輸完成瞬間通知 CPU 處理，最大化處理效能。

📼 3. 雙緩衝機制：消滅影像撕裂

完美並行的工業標準。當 DMA 正在將新數據填入 Buffer A 時，WiFi 任務同時從 Buffer B 發送已完成帧。透過輪替（Ping-Pong）機制，徹底消除影像斷層（撕裂）與讀寫衝突。

Industrial Synchronization Protocol

  • FB_COUNT = 2：配置雙重影幀緩衝區，支撐零延遲讀寫。
  • GRAB_LATEST 模式：處理器自動丟棄過時緩存，強制選取最即時的環境影幀。
  • 原子切換鎖定：利用中斷服務 (ISR) 確保在 Buffer 指針切換瞬間絕不發生競爭。

Ping-Pong Sync Matrix

BUF A: WRITE

單元課程 進階 15 - 影像 DMA

影像處理與 DMA 優化 (Image Processing & DMA Optimization)

歡迎來到 Vibe Coding 進階系列。在本單元中，你將挑戰硬體底層優化的極限。你將學會如何利用 ESP32-S3 的 GDMA (通用直接記憶體存取) 與 LCD_CAM 硬體加速引擎，打造一個高效、低延遲且無撕裂感的「精品視覺驅動」。

🎯 學習目標

  1. 掌握 DMA 原理：理解如何讓硬體自動搬運數據，釋放 CPU 算力。
  2. 優化影像傳輸：實作雙緩衝 (Double Buffering) 機制，消除影像撕裂 (Tearing)。
  3. 量化性能分析：透過 Web 端工具分析影像品質、FPS 與頻寬占用。
  4. 解決硬體邊界問題：處理記憶體對齊 (Alignment) 與硬體描述符鏈路。

🛠️ 任務說明

任務 1：硬體加速擷取器實作 (Hardware Accelerated Image Capture)

配置 GDMA 描述符鏈路，實現影像數據從感知器到記憶體的「零 CPU 介入」搬運。

  • 關鍵檔案：src/cam_dma_optimized.cpp
  • 驗證要點：提交 CPU 負載分析，證明在影像傳輸期間 CPU 仍能穩定執行其他任務。

任務 2：頻寬與品質權衡優化 (Bandwidth & Quality Trade-off)

透過量化數據，找到影像解析度、品質與延遲之間的平衡點。

  • 工具：使用 public/stream_analyzer/ 進行監測。
  • 驗證要點：完成 docs/PERFORMANCE_AUDIT.md 中的審計報告。

任務 3：消滅撕裂之雙緩衝架構 (Tear-Free Double Buffering)

實作「乒乓緩存 (Ping-Pong Buffering)」機制，確保 WiFi 讀取與 DMA 寫入不發生競態。

  • 驗證要點：在 docs/Tear_Free_Report.md 中提交優化前後的影像對比截圖。

2. 作業任務部署細節

任務 1：硬體加速擷取器實作 (Hardware Accelerated Image Capture Lab)

  • 目標：配置 GDMA 描述符鏈路，實現影像數據從感知器到記憶體的「零 CPU 介入」搬運。
  • Classroom 部署建議：
    • DMA 鏈路配置：學員需實作 DMA Descriptors 的鏈式初始化，讓硬體自動處理數據跨頁存取。
    • 中斷回呼調度：僅在 FRAME_DONE 事件發生時喚醒 CPU。
    • 驗證要點：提交一份 CPU 負載分析。證明在影像傳輸期間，主核心依然能以 100Hz 頻率穩定執行控制任務。

任務 2：頻寬權衡權優化分析 (Bandwidth & Quality Trade-off Lab)

  • 目標：透過量化數據，找到「影像解析度/品質」與「傳輸延遲」之間的黃金平衡。
  • Classroom 部署建議：
    • JPEG 二進制嗅探：在網頁端撰寫代碼解析 JPEG Quality Factor (QF)。建立一套 Mbps 實時計算儀表。
    • 性能回報：測試在 QF=30, 50, 85 下的總體系統延遲變動曲線。
    • 官方手冊編撰：學員需在 README 中給出「針對 Vibe-Car 避障場景的最優視覺配置建議」。

任務 3：消滅撕裂之雙緩衝架構 (Tear-Free Double Buffering Lab)

  • 目標：實作工業級的「乒乓緩存 (Ping-Pong Buffering)」機制，徹底消除影像斷層。
  • Classroom 部署建議：
    • fb_count = 2 部署：修改固件並在 PSRAM 中分配雙重 FrameBuffer。
    • 原子性切換鎖定：確保 WiFi 讀取指標與 DMA 寫入指標永不發生競態 (Race Condition)。
    • 對比證據展示：要求提交一張在「快速移動」環境下的對比圖。左邊是優化前的斷層 (Tearing)，右邊是優化後的完美影格。

3. 底層驅動工程師導師點評標準 (Hardware Acceleration Standards)

此單元的價值在於 「對 SoC 硬體邊界功能的極致壓榨」：

  • [ ] DMA 鏈路穩健度：在長時間（30 分鐘以上）的串流環境下，是否會產生數據位移或記憶體洩漏？
  • [ ] 資源利用藝術：是否成功將 PSRAM 的延遲與 DMA 的速度完美結合，釋放出 CPU 算力？
  • [ ] 架構思維深度：提交的代碼是否考慮了「影格跳過 (Frame Skip)」邏輯以因應無線網路抖動？
