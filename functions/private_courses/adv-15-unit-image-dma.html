🚀 1. 揭祕 DMA：硬體數據搬運助手

CPU 不應浪費在搬運像素點上。DMA (Direct Memory Access) 就像專業的搬運工，讓鏡頭數據自動流向記憶體，釋放極其珍貴的 CPU 核心算力用於避障決策。

Hardware Architecture Advantages

  • 非阻塞傳輸：傳輸過程無需 CPU 參與，可同時執行 PID 運算或 AI 識別。
  • PSRAM 直接映射：數據塊繞過繁重總線，直接進入外部大容量記憶體存取。
  • 中斷回呼驅動：僅在影幀傳輸完成瞬間通知 CPU 處理，最大化處理效能。

📼 3. 雙緩衝機制：消滅影像撕裂

完美並行的工業標準。當 DMA 正在將新數據填入 Buffer A 時，WiFi 任務同時從 Buffer B 發送已完成帧。透過輪替（Ping-Pong）機制，徹底消除影像斷層（撕裂）與讀寫衝突。

Industrial Synchronization Protocol

  • FB_COUNT = 2：配置雙重影幀緩衝區，支撐零延遲讀寫。
  • GRAB_LATEST 模式：處理器自動丟棄過時緩存，強制選取最即時的環境影幀。
  • 原子切換鎖定：利用中斷服務 (ISR) 確保在 Buffer 指針切換瞬間絕不發生競爭。
