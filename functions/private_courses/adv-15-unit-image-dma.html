🚀 1. 揭祕 DMA：硬體數據搬運助手

CPU 不應浪費在搬運像素點上。DMA (Direct Memory Access) 就像專業的搬運助手，讓鏡頭數據自動流向記憶體，釋放極其珍貴的 CPU 算力。

System Architecture Advantages

  • 非阻塞傳輸：CPU 在傳輸過程中可同時運算避障邏輯。
  • 直接緩存 (PSRAM)：繞過繁忙的總線，直接進入外部大容量記憶體。
  • 中斷驅動：傳完一幀才通知 CPU，最大化效率。

📼 3. 雙緩衝機制：消滅影像撕裂

完美並行的工業標準。當 DMA 正在填入 Buffer A 時，WiFi 任務同時發送 Buffer B。透過輪替機制，徹底消滅影像斷層（撕裂）現象。

Industrial Sync Rules

  • FB_COUNT = 2：配置兩個框緩衝區。
  • GRAB_LATEST：自動丟棄過期影幀，減少顯示延遲。
  • 中斷回呼：確保在 Buffer 切換的瞬間不發生讀寫競爭。

Sync Matrix

BUFA: WRITING
BUFB: READING

GitHub Classroom 詳細操作指南

本單元教學重點：掌握 SoC 層級的硬體加速。讓學員學會利用 DMA 與雙緩衝區，在資源極度受限的 ESP32 上實現「專業級」影像處理效能。

  1. 重點 1：CPU 的解放 - 強調 DMA 不是把 CPU 變快，而是讓它「不用做雜事」。這是邊緣計算的設計哲學。
  2. 重點 2：記憶體架構觀感 - 深入 SRAM 與 PSRAM 的差異。讓學員體悟到大數據必須擺在外部緩衝的工程必要性。
  3. 重點 3：消滅隨機性 - 雙緩衝區是為了消滅影像撕裂與傳輸不穩定性。這是高品質串流產品的標配。
