VS Code 安裝與基礎配置

模組 6 個單元 初學者 開發者 本機環境

工欲善其事，必先利其器。雖然雲端開發環境（如 github.dev、Codespaces）非常便利， 但如果你要進行離線開發、或是連接實體硬體設備（如 ESP32、感測器、馬達等）， 你仍需要在電腦上安裝一套完整的 VS Code 本地環境。

本模組將引導你完成 VS Code 本機端的下載、安裝、中文化配置與必備擴充套件， 並設定 GitHub 帳號同步，讓你的開發偏好設定永不遺失。

學習目標

完成本模組後，你將能夠：

  • 成功下載並安裝 Visual Studio Code (VS Code) 桌面版。
  • 安裝 Chinese (Traditional) Language Pack 完成介面中文化。
  • 配置 Prettier、Material Icon Theme 與 Error Lens 等必備擴充套件。
  • 掌握 GitHub Settings Sync，實現設定的跨裝置備份與同步。

必要條件

  • 電腦一台（支援 Windows, macOS 或 Linux）
  • 可上網的網路連線，用於下載安裝檔與套件

簡介

在現代的 Vibe Coding 模式中，AI 能幫你寫出大量程式碼，但要將這些程式碼在真實硬體上編譯、執行、除錯， 依然需要一個穩定、高度客製化的「指揮中心」。而這個中心就是本地端的 Visual Studio Code。

情境
小華使用 Codespaces 寫好了一個 ESP32 網頁控制的程式碼。但當他要把程式碼燒錄到手邊的開發板時， 他發現網頁瀏覽器無法直接存取本機的序列埠（Serial Port），也無法直接插上 USB 燒錄。 老師點醒他：「雲端虛擬機在天邊，你的開發板在手邊。這時你需要將代碼同步回本機，用本地安裝的 VS Code 開發。」

本地 VS Code 的優勢在於：它可以完全控制你的電腦資源、直接使用終端機執行任意編譯工具鏈， 並且可以通過 USB 與藍牙與你的硬體直接溝通。這是瀏覽器網頁版目前難以做到的。 本單元將引導你在 15 分鐘內，搭建起這套強大的本地開發環境。

安裝與中文化

打地基的時間到了！我們將循序漸進完成編輯器主體的部署，並設定好親切的繁體中文介面。

安裝步驟

1

下載 VS Code

前往官方網站 code.visualstudio.com，點擊下載適用於你作業系統（Windows/macOS/Linux）的安裝包。

2

執行安裝程序

執行安裝檔，並依指示完成安裝。

Windows 用戶關鍵設定 安裝時的「選取其他工作」步驟，務必勾選 「加入 '開啟以程式碼編輯' 動作到 Windows 檔案總管內容功能表」 (Add "Open with Code")。 這讓你能在任何資料夾直接按右鍵開啟專案，開發效率大幅提升！

繁體中文介面設定

1

打開 Extensions 控制面板

開啟安裝好的 VS Code。點擊左側工具欄最下方的四個方塊圖示 (Extensions) ，或按快捷鍵 Ctrl+Shift+X (Mac 為 Cmd+Shift+X)。

2

搜尋與安裝中文包

在搜尋框輸入 Chinese，找到名為 "Chinese (Traditional) Language Pack" (由 Microsoft 官方發行) 的擴充套件，點擊 Install 安裝。

3

重啟套用

安裝完成後，右下角會彈出重啟提示，點擊 Restart。重啟後介面將切換為繁體中文。

必備擴充套件推薦

VS Code 本身是一台輕快的骨架，但加上適合的外掛 (Extensions) 後，它將成為你的神兵利器。 以下是我們強烈建議每位網頁與物聯網開發者立刻裝上的三個基本擴充套件。

擴充套件介紹

Prettier
代碼格式化（Code Formatter）

自動幫你把縮排亂掉、括號對齊錯誤的程式碼排版整齊。在存檔瞬间（可設定 format on save），讓混亂一秒歸零。

Material Icon Theme
檔案標誌美化（File Icons）

讓檔案樹列表出現漂亮、易辨識的標誌與圖示。HTML 檔案顯示 HTML 圖標，JavaScript 檔案顯示 JS 圖標，方便快速定位檔案類型。

Error Lens
即時除錯反饋（Instant Debug）

直接把代碼中的錯誤、警告以及語法警告提示，以高亮色彩直接顯示在出錯程式碼的行尾，不用將滑鼠移上去懸停，除錯效率提升 200%。

自動存檔排版設定 安裝 Prettier 後，建議進入 VS Code 設定（Ctrl+,），搜尋 format on save 並將其勾選。 如此一來，每次存檔時 VS Code 都會自動以 Prettier 標準幫你重新排版！

實作練習：本地開發環境與同步

請動手完成以下兩項本地端配置任務，這將為你未來的程式開發與 AI 協作打下最穩固的基礎。

  作業 1
約 10 分鐘

安裝 GitHub Copilot

讓 AI 成為你的副駕駛，體驗結對編程（Pair Programming）的強大威力。

  • 點擊左側 Extensions 圖示或按 Ctrl+Shift+X
  • 搜尋 GitHub Copilot 並點擊安裝 (Install)
  • 點擊右下角的登入通知，點選 "Allow" 登入 GitHub 帳號以獲取授權
  • 確認 VS Code 右下角狀態列出現 Copilot 小圖示（無斜線標示），即可開始使用
小提示 GitHub Copilot 對於學生或符合特定資格的開發者提供免費試用；若無訂閱，也可以先安裝，之後再進行啟用。
  作業 2
約 10 分鐘

Git 帳號設定同步

設定 Settings Sync，讓你的編輯器外掛、快捷鍵、主題跟著帳號走。

  • 點擊 VS Code 左下角的人頭 👤 帳戶圖示
  • 點擊 "開啟設定同步... (Turn on Settings Sync...)"
  • 在跳出的選單中點擊 "Sign in & Turn on"，並選擇使用 GitHub 帳號登入
  • 於瀏覽器完成 GitHub 授權後，確認 VS Code 左下角頭像顯示已登入，啟用同步狀態

🚀 單元 01：VS Code 本地安裝與開發環境配置

歡迎來到你的第一個開發者任務！在本單元中，我們將一起完成開發環境的架設。工欲善其事，必先利其器。擁有一個順手的編輯器是成為優秀開發者的第一步。

🎯 學習目標

  • 成功安裝並配置 Visual Studio Code (VS Code)。
  • 掌握 GitHub Setting Sync，讓你的設定跟著帳號走。
  • 學會使用 Live Server 進行網頁即時預覽。
  • 練習透過 Git 提交你的第一份環境檢查報告。

🛠️ 操作步驟

1. 安裝與初始化

如果您尚未安裝 VS Code，請至 官網 下載並安裝。 開啟本資料夾後，您會看到右下角跳出「建議安裝外掛」的提示，請點選 「全部安裝 (Install All)」。

[!TIP] 我們推薦安裝：Chinese (Traditional) Language Pack (中文化) 與 Live Server (網頁預覽)。

2. 設定同步 (Settings Sync)

點擊 VS Code 左下角的 「人頭圖示」，選擇 「開啟設定同步 (Turn on Settings Sync...)」，並使用您的 GitHub 帳號 登入。 這能確保未來不論換到哪台電腦，您的偏好設定都能自動套用。

3. 環境測試 (Live Server)

  1. 在左側檔案瀏覽器點擊 index.html。
  2. 點擊編輯器右下角的 「Go Live」 按鈕。
  3. 您的預設瀏覽器應會自動開啟，並顯示「環境測試成功！」的字樣。

📝 繳交任務

請完成以下兩項任務，以證明您的環境已準備就緒：

任務 1：全視窗截圖 (setup-ui.png)

截下一張 VS Code 的全視窗截圖。

  • 必須包含：左側導覽列、下方的狀態列、以及左下角已登入的人頭圖示。
  • 檔案命名：setup-ui.png。
  • 存檔路徑：直接放入專案根目錄。

任務 2：網頁預覽截圖 (html-preview.png)

截下一張包含 「VS Code 編輯器」 與 「瀏覽器預覽畫面」 並排的圖片。

  • 檔案命名：html-preview.png。
  • 存檔路徑：直接放入專案根目錄。

任務 3：填寫檢查清單

開啟 env-check.md，將您的進度打勾 [x]，並確認截圖已正確顯示。

🚀 提交方式

當您完成上述步驟後，請在終端機 (Terminal) 輸入以下指令：

git add .
git commit -m "feat: complete vscode environment setup"
git push

最後，請回到 GitHub 程式庫頁面並點擊 「Create Pull Request」，這將通知導師進行檢核。

[!IMPORTANT] 如果在啟動 Live Server 時遇到 Port 5500 is already in use 錯誤，請嘗試重啟 VS Code 或在設定中更改 Live Server 的 Port。

針對 01-unit-vscode-setup (VS Code 本地安裝與配置) 單元，由於重點在於「軟體環境安裝」，在 GitHub Classroom 的部署上需要結合「圖片證明」與「設定檔測試」。

以下是針對本單元作業 (Assignments) 的部署建議：

1. 範本程式庫 (Template Repo) 的關鍵配置

為了讓學生一打開程式庫就能「接收到安裝建議」，強烈建議在範本中加入以下設定：

  • 📂 .vscode/extensions.json：這是 VS Code 的「推薦外掛」功能。
    • 內容包含：ritwickdey.liveserver 與 ms-ceintl.vscode-language-pack-zh-hant。
    • 效果：當學生 Clone 並開啟資料夾，右下角會自動跳出「此程式庫建議安裝外掛」的提示，大幅降低學生搜尋的時間。
  • 📂 index.html：放一個簡單常用的 HTML 範例。
    • 效果：這能用來測試 VS Code 是否已正確啟動。如果安裝成功，學生可以使用 Live Server 預覽網頁內容。

2. 作業任務部署細節

任務 1：中文化與同步測試 (L10N & Sync)

  • 驗證方式：螢幕截圖法。
    • 要求學生將當前 VS Code 的全視窗截圖，命名為 setup-ui.png 放入程式庫。
    • 導師檢核點：介面是否為中文？左下角人頭圖示是否顯示登入？
  • 自動化檢核自學 (Autograding)：
    • 設置一個 Action 檢查 setup-ui.png 檔案是否存在。

任務 2：網頁開發環境測試 (Live Server)

  • 驗證方式：報告更新法。
    • 提供一個 env-check.md 檢核表。
    • 要求學生在成功使用 Live Server 開啟網頁後，截下一張含有 「VS Code 編輯器」 與 「瀏覽器預覽畫面」 的圖片 html-preview.png。
    • 將圖片嵌入 env-check.md 中並 Push。

3. 給導師的翻轉教室策略 (Flipped Strategy)

  • 自動化回饋 (Feedback PR)：
    • 利用 Classroom 的自動建立 PR 功能。
    • 導師若看到學生截圖中的 PIO 狀態列顯示 Loading... 或錯誤提示，可直接在截圖處留言指導。
  • 解決「大檔案下載」痛點：
    • 由於 PlatformIO 第一次啟動會下載約 500MB 的組件，在課堂上可能會發生網路擁塞。
    • 部署建議：在 README 中提早釋出下載連結，讓學員在「課前」就先完成下載動作，課中專注於 Sync 邏輯與介面操作。

📁 推薦範本結構 (GitHub Classroom Template)：

.
├── .vscode/
│   └── extensions.json    # 引導學生一鍵安裝所有必備外掛
├── index.html             # 測試環境用的 HTML 範例
├── env-check.md           # 學生需填寫的環境檢查清單
└── README.md              # 作業提交指南與截圖上傳說明

透過這種方式，您不僅能驗證學生是否「裝好了」，更能確保他們學會了如何透過 Git 提交開發環境報告，這對未來的協作開發是非常好的開端。
