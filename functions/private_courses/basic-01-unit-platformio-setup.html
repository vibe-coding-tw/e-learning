為什麼捨棄 Arduino IDE？

Arduino IDE 是許多人接觸嵌入式開發的第一個工具，它的優點是簡單直覺——打開就能寫、一鍵就能燒。然而，當你的專案開始成長（超過幾百行程式碼、需要引用多個第三方程式庫、或是要同時支援多塊不同的開發板），Arduino IDE 的侷限性就會變成開發瓶頸。

PlatformIO 搭配 VS Code 是目前嵌入式開發社群中最受歡迎的專業開發組合。它不是取代 Arduino 框架，而是提供一個更強大的「容器」來運行 Arduino 程式碼——你可以繼續使用 setup()、loop()、digitalWrite() 這些熟悉的 API，同時享受現代 IDE 帶來的效率提升。

Arduino IDE vs PlatformIO 功能比較

功能 Arduino IDE VS Code + PlatformIO
代碼智慧補全 基本提示 IntelliSense（函式簽章、參數提示、自動補全）
多檔案專案 所有 .ino 自動合併，難以管理 標準 C++ 專案結構，src/、lib/、include/ 分層清晰
程式庫管理 手動下載 ZIP 或使用內建管理器 lib_deps 一行設定，自動下載與版本鎖定
多環境編譯 一次只能選一塊板 [env:dev] + [env:release] 同時配置
版本控制整合 無內建支援 Git 完整整合，左側欄即時顯示變更
編譯速度 每次全量編譯 增量編譯（只重新編譯修改過的檔案）

IntelliSense 的價值

IntelliSense 是 VS Code 最強大的功能之一。當你輸入 Serial. 後，它會自動列出所有可用的方法（如 begin()、println()、available()），並且顯示每個方法需要的參數類型與說明文件。這不只是方便——它能顯著減少拼寫錯誤，加速學習新的 API。

關鍵差異 在 Arduino IDE 中，#include <Arduino.h> 是隱式自動引入的，所以你不需要手動撰寫。但在 PlatformIO 的嚴格 C++ 環境中，你必須在每個 .cpp 檔案的開頭顯式加上 #include <Arduino.h>，否則編譯器會找不到 setup()、loop()、Serial 等核心定義。

依賴管理的革新

在 Arduino IDE 中安裝第三方程式庫，你需要手動搜尋、下載 ZIP 檔、解壓到特定資料夾，甚至還要擔心版本衝突。PlatformIO 的 lib_deps 機制徹底解決了這個問題——只需要在 platformio.ini 中加入一行設定：

lib_deps =
bblanchon/ArduinoJson@^7.0.0
adafruit/Adafruit NeoPixel@^1.12.0

編譯時，PlatformIO 會自動下載指定版本的程式庫並放入專案目錄。當你把專案分享給同學或上傳到 GitHub 時，對方只需要 clone 下來就能直接編譯，完全不需要手動安裝任何程式庫。

安裝與初始化

現在讓我們動手安裝開發環境。整個流程分為三個階段：安裝 VS Code 本體、載入 PlatformIO 擴充套件、以及認識專案的核心配置檔。完成後，你就擁有了一個功能齊全的 ESP32 開發環境。

階段一：安裝 VS Code

Visual Studio Code（簡稱 VS Code）是微軟開發的免費跨平台程式碼編輯器，目前是全球最受歡迎的開發工具之一。它的強大之處在於豐富的擴充套件生態系——透過安裝不同的擴充套件，VS Code 可以變成任何語言的完整 IDE。

1

下載 VS Code

前往 code.visualstudio.com，點擊首頁的下載按鈕。網站會自動偵測你的作業系統（Windows / macOS / Linux）並提供對應的安裝檔。

2

執行安裝

Windows 用戶執行 .exe 安裝檔，建議勾選「Add to PATH」選項。macOS 用戶將 .app 拖入 Applications 資料夾即可。

3

首次啟動

啟動 VS Code 後，你會看到歡迎頁面。可以先選擇喜歡的色彩主題（推薦深色主題 Dark+ 以減少眼睛疲勞）。

階段二：安裝 PlatformIO 擴充套件

1

開啟擴充套件面板

點選左側活動列（Activity Bar）的方塊圖示，或使用快捷鍵 Ctrl+Shift+X（macOS 為 Cmd+Shift+X）開啟 Extensions 面板。

2

搜尋並安裝 PlatformIO IDE

在搜尋欄輸入 PlatformIO IDE，找到由 PlatformIO 官方發佈的擴充套件，點擊「Install」按鈕。

3

等待初始化

首次安裝後，PlatformIO 會自動下載 Python 環境與核心工具鏈，這可能需要 5~10 分鐘。完成後，左側活動列會出現一個「外星人」圖示——這就是 PlatformIO 的入口。

安裝驗證 安裝完成後，點擊左下角狀態列，你應該會看到 PlatformIO 的小圖示（一個小房子）。點擊它可以開啟 PlatformIO Home 頁面，從這裡你可以建立新專案、管理程式庫、以及更新工具鏈。

階段三：認識 platformio.ini

每個 PlatformIO 專案的根目錄都有一個 platformio.ini 檔案，它是整個專案的「大腦」——定義了目標板型、編譯框架、序列監控速率、第三方程式庫依賴等所有配置。以下是一個典型的 ESP32 專案配置：

; PlatformIO 專案設定檔
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
monitor_speed = 115200
upload_speed = 921600
lib_deps =
bblanchon/ArduinoJson@^7.0.0
重要提醒 platformio.ini 使用 INI 格式。每個 [env:名稱] 區塊代表一組編譯環境。你可以同時定義多個環境（例如 [env:dev] 和 [env:release]），在編譯或上傳時選擇要使用哪一個。

專案結構巡禮

了解 PlatformIO 專案的目錄結構是高效開發的基礎。與 Arduino IDE 把所有檔案混在一起的做法不同，PlatformIO 採用清晰的分層架構，每個目錄都有明確的職責。這種結構遵循了 C/C++ 專案的工業標準，也為未來的團隊協作與版本控制奠定了基礎。

當你用 PlatformIO 建立一個新的 ESP32 專案時，系統會自動生成以下目錄結構：

my-esp32-project/
├── src/ # 原始碼目錄（你的主要程式碼放這裡）
│ └── main.cpp # 程式進入點
├── include/ # 標頭檔目錄（.h 宣告檔）
├── lib/ # 私有程式庫目錄
├── test/ # 測試程式碼目錄
├── .pio/ # 編譯暫存區（自動生成，勿手動修改）
└── platformio.ini # 專案配置檔

各目錄詳解

目錄/檔案 用途 要點
src/ 存放所有原始碼（.cpp 檔案） 主程式 main.cpp 必須包含 #include <Arduino.h>
include/ 存放標頭檔（.h 宣告檔） 與 src/ 搭配使用，定義函式與結構體的介面
lib/ 存放專案私有的程式庫 當專案程式碼超過 500 行時，建議開始拆分模組到此
test/ 存放單元測試程式碼 PlatformIO 支援在本機或裝置上執行自動化測試
.pio/ 編譯中間產物暫存區 由系統自動管理，應加入 .gitignore 不納入版本控制
platformio.ini 專案核心配置檔 定義板型、框架、波特率、程式庫依賴等所有設定

為什麼必須 #include <Arduino.h>？

這是從 Arduino IDE 轉移到 PlatformIO 時最常遇到的問題。在 Arduino IDE 中，編譯器會在幕後自動將你的 .ino 檔案轉換為合法的 C++ 程式碼，其中就包括隱式插入 #include <Arduino.h>。但 PlatformIO 使用標準的 C++ 編譯器，不會做這種自動轉換——如果你沒有手動引入，編譯器就找不到 setup()、loop()、Serial、digitalWrite() 這些定義，結果就是一整頁的紅色錯誤。

// src/main.cpp — PlatformIO 的標準起手式
#include <Arduino.h>

void setup() {
Serial.begin(115200);
Serial.println("Hello, ESP32!");
}

void loop() {
// 你的主程式邏輯
}
常見錯誤 如果你在編譯時看到 'setup' was not declared in this scope 或 'Serial' was not declared 的錯誤訊息，十之八九是忘記在檔案開頭加上 #include <Arduino.h>。這是 PlatformIO 新手最常踩的坑。

.gitignore 的重要性

當你使用 Git 管理專案時，.pio/ 目錄下的編譯暫存檔案不應被追蹤。PlatformIO 通常會自動生成 .gitignore 檔案，但如果沒有，請確保其中至少包含以下內容：

# .gitignore
.pio/
.vscode/
*.swp

Unit: PlatformIO 環境搭建 (VS Code + PlatformIO Setup)

歡迎來到專業硬體開發的第一步！在本單元中，你將學習如何從 Arduino IDE 轉移到更強大的 VS Code + PlatformIO 開發環境。這不僅是換一個編輯器，更是學習現代軟體工程的專案管理思維。

🎯 學習目標

  1. 掌握 PlatformIO 的專案結構與 platformio.ini 配置。
  2. 理解 C++ 代碼規範與 #include 邏輯。
  3. 學會使用 lib_deps 自動化管理第三方庫。
  4. 掌握多環境配置與調速技巧。

📂 專案結構說明

  • platformio.ini: 核心配置文件，負責硬體定義與依賴管理。
  • src/main.cpp: 你的原始碼存放在這裡。
  • docs/ENV_CHECK.md: 你的環境報告書，完成任務後請在此記錄。
  • assets/: 請將你的實驗截圖存放在此。

任務 1：PlatformIO 首航與語法修正

  • 打開 src/main.cpp。
  • 你會發現代碼有紅字報錯。請修正它，讓 PlatformIO 能夠識別 Arduino 的核心函式。
  • 驗證：按下下方狀態列的 ✔ (Compile) 按鈕，直到出現綠色的 SUCCESS。

任務 2：自動化依賴管家

  • 在 platformio.ini 中加入 bblanchon/ArduinoJson 庫。
  • 再次點擊編譯，觀察 PlatformIO 如何自動下載並安裝庫檔案。
  • 驗證：取消 src/main.cpp 中關於 ArduinoJson 的註解，確保編譯通過。

任務 3：解決亂碼與多環境配置

  • 在 platformio.ini 中設定 monitor_speed = 115200 以解決串口亂碼。
  • 建立一個名為 [env:release] 的新區塊，並將 upload_speed 提高到 921600。
  • 驗證：提交最終的 platformio.ini 檔案。

📝 提交要求

  1. 完成 docs/ENV_CHECK.md 中的所有勾選。
