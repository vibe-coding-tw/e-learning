基礎 1 - 開發環境

VS Code + PlatformIO 環境搭建

邁向專業：IntelliSense、版本控制、多環境管理。寫程式不再是寫作業，而是做工程。

🚀 為什麼捨棄 Arduino IDE？

PlatformIO 讓開發從「寫作業」進化為「做工程」。

✨

代碼補完 (IntelliSense)

自動校正。跳出可用函式，減少 Typo，寫程式不再依賴記憶。

📦

版本管理

專案級函式庫鎖定。避免更新就壞掉的悲劇，確保穩定度。

🌍

多環境編譯

同一套代碼，換個配置就能分別部署到不同開發板或速率。

🛠️ 安裝與初始化

1

安裝 VS Code 本體

下載並安裝 Visual Studio Code。目前最受歡迎的通用型 IDE。

2

載入 PlatformIO 套件

搜尋並安裝 "PlatformIO"。出現「外星人圖標 👽」代表架設成功。

3

認識 platformio.ini

專案的「大腦」，管理板型、驅動速率與函式庫依賴。

Configuration Example
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
monitor_speed = 115200
upload_speed = 921600

📂 專案結構巡禮

src/main.cpp

程式碼主戰場。注意：手動包含 #include <Arduino.h> 是基本功。

lib/

私有函式庫。當程式碼超過 500 行時，建議開始拆解模組。

include/

Header 檔案。與 src 配合建立嚴謹的 C++ 專案結構。

.pio/

編譯暫存區。Git 提交時應忽略此目錄以節省空間。

任務 1：新環境的首航 (First Voyage in PlatformIO)

角色扮演：韌體開發實習生 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 建立倉庫：從 GitHub Classroom 建立個人作業倉庫教 Clone 到本地。
  2. 專案初始化：建立全新專案，選擇 ESP32 Dev Module 與 Arduino。
  3. 代碼移植與修正：搬移代碼，手動加上 #include <Arduino.h>。
  4. 驗證繳交：在 README 貼上 IntelliSense 正常運作截圖。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 編譯成功率：PIO 終端機是否顯示綠色 SUCCESS 狀態？
  • [ ] 專案結構掌握：理解 src/main.cpp 是主程式入口。

任務 2：函式庫管家 (Library Manager Implementation)

角色扮演：軟體套件管理員 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 依賴聲明實作：在 lib_deps 欄位加入 ArduinoJson。
  2. 自動化安裝驗證：觀察 PIO 如何自動從網路抓取函式庫。
  3. 技術報告：在 README 中比較 PIO 管家模式與手動下載優劣。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 配置導向思維：掌握透過配置文件管理依賴的核心規範。
  • [ ] 版本控制意識：能說明 ^ 符號在版本控制中的意義。

任務 3：多環境並行配置 (Multi-Environment Configuration)

角色扮演：自動化工程師 | 難度：⭐️⭐️ | 估計時長：45 min

🛠️ 執行技術流程 (Git Workflow)

  1. 通訊參數優化：設定 monitor_speed = 115200 解決亂碼。
  2. 環境 Profile 分流：建立 [env:release] 設定極速燒錄。
  3. 切換演練：使用狀態列選單針對不同 Profile 進行編譯與上傳。
  4. 最終 Commit：整理 platformio.ini 並 Push。

🎯 教師點評標準 (Tutor Benchmarks)

  • [ ] 環境抽象化理解：體會上傳速度配置對效率的影響。
  • [ ] 排障能力：能自動調整相應速率參數以恢復數據可讀性。
