📂 專案結構巡禮

src/main.cpp

程式碼主戰場。注意：手動包含 #include <Arduino.h> 是基本功。

lib/

私有函式庫。當程式碼超過 500 行時，建議開始拆解模組。

include/

Header 檔案。與 src 配合建立嚴謹的 C++ 專案結構。

.pio/

編譯暫存區。Git 提交時應忽略此目錄以節省空間。

實戰任務：PlatformIO 環境搭建

請動手完成以下三項實作任務，這是驗證本單元學習成果的核心挑戰。 每項任務完成後，請點擊卡片繳交作業。

  作業 1-1
約 60 分鐘

PlatformIO 首航與語法修正

目標：建立專案並移入代碼，加上必要標頭檔，解決 C++ 編譯錯誤，完成首次燒錄驗證。

  • 打開 src/main.cpp，手動引用 Arduino.h 以修正紅字語法報錯
  • 點擊下方狀態列 of ✔ (Compile) 按鈕進行編譯
  • 確認終端機編譯結果出現綠色的 SUCCESS 訊息
  • 截圖「編譯成功」畫面，存入 assets/ 資料夾中
提示說明 PlatformIO 是嚴格的標準 C++ 編譯環境，與 Arduino IDE 不同，它要求所有原始碼檔案必須顯式 #include <Arduino.h>。
  作業 1-2
約 60 分鐘

自動化依賴管理

目標：編輯設定檔引入 ArduinoJson 第三方庫，體驗 PlatformIO 的依賴庫自動同步。

  • 編輯 platformio.ini，在 lib_deps 中加入 bblanchon/ArduinoJson
  • 取消 src/main.cpp 中 ArduinoJson 相關代碼的註解
  • 重新點擊編譯，觀察並確認 PlatformIO 自動下載並安裝庫檔案
  • 確認代碼再次順利編譯通過
  • 截圖「IntelliSense 提示」或「庫自動下載成功」畫面存入 assets/
依賴管理優勢 使用 lib_deps 能讓專案程式庫隨附於設定檔，他人複製你的專案時不需要手動安裝任何庫，環境會自動下載，極大提升專案可移植性。
  作業 1-3
約 45 分鐘

解決亂碼與多環境配置

目標：設定 monitor_speed 解決亂碼，並配置發布環境（release）以提升燒錄速度。

  • 在 platformio.ini 中設定 monitor_speed = 115200 以解決串口監控器亂碼
  • 在 platformio.ini 中建立一個名為 [env:release] 的新區塊
  • 在新環境區塊中將 upload_speed 提高到 921600 以加速燒錄
  • 完成 docs/ENV_CHECK.md 中所有的檢核項目並提交
多環境配置注意 [env:release] 的參數會覆蓋或獨立於 [env:esp32dev]。上傳時請選擇正確的環境 (Environment) 以免設定未生效。

Unit: PlatformIO 環境搭建 (學生作業指引)

歡迎使用本程式庫！本單元將帶領你從 Arduino IDE 轉移到專業的 VS Code + PlatformIO 開發環境。這不僅是換一個編輯器，更是學習現代軟體工程的專案管理思維。

🎯 作業目標

完成本單元後，你應能：

  1. 配置 platformio.ini 以管理硬體環境與第三方程式庫。
  2. 修正 C++ 編譯錯誤並理解 #include 邏輯。
  3. 使用 PlatformIO 的自動化依賴管理功能。

🚀 執行步驟

任務 1：修正 src/main.cpp

  • 打開 src/main.cpp。
  • 修正紅字報錯，確保引用了必要的標頭檔（提示：PlatformIO 需要手動引用 Arduino.h）。
  • 點擊狀態列的 ✔ (Compile) 確保出現 SUCCESS。

任務 2：管理程式庫依賴

  • 編輯 platformio.ini。
  • 在 lib_deps 中加入 bblanchon/ArduinoJson。
  • 重新編譯並觀察 PlatformIO 如何自動下載依賴。
  • 取消 src/main.cpp 中 ArduinoJson 相關代碼的註解並通過編譯。

任務 3：解決串口亂碼與進階配置

  • 在 platformio.ini 中設定 monitor_speed = 115200。
  • 新增一個名為 [env:release] 的區塊，並將 upload_speed 提高到 921600。

📝 提交清單

  • [ ] 完成 docs/ENV_CHECK.md 中的所有檢核項目。
  • [ ] 將「編譯成功」與「IntelliSense 提示」截圖存入 assets/。
  • [ ] 將變更 git commit 並 push 到 GitHub。

詳細的導師指引請參考 .github/tutor_guide.md

Unit: PlatformIO 環境搭建 (VS Code + PlatformIO Setup)

歡迎來到專業硬體開發的第一步！在本單元中，你將學習如何從 Arduino IDE 轉移到更強大的 VS Code + PlatformIO 開發環境。這不僅是換一個編輯器，更是學習現代軟體工程的專案管理思維。

🎯 學習目標

  1. 掌握 PlatformIO 的專案結構與 platformio.ini 配置。
  2. 理解 C++ 代碼規範與 #include 邏輯。
  3. 學會使用 lib_deps 自動化管理第三方庫。
  4. 掌握多環境配置與調速技巧。

📂 專案結構說明

  • platformio.ini: 核心配置文件，負責硬體定義與依賴管理。
  • src/main.cpp: 你的原始碼存放在這裡。
  • docs/ENV_CHECK.md: 你的環境報告書，完成任務後請在此記錄。
  • assets/: 請將你的實驗截圖存放在此。

🚀 任務清單

任務 1：PlatformIO 首航與語法修正

  • 打開 src/main.cpp。
  • 你會發現代碼有紅字報錯。請修正它，讓 PlatformIO 能夠識別 Arduino 的核心函式。
  • 驗證：按下下方狀態列的 ✔ (Compile) 按鈕，直到出現綠色的 SUCCESS。

任務 2：自動化依賴管家

  • 在 platformio.ini 中加入 bblanchon/ArduinoJson 庫。
  • 再次點擊編譯，觀察 PlatformIO 如何自動下載並安裝庫檔案。
  • 驗證：取消 src/main.cpp 中關於 ArduinoJson 的註解，確保編譯通過。

任務 3：解決亂碼與多環境配置

  • 在 platformio.ini 中設定 monitor_speed = 115200 以解決串口亂碼。
  • 建立一個名為 [env:release] 的新區塊，並將 upload_speed 提高到 921600。
  • 驗證：提交最終的 platformio.ini 檔案。

📝 提交要求

  1. 完成 docs/ENV_CHECK.md 中的所有勾選。
  2. 在 assets/ 中存入「IntelliSense 常數提示」與「編譯成功」的截圖。
  3. 將所有變更 git commit 並 push 到 GitHub。
