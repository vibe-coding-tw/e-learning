使用 Copilot Agent Mode (自主代理模式)

這是最直接的結合方式。在 VS Code 中，你可以將 Copilot 切換到 Agent Mode。它是一個「自主程式設計師」，可以跨多個檔案進行修改、執行終端機命令、讀取編譯錯誤並自動修復。

操作流程

  1. 1 進入 Chat 視窗： 打開 Copilot Chat (Ctrl+Alt+I)。
  2. 2 切換模式： 在 Chat 頂部的下拉選單中選擇 "Agent" 模式。
  3. 3 下達任務： 輸入高階指令。例如：「幫我把這個專案的 API 從 REST 改成 GraphQL」。

自主迴圈 (Agentic Loop)

Agent 會先列出計畫 (Plan)，獲得你同意後開始修改檔案、跑測試、看到錯誤後自動修正，直到任務完成。這就是所謂的「自主回饋機制」。

結合 MCP (Model Context Protocol)

你可以透過 MCP 協議，讓 GitHub 的 AI Agent 擁有「手和腳」，去存取外部工具或私有數據。

應用場景

  • 自動查詢 Jira / GitHub Issues
  • 搜尋 Google 即時資訊
  • 讀取電腦本地的特定資料庫

設定方式

// 在 VS Code 設定中開啟

"chat.mcp.enabled": true

// 連接 MCP 伺服器，賦予 API 呼叫能力

自定義 Agent 指令 (AGENTS.md)

如果你希望 Agent 在你的 Workspace 中遵循特定的架構或風格，你可以「調教」它。在專案根目錄建立一個 AGENTS.md。

範例指示 (Instruction Example)

  • - 所有 API 必須使用 TypeScript 定義介面
  • - 所有組件必須使用 Tailwind CSS 進行樣式設定
  • - 測試必須使用 Jest 且覆蓋率需大於 80%

02-unit-agent-mode (AI 代理人與 Vibe Coding 實戰)

本程式庫是專為「AI 代理人與 Vibe Coding」單元設計的實戰作業環境。在 2026 年，開發的核心能力已從單純的編碼轉向「引導自主代理 (Agent Guidance)」。你將學習如何指揮 AI Agent 完成複雜的重構、設定團隊規範，並整合外部工具。

🎯 學習目標

  1. 掌握 AI Agent 的 "Plan-Execute" 工作流程。
  2. 學習透過 AGENTS.md 定義專案級別的 AI 工作規範。
  3. 實踐 MCP (Model Context Protocol) 整合，擴展 AI 的外部感知能力。

🛠️ 環境準備

  • GitHub Copilot 訂閱：請確保你的 VS Code 已更新至支援 Agent Mode 的版本。
  • VS Code 擴充功能：安裝最新版的 GitHub Copilot 與 GitHub Copilot Chat。

📝 作業任務

任務 1：全自動代碼重構 (Agent Refactoring)

  • 背景：src/legacy-service.js 是一段結構混亂的舊代碼。
  • 任務：開啟 Copilot Agent Mode，指令 AI 進行重構。要求 Agent 遵循「職責分離」原則，將 UI 邏輯與業務邏輯拆分。
  • 要求：在重構前，讓 Agent 產出 "Plan" (計畫清單)，並將其複製到 PLAN_LOG.md。

任務 2：定義團隊規範 (Custom Instructions)

  • 背景：為了確保團隊協作中 AI 的行為一致，我們需要建立 AGENTS.md。
  • 任務：編輯本程式庫根目錄下的 AGENTS.md，至少加入一條具體的開發規則（例如：所有導出的函數必須包含 JSDoc 註解）。
  • 驗證：完成後，嘗試讓 Agent 修改 src/legacy-service.js，觀察它是否會自動遵循你在 AGENTS.md 中設定的新規則。

任務 3：MCP 外部工具整合

  • 背景：AI 需要更多外部資訊來協助開發。
  • 任務：參考 MCP_GUIDE.md 串接一個 MCP Server (例如 Google Search 或 GitHub MCP)。
  • 回傳：提交一張截圖 mcp-proof.png，展示 Copilot Chat 視窗中出現了 「Tool Call (工具呼叫)」 的圖示。並在本 README 下方的「MCP 報告」區塊簡述該工具如何幫助你。

📂 程式庫結構

  • src/：存放源代碼，包含待重構的舊代碼。
  • AGENTS.md：存放你定義的 AI 代理人工作規範。
  • PLAN_LOG.md：紀錄 Agent 的執行計畫。
  • MCP_GUIDE.md：MCP 串接參考手冊。

以下是在 GitHub Classroom 部署其作業 (Assignments) 的具體建議：

1. 核心設置與工具要求

  • 軟體要求：學員必須擁有 GitHub Copilot 訂閱，並將 VS Code 的 Copilot 擴充功能更新至支援 Agent Mode 的版本。
  • 範本程式庫 (Template Library) 配置：
    • 📂 src/legacy-service.js：故意放入一段結構混亂、未模組化的舊代碼（例如混合了資料庫操作與 UI 邏輯），供學員在任務 1 中重構。
    • 📂 AGENTS.md (範本)：預置一個包含 TODO 的 Markdown 檔案，引導學員練習定義「團隊開發規範」。
    • 📂 MCP_GUIDE.md：提供常用 MCP Server (如 Google Search, GitHub) 的連接清單。

2. 作業任務部署細節

任務 1：跨檔案手術 - 全自動代碼重構 (Agent Refactoring)

  • 目標：練習讓 Agent 自主規劃並執行跨檔案的大型變動。
  • Classroom 部署建議：
    • 回傳證明：要求學員將 Copilot Agent 產出的 "Plan" (計畫清單) 截圖或複製內容存入 PLAN_LOG.md。
    • 核核重點：查看 Git Commit 歷史。如果是 Agent Mode 產出的提交，通常會有「修正編譯錯誤」的連鎖紀錄，這正是學員需要學習的「觀察 Agent 自修」過程。

任務 2：協作 DNA - AGENTS.md 指令調教 (Custom Instructions)

  • 目標：透過定義專案級別的指令，確保 Agent 在整個 Workspace 的行為一致。
  • Classroom 部署建議：
    • 驗證方式 (細粒度檢核)：要求學員在 AGENTS.md 中設定一個具體的語法規則（例如：所有導出的函數必須包含 JSDoc 註解）。
    • 自動評分 (Autograding)：使用代碼分析工具（如 ESLint 或簡單的 grep）檢查學員提交的代碼是否符合他們在 AGENTS.md 中設定的規則。

任務 3：打破邊界 - MCP 外部工具整合 (MCP Integration)

  • 目標：讓 AI Agent 具備獲取即時資訊或外部工具的能力。
  • Classroom 部署建議：
    • 成果展示：這項任務需要物理環境支持。學員需提交一張截圖 mcp-proof.png，展示 Copilot Chat 視窗中出現了 「Tool Call (工具呼叫)」 的圖示（例如 Agent 正在搜尋外部文檔或查詢 GitHub Issue）。
    • 描述任務：要求學員在 README 描述他們串接了哪一個 MCP Server，以及這如何幫助他們解決開發中的問題。

3. 教育者評估指標 (Tutor Benchmarks)

在 Agent 時代，我們評分的對象從「代碼 (Result)」轉向「日誌 (Trace)」：

  • [ ] Prompt 品質：學員對 Agent 下達的指令是否清晰？是否能有效地將複雜任務拆解？
  • [ ] 約束力檢測：AGENTS.md 內的規則是否能有效減少 AI 產出的幻覺或不一致代碼？
  • [ ] 自主修正意識：當代碼出錯時，學員是否學會讓 Agent 讀取日誌自行修復，而非手動介入？
