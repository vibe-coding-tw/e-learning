🌉 1. 類比與數位的橋樑：什麼是 ADC？

世界是類比 (Analog) 的。光線強弱、氣溫變動，它們都是平滑、連續地存在於實體空間中。

ADC (Analog-to-Digital Converter) 就像一把「數位量尺」，將 0V 到 3.3V 的電壓範圍切成好幾格，來度量目前在哪個刻度上。

🪄 2. 數據映射：Map 函式的數學魔法

為什麼需要 Mapping？感測器讀到的是 0–4095，但馬達控制只需 0–255。我們需要一個「比例轉換器」。

∑ 線性變換公式

\[V_{out} = (V_{in} - In_{min}) \times \frac{(Out_{max} - Out_{min})}{(In_{max} - In_{min})} + Out_{min}\]
mapping-logic

Arduino 語法實境應用

// 將 12-bit 轉為 8-bit PMW

int pwm = map(val, 0, 4095, 0, 255);

// 將讀值轉為真實電量百分比

float battery = map(val, 2800, 4000, 0, 100);

⚠️ 3. ESP32 ADC 的真實面貌

非線性「死區」現象

在電壓接近 0V 與 3.3V 附近時，讀值變化會趨於平緩，這在精準溫控或極限控制時需要補償。

Wi-Fi & ADC2 衝突

當 Wi-Fi 開啟通訊時，ADC2 將被系統占用。開發無人車聯網時，務必選擇 ADC1 (GPIO 32-39)。

  1. 映射開發：實作 map() 函式，將電位計的 0-4095 讀值平滑映射到 LED PWM 控制值 (0-255)。

任務 3：「精準轉向模擬器」 (Motion Control: Steering & Dead-band)

角色扮演：運動控制專家 | 難度：⭐️⭐️ | 估計時長：60 min

🛠️ 執行技術流程 (Git Workflow)

  1. 運動模型建立：將類比手柄（電位計）對應到 -90° 至 +90° 的轉向角模型。
  2. 盲區補償實作：在中心點附近設置 ±5 度的死區 (Dead-band)。只有當旋鈕偏移足夠大時，系統才發出運動指令。
  3. 速差計算：根據目前的虛擬轉角，計算並在 Serial 輸出左右馬達的動態速差百分比。
  4. 最終提交：整併所有算法邏輯並 Push 最終報告，討論盲區設定對操控手感的正面影響。

GitHub Classroom 詳細操作指南
