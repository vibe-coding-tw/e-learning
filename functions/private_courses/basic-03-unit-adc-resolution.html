🌉 1. 類比與數位的橋樑：什麼是 ADC？

世界是 類比 (Analog) 的。光線、溫度，都是平滑、連續的存在。

ADC 就像數位量尺，將 0V 到 3.3V 的電壓切成好幾格，來度量目前在哪個刻度上。

🪄 2. 數據映射：Map 函式的數學魔法

為什麼需要 Mapping？感測器讀到的是 0–4095，但馬達只需 0–255。我們需要「比例轉換器」。

∑ 線性變換公式

\[V_{out} = (V_{in} - In_{min}) \times \frac{(Out_{max} - Out_{min})}{(In_{max} - In_{min})} + Out_{min}\]
mapping-logic

Arduino 語法實境應用

// 將 12-bit 轉為 8-bit PWM

int pwm = map(val, 0, 4095, 0, 255);

// 將讀值轉為真實電量百分比

float battery = map(val, 2800, 4000, 0, 100);

⚠️ 3. ESP32 ADC 的真實面貌

非線性「死區」現象

電壓接近 0V 與 3.3V 時，讀值變化會趨於平緩，精準量測需補償。

Wi-Fi & ADC2 衝突

Wi-Fi 開啟時，ADC2 將被占用。通訊時務必選擇 ADC1 (GPIO 32-39)。

課程單元 基礎 3 - 量化感知

類比感測與信號處理 — ADC 解析度與數據映射

任務目標

在本練習中，你將學習如何處理來自類比感測器（如電位計）的訊號。你將會實作：

  1. 電壓採樣：了解 ESP32 ADC 的 12-bit 解析度與物理特性。
  2. 移動平均濾波器 (Moving Average Filter)：消除信號雜訊，獲得穩定的數據。
  3. 數據映射 (Mapping) 與死區 (Dead-zone) 補償：將原始數值轉換為具備物理意義的控制參數。

硬體準備

  • ESP32 開發板
  • 可變電阻 (Potentiometer) x 1
  • 麵包板與跳線

實驗步驟

1. 電壓採樣實驗 (Digital Ruler)

請參考 include/sensor_id.h 中的引腳配置，搭建電路。 使用三用電表測量不同旋轉角度下的電壓，並將測得的電壓與 Serial Monitor 顯示的 ADC 原始值記錄在 SENSING_REPORT.md 中。

2. 實作移動平均濾波器

在 src/main.cpp 中補全濾波算法。你需要：

  • 維護一個長度為 10 的取樣陣列。
  • 計算陣列內所有數值的平均值。
  • 對比濾波前後的數據波動情況。

3. 數據映射與死區補償

  • 使用 map() 函式將濾波後的數值轉換為 0-100% 的亮度或 -90° 至 +90° 的轉向角。
  • 實作死區邏輯：當旋鈕在中位附近（例如 2048 ± 50）時，強制輸出中心值，防止數據微幅抖動造成的誤觸。

繳交內容

完成後，請確保你的程式庫 (Program Library/Repository) 包含以下內容：

  1. 修改後的 src/main.cpp。
  2. 填寫完整的 SENSING_REPORT.md（包含實驗記錄與 Serial Plotter 截圖）。
  3. 推送 (Push) 你的更新至 GitHub 程式庫。

以下是在 GitHub Classroom 部署其作業 (Assignments) 的具體建議：

1. 範本程式庫 (Template Repo) 配置建議

ADC 作業的核心在於「數據的清洗與轉換」，範本應包含完整的測試與記錄工具，建議包含：

  • 📂 src/main.cpp：算法實作核心。預設 analogRead() 框架，學員需在此補全 map() 比例轉換公式，並實作一個簡單的「移動平均濾波器 (Moving Average Filter)」。
  • 📂 include/calibration.h：定義校準偏移量。學員需在此填入實驗測得的「死區」邊界值（例如：讀值低於 120 時視為 0V）。
  • 📂 README.md：採樣數據報表。提供表格範本，讓學員記錄：「實測電壓 V」 vs 「ADC 原始讀值」。
  • 📂 docs/SENSING_REPORT.md：包含 Serial Plotter 截圖區。學員需對比「濾波前」與「濾波後」的數據跳動狀況，證明算法的有效性。

2. 作業任務部署細節

任務 1：「數位量尺」電壓量測實驗 (Digital Ruler: Voltage Sampling Lab)

  • 目標：建立對 12-bit 解析度的感官認知，並找出晶片底層的物理誤差。
  • Classroom 部署建議：
    • 採樣記錄：學員需配合三用電表，記錄在 0.5V, 1.5V, 2.5V 時 ADC 的原始輸出值。
    • 誤差分析：觀察在電壓極低（接近 0V）與極高（接近 3.3V）時，讀值是否停止變化？（識別 ESP32 的非線性區）。
    • 驗證要點：README 中需包含完整的採樣對應表，並描述讀值與真實電壓的線性偏差。

任務 2：「感測器變量儀」反向映射與濾波 (Sensor Interactor & Filtering Lab)

  • 目標：掌握 map() 的靈活運用，並解決物聯網感測器常見的「數據抖動」問題。
  • Classroom 部署建議：
    • 反向邏輯挑戰：實作「越轉越暗」控制，學員需將 0–4095 映射為 255–0（PWM 佔空比）。
    • 雜訊抑制實作：實作一個 10 次採樣取平均的濾波器。
    • 提交結果：在 Serial Plotter 中對比原始數據與濾波數據。要求：濾波後的曲線必須顯著平穩。

任務 3：「精準轉向模擬器」死區補償 (Steering Simulator & Dead-band Lab)

  • 目標：為自動駕駛或機器人建立具備「容錯能力」的控制邏輯。
  • Classroom 部署建議：
    • 轉角映射：將讀值轉換為 -90° 至 +90° 的轉向角，並在 Serial 中輸出。
    • 死區 (Dead-zone) 補償：設定中心點 ±5 度的死區。當旋鈕保持在中位附近時，穩定輸出 0° 防止馬達微顫。
    • 差速邏輯計算：根據轉角，輸出兩台馬達的速差比例（例如：左轉時，左輪 80%，右輪 100%）。

3. 量化感知專家導師點評標準 (Sensing & Signal Processing Standards)

此單元的價值在於 「對原始數據的批判性處理與算法優化意識」：

  • [ ] 映射準確度：map() 函式的參數設定是否考慮了 12-bit (4095) 的邊界？
  • [ ] 濾波效率：採樣平均邏輯是否簡潔？是否有成功抑制高頻跳動感？
  • [ ] UX 思考深度：死區 (Dead-zone) 的設定是否合理地解決了「實體手感」與「數位控制」的摩擦？
