使用 Copilot Agent Mode (自主代理模式)

這是最直接的結合方式。在 VS Code 中，你可以將 Copilot 切換到 Agent Mode。它是一個「自主程式設計師」，可以跨多個檔案進行修改、執行終端機命令、讀取編譯錯誤並自動修復。

操作流程

  1. 1 進入 Chat 視窗： 打開 Copilot Chat (Ctrl+Alt+I)。
  2. 2 切換模式： 在 Chat 頂部的下拉選單中選擇 "Agent" 模式。
  3. 3 下達任務： 輸入高階指令。例如：「幫我把這個專案的 API 從 REST 改成 GraphQL」。

自主迴圈 (Agentic Loop)

Agent 會先列出計畫 (Plan)，獲得你同意後開始修改檔案、跑測試、看到錯誤後自動修正，直到任務完成。這就是所謂的「自主回饋機制」。

結合 MCP (Model Context Protocol)

你可以透過 MCP 協議，讓 GitHub 的 AI Agent 擁有「手和腳」，去存取外部工具或私有數據。

應用場景

  • 自動查詢 Jira / GitHub Issues
  • 搜尋 Google 即時資訊
  • 讀取電腦本地的特定資料庫

設定方式

// 在 VS Code 設定中開啟

"chat.mcp.enabled": true

// 連接 MCP 伺服器，賦予 API 呼叫能力

自定義 Agent 指令 (AGENTS.md)

如果你希望 Agent 在你的 Workspace 中遵循特定的架構或風格，你可以「調教」它。在專案根目錄建立一個 AGENTS.md。

範例指示 (Instruction Example)

  • - 所有 API 必須使用 TypeScript 定義介面
  • - 所有組件必須使用 Tailwind CSS 進行樣式設定
  • - 測試必須使用 Jest 且覆蓋率需大於 80%

單元任務：Agent 實務演練 (Unit Quests)

Mission 1

The First Loop (VS Code Agent 實務)

角色：AI 編排師 難度：⭐️⭐️

任務目標 / Goal

學生需在實驗專案中啟動 Copilot **Agent Mode**，針對專案中刻意留下的邏輯錯誤或架構缺陷進行修正。重點在於觀察 Agent 如何提出 **Plan**，以及在測試失敗時如何進行自動補償 (Self-healing)。

繳交物件 / Deliverables

  • Agent 提出的執行計畫 (Plan) 截圖
  • 修正後的 PR (Pull Request) 連結
Mission 2

Protocol Master (指令調教與 MCP)

角色：系統架構師 難度：⭐️⭐️⭐️

任務目標 / Goal

在專案根目錄撰寫 `AGENTS.md`，定義該專案的程式風格與架構原則。接著嘗試連接一個 **MCP Server** (如 filesystem 或 web-search)，展示 Agent 如何跨出 IDE 存取外部資訊。

繳交物件 / Deliverables

  • `AGENTS.md` 內容文本
  • MCP 工具呼叫成功的 Console 截圖

在 GitHub Classroom 的實務部署

1

接受挑戰：點擊 Classroom 連結並 clone `ai-agent-mode-lab` 倉庫到本地。

2

環境檢查：確認 VS Code 已開啟 Copilot Chat 並切換至 `Agent` 選單模式。

3

實作與推送：完成任務後透過 Git push 修改。PR 的內容需包含 Mission 要求的計畫說明。

教師點評標準 (Tutor Benchmarks)

Agent 控制精準度
學生是否能有效透過對話（Human-in-the-loop）引導 Agent 修正錯誤，而非盲目接受所有建議。
協作與文檔化
`AGENTS.md` 是否具備清晰的架構指引，並能顯著影響 Agent 的程式碼產出風格。

GitHub Classroom 教學引導方針

第一步：匯入 Template

請將 ai-agent-mode-lab 設定為作業範本，並在 .vscode/settings.json 中預設開啟 Agent 模式參數。

第二步：評分重點 (PR Review)

著重檢查學生的 AGENTS.md 邏輯是否清晰，以及是否學會了「小步快跑」的 Agent 溝通技巧。

下一步建議：檢查 VS Code 更新，並嘗試啟動 Agent 模式！
