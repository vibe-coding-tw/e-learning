卷積的魔術：CNN 如何「看」東西

卷積神經網路（Convolutional Neural Network, CNN）是深度學習中專門處理具有空間結構資料的架構。它的核心靈感來自生物視覺皮層中的感受野（Receptive Field）機制——不同的神經元對視野中不同區域的刺激做出反應，而 CNN 透過可學習的卷積核（Kernel）在輸入資料上滑動，自動提取由低階到高階的特徵表示。

卷積運算的數學本質

卷積操作的核心是一個小型矩陣（通常為 3×3 或 5×5）在輸入影像上逐像素滑動，執行逐元素乘積並加總。對於一個輸入圖片 $I$ 與卷積核 $K$，輸出的特徵圖 $F$ 在位置 $(x, y)$ 的值為：

$F(x,y) = \sum_{i}\sum_{j} I(x+i,\; y+j) \cdot K(i,\; j) + b$

其中 $b$ 是偏置項（bias）。不同的卷積核能偵測不同的空間模式：水平邊緣、垂直邊緣、對角線紋理等。以經典的 Sobel 算子為例：

// Sobel 水平邊緣核 (Gx) 與垂直邊緣核 (Gy)
const SOBEL_X = [
  [-1, 0, 1],
  [-2, 0, 2],
  [-1, 0, 1]
];

const SOBEL_Y = [
  [-1, -2, -1],
  [ 0,  0,  0],
  [ 1,  2,  1]
];

// 邊緣強度 = sqrt(Gx² + Gy²)
let magnitude = Math.sqrt(edgeX * edgeX + edgeY * edgeY);

CNN 的層級架構

一個典型的 CNN 由多個功能層堆疊而成，每一層負責不同層次的抽象特徵提取：

層類型 功能說明 輸出
卷積層 (Conv2D) 使用多組卷積核掃描輸入，提取局部空間特徵 特徵圖 (Feature Maps)
激活函數 (ReLU) 引入非線性：$f(x) = \max(0, x)$，讓網路能學習複雜模式 稀疏化的特徵圖
池化層 (Pooling) 降低空間維度，保留最顯著特徵（常用 Max Pooling 2×2） 壓縮後的特徵圖
全連接層 (Dense) 將高階特徵映射到分類標籤的機率分布 類別機率向量
備註 在語音辨識場景中，CNN 的輸入不是 RGB 彩色圖片，而是梅爾頻譜圖（Mel Spectrogram）——一張以時間為 X 軸、頻率為 Y 軸、能量為色彩深淺的二維「聲音圖片」。CNN 在這張圖上搜尋的不是物體邊緣，而是特定頻率模式隨時間的演變規律。

從影像 CNN 到音訊 CNN

傳統影像 CNN 掃描的是像素空間中的紋理與形狀，而語音 CNN 掃描的是時間-頻率空間中的聲學模式。兩者的數學運算完全相同——差別只在於輸入表示方式。這也是為什麼「先將音訊轉為頻譜圖，再餵入 CNN」是語音分類的黃金標準流程。

提示 當你在 Teachable Machine 訓練語音模型時，背後正是將你的錄音片段轉為頻譜圖，再使用一個輕量 CNN 進行分類。理解這個流程，能幫助你在資料品質不佳時做出正確的除錯判斷。

聲音的指紋：語音特徵提取

人類聽覺系統能在嘈雜環境中瞬間辨識出熟悉的聲音——這仰賴的是大腦對頻率模式的高效解析。在 AI 語音辨識中，我們同樣需要將原始音訊波形轉換為一種能凸顯「聲學特徵」的表示形式，讓機器學習模型能從中找出規律。這個過程稱為特徵提取（Feature Extraction）。

從時域到頻域：傅立葉轉換

原始音訊是一維的時間序列——振幅隨時間變化的波形。然而，同一段語音中可能同時包含多種頻率成分（基頻、泛音、噪音等）。快速傅立葉轉換（FFT）能將一段時間窗口內的音訊訊號分解為其組成頻率，產生頻譜（Spectrum）。

實務上，我們對整段音訊使用滑動窗口（通常 25ms 窗口，10ms 步進），對每個窗口做 FFT，將所有窗口的頻譜沿時間軸排列，就得到了頻譜圖（Spectrogram）——一張 X 軸為時間、Y 軸為頻率、色彩表示能量強度的二維圖像。

梅爾頻率倒譜係數（MFCC）

人耳對不同頻率的感知並非線性——我們對低頻差異非常敏感（能分辨 200Hz 和 300Hz），但對高頻差異不太敏感（難以區分 8000Hz 和 8100Hz）。梅爾刻度（Mel Scale）模擬了這種非線性感知，將線性頻率映射為人耳感知等距的尺度。

1

預加重 (Pre-emphasis)

對訊號施加高通濾波器 $y(t) = x(t) - \alpha \cdot x(t-1)$，補償高頻能量衰減。

2

分幀加窗 (Framing & Windowing)

將連續訊號切為 25ms 短幀，施加漢明窗（Hamming Window）以減少頻譜洩漏。

3

FFT → 功率頻譜

對每幀計算 FFT，取其功率譜 $|X(k)|^2$。

4

梅爾濾波器組 (Mel Filter Bank)

將功率譜通過一組三角帶通濾波器，轉換至梅爾刻度。通常使用 26~40 個濾波器。

5

對數壓縮 + DCT

取對數後進行離散餘弦轉換（DCT），取前 12~13 個係數即為 MFCC。

// 簡化的 MFCC 特徵提取流程（概念展示）
function extractMFCC(audioBuffer, sampleRate) {
  const frameSize = Math.floor(sampleRate * 0.025); // 25ms
  const hopSize = Math.floor(sampleRate * 0.010);   // 10ms
  const frames = splitIntoFrames(audioBuffer, frameSize, hopSize);
  const spectrogram = frames.map(f => fft(applyHamming(f)));
  const melSpec = applyMelFilterBank(spectrogram, 40);
  return dct(logCompress(melSpec)).slice(0, 13);
}

為什麼 MFCC 是語音辨識的黃金標準

MFCC 的設計巧妙之處在於它同時達成了三個目標：(1) 透過梅爾刻度模擬人耳感知，保留對辨識最重要的頻率特徵；(2) 透過 DCT 去相關化，將高維頻譜壓縮為少數獨立係數；(3) 對說話者的音量變化具有魯棒性（因為取了對數）。這使得不同人說同一個詞，其 MFCC 模式高度相似，便於分類器學習。

注意 在嵌入式裝置（如 ESP32）上執行即時 MFCC 運算時，FFT 的計算量是主要瓶頸。建議使用定點數運算（Fixed-Point Arithmetic）或預編譯的查找表（LUT）來加速梅爾濾波器的運算，將單幀處理時間控制在 5ms 以內。

權重的推論：從雲端到邊緣

深度學習模型的「智慧」全部編碼在其權重矩陣（Weights）之中。訓練過程是在高效能 GPU 上透過反向傳播演算法，花費數小時甚至數天調整數百萬個參數。然而，推論（Inference）——使用已訓練好的模型進行預測——只需要執行一次正向傳播，計算量遠小於訓練。這使得將模型部署到資源受限的邊緣裝置（Edge Device）成為可能。

雲端訓練 vs. 邊緣推論

面向 雲端訓練 邊緣推論 (ESP32)
計算能力 GPU/TPU，TFLOPS 等級 單核 240MHz，MFLOPS 等級
記憶體 16~80 GB VRAM 520 KB SRAM + 4~8 MB PSRAM
延遲 含網路往返 200~500ms 本地推論 10~50ms
隱私 資料需上傳至雲端 所有資料留在裝置端
離線能力 需要網路連線 完全離線運作

模型量化（Quantization）

直接將雲端訓練的模型（通常使用 32-bit 浮點數儲存權重）搬到 MCU 是不可行的——光模型大小就超出可用記憶體。量化是將浮點權重壓縮為 8-bit 整數（INT8）的技術，能將模型大小縮減約 4 倍，同時推論速度提升 2~3 倍，精度損失通常低於 1~2%。

// TensorFlow Lite 量化轉換流程（Python）
import tensorflow as tf

converter = tf.lite.TFLiteConverter.from_saved_model('my_model/')
converter.optimizations = [tf.lite.Optimize.DEFAULT]
converter.target_spec.supported_types = [tf.int8]
tflite_model = converter.convert()

# 輸出：從 2.4 MB → 600 KB，可放入 ESP32 的 Flash

TensorFlow Lite Micro 推論流程

在 ESP32 上執行推論的完整流程包含四個階段：

  1. 載入模型：從 Flash 記憶體讀取 .tflite 量化模型的位元組陣列。
  2. 輸入歸一化：將感測器原始資料（如 MFCC 向量）縮放至模型期望的 $[0, 1]$ 或 $[-128, 127]$ 範圍。
  3. 正向傳播：呼叫 interpreter->Invoke() 執行所有層的矩陣乘法與激活函數計算。
  4. 輸出解碼：讀取輸出張量，找出最高機率的類別標籤及其信心分數。
重要 邊緣 AI 系統的安全性取決於推論延遲。對於語音喚醒（Keyword Spotting）場景，從麥克風收到音訊到輸出辨識結果，整體延遲必須控制在 200ms 以內，否則使用者會感知到明顯的「不靈敏」。在 ESP32-S3 上，典型的 KWS 模型推論時間約為 30~80ms，加上 MFCC 前處理約 15ms，仍有充裕的時間預算。
提示 如果你的模型在量化後精度下降過大，可嘗試量化感知訓練（QAT）——在訓練過程中就模擬量化誤差，讓模型學會適應低精度環境，通常能將精度差距縮小到 0.5% 以內。
