簡介

本單元將教你實作視覺追蹤中最基礎也最實用的控制演算法——比例控制（P Control）。P 控制的核心思想是：修正力道與偏差量成正比。當目標偏離畫面中心越多，馬達轉動越快；接近中心時則自動減速。掌握 Kp 增益的調校方法，是實現平滑追蹤的第一步。

情境
你的攝影機雲台需要自動追蹤一顆移動中的紅色球體。若直接用「偵測到偏左就全速左轉」的邏輯，雲台會劇烈抖動、來回擺盪而永遠無法穩定對準目標。但如果轉動速度與偏差成比例——偏得多就轉快、偏得少就轉慢——動作立刻變得平滑。這個簡單卻強大的原理就是 P 控制，本單元將教你從數學公式到程式實作完整走一遍。

什麼是比例控制 (P-Control)？

比例控制（Proportional Control）是自動控制理論中最基礎、最直覺的控制策略。它的核心法則極為簡潔：控制輸出與誤差成正比。用白話來說——偏得多就修正多，偏得少就修正少，完全對齊就不動。這個看似簡單的原理，卻是所有進階控制演算法（PI、PID）的根基。

數學公式

P 控制器的輸出由一個單一的乘法運算決定：

\( U(t) = K_p \times e(t) \)

其中：

  • \( U(t) \) — 控制輸出（馬達的 PWM 修正量）
  • \( K_p \) — 比例增益常數（由工程師調校）
  • \( e(t) \) — 當前時刻的誤差（目標位置 − 實際位置）
負回饋原理 控制系統的關鍵在於「負回饋」——修正方向永遠與誤差方向相反。誤差為正（目標在右）→ 輸出正值（向右轉）→ 車體靠近目標 → 誤差縮小 → 輸出減小 → 趨向穩定。這個自我修正的循環就是閉迴路控制的本質。

程式實作

// P 控制器核心實作
function pControl(error, Kp) {
let output = Kp * error;

// 限幅保護：防止 PWM 超出硬體範圍
output = Math.max(-255, Math.min(255, output));

return output;
}

// 使用範例：每幀迴圈中呼叫
const Kp = 200;
const error = getNormalizedError(targetX, canvasWidth);
const steering = pControl(error, Kp); // 範圍: [-255, +255]

視覺追蹤中的 P 控制流程

將 P 控制應用於視覺追蹤的完整回饋迴路如下：

1

影像擷取

攝影機以 30 FPS 擷取當前幀，送入影像處理管線。

2

目標偵測

HSV 過濾或 AI 模型找出目標物的質心座標 \((x, y)\)。

3

誤差計算

正規化誤差 = (targetX − centerX) / centerX → 範圍 [-1, +1]。

4

P 控制運算

steering = Kp × error，並執行限幅。

5

馬達輸出

左輪 = baseSpeed + steering，右輪 = baseSpeed − steering，驅動差速轉向。

關鍵理解 P 控制的「智慧」來自於它的自適應特性：離目標越遠，轉得越快；越接近，轉得越慢。這自然地產生了一個「減速靠近」的效果，遠比簡單的「開/關」控制（Bang-Bang）平滑得多。

比例增益 Kp 的行為特徵

\(K_p\) 是 P 控制器中唯一需要調校的參數，但它對系統行為的影響是決定性的。\(K_p\) 本質上定義了系統的「反應靈敏度」——同樣大小的誤差，不同的 \(K_p\) 會產生截然不同的修正力道與動態行為。

Kp 對系統行為的影響

Kp 範圍系統行為時間響應特徵適用場景
過小（如 50）反應遲鈍，收斂極慢上升時間長，無超調不適用（無法追蹤移動物體）
適中（如 150–250）快速收斂，輕微超調上升時間短，1–2 次震盪後穩定正常追蹤模式
過大（如 500+）劇烈震盪，永不收斂持續等幅震盪（極限環）不適用（系統失控）
臨界值恰好在穩定邊界等幅正弦震盪用於 Ziegler-Nichols 調校法

震盪的物理成因

為什麼 \(K_p\) 過大會導致震盪？這是因為系統存在時間延遲。從攝影機擷取影像、處理辨識、到馬達實際轉動，整個流程需要 30–100ms。在這段延遲期間，車體仍在以舊指令移動。當 \(K_p\) 很大時，系統會「過度修正」——當馬達反應過來時，車體已經衝過了目標點，誤差反號，於是又往反方向猛修正，形成來回擺盪。

工程風險 持續震盪不僅無法追蹤目標，還會快速消耗電池、磨損機械結構、產生過大的電流突波。在實際硬體上測試時，應先從小的 \(K_p\) 開始，逐步增大，並隨時準備手動斷電。

臨界增益法（Ziegler-Nichols 簡化版）

尋找最佳 \(K_p\) 的經典方法之一：

  1. 將 \(K_p\) 設為很小的值，觀察系統是否能追上目標。
  2. 逐步增大 \(K_p\)，直到系統開始出現持續等幅震盪。此時的 \(K_p\) 稱為「臨界增益」\(K_u\)。
  3. 最終使用的 \(K_p\) 設定為 \(K_u\) 的 50–60%，確保系統在穩定區間內運作。
// Kp 調校實驗流程
let Kp = 50; // 從保守值開始

// 逐步增大直到觀察到等幅震盪
// Kp = 100 → 收斂正常
// Kp = 200 → 輕微超調，仍可收斂
// Kp = 350 → 持續震盪！→ 這就是 Ku

// 最終設定
const Ku = 350;
const Kp_final = Ku * 0.5; // = 175，留有安全餘量
實務經驗 在虛擬模擬環境中找到的 \(K_p\) 值，部署到實體車輛時通常需要降低 20–30%。因為真實世界有更多未建模的干擾因素（地面不平、輪胎打滑、電池電壓下降等）。

穩態誤差 (Steady-State Error)

P 控制雖然直覺有效，但它有一個根本性的限制：當誤差極小時，控制輸出 \(U = K_p \times e\) 也會變得極小，可能小到無法克服系統中的物理阻力（摩擦力、重力分量、馬達死區電壓等）。結果就是系統在「接近」目標後停住，留下一個永遠無法消除的殘餘偏差——這就是穩態誤差（Steady-State Error）。

穩態誤差的物理成因

考慮以下場景：目標物靜止，車體正在追蹤。隨著車體接近目標，誤差逐漸縮小。當誤差小到某個值時：

\( K_p \times e_{ss} = F_{friction} \)

此時控制輸出恰好等於摩擦力，車體達到力平衡而停止移動——但誤差 \(e_{ss}\) 並不為零。增大 \(K_p\) 可以縮小穩態誤差，但會犧牲穩定性（更容易震盪）。這就是 P 控制無法完美解決的權衡困境。

// 穩態誤差的數學推導
// 在力平衡狀態下：
e_ss = F_friction / Kp

// 範例計算：
// 若摩擦力等效為 PWM 20，Kp = 200
e_ss = 20 / 200 = 0.1 // 正規化誤差 10%
// 對於 320px 寬的畫面 → 殘餘偏移 = 16 像素

穩態誤差的實際觀察

Kp 值理論穩態誤差對應像素偏移 (320px)追蹤品質
1000.20 (20%)32 px明顯偏離，不可接受
2000.10 (10%)16 px尚可接受
4000.05 (5%)8 px較為精確，但已接近震盪邊界
核心缺陷 P 控制在理論上不可能將穩態誤差完全歸零（除非 \(K_p → ∞\)，此時系統必然不穩定）。這不是程式的 Bug，而是比例控制的數學本質決定的。

解決方案預告：積分控制 (I Control)

要徹底消除穩態誤差，需要一個能夠「記住過去累積偏差」的機制。當系統偏離目標一段時間時，即使瞬間誤差很小，累積值持續增大，最終能產生足夠的修正力道克服摩擦力。這就是 PID 控制中 I（Integral，積分）項的角色：

// PI 控制器概念（下一階段將完整實作）
integral += error * dt; // 持續累加誤差
output = Kp * error + Ki * integral;

// 即使 error 很小，integral 持續增長
// 最終 Ki * integral 足以克服摩擦力 → e_ss = 0
學習路線 本單元的重點是讓你深刻「體驗」穩態誤差的存在與其不可避免性。在後續的 PID 模組中，你將學會如何用積分項消除它，以及用微分項 (D) 抑制震盪。完整的 PID 控制器才是工業級追蹤系統的標配。

  • 目標：將抽象數學公式 $U = Kp \times e$ 落實為強健的代碼。
  • Classroom 部署建議：
    • 代碼邏輯檢核： 
      // 學生需實作：比例運算與限幅處理
calculate(error) {
    let output = error * this.kp;
    return Math.max(-255, Math.min(255, output)); // 防止變數溢位
}
    • Autograding：餵入一組誤差與 $Kp$，檢核輸出的 PWM 數值是否精確對應。
    • 驗證報告：要求學員在 README 中列出其對「溢位處理」重要性的簡短說明。

任務 2：臨界增益調音師 (Kp Tuning Lab)

  • 目標：透過實驗觀察「震盪 (Oscillation)」現象，尋找系統穩定邊界。
  • Classroom 部署建議：
    • 實驗流程要求：學員需調整 $Kp$ 滑桿，偵測出虛擬車體剛好開始進入「等幅蛇行」的 $Kp$ 臨界值。
    • 波形提交：要求學員在 README 中貼上兩張波形截圖：
      1. 過小 $Kp$：收斂極慢且無法克服摩擦力的波形。
      2. 臨界 $Kp$：呈現規律正弦波震盪的波形。
    • PR 深度核核：點評其對「為什麼系統會蛇行」的物理分析（答案：相位延遲與過度補償）。

任務 3：閉迴路視覺追蹤壓力測試 (Visual Servoing Integration)

  • 目標：實作全系統閉迴路 (Closed-loop) 控制，觀察「穩態誤差」。
  • Classroom 部署建議：
    • 系統裝配檢核：學員需成功將影像座標誤差串接到 P 控制器的輸入。
    • 誤差平衡測試：要求學員移動追蹤目標，並在 README 中記錄：當追蹤停止時，最終產生的「穩態偏移量 (Steady-State Error)」是多少像素？
    • 理論思考：引導學員討論「為什麼 P 控制無法完全歸零誤差？」。這為下單元引進 I (Integral) 積分控制 埋下伏筆。

3. 控制系統導師點評標準 (Feedback System Benchmarks)

此單元的價值在於 「對回饋循環的感知與預測能力」：

  • [ ] 數學實作準確度：代碼是否準確執行了比例運算？是否具備「限幅」這種工程保護意識？
  • [ ] 實驗嚴謹度：是否能透過觀察波形來定義「臨界穩定性」？
  • [ ] 缺陷洞察力：是否理解 P 控制在靜態摩擦力面前的無力感？（這標誌著學員具備了向 PID 高階調控邁進的資格）。
