數據的一趟旅程：
HTTP 封包結構與生命週期

拆解 HTTP 通訊細節。跟隨數據包的跳轉，掌握網路效能診斷。

📦 1. 封包裡裝了什麼？

底層都是透過一封封「HTTP 報文 (Message)」在傳遞。

REQ

Request (請求)

包含 Method、Headers 與 Body。

RES

Response (回應)

關鍵在於 Status Code 與回應正文。

Protocol Anatomy

POST /api/move HTTP/1.1

Host: 192.168.4.1

Content-Type: application/json

{ "dir": "F", "speed": 100 }

// 控制馬達的 Payload

🚀 2. 請求生命週期

1

DNS / IP

將名稱轉為 IP。

2

TCP Connect

建立可靠握手。

3

Send Payload

傳輸指令 Body。

4

Render Result

接收回應更新 UI。

⚡ 3. Keep-Alive 的力量

頻繁控制時，傳統「一發一斷」會浪費大量握手時間。

Optimized Connectivity

  • Keep-Alive：讓 TCP 暢通。1 次握手，多次指令。
  • TTFB：診斷 ESP32 負載的黃金標準。
  • 批次通訊：降低 JSON 轉義開銷是效能關鍵。

Latency Inspector

"差 200ms 的延遲，就是撞牆與過彎區別。"
