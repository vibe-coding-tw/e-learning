# Car Basic ESP32 Motor Curriculum Plan
**Last updated**: 2026-06-14

這份文件先不修改任何 `content-repo` 課程 HTML 內容，而是把目前 `car-basic-*` 30 個基礎單元整理成一條完整的「透過 ESP32 精準控制馬達」學習路線。

目標是讓學生從「認識 ESP32 與基本硬體輸出」一路走到「能穩定、精準、可回傳狀態地控制馬達」，並且把每一個單元的作業都設計成下一關可以接續使用的中間成果。

---

## 1. 核心設計原則

1. 先建立硬體與控制語言。
   - 先讓學生理解 ESP32、腳位、PWM、H-Bridge、感測器、定時器與非同步流程。
   - 再進入更高層的通訊協定、Wi-Fi、BLE、Web 控制與 OTA。

2. 先做精準控制，再做遠端控制。
   - 精準馬達控制的核心不是「能轉」，而是「能穩、能準、能重現」。
   - 所以前半段要強調輸出精度、控制迴路、校正、濾波、狀態一致性。

3. 每個單元只引入一個主要硬體控制概念。
   - 例如某關只教 PWM duty cycle，不要同時塞進 API 架構、Wi-Fi 與 BLE。
   - 這樣學生比較不會把馬達控制學成一堆零碎 API 背誦。

4. 每一關都要能產出中間控制成果。
   - 例如 pin map、PWM 參數、響應曲線、狀態機、採樣結果、封包格式、連線狀態、OTA 驗證表。
   - 後一關應該能直接接續前一關的成果。

5. 這份文件只定義路線圖。
   - 先不改課程內容。
   - 等路線圖確認後，再回寫到各單元的 `assignment-guide` / `tutor-guide`。

## 2. 30 個單元的漸進式路線

### 01. PlatformIO Setup

- 現有重點：開發環境、套件管理、序列埠排錯。
- 精準馬達控制練習目標：讓學生可以穩定編譯、燒錄與讀取序列監控。
- 作業可加的練習：
  - 建立最小可運行 ESP32 專案。
  - 記錄序列埠、板子型號與編譯設定。
- 產物：
  - 可燒錄的基礎專案
  - 環境檢查報告

### 02. ESP32 Architecture

- 現有重點：ESP32 結構、開機流程、硬體 DNA。
- 精準馬達控制練習目標：理解 MCU 的基本運作層次。
- 作業可加的練習：
  - 標記 CPU、記憶體、外設、GPIO 的角色。
  - 理解為什麼控制馬達需要周邊模組而不是單純 `digitalWrite()`。
- 產物：
  - ESP32 硬體架構圖

### 03. Pinout

- 現有重點：腳位限制、可用資源、GPIO 安排。
- 精準馬達控制練習目標：規劃馬達驅動、感測器、通訊腳位。
- 作業可加的練習：
  - 選出 H-bridge、PWM、編碼器、按鈕、LED 的腳位分配。
  - 避開啟動保留腳與衝突腳位。
- 產物：
  - Pin map / wiring plan

### 04. PWM Basics

- 現有重點：PWM、duty cycle、輸出控制。
- 精準馬達控制練習目標：掌握馬達速度控制的最小核心。
- 作業可加的練習：
  - 用 LED 或示波器觀察不同 duty cycle 的差異。
  - 把 PWM 的變化和速度感知建立對應。
- 產物：
  - PWM 參數表
  - 基本速度梯度紀錄

### 05. LEDC Syntax

- 現有重點：ESP32 LEDC API、通道與頻率設定。
- 精準馬達控制練習目標：把 PWM 從概念變成可重複的 API。
- 作業可加的練習：
  - 建立 `motorSetSpeed()` 包裝函式。
  - 規劃不同頻率與解析度的配置。
- 產物：
  - 馬達速度 API 草稿

### 06. H-Bridge

- 現有重點：H-bridge 驅動、正反轉、真值表。
- 精準馬達控制練習目標：把單向 PWM 擴展成可前進/後退的完整控制。
- 作業可加的練習：
  - 實作前進、後退、煞車、滑行的狀態表。
  - 檢查方向控制與 PWM 輸出是否互相衝突。
- 產物：
  - H-bridge 控制真值表
  - 方向控制 API

### 07. Drivers & Ports

- 現有重點：硬體連線、驅動器、序列排障。
- 精準馬達控制練習目標：讓學生能把實體接線與程式輸出對上。
- 作業可加的練習：
  - 對照馬達驅動板、ESP32、電源、共地、感測器接線。
  - 記錄燒錄失敗與硬體故障的排除流程。
- 產物：
  - 接線診斷報告

### 08. Response Curves

- 現有重點：線性與指數反應曲線。
- 精準馬達控制練習目標：讓速度變化更符合人類操作直覺。
- 作業可加的練習：
  - 讓低速區更細緻、高速區更平滑。
  - 比較不同曲線對操控手感的影響。
- 產物：
  - 反應曲線比較表
  - 最終曲線參數

### 09. Sampling Rate

- 現有重點：取樣頻率、訊號處理、資料節流。
- 精準馬達控制練習目標：建立穩定的控制更新頻率。
- 作業可加的練習：
  - 設計馬達控制更新週期。
  - 比較過快與過慢的控制更新差異。
- 產物：
  - 控制頻率選擇報告

### 10. FSM

- 現有重點：有限狀態機、障礙物邏輯、安全模式。
- 精準馬達控制練習目標：把控制流程做成可預測的狀態系統。
- 作業可加的練習：
  - 定義 idle / accelerating / cruising / braking / emergency stop。
  - 讓每個狀態對應明確的馬達輸出。
- 產物：
  - 馬達控制 FSM 圖

### 11. State Consistency

- 現有重點：非同步環境下的狀態一致性、ACK、重送。
- 精準馬達控制練習目標：避免控制命令與實際馬達狀態不同步。
- 作業可加的練習：
  - 為控制命令加版本號或序號。
  - 模擬掉包、延遲、重送後的狀態修正。
- 產物：
  - 命令版本控制規格
  - ACK / retry 策略

### 12. Joystick Mapping

- 現有重點：歸一化、映射、死區、節流。
- 精準馬達控制練習目標：把人手操作轉成可控的速度與轉向值。
- 作業可加的練習：
  - 將搖桿座標映射到左右輪速度。
  - 建立 dead zone 防止抖動。
- 產物：
  - 控制輸入到馬達輸出映射表

### 13. Unicycle Model

- 現有重點：單輪車模型、差速運算。
- 精準馬達控制練習目標：把轉向與速度變成左右馬達差速。
- 作業可加的練習：
  - 實作左右輪混合器。
  - 比較直線、轉彎、原地轉向的差速公式。
- 產物：
  - 差速混合公式

### 14. ADC Resolution

- 現有重點：類比感測、解析度、數值映射。
- 精準馬達控制練習目標：讓學生成熟理解輸入精度如何影響控制穩定度。
- 作業可加的練習：
  - 觀察 ADC 解析度對控制手感的影響。
  - 將感測值做校正與映射。
- 產物：
  - 感測校正表

### 15. Pull-up & Debounce

- 現有重點：按鈕抖動、上拉、去抖。
- 精準馬達控制練習目標：確保啟動、停止、模式切換按鍵可靠。
- 作業可加的練習：
  - 為啟動鍵、煞車鍵、模式鍵設計去抖流程。
  - 測試抖動對控制命令的影響。
- 產物：
  - 按鍵穩定策略

### 16. Millis

- 現有重點：非阻塞延遲、事件排程。
- 精準馬達控制練習目標：用時間切片處理控制更新與安全檢查。
- 作業可加的練習：
  - 把馬達輸出更新改成非阻塞 loop。
  - 同時維持心跳與安全停機檢查。
- 產物：
  - 非阻塞控制框架

### 17. Hardware Timer

- 現有重點：硬體定時器、ISR、看門狗。
- 精準馬達控制練習目標：把控制週期做成更穩定的底層節拍。
- 作業可加的練習：
  - 讓馬達控制以固定週期觸發。
  - 比較 timer 與 millis 的精準度差異。
- 產物：
  - 固定週期控制框架

### 18. HTTP Request

- 現有重點：HTTP 請求、回應、狀態碼。
- 精準馬達控制練習目標：理解手機 Web 控制如何送命令給 ESP32。
- 作業可加的練習：
  - 透過 HTTP 傳送速度、方向、煞車命令。
  - 解析回應碼與錯誤。
- 產物：
  - 控制命令 HTTP API 草稿

### 19. HTTP Lifecycle

- 現有重點：請求生命週期、keep-alive、性能。
- 精準馬達控制練習目標：讓控制命令傳輸更穩定、延遲更可預測。
- 作業可加的練習：
  - 比較短連線與長連線對控制手感的影響。
  - 觀察頻繁送命令時的性能瓶頸。
- 產物：
  - HTTP 控制延遲觀察表

### 20. Async WebServer

- 現有重點：ESP32 非同步 Web Server。
- 精準馬達控制練習目標：建立可同時回應 UI 與控制命令的伺服器。
- 作業可加的練習：
  - 非同步處理控制請求與狀態頁。
  - 避免阻塞馬達控制 loop。
- 產物：
  - Async 控制伺服器骨架

### 21. Fetch API

- 現有重點：前端非同步請求、資料提交、timeout。
- 精準馬達控制練習目標：從 Web 前端送控制命令到 ESP32。
- 作業可加的練習：
  - 送出速度 / 方向 / stop 命令。
  - 增加 timeout 與錯誤回報。
- 產物：
  - 前端控制請求範例

### 22. CORS Security

- 現有重點：跨網域、安全標頭、防護。
- 精準馬達控制練習目標：確保前端控制頁能安全地呼叫 ESP32 API。
- 作業可加的練習：
  - 設定允許的來源。
  - 比較允許與阻擋時的行為。
- 產物：
  - CORS policy 說明

### 23. Wi-Fi AP vs STA

- 現有重點：ESP32 AP / STA 雙模式。
- 精準馬達控制練習目標：讓設備能獨立發網，也能接既有 Wi-Fi。
- 作業可加的練習：
  - 建立 AP 模式供初次配網。
  - 切換 STA 後保持控制可用。
- 產物：
  - 連線模式切換規格

### 24. BLE Properties

- 現有重點：BLE read / write / notify。
- 精準馬達控制練習目標：把馬達控制命令設計成 BLE characteristic 操作。
- 作業可加的練習：
  - 定義哪些 characteristic 用於寫入控制，哪些用於回報狀態。
  - 用 notify 回傳當前速度或錯誤狀態。
- 產物：
  - BLE characteristic 對照表

### 25. Advertising & Connection

- 現有重點：BLE 廣播、連線、RSSI。
- 精準馬達控制練習目標：讓手機能找到正確的 ESP32 車子。
- 作業可加的練習：
  - 設計唯一裝置名稱。
  - 利用廣播訊號與連線流程建立配對基礎。
- 產物：
  - 廣播命名與配對規格

### 26. GATT Structure

- 現有重點：GATT 階層、服務、特徵值。
- 精準馬達控制練習目標：把控制資料結構化成可擴充的 BLE 服務設計。
- 作業可加的練習：
  - 依照控制與回報分服務。
  - 把馬達控制、電量、狀態、遙測拆開。
- 產物：
  - GATT schema

### 27. BLE Properties Advanced

- 現有重點：更完整的 read / write / notify 應用。
- 精準馬達控制練習目標：讓控制命令、馬達狀態與警報可以互相同步。
- 作業可加的練習：
  - 下達控制命令後，透過 notify 回報執行結果。
  - 比較單向與雙向回饋的差異。
- 產物：
  - 指令回饋協定

### 28. BLE Advertising Mechanism

- 現有重點：廣播封包、連線入口、訊號距離。
- 精準馬達控制練習目標：讓學生理解實體裝置被手機「看見」之前發生什麼事。
- 作業可加的練習：
  - 設計廣播內容與識別資訊。
  - 解讀 RSSI 對控制體驗的影響。
- 產物：
  - 廣播資料紀錄

### 29. OTA Principles

- 現有重點：OTA 流程、分區、進度條。
- 精準馬達控制練習目標：讓韌體更新不破壞控制功能。
- 作業可加的練習：
  - 在更新前後確認馬達控制仍可正常啟動。
  - 記錄 OTA 與控制服務共存的流程。
- 產物：
  - OTA update checklist

### 30. OTA Security

- 現有重點：密碼保護、版本檢查、裝置鎖定。
- 精準馬達控制練習目標：讓學生知道如何安全部署可控制馬達的韌體。
- 作業可加的練習：
  - 防止未授權韌體覆蓋。
  - 加入版本與裝置模型驗證。
- 產物：
  - 安全更新規範

## 3. 建議的分階段學習結構

### Phase 1: 硬體認知與輸出控制

- 單元：1 到 7
- 重點：能正確安裝、接線、理解腳位、輸出 PWM、驅動 H-Bridge。
- 這一段先讓學生知道「馬達是怎麼被 ESP32 推動的」。

### Phase 2: 控制精度與穩定性

- 單元：8 到 17
- 重點：反應曲線、採樣率、FSM、狀態一致性、去抖、非阻塞、硬體 timer。
- 這一段的主題是「如何讓馬達不是亂轉，而是準確地轉」。

### Phase 3: Web 控制與即時請求

- 單元：18 到 23
- 重點：HTTP、Async WebServer、Fetch、CORS、Wi-Fi AP / STA。
- 這一段讓學生把手機或瀏覽器變成控制端。

### Phase 4: BLE 精準控制與部署

- 單元：24 到 30
- 重點：BLE properties、advertising、GATT、notify、OTA、security。
- 這一段才真正把「手機透過 BLE 精準控制馬達」做成完整產品。

## 4. 每個單元作業的共同設計原則

1. 每關只多一個新的控制觀念。
   - 例如只加 PWM 參數，不同關再加閉迴路、再加通訊。

2. 每關都要留下可重用的中間產物。
   - 設定表、公式、狀態機、封包 schema、API spec、測試紀錄。

3. 先寫模擬，再碰實機。
   - 例如先用 LED 模擬 PWM，再改成馬達。
   - 先用假資料做控制封包，再連到真 ESP32。

4. 控制精準度要可被量化。
   - 最好每關都有測量項目：誤差、延遲、抖動、頻率、反應曲線。

5. 安全機制要從中段就開始加入。
   - 不是只在最後一關才講。
   - 馬達控制一定要有 `stop`、timeout、fail-safe、版本驗證與模式切換。

## 5. 建議的落地順序

如果接下來要開始把這份規劃回寫到課程內容，我建議先做這三步：

1. 先把 1 到 10 關的作業敘述補上「精準控制」語境。
2. 再把 11 到 17 關的作業補成真正的控制迴路與狀態一致性規格。
3. 最後把 18 到 30 關收斂成完整的 Web / BLE / OTA 控制流程。

這樣學生不會一開始就被通訊與部署壓住，而是先把馬達控制的物理層、時間層、資料層建立起來，最後再接上手機與遠端控制。
