1. 完成購買後，該如何上課?

登入後前往課程頁面，按鈕會變更為「進入課程內容」。首次註冊還可獲得一個月的「入門課程」免費體驗權限！

2. 沒辦法順利看課程怎麼辦？

請確認是否已登入。若權限未更新，請來信 info@vibe-coding.tw 我們將優先為您處理。

3. 課程使用期限？

付費課程擁有一年的使用權限。系統會在到期前約 7 天與 1 天發送 Email 提醒。

4. 課前設備準備？

我們準備了詳細的軟硬體清單，請務必先查閱 課前準備事項。

5. 老師會提供什麼協助？

老師負責線上解答與技術指導，協助您從看不懂題目、卡關除錯到能獨立解題；分數由系統自動評分回寫。

6. 硬體教材運送方式？

支援超商取貨 (7-11/全家)。確認付款後通常在 2 個工作天內出貨。

7. 追蹤學習進度？

登入「課程儀表板 (Dashboard)」即可查看時數統計、自動評分分數、卡點狀態與老師指導建議。

8. 智慧導航 (Smart Redirect)？

點擊實作按鈕將直接導向 GitHub。按住 Shift 點擊可手動開啟提交視窗。

9. 作業繳交與通知流程？

點作業連結會先建立「開始作業」紀錄；完成後再於平台「正式提交」，老師會收到通知開始指導。分數由 GitHub 自動評分回寫，系統會通知您查看最新結果。Assignments 分頁以 Repo 的 README.md 為主說明來源。

10. 被推薦為合格導師後要做什麼？

當您收到「被推薦為合格導師候選」通知後，請先到 Dashboard 填寫該單元的 GitHub Classroom 邀請連結（classroom.github.com/a/...）。送出後系統才會通知管理員進入審核流程。

11. 付費方式有哪些？

支援 綠界科技 (ECPay) 線上刷卡與 ATM 轉帳。交易皆受 SSL 加密保護。

12. 退費政策？

本服務為數位內容，開通後不適用七天鑑賞期。請利用預覽功能評估後再行購買。

GitHub Classroom 深度整合

1
自動建立程式庫

系統會為您建立專屬 Repo；實際可見範圍依 GitHub Classroom 與組織權限設定為準。

2
模板內容同步

Starter Code、測試檔、README 與導師指引（tutor-guide）會依單元模板同步準備。

3
自動化評分

若該作業模板有設定 GitHub Actions / Autograding，push 後會由 GitHub 自動評分，結果同步回寫到 Vibe Coding Dashboard。

🛠️ 深度排錯指南：接受邀請後沒看到程式庫？

這是最常見的卡關點：作業存在於「組織」內，若未加入組織則無法存取私有 Repo。

方法一：直接前往組織頁面（最快）

登入 GitHub 後直接輸入組織網址：
https://github.com/您的組織名稱

若看到「You have been invited...」橫幅，請點擊 Join。

方法二：檢查 GitHub 通知

點擊 GitHub 右上角鈴鐺圖示，尋找邀請通知並按下 Accept。

方法三：檢查 Email

搜尋標題包含「Invitation to join...」的信件，點擊 Join 按鈕。

4
參與單元內容改進

若 README、測試條件或作業說明有不清楚之處，您可提出「Unit Repo 改進提案」，和 Tutor / Admin 一起持續改善每個單元。

5
內容分工一眼看懂

Assignments 看 README；導師 Settings 看 tutor-guide。兩者會透過協作流程保持一致。
