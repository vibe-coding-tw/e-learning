STUDENT SUCCESS GUIDE

開啟您的學習實作之旅

這裡整理了從購買、上課到作業繳交的完整指引，協助您在 Vibe Coding 獲得最棒的開發體驗。

最近更新：GitHub 原生整合 (Native API)

為提供更順暢的作業分發與學習體驗，平台已全面推出「GitHub 原生 API」整合模式。此模式能完全替代傳統的外部邀請機制，解決因加入 GitHub 組織 Pending 導致學員無法存取專案的問題：

  • 全自動快速建庫：在作業頁填寫 GitHub 帳號後，系統會在 3-8 秒內背景自動從教材樣板庫（Template Repo）複製並建立您的專屬私有 Repository。
  • 免除加入組織限制：系統會以「外部協作者 (Collaborator)」身份直接授權您的專屬專案 Repo 寫入權限，不需加入任何 GitHub 組織。您只需在電子信箱或 GitHub 鈴鐺通知中接受邀請，即可立即開始實作！
  • Feedback PR 師生交流：建庫時系統會自動建立不合併的 feedback 分支與 PR。您每次 push 提交程式碼時，自動測試的分數與導師的逐行 Code Review 評論都會動態顯示在該 PR 中。

最近更新：平台擴張與體驗升級 (Platform Expansion)

我們已正式上線新版課程路由架構與品質標準化：

  • 智慧路徑導航：改用全新的 learning-path.html 動態學習目錄渲染路由，將所有課程單元的進度與入口統一整合。
  • MS Learn 標準化學習卡：所有單元教材均重構為標準化排版，統一具備 Breadcrumbs 麵包屑導航、進度追蹤側邊欄、Quiz 隨堂測驗以及單元 Summary。
  • 多語系支援 (I18N) 預備：底層資料模型與教材分發架構已完成多語系儲存規格升級，為未來提供國際化英語教材鋪平道路。

1. 付費方式有哪些？

支援 綠界科技 (ECPay) 線上刷卡與 ATM 轉帳。交易皆受 SSL 加密保護。

2. 硬體教材運送方式？

支援超商取貨 (7-11/全家)。確認付款後通常在 2 個工作天內出貨。

3. 課前設備準備？

我們準備了詳細的軟硬體清單，請務必先查閱 準備課程。

4. 完成購買後，該如何上課?

登入後前往課程頁面，按鈕會變更為「進入課程內容」。首次註冊還可獲得一個月的「入門課程」免費體驗權限！

5. 老師會提供什麼協助？

老師負責線上解答與技術指導，協助您從看不懂題目、卡關除錯到能獨立解題；分數由系統自動評分回寫。

6. 追蹤學習進度？

登入「課程儀表板 (Dashboard)」即可查看時數統計、自動評分分數、卡點狀態與老師指導建議。

在課程頁右下角點「師生互動與卡點支援」，可直接回報卡點、查看學習狀態節點，並快速開啟 Dashboard。

7. 智慧導航 (Smart Redirect)？

點擊「前往教室寫作業」後，系統會先確認導師 Promotion code 與授權狀態。通過後，系統會跳出 GitHub 帳號確認框並在 3-8 秒內背景自動為您建立專屬 Repository (原生 API 模式)。按住 Shift 點擊可以手動開啟自訂提交視窗。

8. 導師綁定要輸入什麼？

點擊作業卡下方的「前往教室寫作業」按鈕後，會先出現綁定視窗。請輸入導師提供的 Promotion code；若留白，系統會自動使用預設導師。若有輸入，系統會驗證該 code 是否為此單元合格導師。

9. 作業繳交與通知流程？

點擊「前往教室寫作業」會先建立「開始作業」紀錄。平台會詢問並記錄您的 GitHub 帳號，隨後在背景自動建立專屬專案 Repo 並邀請您為專屬協作者。

您僅需前往電子信箱（搜尋 "Invitation to collaborate"）或點擊 GitHub 右上角的鈴鐺通知接受邀請，即可獲得該作業 Repo 的讀寫權限並開始實作。

分數皆由 GitHub Actions 自動評分，評分結果會在您 push 提交程式碼後即時同步回寫，系統會自動通知您查看最新結果與導師回饋。

10. 課程使用期限？

付費課程擁有一年的使用權限。系統會在到期前約 7 天與 1 天發送 Email 提醒。

11. 退費政策？

本服務為數位內容，開通後不適用七天鑑賞期。請利用預覽功能評估後再行購買。

12. 沒辦法順利看課程怎麼辦？

請確認是否已登入。若權限未更新，請來信 info@vibe-coding.tw 我們將優先為您處理。

有任何問題都可來信，我們會盡快協助您排除。

GitHub 原生作業分發與深度整合

1
自動建立專屬 Repository

在平台確認您的 GitHub 帳號後，系統會在 3-8 秒內背景自動為您複製教材並建立專屬私有 Repo，並直接以外部協作者身份授權 Push 權限。

2
模板內容與教材同步

Starter Code、單元測試、README 與導師指導手冊 (tutor-guide) 會自動複製到您的專屬 Repository。

3
自動化評分與成績回寫

當您 commit 並 push 程式碼時，GitHub Actions 會自動執行單元測試，並透過 Webhook 將分數即時回寫至平台的 Dashboard。

4
Feedback PR 師生代碼審查

系統會為您建立一個永遠不合併的 Feedback Pull Request。您可以在此與導師針對每一行程式碼進行提問、討論與評閱。

🛠️ 深度排錯指南：無法存取您的作業程式庫？

若您點擊後無法存取私有 Repository，通常是因為尚未接受協作者邀請：

接受外部協作者邀請的管道：

  1. GitHub 鈴鐺通知：點擊 GitHub 網頁右上角鈴鐺通知，找到並接受作業專案的邀請。
  2. 電子信箱邀請信：檢查您的 GitHub 註冊信箱，搜尋 "Invitation to collaborate" 並點擊 Accept invitation。
  3. 直接連結接受：您也可以直接造訪作業 Repository 網址（例如：https://github.com/您的導師組織/作業庫名稱/invitation）來手動接受協作者邀請。

1

第一步：建立專屬 Repository 與綁定導師

點擊作業卡下方的「前往教室寫作業」後，系統會引導您填寫或確認導師代碼（Promotion Code）。確認後，系統將自動為您建立與開通專屬的 GitHub 程式作業庫。您可以在本機使用 Git、VS Code 或直接在瀏覽器中使用 GitHub Codespaces 開始實作。

2

第二步：Push 程式碼與觸發自動測試

撰寫完成後，只要將程式碼 git push 送回 GitHub，即會自動觸發該單元的背景自動化評分（GitHub Actions）。系統將針對實作功能進行完整性與語法驗證，無須任何手動操作。

3

第三步：儀表板追蹤評分與回報卡點

您可以隨時登入您的個人學習儀表板（Dashboard）查看最新的自動測試評分。如果程式遇到了無法排除的 bug，您可直接點擊「我卡住了（回報卡點）」並填寫嘗試紀錄，將具體的錯誤與除錯過程發送給專屬導師。

4

第四步：專屬導師線上即時點評與指引

您的專屬導師在接收到卡點或作業提交通知後，會針對您的程式進行人工審查，提供個人化的指引（包含 L1 至 L3 提示等級）與下一步改善計畫。這能協助您釐清盲點，而不直接公布標準答案。

5

第五步：依引導優化程式並再次挑戰滿分

配合導師的回饋建議修正程式邏輯後，再次 push 程式碼即可重新獲得評分。透過「自主實作 ➜ 卡點求助 ➜ 精準引導 ➜ 再次挑戰」的完整閉環，讓您獲得最札實的程式開發與問題解決能力！
