課程使用說明

這裡整理了課程使用方式與相關服務說明

🚀 完整課程使用指南

1

註冊 / 登入會員

首先，請點擊網站右上角的 「登入」 按鈕。我們支援 Google 快速登入，或使用 Email 註冊帳號。擁有帳號後，系統才能記錄您的購買與學習進度。

✨ 新手好禮：首次註冊後，您可以免費體驗一個月的入門課程！

2

瀏覽並挑選課程

前往 入門課程 或 基礎實作 頁面，瀏覽詳細的課程大綱與介紹。確認符合您的需求後，準備進行購買。

3

加入購物車與填寫推薦

在課程頁面點擊 🛒 加入購物車。若您有導師提供的 GitHub Classroom 連結，請在此時輸入，系統將自動為您指派該導師。

4

確認訂單與付款

進入購物車頁面確認金額無誤，點擊「前往付款」。系統將跳轉至 綠界科技 ECPay 安全支付頁面，您可選擇信用卡或 ATM 轉帳。

5

導師綁定與開始上課

付款成功後，前往課程頁點擊 「進入課程內容」。若您先前未填寫推薦連結，系統會在您點擊「作業卡片」時跳出對話框，請補填老師提供的 Classroom 連結即可即時完成綁定。

💡 智慧導航功能：首次點擊會引導您領取 GitHub Classroom 作業；一旦領取完成，後續點擊將直接開啟您的 個人專屬實作程式庫，大幅縮短開發流程。

如有任何操作疑問，歡迎參考下方常見問題或直接聯繫客服信箱。

🎓 1. 完成購買後，該如何上課?

上課流程非常簡單：

  1. 請點擊網站右上角的「登入」按鈕。
  2. 登入您的帳號（建議使用 Google 快速登入）。
  3. 前往課程頁面，原本的「加入購物車」按鈕會轉變為「進入課程內容」。
  4. 點擊該按鈕，即可開始觀看課程影片與實作教材。

如果您已經購買課程，但遇到按鈕狀態未更新或權限問題，請別擔心。

請直接來信與我們聯繫，我們將優先為您處理：

⏳ 3. 課程的使用期限與服務？

完成課程付款後，您將擁有一年的課程使用權限。

系統會為您分配一位指導老師，協助您解答學習過程中的疑問。

提醒 課程到期前一個月，系統會自動發送 Email 提醒。

為了確保最佳學習體驗，我們整理了一份詳細的電腦軟體與硬體清單。

請在開始上課前詳細閱讀：

👉 查看「課前準備事項」

👩‍🏫 5. 老師會提供什麼協助？

老師的主要職責包含：

  • 線上問題解答與技術指導。
  • 批改作業並提供具體的優化反饋。
  • 協助您在 GitHub Classroom 上的實作挑戰。

📦 6. 如何購買硬體教材與運送方式？

您可以在 課前準備 頁面購買 ESP32 套件或 Car Platform 實作車。

  • 超商取貨：結帳時可選取 7-11、全家等門市。
  • 付款完成即寄出：我們通常在確認訂單後的 2 個工作天內出貨。

📊 7. 如何追蹤我的學習進度？

登入後，在課程中點擊「課程儀表板 (Dashboard)」即可查看：

  • 學習時數統計：掌握各單元的累積學習時間。
  • 作業批改狀態：查看老師的分數評語與退回調整建議。

📝 8. 點擊「實作任務」後會發生什麼？

系統會根據您的狀態自動判斷顯示內容：

  • 自動跳轉 (Smart Redirect)：若您已付費且單元有 GitHub 連結，點擊後會直接開啟新分頁引導您去 GitHub 領取考卷，省去多餘步驟。
  • 連結您的授課老師：若系統偵測到您尚未綁定導師，會彈出視窗請您輸入老師提供的 Classroom 連結。
  • 提交作業對話框：若您已完成領取（或按住 Shift 點擊），則會開啟對話框讓您貼上成果連結並留言給老師。

🔔 9. 作業繳交後，老師會如何批改？

繳交作業後的完整回饋流程如下：

  • 即時通知老師：當您點擊「送出作業」後，系統會立即透過 Email 通知負責您的導師。
  • 專業評分與回饋：老師會在管理後台查閱您的程式碼，並輸入分數與具體的優化建議。
  • 自動通知學生：評分完成後，系統會自動寄送 Email 通知您，您可隨時在 Dashboard 查閱結果。

🔧 9. 接受 GitHub Classroom 邀請後，背後發生了什麼事？

當您點擊老師提供的連結並按下 「Accept this assignment」 後，系統會自動執行以下步驟（就像老師發下數位考卷，系統自動幫您印好考卷並寫上名字）：

  1. 自動建立程式庫：系統依據模板建立一個命名為 作業名稱-您的帳號 的專屬程式庫，您有寫入權限、老師有管理權限。
  2. 複製模板內容：Starter Code、測試檔案與 README 會自動複製到新程式庫中。
  3. 開始作業：網頁顯示「Your assignment repository has been created」後，git clone 到本地、完成程式碼、git push 上傳。
  4. 自動評分（若啟用）：每次 push 時，GitHub Actions 會自動執行測試，結果可在 Repo 的「Actions」分頁查看（✅ 通過 / ❌ 失敗）。
  5. Feedback PR：系統自動建立「Feedback」Pull Request，老師可在此留下程式碼行級別的建議。

⚠️ 接受邀請後沒看到程式庫？

  • 伺服器延遲：GitHub 忙碌時建立程式庫需 1-2 分鐘，請重新整理頁面稍候。
  • 組織權限問題：若您尚未加入 Classroom 所屬的 Organization（組織），系統無法賦予權限。請確認已接受組織邀請。
