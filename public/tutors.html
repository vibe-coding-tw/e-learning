導師實務操作流程

我們簡化了所有技術細節，讓導師能專注於最核心的教學與引導。

1
進入管理後台

2

當學生完成導師綁定、回報卡點或有自動評分更新時，系統會依事件寄送 Email 通知您。分數由系統自動回寫，導師可專注於卡點診斷、除錯引導與下一步任務安排，並在 Settings 維護 tutor-guide 指引品質。

4
透明分潤管理

查看即時 Referral 紀錄與分潤佣金，並提供您的 Promotion code 給學員於作業頁綁定。

最近更新：GitHub 原生整合 (Native API)

為提供更順暢的作業分發體驗，我們已推出「原生 API」整合模式。此模式能完全替代傳統的組織邀請派發機制，解決因組織成員邀請 Pending 導致學生無法存取專案的問題：

  • 全自動建庫與授權：學生一鍵點擊「前往 GitHub 寫作業」，系統將在 3-8 秒內於背景自動複製樣板（Template Repo），建立學生專屬私有 Repo，並直接以外部協作者（Collaborator）身份授權學生的 Push 權限。
  • 分支保護與 Dynamic Review：系統建庫時會自動建立 feedback 分支與不合併的 PR（Pull Request），學生所有的 push 提交都會動態更新，導師可直接在程式碼行旁進行 Code Review。
  • 獨立計費與多導師隔離：每位導師可在各自的 GitHub 組織 (Org) 下 Fork 官方教材，並在 Settings 提供個人 GitHub PAT 權杖、組織名稱與樣板名稱，實現班級與 Actions 計費額度的完全獨立。

最近更新：平台擴張計畫 (Platform Expansion)

我們已正式發布平台擴張與多通路營運策略，使 Vibe Coding 平台具備更強大的跨國、跨通路擴展能力：

  • 架構升級與 entryUnitId：淘汰 Legacy 課程檔案的 *-master-*.html 頁面，改用 Firestore 中定義的 entryUnitId 與 courseKey 作為統一的規範入口，並透過 learning-path.html 進行動態目錄與路由渲染。
  • 教材品質標準化 (MS Learn Standard)：所有 104 個教材單元均完成重構，統一具備 Breadcrumbs、進度追蹤側邊欄、Quiz 測驗頁面及 Summary。
  • 參數化多級分潤機制：在 Firestore 建立 revenue_share_policies 集合，將舊有單一 Partner 模式升級為多級角色分潤（包含 Tutor / Agent / CourseDev 角色），支援 upline 抽成、區域化定價及利潤安全防護。

專業導師與 API 整合小撇步

我們已全面改用「原生 GitHub API 模式」進行作業指派與管理：

系統會透過 API 將學生直接新增為該單元專屬 Repository 的外部協作者（Collaborator）。這能帶來以下優勢：

  • 免除組織邀請卡關：學生不需接受任何「組織邀請」就能直接開始寫作業，徹底避免了學生因忘記接受組織邀請而造成的 Repository Access Issue。
  • 全自動建庫流程：背景自動複製樣板（Template Repo）至您的 GitHub 組織，並完成學生帳號授權，整個過程僅需 3-8 秒。
  • 一班一組織安全隔離：導師可在 Settings 欄位中配置專屬的 GitHub Org、樣板 Repo 與 個人 Access Token (PAT)，確保班級與計費額度完全隔離。

🧭 教學介入流程圖

Tutor 不負責手動打分，重點在診斷卡點、設計下一步練習，並持續追蹤學生是否真的跨過關鍵障礙。

STEP 1
收到提交通知，先確認學生目標
從 Email / Dashboard 進入該學生單元，先看他這次想完成哪個任務、目前做到哪裡，避免一開始就直接給答案。
↓
STEP 2
查看分數、卡點與嘗試紀錄
先判斷是觀念問題、流程問題，還是環境問題。分數只用來定位現況，真正重點是找出「卡住的最小步驟」。
↓
STEP 3
提示階梯介入，給 nextAction
依序從「方向提示 → 關鍵觀念 → 範例片段」逐層介入，最後只留一個明確下一步，讓學生知道現在就該做什麼。
↓
STEP 4
學生修正後再次 Push 驗證
請學生照 nextAction 完成修正並 Push；系統會再跑自動評分。Tutor 這時看的是「是否真正理解」，不只是分數有沒有上升。
↓
STEP 5
追蹤提升，必要時升級介入
若連續數次仍卡同一點，改成更高強度介入（拆題、結構化除錯、同步示範）。目標是把學生帶到可獨立完成下一題。

🧩 課程單元作業協作責任

現在每個單元的共同改進範圍不只 README，而是整個 repository：README、tutor-guide、tests 與 workflows。Tutor 是內容一致性的第一線把關者。

  • Assignments 分頁：學生看到的是 README.md。
  • Settings 導師區：導師操作以 tutor-guide.md 為主要引導。
  • 收到學生提案後，請一起檢查 README、tutor-guide、測試規則是否一致，再進行 review。
