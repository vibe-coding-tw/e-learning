導師實務操作流程

我們簡化了所有技術細節，讓導師能專注於最核心的教學與引導。

1
進入管理後台

登入後進入「儀表板 (Dashboard)」，系統會自動識別導師權限並開啟管理視窗。

2
即時通知與批改

當學生提交作業時，系統會立即寄送 Email 通知您，您可以直接點擊連結進行評分與提供建議。

3
發掘優秀人才

針對表現優異的學員，可點擊「推薦為合格導師」，建立您的教學接班團隊。

4
透明分潤管理

查看即時 Referral 紀錄與分潤佣金，並取得推薦連結發放給您的學員。

自動化連動指派 (Cascade Assignment)

當學生已具備付款授權，且您在該課程單元具合格導師資格時，系統會依推薦連結執行課程包全單元連動指派（Cascade Assignment）。

智慧導航 (Smart Redirect)

一旦學生領取作業，系統會自動記錄其個人程式庫，後續點擊將直接導航至實作環境，顯著降低操作門檻。

📋 GitHub Classroom 背後的自動化機制

當學生點擊您的邀請連結按下「Accept this assignment」後，以下流程會自動完成。了解這些機制有助於您更精準地排除學生端問題。

📦
自動建立專屬程式庫

系統根據您設定的模板，自動建立命名為 作業名稱-學生帳號 的 Repo。學生獲得 Write 權限，您擁有 Admin 權限。

📄
模板內容自動複製

Starter Code、Unit Tests、README.md 及 Autograding 設定檔（.github/workflows）全部同步複製到新 Repo。

⚙️
GitHub Actions 自動評分

若該作業模板有設定 GitHub Actions / Autograding，學生每次 git push 後會由 GitHub 執行測試，結果可在 Classroom 控制台查看。

💬
Feedback Pull Request

系統自動建立「Feedback」PR，您可直接在程式碼行旁留言，提供精準的行級別回饋建議。

⏰
Deadline 與遲交標記

您可設定作業截止日，超過時間的 Push 會被系統自動標記為遲交，方便管理多班級作業。

🔒
隱私與防抄襲

除非您設定為 Public，否則每個學生的 Repo 僅該生與您可見，有效保護隱私與預防抄襲。

🎓
師資推薦機制

若發現優秀學員，可在評分時勾選「推薦成為合格教師」，推薦將送至系統管理員進行最終審核。

📦
硬體與分潤管理

管理員可在後台管理實體教材出貨進度。導師則可即時查閱分潤紀錄並取得個人推薦連結。

如何確認學生是否成功加入？

這是導師最常遇到的卡關點：學生接受了作業，但沒有加入「組織 (Organization)」，導致無法存取私有儲存庫。

給導師的檢查步驟：

  1. 進入您的 GitHub 組織首頁。
  2. 點擊上方的 "People" 分頁。
  3. 搜尋學生的帳號：
    • 若在 "Pending invitations" 代表還沒按接受。
    • 若找不到，請點擊 "Invite member" 手動補發。
專業導師小撇步

在 GitHub Classroom 設定中，有一個選項是 「自動邀請學生加入組織」。開啟此功能後，學生接受作業連結的同時，系統就會自動補發組織邀請，大幅降低溝通成本。

📌 快速排解建議：

請學生直接前往組織網頁（github.com/您的組織名），通常頁面頂端就會出現接受邀請的藍色橫幅。
