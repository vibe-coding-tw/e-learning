導師實務操作流程

我們簡化了所有技術細節，讓導師能專注於最核心的教學與引導。

1
進入管理後台

登入後進入「儀表板 (Dashboard)」，系統會自動識別導師權限並開啟管理視窗。

2
即時通知與教學介入

當學生完成作業綁定、回報卡點或有自動評分更新時，系統會依事件寄送 Email 通知您。分數由系統自動回寫，導師可專注於卡點診斷、除錯引導與下一步任務安排，並在 Settings 維護 tutor-guide 指引品質。

3
發掘優秀人才

針對表現優異且達成 100 分門檻的學員，可點擊「推薦為合格導師」。推薦送出後，系統會自動在導師設定的 GitHub 組織中為學生建立專屬 Repository，完成後通知 Admin 進行最終審核。

4
透明分潤管理

查看即時 Referral 紀錄與分潤佣金，並提供您的導師代碼給學員於作業頁綁定。

📋 GitHub 作業分發與自動化機制

本平台採用「作業連結整合」作為標準作業分發系統。此系統專為極致順暢的教學體驗設計，可自動為學生建立私有 Repository 並授予存取權限，而近期的流程更新也已直接納入這套日常操作。

⚡

GitHub 作業連結分發模式

學生在網頁上一鍵開通作業，系統自動在背景建立作業儲存庫並直達開發環境，徹底解決了傳統流程需要額外確認而導致無法及時存取私有 Repository 的卡關問題。

  • ✓ 免額外確認卡關：以直接授權的方式給予學生存取權限，免除學生等待額外授權的繁瑣步驟。
  • ✓ 超低操作門檻：學生 3-8 秒即自動生成 Repo，直接開啟寫作業，省去跳轉至第三方平台的複雜程序。
  • ✓ 完全隔離與安全：導師可在後台填寫自訂的 GitHub Org、樣板 Repo 與 個人 Access Token (PAT)，支援一班一組織的安全隔離。

🧭 教學介入與回饋流程

Tutor 不負責手動打分，重點在診斷卡點、設計下一步練習，並持續追蹤學生是否真的跨過關鍵障礙。

STEP 1
收到提交通知，先確認學生目標
↓
STEP 2
查看分數、卡點與嘗試紀錄
↓
STEP 3
提示階梯介入，給 nextAction
↓
STEP 4
學生修正後再次 Push 驗證
1

先確認學生當下目標

從 Email / Dashboard 進入該學生單元，先看他這次想完成哪個任務、目前做到哪裡，避免一開始就直接給答案。

2

判斷卡點類型

先判斷是觀念問題、流程問題，還是環境問題。分數只用來定位現況，真正重點是找出「卡住的最小步驟」。

3

逐層給 nextAction

依序從「方向提示 → 關鍵觀念 → 範例片段」逐層介入，最後只留一個明確下一步，讓學生知道現在就該做什麼。

4

跟進修正後的結果

請學生照 nextAction 完成修正並 Push；系統會再跑自動評分。Tutor 這時看的是「是否真正理解」，不只是分數有沒有上升。

追蹤與升級介入

若連續數次仍卡同一點，就改成更高強度介入，例如拆題、結構化除錯或同步示範，目標是把學生帶到可獨立完成下一題。

🧩 課程單元作業協作責任

現在每個單元的共同改進範圍不只 README，而是整個 repository：README、assignment-guide、tutor-guide、tests 與 workflows。Tutor 是內容一致性的第一線把關者。

  • Assignments 分頁：直接讀取課程頁隱藏區塊 <section id="assignment-guide">。
  • Settings 導師區：直接讀取課程頁隱藏區塊 <section id="tutor-guide">。
  • 收到學生提案後，請一起檢查 README、assignment-guide、tutor-guide、測試規則是否一致，再進行 review。
