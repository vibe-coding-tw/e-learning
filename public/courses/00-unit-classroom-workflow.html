GitHub Classroom 工作流

從接受邀請到自動評分。掌握高效的雲端教室操作流程，確保你的每一筆 Commit 都是具備價值的學習足跡。

學員操作：5 步驟輕鬆交作業

1

接受邀請

點擊專屬連結並授權 GitHub。

2

Clone 倉庫

將影印好的作業下載至本機。

3

開始實作

在代碼中透過 Vibe 引導 AI。

4

Push 同步

上傳代碼，啟動自動化測試。

5

確認結果

查看 ✅ 指標，確認任務達成。

Vibe 時代的提交規範

在 AI 協助開發的時代，一次性提交（複製貼上）會導致學習過程不對稱。我們建議採用 「微提交 (Micro-commits)」 策略：

範例提交訊息：
• git commit -m "[AI-Generated] 實作 BLE 連線基礎邏輯"
• git commit -m "[Human-Modified] 修正 MTU 限制下的分段傳輸 Bug"
• git commit -m "[Review-Only] 完成單元測試驗證"

💡 每一筆 Commit 都是你的開發思考痕跡。

自動評分 (Autograding) 邏輯

系統會模擬真實的產品驗收，對你的程式碼進行功能測試。

✅
通過測試
邏輯符合預期，滿足產品規格。
❌
測試失敗
點擊紅叉查看報錯，修正後再次 Push。

核心教學策略：從「寫代碼」轉向「架構設計」

Prompt 審查作業

除了提交代碼，你也需要記錄與 AI 的對話過程。我們關注的是你如何精確描述需求，以及如何糾正 AI 的「幻覺」。這才是 Vibe Coding 真正的基本功。

規格說明書 (Spec) 的定義權

訓練撰寫高品質的 Markdown 文檔，定義介面、資料結構與流程。在 2026 年，定義問題的能力遠比寫出語法更重要。
