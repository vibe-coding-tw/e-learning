1. What payment methods are available?

We support ECPay online credit card payments and ATM transfers. All transactions are protected by SSL encryption.

2. How are hardware materials shipped?

We support convenience store pickup (7-11/FamilyMart). Orders are usually shipped within 2 business days after payment confirmation.

3. What equipment do I need before class?

We have prepared a detailed list of software and hardware. Please be sure to check the Preparation Course。

4. After purchasing, how do I access the course?

After logging in, go to the course page. The button will change to "Enter Course Content". First-time registrants also get one month of free access to the "Introductory Course"!

5. What kind of assistance will the instructor provide?

The instructor is responsible for online Q&A and technical guidance, helping you go from not understanding problems and getting stuck debugging to being able to solve problems independently; scores are automatically graded and written back by the system.

6. How do I track my learning progress?

Log in to the "Course Dashboard" to view time statistics, automatically graded scores, stuck points, and instructor guidance.

In the bottom right corner of the course page, click "Instructor-Student Interaction and Stuck Point Support" to directly report stuck points, view learning status nodes, and quickly open the Dashboard.

7. Smart Redirect?

After clicking "Go to Classroom to Do Homework", the system will first confirm the instructor's promotion code and authorization status. After passing, the system will pop up a GitHub account confirmation box and automatically create a dedicated Repository for you in the background within 3-8 seconds (native API mode). Hold down Shift and click to manually open a custom submission window.

8. What should I enter for mentor binding?

After clicking the "Go to Classroom to do Homework" button below the assignment card, a binding window will appear. Please enter the Promotion code provided by your mentor. If left blank, the system will automatically use the default mentor. If entered, the system will verify if the code belongs to a qualified mentor for this unit.

9. Assignment submission and notification process?

Clicking "Go to Classroom to do Homework" will first create a "Start Assignment" record. The platform will ask for and record your GitHub account, then automatically create a dedicated project Repo in the background and invite you as a dedicated collaborator.

You only need to go to your email (search for "Invitation to collaborate") or click the bell notification in the upper right corner of GitHub to accept the invitation, then you will get read and write access to the assignment Repo and can start implementing.

Scores are automatically graded by GitHub Actions. Grading results will be written back in real-time after you push your code. The system will automatically notify you to check the latest results and mentor feedback.

10. Course validity period?

Paid courses have a one-year validity period. The system will send email reminders approximately 7 days and 1 day before expiration.

11. Refund policy?

This service is digital content and is not subject to a seven-day cooling-off period after activation. Please use the preview function to evaluate before purchasing.

12. What if I can't view the course properly?

Please confirm if you are logged in. If your permissions have not been updated, please email info@vibe-coding.tw and we will prioritize your request.

If you have any questions, please feel free to email us, and we will assist you as soon as possible.

🛠️ In-depth Troubleshooting Guide: Cannot Access Your Assignment Repository?

If you cannot access your private repository after clicking the link, it's usually because you haven't accepted the collaborator invitation yet:

Ways to accept an external collaborator invitation:

  1. GitHub Bell Notification:Click the bell icon in the top right corner of the GitHub webpage, find and accept the invitation for your assignment project.
  2. Email Invitation:Check the email address registered with your GitHub account, search for "Invitation to collaborate," and click "Accept invitation."
  3. Direct Link Acceptance:You can also directly visit the assignment repository URL (e.g.,https://github.com/your-mentor-organization/assignment-repo-name/invitation) to manually accept the collaborator invitation.

🔄 Two-Way Student-Teacher Interaction Process

Centered around "Practice -> Obstacle -> Guidance -> Re-challenge," providing you with the most efficient programming learning experience.

1

Step 1: Create a dedicated Repository and bind your tutor

After clicking "Go to Classroom to do homework" below the assignment card, the system will guide you to fill in or confirm your tutor's promotion code. After confirmation, the system will automatically create and activate your dedicated GitHub code repository for assignments. You can start implementing locally using Git, VS Code, or directly in your browser using GitHub Codespaces.

2

Step 2: Push code and trigger automated testing

Once you've finished writing your code, simply git push back to GitHub to automatically trigger the unit's background automated grading (GitHub Actions). The system will perform integrity and syntax validation on the implemented features without any manual intervention.

3

Step 3: Track scores on the dashboard and report blockers

You can log in to your personal learning dashboard at any time to view the latest automated test scores. If your code encounters a bug that you cannot resolve, you can directly click "I'm stuck (report blocker)" and fill in your attempts, sending the specific error and debugging process to your dedicated tutor.

4

Step 4: Dedicated tutor provides real-time online feedback and guidance

Upon receiving a blocker or assignment submission notification, your dedicated tutor will manually review your code and provide personalized guidance (including L1 to L3 hint levels) and a plan for improvement. This helps you clarify blind spots without directly revealing the standard answer.

5

Step 5: Optimize code based on guidance and challenge for a perfect score again

After revising the code logic based on your tutor's feedback, push the code again to get re-graded. Through the complete closed loop of "independent implementation ➜ seeking help for blockers ➜ precise guidance ➜ challenging again," you will gain solid programming development and problem-solving skills!
