什麼是 GitHub Classroom？

想像老師在準備一份實作作業。在傳統方式下，老師需要一張張影印講義發給全班。 在 GitHub Classroom 中，老師製作一個「作業模板（Template）」， 然後讓系統自動為每個學生「影印」一份私人的儲存庫。

  • ✓ 每個學生都有獨立的練習空間
  • ✓ 老師能即時看到全班的進度
  • ✓ 支援自動評分機制

學生端：5 步驟輕鬆交作業

1

接受邀請

點擊老師提供的 Classroom 連結並登入 GitHub。

2

複製儲存庫

使用 git clone 將作業下載到電腦中。

3

開始實作

在 VS Code 或 Codespaces 中撰寫程式碼。

4

提交推送到雲端

使用 git push 將成果上傳完成繳交。

5

確認評分結果

查看 ✅ 指標，確認自動化測試是否通過。

💡 小撇步： 隨時 Push，不用等最後一刻！紀錄開發歷程也是學習的一部分。

自動評分 (Autograding)
即時回饋與驗證

當你 Push 程式碼時，系統會自動啟動虛擬環境執行測試。這能幫助你立刻知道程式邏輯是否正確。

✅

通過測試

恭喜！你的程式符合預期輸出或通過單元測試。

❌

測試未通過

別灰心，點擊紅叉查看報錯訊息，修正後再次 Push 即可。

# autograding.yml 範例
name: GitHub Classroom Autograding
on: [push]
jobs:
  build:
    name: Autograding
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Python Tests
        run: python3 test_hw1.py

老師與導師端 (Tutor Guide)

如何利用 Classroom 大幅提升教學效率？

1

建立組織 (Organization)

為你的課程建立一個「虛擬教室」，統一管理所有學生儲存庫。

2

準備範本 (Template)

寫好基礎架構、留空位，並在 README.md 寫下評分標準。

3

發布邀請

生成專屬連結，學生點擊後自動影印一份作業、自動開工。

💡 給老師的小建議

善用 Pull Requests 進行批改：
勾選 "Enable feedback PRs"，你就能直接在學生的程式碼行數留評語，互動感極佳！

運用隱藏測資：
防止學生針對測試案例「硬寫」輸出。增加 Linting 檢查，確保代碼風格統一。
